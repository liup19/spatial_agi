## 核心信息

### 摘要

Bird's-Eye-View (BEV) 感知是自动驾驶的基石，提供了一个统一的空间表示，通过融合周围视图的图像来实现下游任务的推理，如语义分割、3D目标检测和运动预测。然而，大多数现有的BEV感知框架采用端到端训练范式，其中图像特征被直接转换为BEV空间，仅通过下游任务监督进行优化。这种公式将整个感知过程视为黑盒，通常缺乏明确的3D几何理解和可解释性，导致次优性能。

在本文中，我们主张显式的3D表示对于准确的BEV感知至关重要，并提出了 **Splat2BEV**，一个基于高斯溅射的BEV任务框架。Splat2BEV旨在学习既语义丰富又几何精确的BEV特征表示。

我们首先预训练一个高斯生成器，它能够从多视图输入中显式重建3D场景，从而生成几何对齐的特征表示。然后将这些表示投影到BEV空间，作为下游任务的输入。在nuScenes和Argoverse数据集上的广泛实验表明，Splat2BEV实现了最先进的性能，验证了将显式3D重建整合到BEV感知中的有效性。

### 关键贡献

1. **首次提出显式重建的重要性**: 提出Splat2BEV，一个新颖的高斯溅射辅助的BEV感知框架，首次展示了显式重建在增强BEV感知中的重要性。

2. **融入视觉基础模型**: 融入视觉基础模型以将细粒度的视觉线索注入到重建中，改善了BEV特征表示的质量和表达能力。

3. **最先进性能**: 在nuScenes和Argoverse1数据集上的广泛定量和定性实验表明，Splat2BEV实现了最先进的性能，并一致地优于基线方法。

## 问题背景与动机

### BEV感知的重要性

Bird's-Eye-View (BEV) 感知已成为3D场景理解和自动驾驶的基础。通过将来自多个周围视图的信息聚合到统一的俯视表示中，BEV感知允许对物体位置、运动轨迹和环境布局进行一致的推理。这种空间结构化的表示作为下游模块的有效桥梁，包括分割、检测和规划。

### 现有方法的局限

尽管BEV生成的方法论方法有所不同，但这两条工作都遵循端到端训练范式，其中BEV特征仅通过下游任务监督学习。这种BEV特征的隐式学习过程导致生成不精确和低质量的BEV特征图，最终导致关键的下游自动驾驶任务性能次优。

虽然更多最近的工作试图引入额外的监督，但它们仍然未能解决核心问题——缺乏显式表示，因为它们不强制执行能够产生高度精确和准确的BEV特征图的几何对齐BEV重建。

### 核心洞察

本文的核心洞察是：**显式的3D表示对于准确的BEV感知至关重要**。

传统方法将感知过程视为黑盒，缺乏明确的3D几何理解和可解释性。相比之下，Splat2BEV通过引入3D高斯溅射作为显式中间表示，在学习过程中明确重建3D场景，从而生成几何对齐的特征表示。

这种显式重建的优势包括：
1. **几何一致性**: 学习的特征自然与底层场景几何对齐
2. **可解释性**: 可以可视化3D重建和特征场
3. **语义丰富性**: 通过基础模型蒸馏注入细粒度语义信息
4. **性能提升**: 在多个下游任务上实现显著性能提升

## 核心算法原理

### Splat2BEV框架概述

Splat2BEV是一个基于3D高斯溅射的BEV感知框架，通过显式重建场景来实现准确的鸟瞰图感知。框架的核心思想是：**先明确重建3D场景，再投影到BEV空间**，而不是直接从图像到BEV的端到端隐式映射。

### 三阶段训练流程

Splat2BEV采用三阶段训练策略，逐步优化几何、语义和任务特定的表示：

#### 阶段1: 高斯生成器预训练

**目标**: 学习一个前馈高斯生成器，能够从多视图输入重建3D场景

**输入**: 多视角透视图像 I = {I_i}_{i=1}^k

**输出**: 一组3D高斯 G = {G_j}_{j=1}^n，其中每个高斯 G_j = (μ_j, Σ_j, σ_j, SH_j, f_j)

**损失函数**:
```
L_total = L_render + λ1 * L_depth^L1 + λ2 * L_depth^SILog + λ3 * L_feat
```

各损失项详解：

1. **渲染损失 (L_render)**: MSE损失，确保重建场景与输入视图一致
   ```
   L_render = (1/k) * Σ_i ||Ĉ_i - C_i^gt||_2^2
   ```

2. **深度损失 (L_depth)**: 使用Metric3Dv2生成的度量深度作为监督
   - L1损失: 像素级度量深度准确性
   - SILog损失: 确保尺度不变的几何一致性
   ```
   L_depth^L1 = (1/k) * Σ_i ||Ď_i - D_i^ref||_1
   L_depth^SILog = (1/k) * Σ_i [ (1/n)Σ_p δ_i(p)^2 - (1/n^2)(Σ_p δ_i(p))^2 ]
   δ_i(p) = log Ď_i(p) - log D_i^ref(p)
   ```

3. **特征损失 (L_feat)**: 从冻结的DINO编码器蒸馏密集视觉线索
   ```
   L_feat = (1/k) * Σ_i (1 - cos(Ĥ_i, F_i^DINO))
   ```

#### 阶段2: 任务头训练

**目标**: 冻结高斯生成器，训练BEV编码器和分割头

**过程**:
1. 将重建的3D高斯正交投影到BEV平面
2. 将几何对齐的BEV表示输入BEV编码器
3. 训练分割头执行下游任务（如车道、车辆、行人分割）

**损失函数**:
```
L_BEV = L_focal + λ4 * L_center + λ5 * L_offset
```

- **Focal损失**: 准确分割掩码
- **中心性L2损失**: 保持预测中心的空间对齐
- **偏移L1损失**: BEV域中的偏移对齐

#### 阶段3: 联合微调

**目标**: 端到端联合优化所有组件，协调几何、语义和任务特定线索

**过程**:
- 解冻高斯生成器
- BEV分割损失的梯度可以流回高斯生成器
- 允许重建的3D场景和特征场适应任务级目标

**优势**: 实现了几何、语义和任务特定表示的最佳协调

### 高斯生成器架构

高斯生成器由两个分支组成：

1. **多视图分支** (基于UniMatch)
   - 输入: 多视图图像
   - 输出: 多视图特征和代价体积
   - 作用: 捕获跨视图对应关系

2. **单视图分支** (基于ViT-S)
   - 输入: 单视图图像
   - 输出: 单目特征
   - 作用: 增强多视图表示

**融合**: 多视图特征和代价体积与单目特征连接，作为U-Net的输入，预测深度图和每个高斯的参数

**输出预测**:
- 深度图 (用于高斯均值位置)
- 不透明度 (σ)
- 协方差 (Σ)
- 颜色/球谐系数 (SH)
- 特征向量 (f)

### 3D高斯溅射基础

Splat2BEV采用3D高斯溅射（3DGS）作为场景表示。3DGS使用一组各向异性高斯 {G_i}_{i=1}^n 来建模场景，其中每个高斯 G_i = (μ_i, Σ_i, σ_i, SH_i) 的特征包括：

- **均值 μ_i**: 3D空间中的中心位置
- **协方差 Σ_i**: 控制高斯在3D空间中的形状和方向
- **不透明度 σ_i**: 控制高斯的透明度
- **球谐系数 SH_i**: 编码视图依赖的颜色

**渲染过程**:
3DGS表示的场景可以通过可微分溅射过程从任意视点渲染，其中3D高斯被投影到图像平面作为2D椭圆足迹，它们的颜色被累积形成像素强度。

对于每个像素p，最终渲染的颜色C(p)通过α混合计算：
```
C(p) = Σ_{i=1}^N T_i * σ_i * c_i
其中 T_i = Π_{j<i} (1 - σ_j)
```

这里，c_i表示从SH_i解码的视图依赖颜色，σ_i表示深度排序后高斯的不透明度。

### BEV投影

不同于高斯溅射中用于新视图渲染的透视投影，BEV投影采用正交投影以保持俯视图中的几何一致性。

**正交投影矩阵**:
```
M_ortho = [fx  0   0   cx
           0   fy  0   cy
           0   0   0   1]
```

**投影过程**:
```
[u v 1]^T = M_ortho * [G_X G_Y G_Z 1]^T
```

其中fx、fy、cx、cy表示BEV相机参数；G_X、G_Y、G_Z表示相机空间坐标中3D高斯的均值；u、v表示图像平面上的2D高斯均值。

**协方差投影**:
由于正交投影是线性变换，3D协方差矩阵Σ_3D的投影是直接的。从正交投影矩阵M_ortho得到雅可比矩阵J_ortho：
```
J_ortho = [fx  0   0
           0   fy  0]
```

3D高斯协方差到2D的投影定义为：
```
Σ_2D = J_ortho * Σ_3D * J_ortho^T
```

其中Σ_2D是图像平面上的投影2D协方差矩阵。

## 技术细节

### 输入图像处理

在实验中，多视图输入图像被调整为224 × 480或448 × 800的分辨率，具体取决于实验设置。除非另有说明，默认输入分辨率为224 × 480。

### 损失系数设置

**预训练阶段损失系数** (在L_total中):
- λ1 = 0.2 (深度L1损失权重)
- λ2 = 0.8 (深度SILog损失权重)
- λ3 = 0.1 (特征损失权重)

**BEV任务损失系数** (在L_BEV中):
- λ4 = 2.0 (中心性损失权重)
- λ5 = 0.1 (偏移损失权重)

### BEV投影参数

**空间分辨率**: BEV表示具有200×200的空间分辨率，BEV编码器输出的BEV特征图保持相同分辨率。

**焦距设置**: BEV焦距设置为fx = fy = 2，计算为BEV像素分辨率（200）与BEV空间范围（100m）之间的比率。

**像素偏移**: 像素偏移设置为cx = cy = 100，对应BEV像素分辨率的一半。

**外参**: BEV相机定位在车辆坐标系中沿z轴+3m的位置。

### 训练策略详解

**阶段1: 高斯生成器预训练**
- 多视图分支学习率: 2×10^-4
- 单视图分支: 从Depth Anything V2预训练权重初始化，采用较小的学习率2×10^-6以保留预训练知识
- 训练轮数: 10 epochs

**阶段2: 任务头训练**
- 冻结高斯生成器
- 仅优化BEV编码器和分割头
- 学习率: 2×10^-4
- 训练轮数: 10 epochs

**阶段3: 联合微调**
- 单视图分支学习率保持为2×10^-6
- 所有其他组件学习率: 2×10^-6
- 训练轮数: 20 epochs

### 基础模型集成

#### Metric3Dv2: 度量深度监督

为了解决仅基于渲染损失的重建挑战（由于车辆周围的视图相机设计为最大化环境覆盖，相邻帧之间重叠区域有限），我们采用深度基础模型Metric3Dv2生成参考度量深度进行额外监督。

#### DINO: 视觉特征蒸馏

为了解决现有BEV感知模型仅依赖下游任务损失提供监督信号的问题，导致中间BEV特征约束较弱且不保证几何或语义有意义，我们：
- 为每个高斯G_j附加可学习特征向量f_j ∈ ℝ^C
- 从冻结的DINO编码器蒸馏密集视觉线索
- 使用余弦相似性损失使渲染特征图与DINO编码器提取的密集特征图对齐

这种蒸馏使每个高斯特征向量f_j与DINO提取的高级语义对齐，产生几何基础和语义有意义的特征。

### 关键设计选择

#### 为什么采用正交投影？

1. **保持几何一致性**: 俯视图需要准确的空间关系，正交投影消除了透视畸变
2. **简化计算**: 线性变换使协方差投影直接且高效
3. **与BEV任务对齐**: 俯视图本质上是一个正交投影

#### 为什么采用三阶段训练？

1. **渐进式优化**: 首先学习良好的几何和语义表示，然后适应任务
2. **避免灾难性遗忘**: 预训练阶段确保基础模型知识得到保留
3. **模块化训练**: 每个阶段可以独立优化，提高训练稳定性

#### 为什么使用DINO特征蒸馏？

1. **语义丰富性**: DINO提供高质量的视觉特征表示
2. **自监督预训练**: 不需要额外标注
3. **泛化能力**: 预训练模型具有强大的泛化能力

## 实验结果

### 数据集与评估指标

#### 数据集

**nuScenes数据集**: 自动驾驶的综合基准，包括在波士顿和新加坡在各种天气和光照条件下收集的1000个驾驶场景。700个场景用于训练，150个用于验证，150个用于测试。

**Argoverse 1数据集**: 另一个在匹兹堡和迈阿密收集的大规模自动驾驶数据集。包含113个具有同步多视图相机图像和激光雷达点云的驾驶日志。

所有训练都在训练分割上进行，所有定量结果都在验证集上报告。

#### 评估指标

遵循先前工作的既定实践，我们在所有任务中报告交并比（Intersection-over-Union, IoU）作为主要评估指标。

### 实验任务设置

#### 单类分割实验

在nuScenes上对三个类别分别进行实验，即车辆分割、行人和车道分割，遵循先前工作的设置。

#### 多类分割实验

在nuScenes和Argoverse 1上都进行多类BEV分割实验，遵循先前工作的设置。

### 定量比较结果

#### 车辆分割

遵循先前工作的评估设置，我们使用两种不同的输入分辨率：224×480和448×800。我们还包括了可见性过滤实验（即仅考虑可见性>40%的物体）。

**关键结果**:
- Splat2BEV在各种实验设置下都优于所有基线方法
- 在高分辨率（448×800）设置下表现更优
- 显式重建带来的几何准确性对大型物体（车辆）特别有益

#### 行人分割

对于行人分割，遵循先前工作的评估协议，我们仅考虑可见性>40%的行人。

**关键结果**:
- Splat2BEV相比基线方法实现了8%的改进
- 由于行人占据的区域比车辆小，准确的2D-3D对应关系至关重要
- 即使微小的投影误差也可能导致完全非重叠的分割结果
- 我们方法的优越性能证明了其通过显式3D重建实现更准确几何映射的能力

#### 车道分割

**关键结果**:
- Splat2BEV相比先前方法实现了21.4%的显著改进
- 车道是自动驾驶场景中最大和最结构化的组件之一
- 车道分割是一个特别受益于显式3D重建的任务
- 准确的几何重建对于捕捉车道的细长和结构化性质至关重要

#### 多类分割

在nuScenes和Argoverse 1上的多类分割实验展示了我们方法的泛化能力和鲁棒性。

**关键发现**:
- Splat2BEV在所有评估任务中始终优于现有的BEV分割框架
- 这些改进突显了将显式3D重建整合以学习几何对齐表示的有效性
- 这样的表示使Splat2BEV能够实现更准确和可解释的特征对齐
- 既对小尺度物体（即行人）也大规模结构元素（即车道）都有益处

### 定性分析结果

#### 重建质量可视化

可视化重建的3D场景及其对应的特征场，以及细粒度的细节，例如：
- 视图1中的斑马线
- 视图2中的垃圾桶
- 视图3中的行人
- 视图4中的禁止停车线

这些可视化证明了：
- 重建场景的保真度和质量
- 特征场的语义连贯性
- 对细粒度细节的捕捉能力

#### BEV表示质量分析

为了检查高斯生成器预训练期间学习的特征质量（即阶段1），我们直接评估在无联合微调情况下的Splat2BEV性能，其中高斯生成器保持冻结。

**惊人发现**: 即使没有联合优化，我们的方法已经达到了可与几个基线相当的性能，表明高斯生成器预训练和特征蒸馏产生的几何对齐特征既有意义又高质量。

启用联合微调后，性能进一步提升，验证了将学习的显式3D表示整合到下游BEV分割中的好处。

#### 特征可视化对比

将Splat2BEV与GaussianLSS（另一个基于3DGS的框架）对比，可视化两种类型的特征：
1. 直接从3D表示投影的特征
2. BEV编码器产生的BEV特征图

**关键观察**:
- 显式重建导致更加结构化和几何感知的BEV特征
- 特征在空间上更加连贯
- 语义边界更加清晰

### 消融研究

#### 三阶段训练方案

对比提出的三阶段训练方案与没有阶段2的两阶段变体。

**结果**: 三阶段训练方案表现更优，表明渐进式优化的重要性。

#### DINO特征蒸馏

消融DINO特征蒸馏的效果。

**结果**: 移除DINO特征蒸馏导致性能下降，证明基础模型特征的重要性。

#### BEV相机高度

由于我们的方法显式重建并将场景投影到虚拟BEV空间，BEV相机高度的选择至关重要。
- BEV相机设置太低可能导致场景中缺失几何或物体
- 设置太高可能引入额外的干扰因素（如树木和杆子），损害下游任务性能

**消融结果**: 将虚拟BEV相机高度设置为3米实现了最佳权衡，有效捕获必要的地面级几何结构，同时抑制无关的高空结构。

### 核心性能提升总结

| 任务 | 性能提升 | 关键原因 |
|------|---------|---------|
| 车辆分割 | 显著优于基线 | 大型物体受益于精确几何重建 |
| 行人分割 | 8%提升 | 准确的2D-3D对应关系对小物体至关重要 |
| 车道分割 | 21.4%提升 | 结构化元素最大程度受益于显式重建 |
| 多类分割 | 一致优于基线 | 几何对齐表示的泛化能力 |

## 个人思考与见解

### 最有趣的发现

#### 1. 显式重建的巨大收益

在端到端学习的时代，Splat2BEV证明了显式重建的价值。这有点"反直觉"，因为端到端学习通常被视为更"先进"的方法。但结果表明，**显式的几何约束和可解释性带来了显著的性能提升**。

这让我思考：在复杂的AI系统中，我们是否过分强调了端到端的优雅性，而忽略了显式表示和中间监督的价值？特别是在空间感知这样的任务中，几何一致性是本质的，而不仅仅是优化目标。

#### 2. 三阶段训练的智慧

三阶段训练（预训练 → 任务头训练 → 联合微调）看似简单，但体现了深刻的设计智慧：

- **预训练阶段**: 确保学习到良好的几何和语义表示
- **任务头训练**: 让模型专注于下游任务，不被过早的梯度更新破坏基础表示
- **联合微调**: 协调几何、语义和任务线索

这类似于人类学习：先学习基础知识，再学习特定技能，最后整合两者。这种渐进式学习范式在Spatial AGI中可能有广泛应用。

#### 3. 基础模型蒸馏的有效性

DINO特征蒸馏的简单性令人惊讶。仅仅通过余弦相似性损失，就能将自监督视觉特征注入到3D几何表示中。

这表明基础模型具有强大的通用性，它们的特征可以在不同的任务和表示之间迁移。对于Spatial AGI来说，这意味着我们可以：
- 利用预训练的基础模型为空间感知提供丰富的先验
- 通过简单的蒸馏方法将知识迁移到特定任务
- 大大减少对标注数据的依赖

### 最意外的结果

#### 1. 冻结高斯生成器仍有良好性能

最意外的发现是：**即使没有联合微调（冻结高斯生成器），Splat2BEV的性能已经可与几个基线相当**。

这说明了：
1. 几何对齐特征本身就有价值
2. 预训练阶段学习到的表示质量很高
3. 端到端优化并非总是必要的

这让我思考：在Spatial AGI中，我们是否可以设计一个"感知内核"，学习通用的空间表示，然后针对不同任务只需轻量级适配？这类似于操作系统的内核和应用架构。

#### 2. 车道分割的巨大提升

车道分割相比其他任务获得了更大的提升（21.4%）。这很有趣，因为：
- 车道是最结构化的道路元素（长、细、规则）
- 车道检测对几何精度要求极高
- 传统方法在车道检测上一直有挑战

这证明了显式几何重建对于结构化、细长、几何约束强的任务特别有效。这对Spatial AGI的设计有启发：对于不同类型的空间元素，可能需要不同类型的表示和推理方法。

### 最有价值的启发

#### 1. 空间感知的几何-语义统一

Splat2BEV展示了如何将几何重建（3D高斯）和语义理解（DINO特征）统一在一个框架中。这对Spatial AGI来说是核心问题：**如何将几何、语义、功能、认知等不同层次的空间信息统一表示和推理？**

可能的探索方向：
- 多层次的统一空间表示（几何层、语义层、功能层、认知层）
- 不同层次之间的对齐和交互机制
- 跨层次的信息流和推理机制

#### 2. 可解释性与性能的平衡

Splat2BEV表明，显式表示和可解释性不一定牺牲性能。相反，可解释的几何约束可能提升性能。

这对Spatial AGI很重要，因为：
- 可解释性对于安全关键应用（如自动驾驶、医疗机器人）至关重要
- 可解释的系统更容易调试和改进
- 可解释性增强人类对AI系统的信任

#### 3. 基础模型与任务特定模型的协作

Splat2BEV展示了如何有效利用基础模型（DINO、Metric3Dv2）的能力。这对Spatial AGI有重要启示：

Spatial AGI应该：
- 利用基础模型的通用能力（视觉、语言、推理）
- 针对空间感知任务进行专门优化
- 设计基础模型与任务模型之间的协作机制

这类似于人类智能：我们利用通用的认知能力（视觉、语言、推理），但需要在特定领域（如开车、手术）进行专门训练。

### 对Spatial AGI未来的思考

#### 1. 空间表示的演进

当前的3D高斯溅射表示是一个重要进步，但可能只是空间表示演进中的一个阶段。未来可能的发展：

- **更丰富的原语**: 3D高斯可能扩展到更丰富的空间原语（平面、圆柱、体素等）
- **层次化表示**: 从局部到全局的多层次空间表示
- **动态表示**: 支持动态场景和时序推理的表示
- **多模态统一**: 统一视觉、触觉、听觉等多模态空间信息的表示

#### 2. 空间推理的深度

Splat2BEV专注于空间感知（重建、表示），但对空间推理（关系、推理、规划）的支持有限。未来需要：

- **空间关系推理**: 推理物体之间的空间关系（相邻、包含、遮挡等）
- **因果推理**: 理解空间现象的因果关系（为什么物体会在这里？）
- **反事实推理**: 推理"如果"情况下的空间变化
- **空间规划**: 基于空间理解的长期规划和决策

#### 3. 开放世界空间智能

当前方法主要在封闭、静态环境中工作。未来的Spatial AGI需要在：

- **动态环境**: 处理移动物体、变化场景
- **开放世界**: 应对未见过的物体、场景、情况
- **长期适应**: 在长期运行中持续学习和适应
- **多代理协作**: 多个Spatial AGI系统的协作和通信

#### 4. 效率与可扩展性

Splat2BEV的性能不错，但计算开销可能较大。未来的Spatial AGI需要：

- **高效推理**: 在资源受限设备上实时运行
- **增量学习**: 支持在线学习和更新
- **分布式系统**: 支持大规模分布式空间感知
- **边缘计算**: 在边缘设备上实现部分计算

## 核心发现总结

### 1. 显式3D重建显著提升BEV感知性能

**发现**: 通过引入3D高斯溅射作为显式中间表示，Splat2BEV在车辆、行人、车道分割等任务上显著优于传统的端到端隐式方法。

**关键证据**:
- 车道分割提升21.4%
- 行人分割提升8%
- 车辆分割显著优于所有基线
- 多类分割一致优于基线方法

**意义**: 这证明了在BEV感知中，显式的3D几何表示和约束比纯数据驱动的端到端学习更有效。这为Spatial AGI设计提供了重要启示：**空间感知任务应该利用显式的几何表示和约束**。

### 2. 几何对齐的特征表示具有内在价值

**发现**: 即使冻结高斯生成器（不进行端到端联合优化），Splat2BEV的性能仍可与基线方法相当，这表明几何对齐的特征本身就有很高的质量。

**关键证据**:
- 冻结高斯生成器的性能可与多个基线相当
- 联合微调后性能进一步提升
- 特征可视化显示清晰的语义边界

**意义**: 这表明我们可以设计一个"空间感知内核"，学习通用的几何对齐空间表示，然后针对不同任务只需轻量级适配。这对Spatial AGI的模块化设计有重要启发。

### 3. 基础模型蒸馏有效提升语义质量

**发现**: 通过简单的余弦相似性损失，将DINO视觉特征蒸馏到3D高斯表示中，显著提升了特征的语义质量和下游任务性能。

**关键证据**:
- DINO特征蒸馏带来显著性能提升
- 消融研究证明其必要性
- 可视化显示语义清晰的重建

**意义**: 这展示了基础模型在空间感知中的应用潜力。Spatial AGI应该充分利用基础模型的通用能力，通过蒸馏等方法将知识迁移到特定任务中。

### 4. 结构化元素最大程度受益于显式重建

**发现**: 车道分割获得最大提升（21.4%），表明结构化、几何约束强的元素（如车道）特别受益于显式几何重建。

**关键证据**:
- 车道分割提升21.4%，远超其他任务
- 车道是道路场景中最结构化的元素
- 车道检测对几何精度要求极高

**意义**: 这启发Spatial AGI应该针对不同类型的空间元素采用不同的表示和推理方法。结构化元素（如建筑、道路、家具）可能需要精确的几何表示，而非结构化元素（如植被、人群）可能需要更灵活的表示。

### 5. 渐进式训练策略的有效性

**发现**: 三阶段训练策略（几何预训练 → 任务头训练 → 联合微调）比端到端训练更有效，确保了几何、语义和任务线索的协调优化。

**关键证据**:
- 三阶段训练优于两阶段变体
- 每个阶段都有明确的目标和效果
- 渐进式优化提高了训练稳定性

**意义**: 这为Spatial AGI的训练策略提供了范本。复杂的空间智能系统可能需要分阶段训练，先学习通用能力，再适应特定任务，最后端到端优化。

## 相关工作

### BEV感知方法

#### 几何基于的反投影

**代表工作**: Lift-Splat-Shoot (LSS) [Philion等人, 2020]

**方法**:
- 预测每个像素的深度
- 使用相机内参和外参将图像特征几何反投影到3D点
- 在BEV平面聚合3D点形成BEV特征图

**局限**:
- 性能严重依赖预测深度的质量
- 训练仅依赖下游任务监督，深度估计可能不稳定或不准确

**改进方法**:
- **BEVDepth** [Li等人, 2023]: 利用激光雷达导出的深度图进行监督
- **BEVStereo** [Li等人, 2023]: 使用立体匹配技术增强深度估计质量

#### 学习基于的反投影

**代表工作**: PETR, PETRv2, BAEFormer, Cross-Attention等

**方法**:
- 隐式学习从2D到3D坐标的映射
- 使用基于注意力的架构或MLP直接将图像特征与3D空间位置关联
- 将3D位置信息编码到可学习查询中，使用交叉注意力聚合多视图图像特征

**局限**:
- 完全隐式，缺乏几何推理和可解释性
- 不保证几何一致性

#### 从学习的3D体积投影

**代表工作**: BEVFormer, PointBEV

**方法**:
- 构建预定义的3D体积
- 在3D体积内学习密集特征表示
- 将学习的3D特征体积投影或坍塌到BEV平面

**BEVFormer** [Li等人, 2024]:
- 学习捕获跨场景空间结构和语义的3D特征网格
- 投影或坍塌3D特征体积到BEV平面生成俯视表示

**PointBEV** [Chambon等人, 2024]:
- 引入粗到细的训练策略，专注于感兴趣区域以减少内存消耗

**局限**:
- 仅使用下游任务损失训练，没有显式3D重建
- 缺乏可解释性

#### Splat2BEV的创新

与现有方法相比，Splat2BEV的创新在于：
1. **显式3D重建**: 在训练和推理期间都引入显式3D重建
2. **3D高斯溅射**: 使用3D高斯溅射作为中间表示
3. **基础模型蒸馏**: 从DINO等基础模型注入语义先验
4. **几何对齐**: 学习的表示与底层场景几何对齐

### 自动驾驶中的高斯溅射

#### 导航 (Navigation)

**代表工作**: SplatAM, Gaussian Splatting for SLAM, GS, Photo, RTG

**应用**:
- 将3D高斯溅射集成到机器人导航和SLAM系统中
- 作为映射组件
- 扩展传统3D高斯溅射以支持在线重建
- 在探索期间实现连续的场景更新

**贡献**: 证明了高斯原语可以为自动驾驶场景中的映射提供紧凑和可微分的表示

#### 仿真 (Simulation)

**代表工作**: Street, DrivingGaussian, HUGS, SplatAD

**应用**:
- 利用3D高斯溅射重建真实环境用于自动驾驶仿真
- 构建照片般逼真和几何一致的驾驶场景
- 将大规模户外环境转换为可编辑的高斯表示
- 支持高效渲染、可控场景操作和传感器仿真
- 用于训练和评估自动驾驶的感知和规划模型

**贡献**: 展示了高斯溅射在场景重建和仿真中的强大能力

#### BEV感知

**代表工作**: GaussianLSS, GaussianBEV

**方法**:
- 引入3D高斯溅射作为中间表示
- 实现更高效的BEV特征聚合

**局限**:
- 将高斯仅视为特征载体，而不是执行显式3D场景重建
- 学习过程仍然仅依赖下游任务监督
- 限制了几何可解释性

#### Splat2BEV的区别

与GaussianLSS和GaussianBEV相比，Splat2BEV的关键区别：
1. **显式重建**: 不仅使用高斯作为特征载体，而是显式重建3D场景
2. **几何监督**: 使用渲染损失和深度监督显式优化几何质量
3. **基础模型**: 融合基础模型知识，提升语义质量
4. **可解释性**: 提供明确的几何重建和特征场可视化

### 空间表示学习

#### 神经辐射场

**代表工作**: NeRF, Instant-NGP

**方法**:
- 使用神经网络隐式表示场景
- 通过体积渲染合成新视图

**局限**:
- 训练和推理慢
- 难以编辑和交互
- 缺乏显式几何表示

#### 3D高斯溅射

**优势**:
- 实时渲染
- 显式几何表示
- 可微分
- 支持编辑

**Splat2BEV的应用**:
- 利用3D高斯溅射的实时性和显式表示
- 通过正交投影生成BEV表示
- 学习几何对齐的特征

### 特征学习与蒸馏

#### 自监督视觉特征

**代表工作**: DINO, MoCo, SimCLR

**方法**:
- 通过自监督学习视觉特征
- 无需标注数据学习丰富的视觉表示

**Splat2BEV的应用**:
- 使用冻结的DINO编码器提取特征
- 通过余弦相似性损失蒸馏DINO特征到3D高斯
- 提升特征的语义质量

#### 深度估计基础模型

**代表工作**: Metric3Dv2, Depth Anything

**方法**:
- 大规模预训练的深度估计模型
- 提供高质量的度量深度估计

**Splat2BEV的应用**:
- 使用Metric3Dv2生成参考度量深度
- 提供额外的几何监督
- 改善重建的几何准确性
