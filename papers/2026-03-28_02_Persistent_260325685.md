## 📄 论文概览

### 核心问题

**动作条件化的机器人世界模型**在生成未来视频帧时表现出色，但在自回归部署时表现不佳：

- 主要针对**短期预测**进行了优化
- 在自回归部署时**崩溃**
- 每个预测片段作为下一轮的上下文，导致错误**累积**和视觉质量**快速退化**

### 研究动机

1. **世界模型的优势**：
   - 给定机器人动作序列，生成操作场景的**未来视频帧**
   - 适用于难以用传统物理引擎建模的任务

2. **自回归部署的挑战**：
   - 错误累积效应
   - 视觉质量快速退化
   - 需要新的训练方法

3. **现有方法的不足**：
   - 优化目标与部署方式不匹配
   - 需要重新设计整个模型架构

## 🛠️ 核心方法

### 方法1: RL后训练方案

#### 1.1 RL训练目标设计

**核心思想**：在**自回归rollout**上训练世界模型，而不是ground-truth历史

**技术创新**：
1. **对比RL目标**：使用对比学习的RL目标来稳定训练
2. **收敛保证**：迁移到自回归设置时**保持收敛保证**
3. **训练-部署一致性**：训练和部署使用相同的数据流

**技术细节**：

```python
# 伪代码：RL训练循环
for rollout_state in autoregressive_rollouts:
    # 1. 生成多个候选未来
    candidate_futures = []
    for future_horizon in future_horizons:
        future = model.generate_future(
            current_state=rollout_state,
            action_sequence=planned_actions,
            horizon=future_horizon
        )
        candidate_futures.append(future)

    # 2. 对比学习和强化学习
    contrastive_loss = compute_contrastive_loss(
        candidates=candidate_futures
    )

    rl_loss = contrastive_loss + reward_terms

    # 3. 优化世界模型参数
    optimizer.step(rl_loss)
```

#### 1.2 对比RL目标的迁移

**理论基础**：
- 对比学习的收敛保证在自回归场景中仍然有效
- 通过强化学习信号促进更好的长期预测

**实现细节**：
- 使用**多个候选未来**进行比较
- 选择**高保真预测**进行强化
- 避免**低保真预测**的累积

### 方法2: 多候选未来训练协议

#### 2.1 动态长度生成

**核心思想**：从同一个rollout状态生成**多个候选未来**

**技术创新**：
1. **可变未来长度**：探索不同长度的预测
2. **多样性采样**：通过不同的action序列生成多样性
3. **质量选择**：强化高保真预测

**技术实现**：

```python
# 伪代码：多候选未来生成
def generate_multiple_candidates(state, action_pool, num_candidates=5):
    candidates = []

    for i in range(num_candidates):
        # 随机选择action子序列
        selected_actions = sample_action_subsequence(
            action_pool,
            min_length=random.randint(5, 10),
            max_length=random.randint(10, 20)
        )

        # 生成未来预测
        future = model.generate(
            start_state=state,
            action_sequence=selected_actions
        )

        candidates.append(future)

    return candidates
```

#### 2.2 高保真强化

**核心思想**：**奖励高保真预测，惩罚低保真预测**

**技术细节**：
1. **多视图视觉质量奖励**
2. **特征一致性检查**
3. **时序连贯性约束**

### 方法3: 高效多视图视觉质量奖励

#### 3.1 多视角感知指标

**核心思想**：结合互补的感知指标，提高奖励的准确性

**技术细节**：

| 指标 | 作用 | 优势 |
|------|------|------|
| **LPIPS** | 感知相似度 | 捕捉人类视觉感知差异 |
| **SSIM** | 结构相似度 | 捕捉局部结构信息 |
| **PSNR** | 峰值信噪比 | 捕捉全局质量 |
| **特征一致性** | 特征空间相似度 | 捕捉语义一致性 |

#### 3.2 多视角融合策略

**核心思想**：在**片段级别**聚合奖励信号，实现密集、低方差训练

**技术创新**：

1. **跨视角融合**：
   - 将不同相机的指标进行加权融合
   - 自适应权重学习

2. **片段级聚合**：
   ```python
   # 伪代码：多视图奖励计算
   def compute_multi_view_reward(prediction, ground_truth, cameras):
       rewards = []

       for camera in cameras:
           # 计算该相机的指标
           lpips_reward = compute_lpips(
               prediction[camera],
               ground_truth[camera]
           )
           ssim_reward = compute_ssim(
               prediction[camera],
               ground_truth[camera]
           )

           # 加权融合
           combined_reward = alpha * lpips_reward + beta * ssim_reward
           rewards.append(combined_reward)

       # 片段级聚合
       final_reward = aggregate_rewards(
           rewards,
           method='weighted_average'
       )

       return final_reward
   ```

3. **低方差训练**：
   - 通过多视角融合减少方差
   - 提供更稳定的训练信号
