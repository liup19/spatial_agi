## 摘要解读

全景图像因其宽广的视场角，在机器人和视觉领域正被越来越多地使用。然而，将3D高斯泼溅（3DGS）扩展到全景相机模型仍然具有挑战性，因为现有的公式是为透视投影设计的，而简单的适配往往会引入失真和几何不一致性。

本文提出了**Spherical-GOF**，这是一个基于高斯不透明度场（Gaussian Opacity Fields, GOF）的全向高斯渲染框架。与基于投影的光栅化不同，Spherical-GOF在球形光线空间中的单位球上直接执行GOF光线采样，从而实现了全景渲染的一致光线-高斯交互。

为了使球形光线投射既高效又稳健，我们推导了一个保守的球形边界规则，用于快速的光线-高斯剔除，并引入了一个球形过滤方案，使高斯足迹适应失真变化的全景像素采样。

在标准全景基准测试（OmniBlender和OmniPhotos）上的大量实验表明，该方法具有竞争力的光度质量和显著改善的几何一致性。与最强的基线相比，Spherical-GOF将深度重投影误差降低了57%，循环内点比率提高了21%。定性结果显示了更清晰的深度和更连贯的法线图，对全局全景旋转具有强大的鲁棒性。

我们进一步在**OmniRob**上验证了泛化能力，这是本文中引入的真实世界机器人全景数据集，包括UAV（无人机）和四足平台。源代码和OmniRob数据集将在https://github.com/1170632760/Spherical-GOF发布。

## 一、核心算法原理

### 1.1 问题背景与动机

#### 1.1.1 全景图像的优势
 panoramic相机（360°相机）具有以下独特优势：

1. **宽视场角覆盖**：能够一次捕捉360°的全景环境，大幅减少图像采集数量
2. **高效的环境感知**：在机器人导航、环境监控等应用中，全景相机提供全面的环境信息
3. **较少的盲区**：相比传统的针孔相机，全景相机大幅减少了视觉盲区

#### 1.1.2 3D高斯泼溅的局限
传统的3DGS虽然具有实时渲染速度的优势，但在全景图像上存在根本性挑战：

**核心问题**：
- 3DGS使用局部仿射近似来计算投影后的协方差矩阵
- 这种近似假设投影后的3D高斯在图像平面上仍保持为椭圆高斯
- 对于全景相机（特别是equirectangular投影，ERP），这种假设在极地区域严重失效

**数学表达问题**：
```
Σ'_i = J_i * (R^c_w * Σ_i * (R^c_w)^T) * J_i^T
```
其中：
- Σ_i: 3D高斯的协方差矩阵
- J_i: 投影的雅可比矩阵（在x_i^c处评估）
- R^c_w: 世界到相机的旋转矩阵

问题在于：全景投影（如ERP）高度非线性，雅可比矩阵的局部线性近似在极地区域误差极大。

#### 1.1.3 NeRF vs 3DGS
- **NeRF**: 由于光线采样的本质，扩展到全景模型相对直接
- **3DGS**: 光栅化流程与相机投影模型紧密耦合，扩展困难

### 1.2 Gaussian Opacity Fields (GOF) 基础

#### 1.2.1 GOF的核心思想
GOF是一个基于光线的高斯渲染公式，避免了屏幕空间投影近似。其关键特点：

1. **光线空间评估**：直接沿相机光线评估每个高斯的密度贡献
2. **无需投影近似**：不依赖特定相机模型的线性化
3. **几何一致性**：提供明确定义的深度和法线输出

#### 1.2.2 GOF的数学表达

**输入**：
- 相机中心: o ∈ ℝ³
- 光线方向: r ∈ ℝ³
- 光线上任意点: x = o + t*r，其中t为光线深度

**光线-高斯交互**：

对于第i个3D高斯，给定均值x_i、旋转R_i和缩放矩阵S_i = diag(s_i)，将光线变换到高斯局部坐标系：

```
o_i = S_i^(-1) * R_i * (o - x_i)  -- 相机中心到高斯中心的向量
r_i = S_i^(-1) * R_i * r           -- 光线方向
x_i(t) = o_i + t * r_i             -- 光线上的点
```

**1D高斯响应**：

将局部坐标系的点代入高斯公式，得到沿光线的1D响应：

```
G_i^1D(t) = exp(-1/2 * x_i(t)^T * x_i(t))

展开后得到二次函数：
G_i^1D(t) = exp(-1/2 * (r_i^T * r_i * t^2 + 2 * o_i^T * r_i * t + o_i^T * o_i))
```

这是一个关于t的二次函数，其最大值在：
```
t* = -B/A
其中 A = r_i^T * r_i
    B = o_i^T * r_i
```

**关键洞察**：由于GOF沿光线评估高斯贡献，渲染不再依赖于特定的投影模型，这使其能够无缝扩展到全景成像，而不会引入投影近似误差。

### 1.3 Spherical-GOF的创新

#### 1.3.1 球形光线空间采样

**坐标系定义**：
- z轴指向前方
- x轴指向右侧
- 与针孔相机惯例一致

**ERP投影定义**：

对于3D点x = (x_x, x_y, x_z)，其经度和纬度为：

```
φ = atan2(x_x, x_z) ∈ [-π, π]      -- 经度
ϕ = arcsin(-x_y / ||x||_2) ∈ [-π/2, π/2]  -- 纬度
```

投影到equirectangular图像：

```
Proj(x) = (W/(2π) * φ + W/2, -H/π * ϕ + H/2)^T
```

#### 1.3.2 保守的球形边界策略

**问题**：为了高效渲染，需要确定每个高斯影响的瓦片范围

**挑战**：
- 在原始3DGS中，使用投影雅可比估计范围，不适用于全景成像
- 直接计算各向异性高斯在全景图上的精确经纬度范围极其困难

**解决方案**：将每个高斯近似为球体

1. **球体直径**：由高斯的最长主轴确定
2. **保守边界**：计算该球体覆盖的经纬度的保守上下界
3. **保守性**：虽然估计的范围比精确投影更宽，但保证不会裁剪任何有效的光线-高斯贡献

**优势**：
- 计算高效
- 保证不丢失几何信息
- 适用于所有全景相机模型

#### 1.3.3 纬度相关的优化策略

**问题发现**：由于全景投影的纬度相关失真，相同3D尺寸的高斯根据其纬度在图像平面上可能占据显著不同的面积

**后果**：高纬度的高斯倾向于比低纬度的高斯累积大得多的梯度

**解决方案**：修改高斯梯度使其具有纬度依赖性

**纬度权重定义**：
```
w_lat = clamp(cos(ϕ), ε)
```

将此权重乘以致密化分数，以抑制极点附近的过度分裂。

#### 1.3.4 球形过滤方案

**问题**：对于ERP全景图，角分辨率随纬度变化，相同尺寸的高斯可能有非常不同的像素足迹

**后果**：子像素足迹会导致混叠和不稳定性

**解决方案**：根据全景图的角分辨率为每个高斯分配各向同性过滤半径f_i

**详细计算**：

1. **相机空间距离**：r = ||x_cam||_2
2. **纬度**：ϕ = arcsin(-x_y / r)

给定图像分辨率W×H，垂直和水平角分辨率为：

```
Δθ_lat = π / H
Δθ_lon = (2π / W) * cos(ϕ)
```

取最大值：
```
Δθ = max(Δθ_lat, Δθ_lon)
```

候选像素支持：
```
f_cand = r * Δθ
```

对所有可见相机取最大值，并应用常数因子κ：
```
f = κ * max_cams(f_cand)
```

**高斯缩放膨胀**：

给定每轴缩放s ∈ ℝ³，用各向同性半径f膨胀高斯缩放：

```
s~ = sqrt(s ⊙ s + f² * 1)
```

其中⊙表示Hadamard积，1 ∈ ℝ³是全1向量。

**不透明度补偿**：

由于膨胀改变了高斯体积，我们需要补偿不透明度以保持密度一致性：

```
o ← o * sqrt(∏(s ⊙ s) / ∏(s~ ⊙ s~))
```

其中∏(·)表示向量分量的乘积。

**效果**：
- 在远距离或高纬度区域维持足够的图像支持
- 抑制数值不稳定性和混叠

### 1.4 全景感知的几何损失

#### 1.4.1 为什么需要几何正则化？

**问题**：仅靠光度监督优化几何仍然是病态的

**现象**：
- 外观变化可以部分地用几何来解释
- 导致深度和法线中的高频伪影

**例子**：纹理边缘可能被错误解释为深度变化，产生"涟漪"伪影

#### 1.4.2 总损失函数

```
L = L_rgb + λ_dn * L_dn + λ_j1 * L_jump1 + λ_j2 * L_jump2
```

其中：
- L_rgb: 高斯渲染中使用的标准重建损失
- L_dn: 深度-法线一致性损失
- L_jump1: 一阶深度跳跃正则化
- L_jump2: 二阶深度跳跃正则化
- λ_*: 各项的权重系数

#### 1.4.3 基于不透明度的有效掩膜

**定义**：
```
Ω = {u | α(u) > τ}
```

其中：
- u: 像素坐标
- α(u): 像素u处的累积不透明度
- τ: 固定阈值

**意义**：几何正则化仅在具有足够渲染不透明度的像素上有意义。

#### 1.4.4 深度-法线一致性损失

**目标**：鼓励渲染的法线图N(u)与从渲染深度计算的法线图N^d(u)之间的一致性

**步骤**：

1. **反投影**：用渲染深度将像素u反投影为3D点p(u)
2. **深度诱导法线计算**：
```
N^d(u) = norm(Δ_x p(u) × Δ_y p(u))
```
其中Δ_x和Δ_y表示使用相邻像素沿图像x和y方向的一阶有限差分

3. **角度差异最小化**：
```
L_dn = Σ_{u∈Ω} w_lat(ϕ) * (1 - |N(u)^T * N^d(u)|)
```

其中w_lat(ϕ)是纬度权重（公式11），用于补偿ERP的纬度相关失真并平衡不同纬度的贡献。

#### 1.4.5 深度跳跃正则化

**目标**：抑制深度振荡

**对数深度定义**：
```
z(u) = log(max(D(u), ε))
s(ϕ) = max(cos(ϕ), ε)
```

**一阶差分**：
```
Δ_x z(u) = (z(u + Δ_x) - z(u)) / s(ϕ)
Δ_y z(u) = z(u + Δ_y) - z(u)
```

**铰链响应**：
```
E_x(u) = max(|Δ_x z(u)| - τ_1, 0)
E_y(u) = max(|Δ_y z(u)| - τ_1, 0)
```

**边缘感知权重**（从输入全景I计算）：
```
w_x(u) = exp(-β * ||∂_x I(u)||)
w_y(u) = exp(-β * ||∂_y I(u)||)
```

**一阶跳跃损失**：
```
L_jump1 = Σ_{u∈Ω} w_lat(ϕ) * (w_x(u) * E_x(u) + w_y(u) * E_y(u))
```

**二阶跳跃损失**：类似地惩罚二阶对数深度差异，使用相同的水平ERP校正以减少涟漪状伪影。

**设计洞察**：
- 使用对数深度处理大深度范围
- 纬度相关的s(ϕ)补偿ERP失真
- 边缘感知权重保护真实深度跳跃
- 二阶损失进一步平滑深度图

#### 1.5.1 初始化

## 三、创新点与局限性

### 3.1 核心创新点

#### 3.1.1 技术创新

**创新1：球形光线空间GOF采样框架**

**技术细节**：
- 直接在单位球上执行GOF光线采样
- 避免了平面投影的局部线性化误差
- 实现了全景渲染的一致光线-高斯交互

**创新性**：
- 首次将GOF框架成功扩展到全景成像
- 完全避免了投影近似，从根本上解决了3DGS在全景图像上的几何不一致性
- 适用于所有全景相机模型（ERP、环视、立方体等）

**技术突破**：
- 传统方法（OmniGS、360-GS）使用全景投影的雅可比矩阵进行协方差传播
- 这些方法仍然依赖局部仿射近似，在极地区域失效
- Spherical-GOF在光线空间中直接评估，完全绕过投影近似

**创新2：保守的球形边界策略**

**技术细节**：
- 将各向异性高斯近似为球体
- 计算球体的经纬度保守边界
- 支持高效的光线-高斯剔除

**创新性**：
- 解决了全景成像中高斯影响范围估计的难题
- 保守性保证不丢失几何信息
- 计算高效，适用于实时渲染

**优势对比**：
- 原始3DGS：使用投影雅可比，不适用于全景
- SPaGS：使用包围盒光栅化，但需要额外处理
- Spherical-GOF：球形边界，自然适配全景几何

**创新3：球形过滤方案**

**技术细节**：
- 根据全景图角分辨率自适应调整高斯足迹
- 膨胀高斯缩放，确保最小像素支持
- 补偿不透明度以保持密度一致性

**创新性**：
- 显式处理ERP投影的纬度相关失真
- 避免子像素足迹导致的混叠
- 跨纬度的稳定训练

**数学贡献**：
```
f = κ * max_cams(f_cand)
s~ = sqrt(s ⊙ s + f² * 1)
o ← o * sqrt(∏(s ⊙ s) / ∏(s~ ⊙ s~))
```

**创新4：球面度量一致的几何正则化**

**技术细节**：
- 深度-法线一致性损失
- 一阶和二阶深度跳跃正则化
- 纬度相关的权重补偿ERP失真

**创新性**：
- 减少高频外观纹理对几何的影响
- 抑制深度涟漪伪影
- 显著提升几何一致性（DRE降低57%，CIR提高21%）

**设计洞察**：
- 边缘感知权重保护真实深度不连续
- 对数深度处理大深度范围
- 逐渐启用几何正则化稳定训练

#### 3.1.2 应用创新

**创新5：OmniRob数据集**

**数据集特点**：
1. **真实机器人平台**
   - UAV（无人机）平台：Antigravity UAV
   - 四足机器人平台：Unitree Go2

2. **多样化相机配置**
   - 全景相机：完整ERP全景图
   - 环视相机：有限垂直视场[-39°, 6°]
   - 伪环视：裁剪UAV全景图到[-40°, 20°]

3. **真实场景复杂性**
   - 异构视角
   - 不同相机参数化
   - 现实世界的噪声和伪影

**创新价值**：
- 首个机器人全景3D重建评估基准
- 支持跨相机模型的泛化验证
- 推动Spatial AGI在机器人领域的应用

#### 3.1.3 性能创新

**创新6：旋转鲁棒性**

**实验结果**：
- θ = 90°时，PSNR仅降低约7%
- OmniGS在θ = 90°时PSNR降低约32%
- LPIPS保持稳定

**技术原因**：
- 光线空间评估对全局旋转不敏感
- 避免了投影空间的局部线性化

**应用价值**：

### 3.2 技术局限性

#### 3.2.1 理论局限

**局限1：球形边界近似**

**问题**：
- 将各向异性高斯近似为球体可能过于保守
- 影响范围估计不够精确
- 可能影响渲染效率

**影响**：
- 更宽的影响范围导致更多的光线-高斯交互计算
- 计算开销增加
- 但保证了几何正确性

**潜在改进**：
- 更精确的各向异性球形边界计算
- 自适应的粗粒度细分
- 层次化的剔除策略

**局限2：球形过滤的均匀性**

**问题**：
- 各向同性过滤半径可能不适配各向异性高斯
- 在某些方向可能过度膨胀
- 在某些方向可能膨胀不足

**影响**：
- 空间分辨率在某些方向降低
- 可能引入轻微的模糊
- 影响细节保持

**潜在改进**：
- 各向异性的球形过滤
- 方向自适应的过滤策略
- 局部内容感知的调整

**局限3：几何正则化的启发式**

**问题**：
- 深度-法线一致性损失基于启发式
- 深度跳跃正则化需要手动调整阈值
- 纬度权重是简化模型

**影响**：
- 在某些场景可能过度平滑
- 边缘保留和噪声抑制的权衡
- 需要根据场景调整超参数

**潜在改进**：
- 学习到的几何先验
- 自适应的阈值选择
- 端到端的正则化权重学习

#### 3.2.2 实践局限

**局限4：训练效率**

**问题**：
- 当前实现约需1小时训练
- 比SPaGS（数十分钟）慢
- 限制了实时应用的可行性

**原因分析**：
- 球形边界计算比投影近似复杂
- 几何正则化增加了计算开销
- 当前实现可能未充分优化

**潜在改进**：
- GPU加速的球形几何计算
- 更高效的光线采样策略
- 混合精度训练

**局限5：分辨率限制**

**问题**：
- 全景图像的大视场角导致有限空间分辨率
- 远距离几何细节丢失
- 影响细粒度空间理解

**原因分析**：
- 固定像素数覆盖360°×180°视场
- 角分辨率随纬度变化
- 远距离物体占像素少

**潜在改进**：
- 多分辨率高斯表示
- 自适应细分策略
- 焦点区域增强

**局限6：动态场景**

**问题**：
- 当前方法假设静态场景
- 不支持动态物体
- 限制了在真实世界应用

**原因分析**：
- 优化目标基于光度一致性
- 动态物体违反多视图一致性假设
- 缺少时序建模

**潜在改进**：

#### 4.1.1 全向基准测试（OmniBlender和OmniPhotos）

**OmniPhotos（真实世界）**：

  - 渲染质量：保持竞争力
  - 几何一致性：全面领先
  - 最低DRE，最高CIR

**关键洞察**：
1. 渲染质量（PSNR/SSIM/LPIPS）所有方法竞争
2. Spherical-GOF在几何一致性（DRE/CIR）上显著领先

#### 4.1.2 旋转鲁棒性实验（表II）

**实验设置**：

- 测试时添加全局旋转±θ

**结果对比**：

**OmniGS（投影基线）**：
- θ = 0°: 基准

**ODGS（投影基线）**：
- θ = 0°: 基准
- θ = 90°: PSNR降低约16%

**SPaGS（光线基线）**：
- θ = 0° 到 90°: 基本不变

**Spherical-GOF**：
- θ = 0° 到 90°: PSNR仅降低约7%，LPIPS保持稳定

**定性观察**（图2）：
- 投影方法在高旋转下模糊增加
- 高纬度细纹理崩塌为针状条纹
- 光线空间方法保持稳定

**技术原因**：
- 投影近似的有效采样模式随全景方向变化
- 光线空间评估独立于旋转

#### 4.1.3 消融实验（表III）

**对比配置**：
1. **Full**: 完整方法
2. **w/o L_dn**: 去掉深度-法线一致性损失
3. **w/o L_jump**: 去掉深度跳跃正则化
4. **w/o L_dn and L_jump**: 去掉所有几何正则化

**结果**：
- 去掉L_jump: DRE增加，CIR降低（几何一致性明显下降）
- 去掉L_dn: DRE/CIR可能略有降低（代理指标特性）
- 定性观察（图3）：L_dn和L_jump显著平滑平面区域并抑制涟漪伪影

**关键发现**：
- 深度跳跃正则化对几何一致性至关重要
- 深度-法线一致性改善视觉几何质量
- 组合效果最好

#### 4.1.4 OmniRob实验（表IV）

### 4.2 定性实验结果

#### 4.2.1 OmniBlender定性比较（图1）

**观察**：
1. **RGB渲染**：所有方法视觉上合理
2. **深度图**：显著差异
   - 投影基线：高频失真，纹理对齐的涟漪
   - Spherical-GOF：平滑深度，干净的不连续性
3. **法线图**（从深度导出）：
   - 投影基线：噪声，与外观纹理相关
   - Spherical-GOF：平面区域更稳定

**技术原因**：
- 大平面使几何伪影特别明显
- 基于投影的方法在高失真区域失效
- 几何正则化减少纹理对几何的影响

## 五、个人思考与深度分析

### 5.1 最有趣的发现

#### 5.1.1 光线空间vs投影空间的根本差异

**发现**：光线空间评估对全景旋转的稳定性远超投影空间方法

**分析**：
- **投影空间**：局部线性化假设在高失真区域（极地）失效
- **光线空间**：直接在球面上采样，完全避免投影近似
- **结果**：90°旋转时，OmniGS的PSNR下降32%，Spherical-GOF仅下降7%

**深层意义**：
- 这不仅是一个技巧，而是范式转变
- 对于Spatial AGI，相机无关的表示可能更通用
- 不同传感器可以在统一的光线空间中表示

**类比思考**：
- 这类似于在数学中选择正确的坐标系
- 极坐标问题用极坐标更简单
- 全景问题在球形空间更自然

#### 5.1.2 纹理对几何的干扰

**发现**：光度监督的病态性导致纹理错误解释为几何

**观察**：
- 投影基线在平面区域产生纹理对齐的涟漪
- 几何正则化显著减少这些伪影
- 深度跳跃正则化特别有效

**分析**：
- 问题的病态性：同一外观可由不同几何解释
- 纹理边缘可能被优化器解释为深度变化
- 几何先验是必需的

**Spatial AGI启示**：
- 纯光度监督不足以学习一致几何
- 需要几何先验或正则化
- 自监督需要更多样的约束

**类比**：
- 类似于人类视觉中的错觉
- 2D图像不足以确定3D结构
- 需要运动、立体线索、物理先验

### 5.2 最意外的结果

#### 5.2.1 旋转鲁棒性的稳定性

**意外**：光线空间方法对旋转几乎完全不变

**数据**：
- SPaGS: θ = 0°到90°，基本不变
- Spherical-GOF: θ = 0°到90°，PSNR仅降7%

**为什么意外**：
- 90°是极大的旋转
- 期望即使光线空间方法也会有退化
- 但退化远小于投影方法

**技术解释**：
- 光线空间评估独立于全景方向
- 球形边界计算旋转对称
- 几何正则化不受旋转影响

**Spatial AGI意义**：
- 机器人可以从任意方向观察环境
- 全景图的重新定向不影响重建质量
- 支持动态视角生成和探索

#### 5.2.2 训练效率的权衡

**意外**：虽然Spherical-GOF训练更慢（1小时 vs 数十分钟），但几何提升巨大

**权衡**：
- 训练时间: 约3-5倍增加
- DRE: 57%降低
- CIR: 21%提高

**思考**：
- 通常，质量提升伴随效率损失
- 但这里几何质量提升显著
- 对于下游应用，这可能完全合理

**Spatial AGI启示**：
- 需要任务驱动的优化目标
- 不同应用有不同的质量/效率权衡
- 几何质量对某些应用更重要

**未来方向**：
- 可以通过优化改善训练效率
- 训练是离线成本，推理需要实时
- 混合方法可能最佳

### 5.3 最有价值的启发

#### 5.3.1 几何质量优先

**启发**：对于Spatial AGI，几何质量可能比渲染质量更重要

**证据**：

### 6.2 评估指标

#### 6.2.1 渲染质量指标

**PSNR (Peak Signal-to-Noise Ratio)**：
- 衡量像素级重建质量
- 传统图像质量指标
- 适合评估光度一致性

**SSIM (Structural Similarity Index)**：
- 衡量结构相似性
- 对比人类感知
- 更好的相关性

**LPIPS (Learned Perceptual Image Patch Similarity)**：
- 基于深度学习特征
- 使用AlexNet骨干
- 感知相似性度量

#### 6.2.2 几何一致性指标

**DRE (Depth Reprojection Error)**：

**定义**：对于视图i中的像素u，深度为D_i(u)：
1. 反投影到3D
2. 变换到视图j
3. 重新投影得到对应点u'和预测深度D_proj(u')
4. 在u'处采样视图j的渲染深度D_j(u')

**计算**：
```
e(u) = |D_proj(u') - D_j(u')| / (D_j(u') + ε)
```

**报告**：所有有效重叠像素的平均值

**意义**：测量跨视图深度漂移的严重性

**CIR (Cycle Inlier Ratio)**：

**定义**：测量像素在前向-后向重投影循环中保持一致的比例

**过程**：
1. 从视图i的u开始
2. 重投影到视图j得到u'
3. 重投影u'回视图i得到û
4. 如果||û - u||_2 < τ_cyc（2像素），则u是内点

**计算**：有效重叠像素和视图对的内点比例

**意义**：反映视图一致几何的覆盖率

**对比**：
- DRE: 深度漂移的严重性
- CIR: 一致几何的覆盖范围

### 6.3 核心实验结果

#### 6.3.1 定量结果总结

**OmniBlender-Indoor**：
- DRE降低: 62.7% vs SPaGS
- CIR提高: 22.6% vs SPaGS
- 渲染指标: 竞争性

**OmniBlender-Outdoor**：
- DRE: 显著降低

- 渲染指标: 竞争性

**OmniPhotos**：

- CIR: 最高（所有方法）
- 渲染指标: 保持竞争力

**旋转鲁棒性**（θ = 90°）：

#### 6.3.2 定性观察

### 6.4 消融实验

#### 6.4.1 配置

- **Full**: 完整方法

## 八、相关工作与对比

### 8.1 3D Gaussian Splatting相关

#### 8.1.1 经典3DGS

**Kerbl et al. (2023)**:
- 创始工作，实现实时渲染
- 使用各向异性3D高斯
- EWA-splatting投影公式

**局限**：
- 依赖局部仿射近似
- 设计用于针孔相机
- 宽视场角失效

#### 8.1.2 几何改进方法

**2D Gaussian Surfels (2DGS)** (Huang et al., 2024):
- 表示为局部表面补丁
- 更准确的深度和法线估计
- 但仍依赖投影

**Gaussian Opacity Fields (GOF)** (Yu et al., 2024):
- 光线空间评估
- 体积占用风格
- 明确定义的深度和法线输出

**HTGS** (Hahlbohm et al., 2025):
- 透视正确的光线评估
- 避免矩阵求逆
- 稳定优化

**Spherical-GOF**:
- 扩展GOF到全景成像
- 球形光线空间采样
- 几何质量优先

### 8.2 全景3D重建相关

#### 8.2.1 NeRF-based方法

**EgoNeRF** (Choi et al., 2023):
- 扩展NeRF到全景图像
- 直接光线评估
- 但训练慢

**其他全景NeRF**:
- 相对直接扩展
- 继承NeRF的慢渲染
- 适合静态场景

#### 8.2.2 3DGS-based方法

**OmniGS** (Li et al., 2025):
- 推导全景投影雅可比
- 传播协方差
- 仍依赖局部仿射近似

**ErpGS** (Ito et al., 2025):
- ERP图像渲染增强
- 3D高斯正则化
- 投影基线

**360-GS** (Bai et al., 2025):
- 布局引导的全景高斯泼溅
- 室内漫游优化
- 投影基线

**ODGS** (Lee et al., 2024):
- 投影到局部切平面
- 映射回全景图
- 引入额外开销和缝合问题

**SPaGS** (Li et al., 2025):
- 扩展HTGS到球形全景
- 全方向光线泼溅相交
- 包围盒基光栅化

**Spherical-GOF**:
- 光线空间评估（类似SPaGS）
- GOF基础（更好的几何）
- 几何正则化（减少纹理干扰）
- 训练时间更长但几何质量显著提升

### 8.3 对比表格

| 方法 | 基础 | 投影 | 训练效率 | 几何质量 | 旋转鲁棒性 |
|------|------|------|----------|----------|------------|
| 3DGS | 3DGS | 针孔 | 快 | 中 | N/A |
| OmniGS | 3DGS | 全景 | 中 | 低 | 低 |
| ErpGS | 3DGS | ERP | 中 | 中 | 中 |
| 360-GS | 3DGS | 全景 | 中 | 中 | 中 |
| ODGS | 3DGS | 切平面 | 中 | 中 | 中 |
| SPaGS | HTGS | 无（光线空间） | 快 | 高 | 高 |
| **Spherical-GOF** | **GOF** | **无（光线空间）** | **中** | **最高** | **高** |

**关键差异**：
- Spherical-GOF在几何质量上显著领先
- 光线空间方法（SPaGS、Spherical-GOF）旋转鲁棒性好
- 训练效率：SPaGS > Spherical-GOF > 其他
