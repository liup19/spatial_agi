## 论文摘要

OmniStream 是一个统一的流式视觉基础模型，旨在解决现代视觉智能体在实时流式环境中面临的挑战。该模型通过引入因果时空注意力和3D旋转位置编码（3D-RoPE），实现了对连续视频流的高效帧级在线处理。通过在29个数据集上的协同多任务预训练，OmniStream 学习到的表征能够同时支持语义理解、动态建模和几何推理，并展现出卓越的跨领域泛化能力。

**核心贡献**：
1. 提出了统一的流式视觉骨干网络，融合因果时空注意力和3D-RoPE
2. 设计了协同多任务训练框架，耦合静态/时序表示学习、流式几何重建和视觉-语言对齐
3. 在冻结骨干网络的严格设置下，实现了跨多个任务的竞争性能
4. 证明了单一流式视觉骨干网络的可行性和有效性

## 研究背景与动机

### 1. 当前视觉基础模型的局限性

现代视觉智能体越来越需要在流式设置中部署：机器人通过相机观察世界，助手观看视频，AR设备处理第一人称视角流。在这些场景中，智能体必须在严格的延迟和内存预算下，从连续流中在线更新其信念。

然而，当前的视觉基础模型仍然**高度碎片化**，每个模型都专门针对以下某一领域进行优化：

#### 1.1 图像语义感知模型

- **代表方法**: DINO, DINOv2, DINOv3, SigLIP, CLIP
- **特点**: 
  - 专注于单帧图像的语义理解
  - 擅长物体识别、分类、分割等任务
  - 学习语义不变性（semantic invariances）
- **局限性**:
  - 缺乏时序建模能力
  - 无法处理动态场景
  - 不适合流式推理

#### 1.2 离线时序建模模型

- **代表方法**: V-JEPA, VideoMAE, I-JEPA, TimeSformer
- **特点**:
  - 专注于视频的时序动态建模
  - 能够捕捉运动信息和时间依赖
  - 通常需要完整的视频序列作为输入
- **局限性**:
  - 非因果性：通常使用双向注意力，依赖未来帧
  - 计算复杂度高：O(T²)的注意力复杂度
  - 不支持实时在线推理
  - 空间定位能力较弱

#### 1.3 空间几何专家模型

- **代表方法**: DepthAnything, VGGT, DUSt3R, Span3R
- **特点**:
  - 专注于3D几何推理和重建
  - 能够预测深度、法线、相机姿态等
  - 提供显式的空间结构信息
- **局限性**:
  - 缺乏高层语义理解能力
  - 通常需要专门的优化过程
  - 难以与其他视觉任务统一

### 2. 现实应用的需求

一个有用的视觉表征必须具备以下三个核心属性：

#### 2.1 通用性（Generality）
- 支持识别、推理和交互
- 跨多个任务的统一表征
- 语义、空间、时序的理解

#### 2.2 因果性（Causality）
- 仅依赖过去和当前帧
- 支持实时在线处理
- 满足流式应用的延迟要求

#### 2.3 结构化（Structured）
- 捕捉外观、几何和运动
- 显式的3D空间理解
- 物理合理的场景表示

### 3. 现有统一方法的不足

#### 3.1 基于生成的统一方法

**代表方法**: Florence, OFA, Unified-IO

**核心思想**: 将所有任务转换为类似文本的令牌生成

**优点**:
- 提供统一的接口
- 便于添加新任务

**缺点**:
- 统一主要发生在输出层面
- 添加新任务通常需要昂贵的重新训练
- 需要重新对输出进行分词
- 可能需要调整生成头架构

#### 3.2 骨干网络统一方法

**核心问题**: 能否学习一个统一的流式视觉骨干网络，其表征足够通用，使得许多下游任务可以在其上解决，而无需修改或微调骨干网络？

**挑战**:
- 如何在统一框架中同时编码语义、时序和几何信息？
- 如何保持因果性以支持实时推理？
- 如何在不同任务间实现有效的知识共享？

### 4. 语言领域的启示

在语言社区，通过下一个令牌预测训练的单个自回归骨干网络已经实现了通用性（GPT-3, GPT-4）。这成功不仅源于规模，更是因为多样化的任务可以共享公共表征和训练接口。

**关键洞察**: 视觉领域的统一不应仅仅是接口层面的，更应该是表征层面的。我们需要一个真正通用的视觉骨干网络，其内部表征本身就蕴含了处理多种任务所需的知识。

### 5. OmniStream的动机

基于以上分析，OmniStream 旨在：

1. **统一语义、动态和几何理解**：打破当前视觉模型碎片化的现状
2. **支持实时流式推理**：满足实际应用中的延迟和内存约束
3. **零样本跨任务迁移**：在冻结骨干网络的情况下支持多样化下游任务
4. **迈向通用视觉理解**：为交互式和具身智能体奠定基础

## 核心创新点

### 1. 因果时空注意力（Causal Spatiotemporal Attention）

#### 1.1 问题陈述

标准的Transformer注意力机制会计算所有令牌之间的交互，这对于视频流存在两个严重问题：

1. **违反因果性**：当前帧不应该"看到"未来帧
2. **计算低效**：每次都需要重新计算历史帧的注意力

#### 1.2 解决方案：因果掩码

OmniStream 引入了严格的因果时序掩码：

**数学形式化**：
```
M_{u,v} = {
    0,           if τ(u) ≥ τ(v)
    -∞,          if τ(u) < τ(v)
}
```

其中：
- `u` 是查询令牌的全局索引
- `v` 是键令牌的全局索引
- `τ(·)` 返回令牌的帧索引

**效果**：
- 帧 `t` 中的每个补丁只能关注帧 `≤ t` 中的补丁
- 永远无法关注未来帧
- 保证严格的时序因果性

#### 1.3 流式推理与KV-Cache

**核心优势**：利用因果性，可以实现高效的增量推理

**推理过程**：
```
当第 t 帧到达时：
1. 计算第 t 帧的查询 (Q)
2. 重用缓存的来自帧 ≤ t-1 的键/值 (K/V)
3. 执行注意力计算
4. 缓存第 t 帧的 K/V 供未来使用
```

**计算复杂度**：
- 传统方法：每步 O(T²)
- KV-Cache方法：每步 O(T)
- 显著降低实时推理的计算开销

**内存效率**：
- KV-Cache 机制避免了重复计算
- 内存增长保持线性而非二次
- 支持处理更长的序列

### 2. 3D旋转位置编码（3D-RoPE）

#### 2.1 背景：2D RoPE

原始 DINOv3 使用 2D RoPE 编码相对空间偏移：
- 在高度和宽度维度上编码空间关系
- 适合单帧图像的 spatial reasoning

#### 2.2 扩展到时空域

OmniStream 将 RoPE 扩展到时空域：

**维度分配策略**：
- 采用 2:3:3 分配比例 (时间:高度:宽度)
- 重新利用每头特征维度 `d_head`
- 尽可能保留预训练的空间结构

**具体实现**：
```
索引分布：
- i ≡ 0 (mod 4): 编码高度 y
- i ≡ 1 (mod 4): 编码宽度 x
- i ≡ 2 (mod 4): 编码高度 y
- i ≡ 3 (mod 4): 编码时间 t
```

**创新点**：
- 时间组件交织到原始 2D RoPE 中
- 遵循 DINOv3 的模式编码 y 和 x
- 每4个维度中包含一个时间维度

#### 2.3 RoPE-Box Jittering

进一步应用 RoPE-box jittering（DINOv3 技术）：
- 提高位置编码的鲁棒性
- 增强对输入变化的适应性
- 防止过拟合到特定位置模式

#### 2.4 3D-RoPE的优势

1. **时空统一编码**：
   - 显式建模"何时-何地"关系
   - 支持跨长距离的时空推理

2. **泛化能力**：
   - 能够泛化到训练中未见过的序列长度
   - 展现出零样本长度外推能力
   - 实验证明可处理长达 110 帧的流（训练窗口仅16帧）

3. **保留空间先验**：
   - 通过维度分配策略保留 DINOv3 的空间先验
   - 避免破坏预训练的 spatial representation

### 3. 统一多任务学习框架

#### 3.1 设计理念

OmniStream 的训练框架基于一个核心洞察：**多样性的学习目标具有协同效应，而非简单叠加**。

三个互补的目标，针对理解的不同维度：

#### 3.2 目标分解

##### (i) 静态与时序表示学习
- **目标**: 学习语义不变性和运动敏感动态
- **方法**: DINOv3风格的学生-教师蒸馏
- **监督信号**:
  - 全局层面：图像/视频级语义一致性
  - 局部层面：补丁级判别性特征
- **因果性约束**: 在因果掩码下执行

##### (ii) 流式几何重建
- **目标**: 注入显式3D约束
- **监督信号**:
  - 深度图 (Depth maps)
  - 射线图 (Ray maps)
  - 相机姿态 (Camera poses)
- **重要性**: 确保表征反映物理场景结构而非仅外观

##### (iii) 视觉-语言对齐
- **目标**: 将视觉令牌与语言概念连接
- **监督信号**:
  - 密集字幕 (Dense captioning)
  - OCR识别
  - 视觉定位 (Visual grounding)
- **作用**: 促进细粒度细节和语义定位，对推理密集型任务至关重要

### 4. 冻结骨干的下游适配

#### 4.1 设计原则

**核心思想**: 基础视觉骨干网络应该是可重用的，无需昂贵的全模型适配。

**实现方式**: 严格冻结设置
- 骨干网络参数完全冻结
- 仅训练任务特定的轻量级模块

#### 4.2 下游任务层次

OmniStream 将下游任务组织为四个复杂度递增的层次：

##### Level 1: 感知（Perception） - 图像与视频

**图像任务**:
- 方法：在冻结特征上附加轻量级头
- 任务示例：
  - ImageNet-1K 分类
  - ADE20K 语义分割
  - NYUv2 单目深度估计

**视频任务**:
- 方法：对冻结的时空特征使用注意力池化
- 任务示例：
  - Kinetics-400 动作识别
  - Something-Something V2 动作识别
  - DAVIS'17 视频目标分割

##### Level 2: 推理（Reasoning） - 视觉-语言模型（VLM）

**方法**:
1. 使用MLP投影器将冻结的视觉令牌映射到语言嵌入空间
2. LLM基于投影的视觉上下文和文本指令生成文本
3. 视觉骨干网络保持冻结

**任务示例**:
- VideoMME：通用视频问答
- VideoMMMU：多模态推理
- PerceptionTest：感知测试
- EgoSchema：第一人称推理
- VSI-Bench：空间智能基准

##### Level 3: 行动（Action） - 视觉-语言-行动模型（VLA）

**方法**:
1. 扩展VLM设置
2. 附加基于MLP的行动专家到LLM输出
3. 从冻结的视觉观察和语言指令预测机器人行动

**任务示例**:
- CALVIN：长视距指令跟随
- Simpler-Env：真实到仿真的泛化

**关键创新**:
- 由于预训练目标（ℒ_ssl, ℒ_geo）显式编码了动态和几何
- 冻结特征提供了从感知到行动的强有力桥梁
- 无需领域特定的视觉微调

## 技术架构详解

### 1. 模型初始化与架构选择

#### 1.1 基础架构：DINOv3 ViT-L

**选择理由**：
1. **强大的空间表征**：DINOv3 在图像理解任务上表现出色
2. **成熟的训练范式**：学生-教师蒸馏框架已经过充分验证
3. **良好的扩展性**：ViT 架构适合扩展到时序域
4. **社区认可度高**：作为视觉基础模型的标准选择

**架构参数**：
- 骨干网络：Vision Transformer Large
- 补丁大小：p × p
- 特征维度：d
- 注意力头数：取决于具体配置
- 层数：标准 ViT-L 深度

#### 1.2 架构修改概览

从图像编码器到流式骨干网络的两个关键修改：

```
DINOv3 (Image Encoder)
    ↓
(1) Causal Spatiotemporal Attention
(2) 3D Rotary Positional Embeddings (3D-RoPE)
    ↓
OmniStream (Streaming Visual Backbone)
```

### 2. 分词与输出设计

#### 2.1 输入分词

对于视频流 `V^T ∈ R^(T×H×W×3)`：

**分词过程**：
1. 将每帧分割为不重叠的 p×p 补丁
2. 每帧产生 h×w 个补丁，其中 h=H/p, w=W/p
3. 线性投影后，为每帧添加特殊令牌

**特殊令牌设计**：
- `[CLS]`：全局语义摘要令牌（每帧1个）
- `[vision_registers]`：4个寄存器令牌（用于细粒度特征）
- `[CAM]`：可选相机令牌（用于姿态估计）

**输入序列**：
```
z^0 ∈ R^(T×(N_s+hw)×d)
```
其中 `N_s = 1(CLS) + 4(registers) + 1(CAM) = 6`

#### 2.2 输出设计

经过多个Transformer块后，提取两类输出：

**2.2.1 密集时空特征图**

```
Z = {z^(ℓ) ∈ R^(T×h×w×d)}_(ℓ∈L)
```
- 从选定的中间层 `L` 提取
- 用于密集预测任务（分割、深度等）
- 保留细粒度的空间和时序信息

**2.2.2 特殊令牌**

```
O = {Z, z_cls, z_cam}
```

- `z_cls ∈ R^(T×d)`：最后一层的[CLS]令牌
  - 提供图像/视频级摘要
  - 用于分类和全局理解

- `z_cam ∈ R^(T×d)`：[CAM]令牌
  - 路由到相机头进行姿态估计
  - 编码相机运动信息

### 3. 因果时空注意力详解

#### 3.1 数学形式化

**掩码注意力公式**：
```
Attn(Q, K, V) = Softmax((QK^T)/√(d_head) + M)V
```

**因果掩码定义**：
```
M_{u,v} = {
    0,           if τ(u) ≥ τ(v)
    -∞,          if τ(u) < τ(v)
}
```

其中：
- `u`：查询令牌的全局索引，范围 [0, N_total-1]
- `v`：键令牌的全局索引，范围 [0, N_total-1]
- `τ(·)`：帧索引映射函数，返回令牌属于哪一帧
- `N_total = T × (N_s + hw)`：总令牌数

#### 3.2 因果性保证

**每帧的注意力模式**：
```
第 t 帧的查询令牌：
- 可以关注帧 1, 2, ..., t 的所有令牌
- 不能关注帧 t+1, ..., T 的任何令牌
```

**可视化示例**（T=3, 每帧2个令牌）：
```
帧1令牌: [✓ ✓ | ✓ ✓ | ✓ ✓]
帧2令牌: [✓ ✓ | ✓ ✓ | ✓ ✓]
帧3令牌: [✓ ✓ | ✓ ✓ | ✓ ✓]
         帧1    帧2    帧3
         ✓: 可关注
```

#### 3.3 流式推理算法

```
输入：历史KV-Cache (K_cache, V_cache)
      新帧 I_t

过程：
1. 对 I_t 进行分词：z_t
2. 计算查询：Q_t = z_t W_Q
3. 计算键和值：K_t = z_t W_K, V_t = z_t W_V
4. 合并键值对：
   K_combined = [K_cache; K_t]
   V_combined = [V_cache; V_t]
5. 执行因果注意力：
   Attn_t = CausalAttention(Q_t, K_combined, V_combined)
6. 更新缓存：
   K_cache ← [K_cache; K_t]
   V_cache ← [V_cache; V_t]

输出：Attn_t
```

**复杂度分析**：
- 传统重新计算：O(T²) 每步
- KV-Cache方法：O(T) 每步
- 内存增长：线性 O(T)

#### 3.4 实际性能优势

根据论文附录的计算分析（表9）：

**对比设置**：
- 设备：单张 NVIDIA H800 GPU
- 批大小：1
- 输入分辨率：224×224

**性能对比**：

| 上下文长度 T | 完全重计算 | OmniStream (KV-Cache) |
|-------------|------------|------------------------|
| 16          | ~0.1s      | ~0.02s                 |
| 32          | ~0.2s      | ~0.03s                 |
| 64          | ~0.998s    | ~0.067s (15× faster)  |
| 128         | OOM        | ~0.12s                 |
| 256         | OOM        | ~0.23s                 |
| 512         | OOM        | ~0.45s                 |

**关键观察**：
1. 完全重计算在 T=128 时发生内存溢出（OOM）
2. OmniStream 保持线性内存增长
3. 在 T=64 时速度提升约15倍
4. 支持处理长达512帧的序列

### 4. 3D旋转位置编码详解

#### 4.1 RoPE基础回顾

**旋转位置编码（Rotary Positional Embedding）**：
- 将绝对位置信息编码为旋转矩阵
- 在注意力计算中隐式建模相对位置
- 避免位置嵌入的外推问题

**2D RoPE（DINOv3）**：
- 在高度（y）和宽度（x）维度上编码空间关系
- 维度分配策略保留空间先验

#### 4.2 3D-RoPE设计

**维度分配策略**：
```
d_head 维度分配 (2:3:3 比例):
- 时间 (t): 2/8 = 1/4 维度
- 高度 (y): 3/8 维度
- 宽度 (x): 3/8 维度
```

**具体索引映射**：
```
对于特征维度索引 i ∈ [0, d_head-1]:

if i % 4 == 0:  编码高度 y
if i % 4 == 1:  编码宽度 x
if i % 4 == 2:  编码高度 y
if i % 4 == 3:  编码时间 t
```

**示例**（d_head=8）：
```
维度 0: y (高度)
维度 1: x (宽度)
维度 2: y (高度)
维度 3: t (时间)
维度 4: y (高度)
维度 5: x (宽度)
维度 6: y (高度)
维度 7: t (时间)
```

#### 4.3 旋转计算

**对于查询令牌 q**：
```
q_rotated = q ⊙ (cos(θ_t,θ_y,θ_x)) + (q_shift) ⊙ sin(θ_t,θ_y,θ_x)
```

其中：
- `θ_t`: 时间维度旋转角度
- `θ_y`: 高度维度旋转角度
- `θ_x`: 宽度维度旋转角度
- `⊙`: 逐元素乘法
- `q_shift`: q 的位移版本

**角度计算**：
```
θ_t = 2π × (t / max_t) × (dim_indices / dim_freq)
θ_y = 2π × (y / max_y) × (dim_indices / dim_freq)
θ_x = 2π × (x / max_x) × (dim_indices / dim_freq)
```

#### 4.4 3D-RoPE的优势

**优势1：时空统一建模**
- 显式编码"何时-何地"关系
- 支持跨时空距离的推理
- 统一空间和时间建模

**优势2：长度外推能力**
- 依赖相对位置而非绝对索引
- 可泛化到训练时未见过的序列长度
- 零样本长度外推

**优势3：保留空间先验**
- 通过精心设计的维度分配保留 DINOv3 空间先验
- 避免破坏预训练的 spatial representation
- 平滑的时序扩展

#### 4.5 RoPE-Box Jittering

**目的**：提高位置编码的鲁棒性

**方法**：
- 在训练时对补丁位置应用随机扰动
- 模拟真实世界中的位置不确定性
- 防止模型过拟合到精确位置

**效果**：
- 增强对输入变化的适应性
- 提高泛化性能
- 与 DINOv3 训练保持一致

## 多任务训练框架

### 1. 总体训练目标

OmniStream 采用加权多任务学习：

```
ℒ_total = λ_ssl · ℒ_ssl + λ_geo · ℒ_geo + λ_cap · ℒ_cap
```

**权重设置**：
- `λ_ssl = 0.1`
- `λ_geo = 1.0`
- `λ_cap = 1.0`

权重设计平衡了不同目标量级的差异。

### 2. 静态与时序表示学习（ℒ_ssl）

#### 2.1 设计理念

**核心思想**：将图像视为退化的流（T=1），实现图像和视频的统一目标。

#### 2.2 DINOv3风格学生-教师蒸馏

**总目标**：
```
ℒ_ssl = ℒ_DINO + ℒ_iBOT + 0.1×ℒ_KoLeo + ℒ_gram
```

##### 2.2.1 DINO损失（ℒ_DINO）

**目标**：捕捉全局语义信息

**实现**：
1. 从一组视图（图像或视频的全局/局部裁剪）中
2. 通过时间均值池化聚合类令牌序列：
   ```
   z̄_cls = (1/T) × Σ_{t=1}^T z_cls,t
   ```
3. 通过学生和教师的 DINO 头投影到"原型分数"
4. 应用softmax得到 p_s 和 p_t
5. 最小化交叉熵：
   ```
   ℒ_DINO = -Σ p_t log(p_s)
   ```

**特点**：
- 鼓励全局语义一致性
- 对比学习框架
- 动量更新教师

##### 2.2.2 iBOT损失（ℒ_iBOT）

**目标**：学习补丁级判别性特征

**实现**：
1. 对学生应用的图像和视频局部补丁进行掩码
2. 不同的掩码策略：
   - 图像补丁：随机掩码（DINO协议）
   - 视频补丁：多块协议（V-JEPA风格）
3. 将 iBOT 头应用于学生的掩码补丁
4. 将 iBOT 头应用于教师的对应补丁
5. 应用softmax和居中
6. 计算交叉熵：
   ```
   ℒ_iBOT = -Σ_{i=1}^{N_mask} p_ti log(p_si)
   ```

**特点**：
- 掩码图像建模（MIM）
- 学习细粒度局部特征
- 区分于DINO的全局焦点

##### 2.2.3 KoLeo正则化（ℒ_KoLeo）

**目标**：鼓励批次内特征均匀分布

**实现**：
给定一批向量 `(x_1, ..., x_n)`：

```
ℒ_KoLeo = -(1/n) × Σ_{i=1}^n log(ρ_{n,i})
```

其中 `ρ_{n,i} = min_{j≠i} ||x_i - x_j||` 是 `x_i` 到批次中其他任何点的最小距离。

**特点**：
- 防止特征塌陷
- 提高表征多样性
- 促进更好的泛化

##### 2.2.4 Gram锚定（ℒ_gram）

**目标**：保持训练期间一致的补丁级特征

**实现**：
1. 定义gram矩阵为当前图像中所有成对点积的矩阵
2. 在训练开始时，选择预训练的 DINOv3 作为 gram 教师
3. 给定 h×w 补丁的图像序列：
   - `X_s ∈ R^(hw×d)`：学生的L2归一化局部补丁特征
   - `X_g ∈ R^(hw×d)`：gram 教师的局部补丁特征
4. 定义损失：
   ```
   ℒ_gram = ||X_s · X_s^T - X_g · X_g^T||_F^2
   ```

**特点**：
- 保留预训练知识
- 仅对全局裁剪启用
- Frobenius范数约束

#### 2.3 因果性约束

所有SSL目标都在因果掩码下执行：
- 确保表示学习尊重时序因果性
- 仅使用过去和当前帧
- 为流式应用做准备

### 3. 流式几何重建（ℒ_geo）

#### 3.1 设计理念

**核心思想**：注入显式3D约束，使表征反映物理场景结构。

#### 3.2 几何头架构

##### 3.2.1 深度头（Depth Head）

**架构**：双DPT模块（DepthAnything-3风格）

**输入**：选定的骨干特征 `{z^(ℓ) ∈ R^(T×h×w×d)}_(ℓ∈L)`

**输出**：
- `D̂ ∈ R^(T×H×W×1)`：深度图
- `R̂ ∈ R^(T×H×W×6)`：射线图

**特点**：
- 密集预测
- 逐帧深度估计
- 包含相机射线信息

##### 3.2.2 相机头（Camera Head）

**架构**：轻量级MLP

**输入**：相机令牌 `z_cam ∈ R^(T×d)`

**输出**：`ĝ ∈ R^(T×9)` 每帧姿态参数
- 旋转四元数 `q ∈ R^(T×4)`
- 平移向量 `t ∈ R^(T×3)`
- 视场角 `f ∈ R^(T×2)`

**假设**：相机主点位于图像中心

#### 3.3 几何损失函数

**总几何损失**：
```
ℒ_geo = ℒ_depth + ℒ_ray + ℒ_points + ℒ_camera
```

##### 3.3.1 深度损失（ℒ_depth）

**带置信度的L1回归**：
```
ℒ_depth = Σ_{t=1}^T (||c_t(D̂_t - D_t)||_1 + ||c_t(∇D̂_t - ∇D_t)||_1 - α log c_t)
```

其中：
- `∇`：空间梯度算子
- `c_t`：学习到的置信度
- `α`：超参数

**特点**：
- L1回归损失
- 额外的梯度项（鼓励平滑）
- 学习置信度权重

##### 3.3.2 射线损失（ℒ_ray）

**标准L1回归**：
```
ℒ_ray = Σ_{t=1}^T ||R̂_t - R_t||_1
```

相机射线定义为射线原点 `o ∈ R^3` 和方向 `d ∈ R^3` 的连接。

##### 3.3.3 点损失（ℒ_points）

**标准L1回归**：
```
ℒ_points = Σ_{t=1}^T ||P̂_t - P_t||_1
```

点图可推导为：
```
P̂_t = ô_t + D̂_t ⊙ d̂_t
```

##### 3.3.4 相机损失（ℒ_camera）

**标准L1回归**：
```
ℒ_camera = Σ_{t=1}^T ||ĝ_t - g_t||_1
```

监督归一化的相机姿态。

#### 3.4 几何数据集

使用多样化的3D/4D场景数据集（约18M帧）：
- Co3Dv2, WildRGBD, BlendedMVS
- MegaDepth, MVS-Synth, PointOdyssey
- HyperSim, TartanAir, DL3DV
- ScanNet, ScanNet++, ArkitScenes
- Virtual Kitti 2, MapFree, Dynamic Replica
- Spring, OmniWorld-Games, Aria Synthetic Environments, Waymo-Open

### 4. 视觉-语言对齐（ℒ_cap）

#### 4.1 设计理念

**核心思想**：将视觉令牌与语言概念连接，支持推理密集的下游任务。

#### 4.2 架构设计

**组件**：
1. MLP投影器：映射视觉令牌到语言嵌入空间
2. 自回归语言解码器：Qwen3-0.6B
3. 训练任务：混合的视觉-语言任务

**视觉令牌**：来自最后一层的 `z^L`

**指令提示**：`x_inst`

#### 4.3 标准语言建模目标

```
ℒ_cap = -Σ_{n=1}^{L_text} log P_text(y_n | z^L, x_inst, y_{<n})
```

其中：
- `P_text`：解码器预测的令牌分布
- `y_n`：目标令牌
- `y_{<n}`：先前生成的令牌

#### 4.4 训练任务

混合视觉-语言任务（约50M帧）：

##### 4.4.1 密集字幕
- 生成详细的图像/视频描述
- 捕捉细粒度视觉细节

##### 4.4.2 OCR识别
- 识别图像中的文本
- 增强细粒度语义理解

##### 4.4.3 视觉定位
- 定位图像中的物体
- 促进细粒度空间推理

#### 4.5 梯度流

**关键创新**：
- 梯度反向传播通过语言解码器进入视觉骨干
- 注入细粒度语义和空间监督
- 证明对VLM和VLA集成至关重要

**效果**：
- 早期视觉-语言对齐防止VLM集成中的灾难性失败
- 增强下游推理能力
- 消除对辅助几何模块的需求

- 运动密集型任务（SSv2）受影响最大

## 局限性分析

### 1. 模型规模限制

#### 1.1 当前模型大小

**现状**：
- 基于 DINOv3 ViT-L（约400M参数）
- 远小于当前最大视觉模型（如DINOv2-G, 1.1B参数）

**局限性**：
- 未充分探索规模效应
- 可能不是性能最优配置
- 与最先进模型（SOTA）的可比性受限

**证据**：
- ImageNet: 84.7% vs DINOv3: 86.7% (-2.0%)
- 某些任务略落后于专门模型

**影响**：
- 不清楚更大模型的性能潜力
- 规模与泛化的关系未充分探索
- 可能低估了统一方法的潜力

#### 1.2 未探索的规模

**可能的改进方向**：
- 扩展到 ViT-H, ViT-G
- 探索不同的架构规模
- 研究规模与性能的缩放定律

**潜在收益**：
- 可能在所有任务上达到SOTA
- 进一步提升泛化能力
- 增强长视距推理

### 2. 训练数据限制

#### 2.1 数据规模

**现状**：
- 200M帧，29个数据集
- 虽然多样，但绝对规模有限

**局限性**：
- 远小于当前最大预训练（如CLIP, 400M+图像）
- 某些数据集可能过小
- 长序列数据相对不足

**影响**：
- 可能未充分利用规模优势
- 长视距推理能力受限
- 泛化到新场景的能力可能受限

#### 2.2 数据多样性

**当前数据组成**：
- 图像：~113M帧
- 视频：~20M帧
- 3D/4D：~18M帧
- 字幕：~50M帧

**局限性**：
- 视频数据相对较少（仅10%）
- 长序列数据不足（固定T=16）
- 真实世界数据有限（大部分是仿真/合成）

**影响**：
- 动态建模可能未充分训练
- 长视距推理能力受限
- 真实世界泛化未知

#### 2.3 数据质量标注

**问题**：
- 不同数据集的标注质量差异大
- 几何标注可能存在噪声
- 字幕质量不一致

**影响**：
- 可能引入学习偏差
- 几何重建精度受限
- 语言对齐可能不完美

### 3. 架构限制

#### 3.1 Transformer的固有局限

**计算复杂度**：
- 虽然KV-Cache降低到O(T)，但仍有T的线性增长
- 对于极长序列（T>1000），可能仍不足

**内存需求**：
- KV-Cache需要存储历史键值对
- 内存需求随T线性增长
- 在资源受限环境中可能受限

**注意力机制**：
- 标准注意力可能不是最优的时序建模
- 可能受益于更高效的架构（如Mamba, State Space Models）

#### 3.2 固定窗口限制

**训练设置**：
- 固定序列长度 T=16
- 虽然可以外推到110帧，但性能可能下降

**局限性**：
- 长视距依赖建模不足
- 长序列记忆能力受限
- 可能不适合需要长期记忆的任务

**证据**：
- 零样本外推到110帧，但未评估性能退化
- 没有训练时更长的序列

#### 3.3 缺乏记忆机制

**当前设计**：
- 仅依赖注意力机制的历史
- 没有显式的记忆模块

**局限性**：
- 长视距记忆能力有限
- 难以记忆远期信息
- 不适合需要长期规划的任务

**可能改进**：
- 添加显式记忆机制
- 探索分层记忆结构
- 研究长视距记忆策略
