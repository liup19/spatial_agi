## 摘要

Latent diffusion models (LDMs)通过在学习的latent空间中操作实现了高保真合成。然而，训练最先进的LDMs需要复杂的阶段：首先需要训练一个tokenizer，然后才能在冻结的latent空间中训练扩散模型。我们提出了UNITE——一个用于统一tokenization和latent扩散的自编码器架构。UNITE由一个Generative Encoder组成，通过权重共享同时作为图像tokenizer和latent生成器。我们的关键洞察是：tokenization和generation可以看作是在不同的条件机制下同一个latent推断问题：tokenization从完全观测的图像推断latents，而generation从噪声与文本或类条件一起推断它们。受此启发，我们引入了单阶段训练程序，通过同一Generative Encoder的两次前向传播联合优化两个任务。共享参数使梯度能够共同塑造latent空间，鼓励形成"通用的latent语言"。跨图像和分子模态，UNITE在不使用对抗损失或任何预训练编码器（如DINO）的情况下达到了接近最先进的性能，在ImageNet 256×256上Base和Large模型分别达到FID 2.12和1.73。我们进一步通过表示对齐和压缩的视角分析了Generative Encoder。这些结果表明，从头开始单阶段联合训练tokenization和generation是可行的。

## 核心贡献

1. **统一的架构设计**：提出了Generative Encoder，通过参数共享统一了tokenization和generation两个任务
2. **单阶段训练**：证明了可以在单次训练中同时优化重建和生成目标，无需分阶段训练
3. **端到端学习**：首次实现了完全从零开始（无预训练encoder、无对抗损失）的latent扩散模型
4. **新的理论视角**：将tokenization和generation重新定义为同一latent推断问题在不同条件下的实例
5. **实证验证**：在ImageNet和QM9分子数据集上达到near-SOTA性能

### Q1: 核心算法原理

**问题**: 这篇文章的核心算法原理是什么？请详细描述：1) 核心思想和动机，2) 主要技术方法，3) 算法流程和关键步骤，4) 输入输出。

**答案**:

#### 1. 核心思想和动机

**传统方法的问题**：
- 现有的Latent Diffusion Models (LDMs)训练需要分两个阶段：
  1. 首先训练一个VAE/VQ-GAN风格的tokenizer（encoder-decoder）
  2. 冻结tokenizer，在latent space训练扩散模型
- 这种分离训练违背了端到端学习的原则
- tokenizer的latent空间没有受到生成目标的优化，可能不是最适合生成的表示

**核心洞察**：
- Tokenization和generation本质上是同一个latent推断问题，只是在不同的条件设置下：
  - **Tokenization**: 从完全观测的图像x推断latent z（强观测条件）
    - 这可以看作是在强观测性下的生成过程：给定数据点x，模型在latents上产生高度集中（接近单点）的分布
  - **Generation**: 从噪声和条件推断latent z（弱观测条件）
    - 这对应弱观测性状态：z必须使用学习的先验从噪声中合成
- 两种操作的主要区别在于可用的信息量：从tokenization的完全观测x到generation的只有先验

**动机**：
- 如果tokenization和generation能同时优化，每个目标可以塑造学习的latent空间
- 生成压力可以将latent空间雕刻成更容易建模的区域
- 重建和推断压力可以保留实例特定信息和语义结构
- 目标是理解当这些目标一起训练时会出现什么——它们的交互是帮助还是伤害

#### 2. 主要技术方法

**UNITE架构组成**：

UNITE系统只包含两个模块：

1. **Generative Encoder (GEθ)**
   - 一个共享参数的Vision Transformer (ViT)网络
   - 以两种模式操作：
     - **Tokenization模式**: 将输入图像x映射到latent tokens z = GEθ(x)
     - **Generation模式**: 去噪corrupted latents产生 ẑ0 = GEθ(zt, t)
   - 相同的网络同时作为tokenizer和多步latent去噪器
   - 参数θ在两个目标间共享

2. **Decoder (Dψ)**
   - 将latents映射回图像空间的标准decoder
   - 用于重建图像

**Latent表示机制**：
- 使用固定的K个register tokens表示latent z
- 这避免了固定grid的限制，提供了更灵活的latent表示
- Register tokens在tokenization和generation中以不同方式初始化

#### 3. 算法流程和关键步骤

**Tokenization路径**（编码）：
1. **输入准备**：
   - 将图像分成patches（ViT的标准操作）
   - 创建K个register tokens，初始化为独立同分布的Gaussian噪声 𝒩(0, I)
   - 将image patch tokens与registers连接成一个序列

2. **前向传播**：
   - 通过self-attention处理连接的序列（GEθ的第一次前向传播）
   - 丢弃patch tokens，只保留更新后的registers
   - 这些更新的registers吸收了来自patches的相关信息

3. **输出**：
   - 更新的registers作为图像latents z0
   - z0传递给decoder重建图像

**Generation路径**（去噪）：
1. **Latent corruption**：
   - 使用rectified flow/flow-matching过程corrupt clean latents z0
   - 在噪声水平t构建noisy latents: zt = t*z0 + (1-t)*ε
   - 其中ε ~ 𝒩(0, I)，t ~ 𝒰[0, 1]

2. **Register初始化**：
   - 用zt初始化相同的K个registers
   - 这次不连接任何image patches

3. **去噪前向传播**：
   - 通过GEθ（现在是generation模式）处理registers
   - 条件化在噪声水平t和可选的类别信息
   - GEθ预测去噪目标

4. **输出**：
   - 预测的clean latents ẑ0 = GEθ(zt, t)
   - 去噪器输出与tokenization输出在同一空间

**关键设计选择**：

1. **Stop-gradient机制**：
   - 在构建用于去噪的zt之前，从tokenization输出z0停止梯度
   - 这防止degenerate solutions，即去噪器目标不使latent空间坍塌
   - 使用sg(·)表示stop-gradient操作

2. **归一化策略**：
   - GEθ的最终层是一个归一化模块
   - 实证发现，具有可学习scale和shift参数的LayerNorm效果最好
   - 这使得clean latents z0、去噪预测ẑ0和推理时的模型输出都归一化

3. **重建噪声**：
   - 在解码之前给z0注入小的Gaussian噪声 z̃0 = z0 + σε
   - 重建噪声尺度σ = 0.7
   - 这提高了decoder的鲁棒性

#### 4. 输入输出

**Tokenization模式输入输出**：
- **输入**: 图像x（patch tokens + K个registers）
- **输出**: Latent表示 z0（更新后的registers）
- **最终输出**: 重建图像 x̂ = Dψ(z̃0)

**Generation模式输入输出**：
- **输入**: Noisy latent zt + 噪声水平t + 可选条件（类别标签、文本）
- **输出**: 去噪latent预测 ẑ0
- **最终输出**: 生成图像 x̂ = Dψ(ẑ0)

**训练输入**：
每个训练迭代需要：
- 一个批次的数据图像
- 采样噪声水平t
- 采样噪声ε

**推理输入**：
- Tokenization推理: 单个图像x
- Generation推理: 类别标签/文本 + 初始噪声latents

**训练目标函数**：

1. **Reconstruction Loss**：
   ```
   ℒrecon = ||x̂ - x||1 + LPIPS(x̂, x)
   ```
   - L1 loss + 感知损失（LPIPS使用预训练VGG）
   - 确保重建质量

2. **Flow Matching Loss**：
   ```
   ℒflow = 𝔼t,ε[||ẑ0 - sg(z0)||²]
   ```
   - Rectified flow匹配
   - sg(z0)表示stop-gradient
   - 优化去噪器预测clean latent

3. **Total Loss**：
   ```
   ℒ = ℒrecon + ℒflow
   ```
   - 联合优化两个目标

**训练动力学**：

- 每个训练迭代进行两次前向传播：
  1. 图像条件前向：产生重建的clean latents
  2. 仅latent前向：去噪corrupted版本的latents
- 系统通过联合优化像素空间重建目标（通过Dψ）和latent空间flow-matching目标（通过GEθ）单阶段端到端训练
- 重建和生成目标的联合训练表现出"adversarial"行为：
  - 两个损失不一定单调一起下降
  - 生成保真度的改进甚至可能与去噪损失的增加同时发生
  - 目标不是将所有损失驱动到零，而是达到stable training dynamics

**推理过程**：

- Tokenization推理：
  - 单次前向传播将输入图像映射到其latent表示
  - 直接可用作编码器

- Generation推理：
  - 从类别标签和noisy latent registers开始
  - 通过GE的多次前向传播迭代细化它们成clean、可解码的latents
  - 使用标准ODE solver进行rectified flow

### Q2: 与Spatial AGI的关系

**问题**: 这篇文章与通用空间智能（Spatial AGI）有什么关系？请分析：1) 如何理解和表示空间，2) 如何处理空间关系，3) 对Spatial AGI有什么启发，4) 可以应用到哪些Spatial AGI场景（机器人、AR/VR等）。

**答案**:

#### 1. 如何理解和表示空间

**Latent空间的本质**：
- UNITE将高维视觉数据（图像）压缩到低维latent空间
- 这是一种对空间信息的抽象表示，捕捉了原始像素的semantically meaningful structure
- Latent既保留了spatial details（通过reconstruction loss），又具备generative smoothness（通过flow matching）

**Register Tokens作为空间表示单元**：
- 不使用固定grid，而是用K个可学习的register tokens表示latent
- 这些tokens通过attention机制可以灵活地表示不同粒度的spatial information
- 类似于人类spatial cognition中的"attentional landmarks"

**ViT架构对空间结构的保留**：
- 使用Vision Transformer处理图像patches
- Patch-level attention保留了local spatial structure
- 深层attention捕获global spatial configuration
- 相比CNN，ViT的attention提供更灵活的spatial reasoning能力

**多尺度空间理解**：
- ViT的层次结构自然支持多尺度spatial representation
- 浅层：fine-grained local spatial features
- 深层：coarse global spatial semantics
- 类似人类spatial perception从细节到整体的处理

#### 2. 如何处理空间关系

**Attention机制的空间建模**：
- Self-attention允许每个patch与其他所有patch交互
- 能够建模复杂的spatial关系：距离、遮挡、纹理关联等
- 通过attention weights可以显式表示不同spatial区域的重要性

**迭代去噪的空间一致性**：
- Rectified flow从噪声逐步构建latent
- 每个去噪步骤都受到之前步骤的约束，确保spatial consistency
- 类似人类spatial imagination：从粗略轮廓到细节完善

**Reconstruction与Generative的平衡**：
- Reconstruction pressure：保留spatial details，确保每个像素的准确性
- Generative pressure：使表示对spatial variations鲁棒，确保泛化能力
- Joint training找到平衡点：既rich又robust的spatial representation

**Conditional Generation的空间控制**：
- 通过类别标签、文本等条件控制生成的spatial content
- Latent space支持条件化：p(z|condition)
- 允许精确控制生成的spatial attributes（物体类别、布局、风格）

#### 3. 对Spatial AGI的启发

**a) 统一表示学习的范式**

**核心启示**：
- UNITE展示了如何将perception（tokenization）和generation统一到一个框架
- 这对Spatial AGI至关重要：智能体需要同时具备理解和模拟能力
- 共享的latent表示可以作为spatial reasoning的统一接口

**理论意义**：
- Perception和generation不是独立的模块，而是同一系统在不同操作模式下的表现
- 这符合人类spatial intelligence：我们能看，也能在心中"看"
- Latent space成为spatial cognition的"工作记忆"

**实际优势**：
- 单一模型完成两个任务，减少了参数和计算开销
- 统一的representation确保perception和generation的一致性
- 避免了perception-generate pipeline中的表示mismatch

**b) 端到端学习的价值**

**从零学习的力量**：
- 不依赖DINO等预训练visual encoder
- 这对新兴的spatial domain特别有价值
- 机器人、AR/VR等领域的spatial data往往缺乏大规模预训练模型

**快速适应新环境**：
- 单阶段训练意味着可以online更新整个系统
- 新的spatial observation可以直接更新latent space
- 无需重新训练整个pipeline

**简化部署**：
- 单个模型而非多个模块的cascade
- 减少了系统复杂度和调试难度
- 对边缘设备部署更友好

**c) Robust Latent空间的启示**

**Dual Optimization的理解**：
- Joint training产生的latent既dense（保持细节）又robust（对抗噪声）
- 这对应Spatial AGI中perception的不确定性：
  - 现实世界的spatial observation总是noisy、不完整的
  - 需要在这种条件下保持稳定的spatial understanding

**Adversarial Dynamics的洞察**：
- 重建和生成损失的trade-off不是零和游戏
- 目标是找到stable equilibrium而非最小化单个损失
- 这对多目标spatial任务（识别+规划+控制）有启发

**Generalization vs Specificity的平衡**：
- Latent space需要在individual-specific information和generic structure间平衡
- 类似人类spatial memory：既记住具体场景，又抽象出通用规律
- UNITE通过loss weighting实现这种平衡

**d) Attention-Based Spatial Reasoning**

**Implicit Spatial Reasoning**：
- Attention机制隐式地执行spatial reasoning
- 每个token可以attend到任何位置，灵活地建模spatial dependencies
- 避免了CNN的fixed receptive field限制

**Explainability潜力**：
- Attention weights可以可视化为spatial saliency maps
- 为spatial decisions提供interpretability
- 对human-AI协作和调试很重要

**Compositionality**：
- Register tokens可以看作compositional units
- 通过attention组合这些单元可以表示复杂的spatial configurations
- 类似语言的compositional semantics扩展到spatial domain

#### 4. 可以应用到哪些Spatial AGI场景

**a) 机器人视觉与操作**

**场景理解**：
- 机器人观测场景 → tokenization产生spatial latent
- Latent捕捉物体位置、关系、可操作性
- 用于任务规划：哪些物体可以抓取、路径是否可行

**操作规划**：
- 在latent space中search可能的action sequences
- 比在pixel space更efficient
- Generation产生预期的spatial outcome（如：物体移动后的场景）

**Visual Servoing**：
- 实时perception: 将摄像头图像编码为latent
- Compare target latent和current latent
- Generation预测所需的action来减小差异

**示例工作流**：
```
1. 机器人观察场景 → GEθ编码为spatial latent z
2. Task目标（如"抓取杯子"） → 映射到target latent z_target
3. 在latent space规划: 如何从z到z_target
4. 执行action → 重新perceive → 更新latent
5. 闭环直到达到z_target
```

**b) AR/VR环境生成**

**沉浸式内容创建**：
- 用户spatial input（手势、注视）编码为latent
- Latent驱动环境响应和内容生成
- 低延迟（因为latent space小）

**场景Completion**：
- 部分扫描的3D环境 → tokenization产生latent
- Generation补全缺失的spatial regions
- 支持稀疏输入的AR体验

**Multi-User Interaction**：
- 每个用户的spatial action编码为latent updates
- 联合latent支持multi-user collaboration
- Consistent的shared spatial environment

**应用示例**：
- **AR导航**：从稀疏深度数据补全室内场景
- **VR社交**：多人共享的空间由latent驱动，确保一致性
- **工业设计**：设计师的spatial意图通过latent实时渲染为3D模型

**c) 3D场景理解与重建**

**多模态空间输入**：
- 统一图像、深度图、点云到同一latent space
- Tokenization处理不同modality的spatial data
- 产生coherent的spatial understanding

**Scene Completion**：
- 不完整的3D扫描 → latent
- Generation补全缺失的几何结构
- Inpainting holes in point clouds或voxel grids

**View Synthesis**：
- 从单视图编码latent
- Generation产生其他视角的渲染
- 类似NeRF，但基于learned latent而非explicit NeRF

**潜在扩展**：
- **Point Cloud Processing**: 将点云序列化为register tokens
- **Mesh Generation**: Latent直接解码为triangle meshes
- **Scene Graph Inference**: 从latent提取spatial relationships graph

**d) Sim-to-Real Transfer**

**Simulated Environment Learning**：
- 在simulation中学习spatial representations
- UNITE的joint training确保latent对生成和重建都有效
- 通过统一的latent language transfer到real world

**Domain Adaptation**：
- Real和sim数据都编码到同一latent space
- 在latent space进行domain alignment
- 减少sim-to-real gap

**Continual Learning**：
- 新的spatial observation不断更新latent space
- 无需从头重新训练整个pipeline
- Online adaptation到新环境（不同光照、纹理）

**e) Autonomous Navigation**

**Spatial Memory**：
- 构建环境的latent map
- Navigation时不断更新spatial latent
- 类似人类cognitive map，但更continuous和expressive

**Path Planning in Latent Space**：
- 在latent space中进行path planning
- 考虑obstacles、passability等spatial constraints
- Generation预测路径上的view

**Loop Closure**：
- 比较当前latent和memory中的latents
- 如果相似度超过阈值，检测到loop closure
- 类似SLAM，但在latent space而非feature space

**f) Object Manipulation and Assembly**

**Object-Level Latents**：
- 每个物体有自己的latent representation
- Assembly任务通过组合multiple object latents
- Generation预测assembly结果

**Grasp Prediction**：
- 物体latent编码geometry和texture
- 在latent space搜索最佳grasp configuration
- Generation可视化grasp feasibility

**Disassembly Analysis**：
- Tokenization编码assembly的当前状态
- 在latent space推理disassembly sequence
- Generation预测每步的intermediate state

**g) Human-Robot Interaction**

**Gesture Understanding**：
- 人类手势编码为spatial latent
- Latent映射到robot action
- Generation预测human intent

**Shared Spatial Mental Model**：
- Human和robot共享latent space概念
- Communication通过latent进行，更efficient
- Generation用于clarification（"你的意思是这个场景吗？"）

**Collaborative Manipulation**：
- Multi-agent系统共享latent updates
- Consistent的spatial understanding
- Generation预测other agents的actions

#### 技术挑战与机会

**当前挑战**：

1. **动态场景处理**
   - UNITE目前处理静态图像
   - 扩展到视频需要处理temporal dimension
   - 可能需要将time作为additional conditioning

2. **物理约束**
   - Latent space是否内置physical laws?
   - 当前UNITE通过data implicit学习physics
   - 可能需要explicit physics-informed constraints

3. **因果推理**
   - Spatial AGI需要理解causality
   - Generation不仅是appearance，还有behavior
   - 需要causal models in latent space

4. **可解释性**
   - Latent的语义不明确
   - 难以将latent features映射到human-understandable concepts
   - 需要disentangled representations

5. **计算效率**
   - 两次前向传播增加了训练成本
   - 对real-time applications可能有挑战

**未来机会**：

1. **Multi-Modal Spatial Integration**
   - 扩展到visual + tactile + proprioceptive inputs
   - 统一到同一latent space
   - 更comprehensive的spatial understanding

2. **Hierarchical Spatial Reasoning**
   - 在latent space中构建spatial hierarchy
   - Low-level: local geometry
   - Mid-level: object parts
   - High-level: scene semantics

## 核心技术发现

### 1. 统一Latent Inference视角的提出

**传统思维的局限**：
- 长期以来，AI社区将perception和generation视为独立的任务
- Pipeline是"先学表示（representation learning），再学生成（generative modeling）"
- 这种思维导致复杂的multi-stage训练流程

**UNITE的新视角**：
- Tokenization和generation本质上是同一个问题：latent inference
- 区别仅在于条件（conditioning regime）：
  - Strong conditioning: 完全观测x → 推断z
  - Weak conditioning: 噪声+条件 → 推断z
- 这是一个paradigm shift，从"task-specific"到"task-agnostic representation"

**深层含义**：
- 可能还有其他看似不同的任务，实际上是同一问题的不同conditioning
- 例如：classification和generation都是conditional latent inference
- 为unified architectures打开新思路

### 2. 参数共享的Generative Encoder

**架构创新**：
- 单个网络同时承担编码和去噪两种角色
- 通过不同输入（image patches vs noisy latents）自然区分模式
- 避免了functional redundancy

**实现细节**：
- ViT backbone适合这种统一，因为attention是通用的信息处理机制
- Register tokens作为可学习的latent slots
- 最后的归一化层处理scale差异

**与Separate Encoders对比**：
- 参数量减少：217M vs 分离时更大
- 描述长度（MDL）更短：更compressed的model
- 实验证明：共享参数在rFID/gFID trade-off上更好

### 3. Adversarial Training Dynamics

**意外发现**：
- Reconstruction和generation损失不是单调同步下降的
- 生成质量改进时，去噪损失可能增加
- 训练动力学类似GAN，而非传统优化问题

**理论洞察**：
- 目标不是将所有损失驱动到零
- 而是达到stable equilibrium
- 这个equilibrium下latent空间平衡信息密度和生成鲁棒性

**为什么会出现这种现象**：
- 生成loss下降：latent变得更简单、更predictable
- 但同时：reconstruction quality可能下降（丢失细节）
- 所以重建loss上升，迫使latent保持细节
- 最终达到trade-off的balance point

**对训练实践的启示**：
- 不要early stop因为generation loss不再下降
- 需要监控最终metric（FID、rFID），而非training loss
- Loss weighting需要careful tuning，特别是no-stop-grad设置

### 4. Stop-Gradient机制的关键作用

**作用机制**：
- 从tokenization输出z0停止梯度，再构建noisy zt
- 防止degenerate solutions
- 但仍通过共享参数耦合两个任务

**为什么需要**：
- 如果不停止梯度：denoising会直接"简化"tokenization
- 导致latent空间坍塌，丢失reconstruction fidelity
- Stop-gradient通过参数共享间接耦合，而非直接

**实验验证**：
- Stop-grad: gFID=2.12, rFID=1.1（UNITE best）
- No-stop-grad: gFID=2.24, rFID=0.85（separate encoders）
- 生成变差但重建变好，说明latent失去generative structure

**类比理解**：
- 类似GAN中的discriminator和generator：indirect coupling
- Direct coupling（denoising直接优化z0）会破坏balance
- Indirect coupling（通过共享参数）更stable

### 5. Representation Alignment Analysis

**CKA对齐分析**：
- Tokenization和generation的activations高度对齐
- 特别是在deep layers（接近输出）
- 表明两任务intrinsic alignment

**为什么对齐**：
- 两种模式操作在同一latent space
- 相同的register parameterization
- 一致的归一化和条件化接口

**对Spatial AGI的启示**：
- Perception和generation不是incompatible的tasks
- 它们可以共享computational primitives
- 这对multi-task systems是good news

**Separate vs Shared**：
- 即使separate encoders也表现出high alignment
- 说明tasks本身有intrinsic similarity
- Parameter sharing进一步exploit这种similarity，减少冗余

### 6. 无需预训练和对抗损失

**实践优势**：
- 不需要DINO等预训练encoder
- 不需要GAN discriminator
- Training和deployment更简单

**理论意义**：
- 证明了pure reconstruction + flow matching已经足够
- External supervision不是必须的
- 端到端学习的力量被低估

**对domain transfer的意义**：
- 可以容易迁移到缺乏预训练生态的domain
- 如：molecular graphs、medical imaging、scientific data
- 对Spatial AGI特别重要：robotics、3D data往往没有大型预训练model

**例外**：
- LPIPS loss仍需要pretrained VGG
- 但(a) VGG训练便宜，(b) molecule generation不使用LPIPS
- 可以完全避免external pretrained weights

### 7. Latent Space的Robustness

**Dense yet Robust**：
- 重建需要dense information：保留每个细节
- 生成需要robust information：对小扰动不敏感
- Joint training找到balance

**实验证据**：
- rFID=1.01（competitive with vanilla autoencoder）
- gFID=2.12（near SOTA）
- 说明latent同时支持reconstruction和generation

**对noisy perception的意义**：
- 现实世界perception总是noisy
- 需要在不完整观测下保持stable understanding
- UNITE的latent training了这种robustness

**类比human perception**：
- 人类既能注意细节，又保持概念稳定性
- UNITE的latent体现了similar trade-off
- 为biologically plausible perception models提供思路

## 个人思考

### 1. 最有趣的发现

**Parameter Tying的Power**：

实验证明，即使separate encoders也表现出high CKA alignment，说明tokenization和generation是intrinsically aligned的任务。这很有趣，因为它意味着：

- **Computational redundancy**: Separate encoders在做similar computations
- **Functional reuse**: Attention和MLP sublayers可以被共享而不loss performance
- **MDL perspective**: Shared model has shorter description length

但更有趣的是，weight sharing不仅在效率上有优势，在performance上也更好：
- UNITE: gFID=2.12, rFID=1.1
- Separate: gFID=2.24, rFID=1.38（no-stop-grad设置）

这违反了intuition：通常认为增加model capacity（separate encoders）应该performance更好。但UNITE的发现是，parameter tying通过indirect coupling机制，可能比separate networks的direct coupling更stable和effective。

**为什么？**
- 可能的原因：shared parameters prevent one objective from dominating
- Separate networks may overfit to individual objectives
- Shared encourages balanced latent space that works for both

这个发现对AI architecture design有deep implications: **shared parameters may not be just about efficiency, but also about better multi-objective learning**。

---

### 2. 最意外的结果

**训练动力学**：

最意外的结果是adversarial training dynamics：generation metrics improve even as denoising loss increases。

这在传统supervised learning中是counter-intuitive的。通常我们期望：
- Training loss ↓ → Performance ↑
- Training loss ↑ → Performance ↓

但UNITE shows:
- Training loss (denoising) ↑
- Performance (FID) ↓

**为什么？**
- 假设1：Denoising loss增加是因为latent变得更rich和informative
- Richer latent → 更难去噪（需要更多information）
- 但同时：generative samples更realistic

这像什么？
- 类似teacher-student trade-off
- Training to be "hard to denoise" ensures latent captures all information
- But generative model learns to handle this difficulty
- End result: high-fidelity samples

**对AI训练的启示**：
- Don't just monitor training loss
- Need to evaluate final task metrics
- Multi-objective training may have non-monotonic dynamics
- "Adversarial" behavior might be a feature, not a bug

**端到端学习的力量被低估**：

**DINO (Caron et al., 2021)**:
- Self-distillation, no labels
- Captures local and global structure
- UNITE不从DINO预训练, 但可借鉴

**REPA (Yu et al., 2025)**:
- Align diffusion features to SSL representations
- Improves training efficiency and sample quality
- UNITE: 从零开始, 无SSL supervision

**REPA-E (Leng et al., 2025)**:
- Jointly tune VAE and diffusion to SSL space
- UNITE: 单阶段, 无pretraining

**RAE (Zheng et al., 2025)**:
- Replace VAE encoder with SSL encoder
- Separate decoder for reconstruction
- UNITE: unified encoder-denoiser, no SSL

### Pixel-Space Diffusion Models

**SiD (Hoogeboom et al., 2024)**:
- Trim U-Net skip connections
- Reduce high-resolution feature capacity
- UNITE: works in latent space, different challenges

**PixelFlow (Chen et al., 2025)**:
- Alternate denoising with progressive upsampling
- UNITE: unified tokenization-generation

**JiT (Li and He, 2025)**:
- Plain ViT generator on raw patches
- No tokenizers, pretraining, or auxiliary losses
- UNITE: includes tokenization as part of system

### Concurrent Works

**Unified Latents (Heek et al., 2026)**:
- End-to-end training of tokenizer + latent generator
- Closest to UNITE's separate-weights ablation
- Best performance relies on second-stage fine-tuning
- UNITE: single-stage, no second stage

**Latent Forcing (Baade et al., 2026)**:
- Denoise DINO latents alongside image patches
- Does not learn latent space from scratch
- UNITE: 从零学习latent space

**Self-Flow (Chefer et al., 2026)**:
- Auxiliary self-supervised objective
- Operates in latent space defined by pretrained encoder
- UNITE: jointly learn tokenizer and generator

### A. Implementation Details

**Model Architecture**:
```
Generative Encoder (GEθ):
  - Patch embedding: 16×16 patches, embed dim 768 (Base)
  - Register tokens: 64 learnable tokens
  - Transformer blocks: 12 blocks (Base), 24 blocks (Large)
  - Normalization: LayerNorm with learnable scale/shift
  - Output: 64×768 latent representation

Decoder (Dψ):
  - Input: 64×768 latent
  - Transformer blocks: 8 blocks (Base), 12 blocks (Large)
  - Patch unprojection: 16×16 patches → 256×256 pixels
  - Output: RGB image (3 channels)
```

**Training Pipeline**:
```python
# Pseudocode for UNITE training
def train_step(image_batch):
    # Tokenization pass
    z0 = GEθ(image_batch)  # Encode to clean latents
    z_noisy = z0 + σ * ε  # Add reconstruction noise
    x_recon = Dψ(z_noisy)  # Reconstruct
    loss_recon = L1(x_recon, image_batch) + LPIPS(x_recon, image_batch)

    # Generation pass
    t = Uniform(0, 1)  # Sample noise level
    ε_noise = Normal(0, I)  # Sample noise
    zt = t * stop_grad(z0) + (1-t) * ε_noise  # Corrupt
    z_pred = GEθ(zt, t, class_label)  # Denoise
    loss_flow = L2(z_pred, stop_grad(z0))  # Flow matching

    # Total loss
    loss = loss_recon + loss_flow
    loss.backward()
    optimizer.step()
```

**Inference**:
```python
# Generation inference
def generate(class_label, num_steps=50):
    z = Normal(0, I)  # Initialize noisy latents
    for i in range(num_steps):
        t = i / num_steps  # Linear schedule
        z = GEθ(z, t, class_label)  # Denoise
    x = Dψ(z)  # Decode
    return x
```

### C. Limitations and Future Work

**Current Limitations**:
1. Computational cost: 2x forward passes per iteration
2. Best reconstruction still needs adversarial fine-tuning
3. Latent interpretability not fully understood
4. Scaling to 512+ resolution needs more research

**Future Directions**:
1. **Dynamic Latent Spaces**: Adapt latent dimension to task complexity
2. **Hierarchical UNITE**: Multi-scale latent representations
3. **Spatial Reasoning**: Explicit spatial relations in latent
4. **Robotics Integration**: Real robot experiments
