## 1. 摘要与核心贡献

### 1.1 摘要概述

扩散模型虽然在视觉生成领域取得了革命性进展，但在需要逐步逻辑推理的复杂任务（如迷宫求解、旅行商问题、数独填空）方面仍存在显著局限。当前主流范式将多模态大语言模型（MLLMs）主要作为静态文本编码器集成到扩散框架中，这种做法存在两个关键问题：

1. **单步推理深度不足**：MLLM文本编码器在单次前向传播中无法充分激活链式思维（Chain-of-Thought, CoT）过程，这对于为复杂任务提供准确指导至关重要
2. **静态指导的局限性**：解码过程中的指导保持不变，使得扩散Transformer（DiT）无法渐进地将复杂指令分解为可执行的降噪步骤

针对这些问题，本文提出了**内生链式思维（Endogenous Chain-of-Thought, EndoCoT）**框架。该框架通过两个核心组件解决上述挑战：

1. **迭代思想指导模块（Iterative Thought Guidance）**：通过迭代优化潜在思想状态来激活MLLM的推理潜力，并将这些状态与DiT的降噪过程建立对应关系
2. **终端思想Grounding模块（Terminal Thought Grounding）**：将最终状态与真实答案对齐，确保推理轨迹保持在文本监督之上

在这两个组件的协同作用下，MLLM文本编码器能够提供精心推理的指导，使得DiT能够以渐进的方式执行指导，最终逐步解决复杂任务。

### 1.2 核心贡献总结

本文的主要贡献可归纳为以下三个关键点：

**贡献1：首次实现扩散模型中的真正链式思维推理**
- 据我们所知，这是首个通过迭代潜在状态优化而非单次预计算来实现真正链式思维推理的扩散框架
- 传统方法依赖静态文本编码，无法利用MLLM的动态推理潜力
- EndoCoT使扩散模型能够像人类一样，通过多步迭代逐步解决复杂问题

**贡献2：揭示扩散模型推理能力的根本瓶颈**
- 通过全面的层敏感性和注意力熵分析，定位了推理能力在扩散模型中的产生位置
- 识别了限制先前方法的两个关键瓶颈：
  - 有限的单步推理能力
  - 复杂动态DiT解码中的静态指导失败
- 这些发现为后续研究提供了明确的改进方向

**贡献3：显著的性能提升与可控推理**
- 在复杂视觉推理基准测试上实现了25-40%的性能提升
- 在Maze-32上达到90%准确率，在Sudoku-35上达到95%准确率
- 支持推理时间的可扩展性，通过调整推理步数实现精度与效率的权衡
- 在图像编辑任务中产生更清晰、更可控的转换轨迹

### 1.3 实验表现概览

在四个多样化的推理任务（Maze、TSP、Sudoku、VSP）上进行广泛评估：
- **平均准确率**: 92.1%
- **最强基线超越**: 8.3个百分点
- **复杂任务优势**:
  - Maze-32: 90%（比最强基线提升25%）
  - Sudoku-35: 95%（比最强基线提升40%）
- **泛化能力**: 在不同迷宫大小、数独字体等新颖配置上表现一致

## 2. 研究背景与动机

### 2.1 扩散模型的发展现状

扩散模型（Diffusion Models）自2020年提出以来，在视觉生成领域取得了前所未有的发展：

**技术演进路径**
- **DDPM（Denoising Diffusion Probabilistic Models）**: 奠定了扩散模型的理论基础
- **DDIM（Denoising Diffusion Implicit Models）**: 提高了采样效率
- **Flow Matching**: 提供了更简化的生成建模框架
- **DiT（Diffusion Transformer）**: 将Transformer架构引入扩散模型

**应用领域突破**
- 图像生成：实现照片级真实感和多样性的突破
- 图像编辑：支持精细的局部修改和风格转换
- 视频生成：实现了高质量时序一致的视频合成
- 3D生成：扩展到三维内容的自动创建

**技术特点优势**
1. **生成质量高**: 能够生成高分辨率、高质量的图像
2. **多样性丰富**: 通过随机噪声采样产生多样化的输出
3. **可控制性强**: 通过条件输入实现精准的生成控制
4. **稳定性好**: 训练过程相对稳定，不容易出现模式崩溃

### 2.2 多模态大语言模型的集成趋势

近年来，多模态大语言模型（MLLMs）被广泛集成到扩散框架中，主要作为文本编码器来处理复杂任务：

**代表性MLLMs**
- **Qwen3-VL-8B**: 阿里巴巴推出的多模态大语言模型
- **Ministral**: Mistral AI系列的小型化多模态模型
- **Flux 2**: Black Forest Labs的先进视觉语言模型

**集成方式演变**
1. **早期方法**：使用CLIP等简单的图像-文本编码器
2. **当前主流**：集成大型多模态模型作为文本编码器
3. **近期趋势**：尝试让MLLM参与更深入的推理过程

**集成目的**
- 增强语义理解能力
- 提升逻辑一致性
- 改善跨模态对齐
- 支持更复杂的条件生成

### 2.3 当前方法的根本局限性

尽管MLLMs被成功集成到扩散框架中，但当前范式在处理需要逐步逻辑推理的任务时仍然困难重重。具体表现在：

**任务类型限制**

1. **迷宫求解（Maze Navigation）**
   - 需要在网格中找到从起点到终点的路径
   - 要求考虑墙壁约束和路径优化
   - 错误的选择可能导致死胡同

2. **旅行商问题（Traveling Salesperson Problem, TSP）**
   - 需要访问所有城市并返回起点
   - 要求优化总路径长度
   - 典型的NP难组合优化问题

3. **数独填空（Sudoku Solving）**
   - 需要遵循行、列、块的数字唯一性约束
   - 要求全局一致性推理
   - 错误的填入会导致整个谜题无解

4. **视觉空间规划（Visual Spatial Planning, VSP）**
   - 需要在有障碍物的环境中规划安全路径
   - 要求实时感知和决策
   - 涉及复杂的空间推理

**现有方法的失效机制**

现有方法将这些MLLMs主要作为**静态条件编码器**使用，只在生成开始时计算一次文本嵌入。这种被动的集成方式存在根本缺陷：

1. **未能利用动态推理潜力**
   - MLLMs在单次前向传播中无法完成所有必要的推理
   - 强迫其在一步内编码所有逻辑约束是不现实的
   - 模型的推理潜力被严重浪费

2. **推理过程不透明**
   - 尽管集成了强大的MLLMs，但这些模型在生成过程中并没有实际执行推理
   - 它们在最初的几个降噪步骤中就"承诺"了最终解决方案
   - 之后只是在优化视觉质量，而非改进解决方案的正确性

3. **泛化能力脆弱**
   - 虽然在特定基准测试上指标有所提升
   - 但当泛化到新颖领域时会灾难性地失败
   - 解决方案是基于模式匹配而非真正的认知处理

**实证观察**

作者通过系统分析发现，当前方法的问题根源在于：
- **过早承诺（Early Commitment）**: 在前几步降噪后就确定了最终方案
- **缺乏渐进分解（Lack of Progressive Decomposition）**: 无法将复杂指令分解为可执行步骤
- **静态耦合失败（Static Coupling Failure）**: DiT与MLLM之间的耦合是脆弱且静态的

### 2.4 内生推理的必要性

"内生推理"（Endogenous Reasoning）指的是模型自身具备的推理能力，而非外部注入的。其重要性体现在：

**人类问题解决的启发**
- 人类解决复杂问题时通常采用逐步迭代的方法
- 不会尝试一次性计算出完整解决方案
- 通过反复评估和调整来逼近最优解

**技术优势**
1. **灵活性**: 能够适应不同复杂度的任务
2. **鲁棒性**: 对输入变化和噪声具有更好的抗性
3. **可解释性**: 推理过程更加透明和可追踪
4. **泛化性**: 更容易迁移到新任务和领域

**链式思维的成功先例**
- 在大语言模型（LLMs）中，Chain-of-Thought推理证明了迭代优化的有效性
- 通过逐步细化和调整，模型能够解决更复杂的问题
- Diffusion模型同样需要类似的机制来释放其推理潜力

## 3. 问题定义与挑战

### 3.1 问题形式化定义

**任务定义**

给定一个复杂的视觉推理任务，目标是通过迭代推理过程生成正确的视觉解决方案。具体而言：

**输入**
- 文本提示 \( P \in \mathbb{R}^{L \times d} \)：描述任务的文本指令
- 输入图像 \( I_{\text{input}} \)：任务的初始视觉表示（如迷宫、城市分布、数独盘面等）

**输出**
- 最终解决方案 \( I_{\mathcal{T}} \)：正确的视觉输出（如路径、访问顺序、填好的数独等）
- 可选的中间推理步骤 \( \{I_1, I_2, ..., I_{\mathcal{T}-1}\} \)：展示推理过程

**约束**
1. **逻辑约束**: 解决方案必须满足任务的特定规则（如不走墙壁、访问所有城市、数独规则等）
2. **顺序约束**: 某些步骤之间存在依赖关系
3. **优化约束**: 可能需要优化某些目标（如路径最短）

**推理步骤**

推理过程被建模为 \( \mathcal{T} \) 个步骤的序列。在每个步骤 \( \tau \in \{1, ..., \mathcal{T}\} \)：

1. **状态更新**: 更新潜在思想状态 \( \mathbf{h}_\tau \)
2. **条件生成**: 基于当前状态生成中间视觉输出 \( I_\tau \)
3. **状态传播**: 将当前状态作为下一步的输入

### 3.2 核心技术挑战

**挑战1：单步推理深度不足**

**问题描述**
MLLM在单次前向传播中无法编码所有必要的逻辑约束。

**具体表现**
- 对于复杂任务（如32×32的迷宫），单步编码无法捕获所有拓扑信息
- 在简单任务（如8×8迷宫）中，DiT能够执行空间定位，但生成路径违反物理约束（如穿墙）
- 表明MLLM需要迭代优化其对问题的理解，而不是尝试一次性计算完整解决方案

**理论分析**
- MLLM的计算图深度有限，难以表示长距离依赖
- 单步推理无法展开复杂的因果链
- 注意力机制在单次传递中无法充分聚焦所有相关信息

**实际影响**
- 导致生成的解决方案存在逻辑错误
- 无法处理需要多步验证的任务
- 降低了模型在高复杂度任务上的性能

**挑战2：静态指导的耦合失败**

**问题描述**
即使MLLM能够在单步中产生完美推理，DiT与MLLM之间的信息耦合仍然是脆弱和静态的。

**诊断方法**
通过分析降噪过程中生成空间斑块与真实推理标记之间的交叉注意力熵来诊断。

**观察现象**
- 在高复杂度场景（如32×32迷宫）中，注意力熵图变得全局性高且扩散
- 这揭示了DiT与MLLM之间耦合的严重崩溃
- 当面临密集的空间拓扑时，DiT失去将空间特征锚定到特定逻辑文本标记的能力
- 导致注意力分布平均化，空间定位完全崩溃

**根本原因**
- 静态的一次性文本嵌入注入是不充分的
- 条件信号需要在整个推理过程中动态更新
- DiT需要持续、演化的条件来维持与复杂逻辑约束的对齐

**挑战3：推理与生成的分离**

**问题描述**
现有方法将推理和生成分离，导致无法形成端到端的推理-生成循环。

**传统方法**
- 先用MLLM产生推理文本
- 再用扩散模型基于推理文本生成图像
- 推理过程与生成过程完全解耦

**问题**
- 推理文本可能不准确或不完整
- 生成过程无法反馈到推理过程
- 无法利用生成结果来验证和调整推理

**理想机制**
- 推理和生成应该相互促进
- 生成结果应该能够影响下一步推理
- 形成闭环的推理-生成循环

### 3.3 技术需求分析

为了解决上述挑战，需要满足以下技术需求：

**需求1：动态推理机制**
- 支持多轮迭代推理
- 允许推理状态的持续更新
- 实现推理与生成的交互

**需求2：跨模态对齐**
- 确保语言推理与视觉生成的对齐
- 维持推理轨迹的一致性
- 防止推理状态的漂移

**需求3：可控推理深度**
- 根据任务复杂度调整推理步数
- 支持推理时间的资源分配
- 实现精度-效率的权衡

**需求4：端到端训练**
- 联合优化推理和生成模块
- 学习跨模态的协同推理
- 保留预训练知识的同时适应新任务

### 3.4 评估指标设计

为了全面评估模型性能，需要设计多维度的评估指标：

**主要指标**

1. **准确率（Accuracy）**
   - 解决方案的正确性
   - 是否满足所有约束条件
   - 是否达到优化目标

2. **路径重复率（Path Repetition Rate）**
   - 生成路径与真实路径的重叠比例
   - 衡量空间推理的精确性
   - 适用于迷宫、TSP等路径规划任务

**辅助指标**

3. **推理效率**
   - 推理步骤数量
   - 计算资源消耗
   - 采样时间

4. **泛化能力**
   - 在不同复杂度任务上的表现
   - 对输入变化的鲁棒性
   - 在未见过的配置上的性能

5. **可解释性**
   - 推理过程的清晰度
   - 中间步骤的合理性
   - 错误的可追溯性

## 4. 相关工作回顾

### 4.1 多模态大语言模型中的推理

**4.1.1 文本领域的Chain-of-Thought推理**

链式思维（Chain-of-Thought, CoT）推理在大语言模型中取得了巨大成功：

**核心技术**
- **逐步推理**: 将复杂问题分解为多个子步骤
- **显式思维链**: 生成中间推理步骤，增强可解释性
- **自一致性**: 通过多次采样和投票提高准确率

**扩展策略**
- **测试时扩展（Test-Time Scaling）**: 
  - Tree-of-Thought（Yao等，2023）
  - Graph-of-Thought（Besta等，2024）
  - Monte Carlo方法（Zheng等，2025）
- **思维树探索**: 支持分支和回溯
- **多路径验证**: 通过多样化路径提高鲁棒性

**应用领域**
- 数学和逻辑推理
- 代码生成和调试
- 复杂问题求解

**4.1.2 多模态领域的CoT扩展**

将CoT推理从文本扩展到多模态设置：

**视觉推理**
- OpenAI的"Think with Images"
- 使用视觉内容支持多步推理
- 结合视觉证据进行逻辑推导

**视频推理**
- "Thinking with Video"系列工作
- Zhang等（2025）, Tong等（2025）
- 使用视频时序信息作为推理依据
- 适用于时序理解和预测任务

**混合模态**
- Latent Sketchpad（Zhang等，2025）
- 文本和视觉潜在token的交错自回归生成
- 实现推理和生成的无缝集成

**4.1.3 扩散模型中的推理挑战**

虽然CoT在自回归LLMs中有效，但在扩散模型（DMs）中仍然具有挑战性：

**架构限制**
- DMs通常依赖固定长度的文本编码器（如CLIP）
- 难以将多步推理融入生成过程
- 缺乏类似LLMs的逐步解码机制

**近期进展**
- Wu等（2025）引入视觉语言模型（VLMs）作为编码器
- 提供更丰富的语义表示
- 支持推理条件生成

**技术难点**
- 推理过程与生成过程的同步
- 跨模态状态的对齐
- 推理轨迹的可控性

### 4.2 扩散模型中的推理

**4.2.1 文本推理信号注入**

早期工作探索将文本推理痕迹注入条件输入：

**代表性方法**
- ThinkGen（Jiao等，2025）

## 5. 扩散模型推理能力的系统性分析

为了理解当前扩散模型尽管集成了强大的MLLMs，但在复杂视觉推理任务上仍然失败的原因，本文进行了系统的实证分析。这些分析直接启发并指导了EndoCoT框架的设计，揭示了限制性能的根本瓶颈。

所有实验均在Qwen-Image-Edit-2511（Wu等，2025）上进行。

### 5.1 层级敏感性分析

**5.1.1 分析方法**

为了调查推理能力如何从内部动态涌现，我们对代表性图像编辑模型进行层级敏感性分析。

**分析对象**
- 模型：Qwen-Image-Edit-2511
- 组件：
  1. Vision Encoder（视觉编码器）
  2. Language Model（语言模型，LLM）
  3. Diffusion Transformer（扩散Transformer，DiT）

**量化方法**
- 测量模型三个核心组件的各个不同层块中的激活响应大小
- 计算相对敏感性，识别关键层级

**5.1.2 分析结果**

**关键发现**

如图2(a)所示，峰值敏感性高度集中在：
1. **Vision Encoder内部**
2. **LLM（终端层）与DiT（初始层）之间的架构连接点**

**深入解读**

这种局部化的激活揭示了关键洞察：
- **推理的主要承担者**：逻辑推理的繁重工作主要由MLLM处理
- **架构依赖性**：推理能力高度依赖于LLM与DiT的耦合
- **激活分布**：并非所有层都同等参与推理过程

**对设计的启示**

这一观察强烈建议：
1. **联合微调的必要性**：在训练过程中激活MLLM是充分释放模型视觉推理潜力的关键
2. **协同推理的重要性**：推理需要LLM和DiT的协同工作
3. **架构设计的考虑**：应该优化LLM与DiT之间的连接

这一发现直接指导了EndoCoT的设计选择：**联合微调MLLM和DiT组件**。

### 5.2 单步推理的局限性分析

**5.2.1 观察现象**

虽然MLLM处理了大部分逻辑推理，但观察到MLLM的单次前向传播对于复杂任务是不充分的。

**5.2.2 不同复杂度任务的表现**

**低复杂度场景（如8×8迷宫）**
- DiT能够成功关注生成的轨迹
- 但生成的路径系统性违反物理约束（如穿墙）
- 表明DiT在简单任务中有效执行了空间定位

**高复杂度场景（如32×32迷宫）**
- 模型完全无法生成有效路径
- 推理过程崩溃

**5.2.3 问题诊断**

这表明：
1. **空间定位能力**：DiT在简单任务中成功执行空间定位
2. **约束编码能力**：MLLM缺乏在单次传递中完全编码和执行所有逻辑约束的能力
3. **迭代优化的必要性**：MLLM需要迭代优化其对问题的理解，而不是尝试一次性计算完整解决方案

**5.2.4 理论解释**

**信息瓶颈**
- 单次前向传播的信息吞吐量有限
- 复杂任务需要考虑的约束和可能性空间巨大
- 无法在单步中充分编码所有相关信息

**计算图限制**
- MLLM的计算图深度有限
- 难以展开长距离依赖
- 无法表示复杂的因果链

**注意力限制**
- 注意力机制在单次传递中无法充分聚焦所有相关信息
- 复杂任务需要关注的空间区域和约束众多
- 单步注意力分配不足

**5.2.5 解决方案启发**

这一观察启发了EndoCoT的**多轮潜在空间推理设计**：
- 不要尝试一次计算完整解决方案
- 通过多次迭代逐步优化
- 每次迭代细化对问题的理解

### 5.3 复杂动态DiT解码中的静态指导失败分析

**5.3.1 问题设定**

即使MLLM能够在单步中产生完美推理，仍然观察到第二个关键限制：DiT与MLLM之间的信息耦合是脆弱和静态的。

**5.3.2 诊断方法**

通过分析降噪过程中生成空间斑块与真实推理标记之间的交叉注意力熵来诊断这个问题。

**关键指标**
- 交叉注意力熵（Cross-Attention Entropy）
- 衡量DiT对MLLM推理标记的关注集中程度

**5.3.3 分析结果**

**低复杂度场景（如8×8迷宫）**
- 注意力熵较低，分布集中
- DiT能够将空间特征锚定到特定的逻辑文本标记
- 空间定位工作良好

**高复杂度场景（如32×32迷宫）**
- 注意力熵图变得全局性高且扩散
- 揭示了DiT与MLLM之间耦合的严重崩溃
- 当面临密集的空间拓扑时，DiT失去将空间特征锚定到特定逻辑文本标记的能力
- 导致注意力分布平均化，空间定位完全崩溃

**5.3.4 根本原因分析**

**拓扑复杂度的影响**
- 密集的空间拓扑增加了DiT需要关注的区域数量
- 需要区分的逻辑约束显著增加
- 单一静态的注意力模式无法适应

**静态嵌入的局限性**
- 静态的一次性文本嵌入注入是不充分的
- 条件信号需要在整个推理过程中动态更新
- DiT需要持续、演化的条件来维持与复杂逻辑约束的对齐

**信息衰减**
- 随着解码过程的进行，初始注入的信息逐渐衰减
- 无法维持对复杂约束的持续关注
- 导致推理轨迹的漂移

**5.3.5 失败机制的总结**

静态指导失败的核心机制：
1. **初始注入**：在生成开始时注入静态文本嵌入
2. **信息维持**：随着降噪步数增加，注入的信息逐渐模糊
3. **注意力漂移**：DiT的注意力无法持续锚定到相关推理标记
4. **最终崩溃**：在高复杂度场景下，注意力分布完全发散，导致推理失败

### 5.4 综合洞察与设计原则

通过图2(a-c)的分析，我们可以总结三个关键洞察：

**洞察1：MLLM的推理潜力未充分利用**
- MLLM具有很强的推理潜力
- 但无法在单次前向传播中充分利用
- 需要迭代机制来充分激活

**洞察2：DiT的空间定位需要动态条件**
- DiT在空间定位方面表现出色
- 但需要动态、演化的条件来维持与复杂逻辑约束的对齐
- 静态条件在高复杂度场景下失效

**洞察3：静态文本注入的局限性**
- 当前的静态、一次性文本注入范式对于多步推理任务是有限的
- 需要持续的、动态的条件更新
- 推理和生成过程应该相互促进

**设计原则**

基于这些洞察，EndoCoT采用以下设计原则：
1. **迭代潜在推理**：通过迭代优化潜在状态实现真正的推理过程
2. **动态条件更新**：在推理过程中持续更新条件信号
3. **联合优化**：联合训练MLLM和DiT，实现协同推理
4. **端到端集成**：将推理和生成无缝集成在统一框架中

这些洞察和原则直接指导了EndoCoT框架的设计，我们将在下一节详细介绍其方法论。

## 6. 方法论：EndoCoT框架

基于第3节的系统性分析，我们提出了**内生链式思维（Endogenous Chain-of-Thought, EndoCoT）**框架，该框架使扩散模型能够在生成过程中执行自我引导的推理。

如图3所示，我们的方法通过在MLLM中迭代优化潜在思想状态来创建CoT推理过程，将这些推理状态与显式文本监督对齐，并使用渐进式训练策略首先构建推理能力，然后优化输出质量。

### 6.1 预备知识

#### 6.1.1 流匹配（Flow Matching）

流匹配（Flow Matching, FM）通过在两个分布之间构建线性概率路径，提供了一种简化的生成建模框架。

**基本定义**

设 \( X_0 \sim \pi_0 \) 表示干净数据样本，\( X_1 \sim \pi_1 \) 表示来自先验的高斯噪声。

**线性轨迹**

在连续时间步 \( t \in [0,1] \) 上，\( X_t \) 在 \( X_0 \) 和 \( X_1 \) 之间插值：

\[
X_t = t X_1 + (1-t) X_0 \quad \text{(1)}
\]

**向量场**

通过对 \( X_t \) 关于时间步 \( t \) 求导，得到真实向量场 \( u_t(X_t) \)：

\[
u_t(X_t) = \frac{d X_t}{dt} = X_1 - X_0 \quad \text{(2)}
\]

**神经网络近似**

神经网络 \( v_\theta(X_t, t, c) \) 近似这个向量场，其中 \( c \) 是可选的条件变量。

**训练目标**

流匹配的训练目标 \( \mathcal{L}_{\text{FM}} \) 为：

\[
\mathcal{L}_{\text{FM}} = \mathbb{E}_{t, X_0 \sim \pi_0, X_1 \sim \pi_1} \left\| v_\theta(X_t, t, c) - u_t(X_t) \right\|^2 \quad \text{(3)}
\]

#### 6.1.2 架构基础

**多模态大语言模型（MLLM）**
- 表示为 \( f_\phi \)
- 包含嵌入层、投影层和Transformer层
- 负责高级逻辑推理和语义理解

**扩散Transformer（DiT）**
- 表示为 \( f_\psi \)
- 基于Transformer架构的扩散模型
- 负责空间定位和视觉生成

**联合训练**
- 应用LoRA微调到 \( f_\phi \) 和 \( f_\psi \)
- 实现条件生成和生成阶段的协同推理
- 保留预训练知识的同时适应新任务

### 6.2 EndoCoT核心方法

当前扩散模型对于复杂、多步骤任务的推理能力有限。最近的分析表明，这些模型具有与其参数规模相匹配的推理潜力。文本编码器架构的进步（Black Forest Labs 2025; Wu等，2025）提供了释放这种潜力的结构基础。我们提出了内生链式思维（EndoCoT），一个使扩散模型能够执行自主链式思维推理的框架。

#### 6.2.1 迭代思想指导

**核心思想**

我们将迭代思想指导过程构建为条件潜在状态的迭代优化。

**与标准方法的对比**

- **标准扩散模型**：通过条件化静态文本嵌入生成输出
- **我们的方法**：执行 \( \mathcal{T} \) 个推理步骤，每步优化视觉输出和条件信号

**人类启发**

直观地，这个过程模仿人类问题解决：
- 不是试图一次生成完整解决方案
- 而是迭代优化对问题和拟议解决方案的理解
- 每个推理步骤构建在前一个之上，允许模型逐步探索解决方案空间

**符号系统**

设：
- \( f_\phi \)：MLLM（排除嵌入和投影层）
- \( f_\psi \)：扩散Transformer（DiT）

\( \mathbf{P} \in \mathbb{R}^{L \times d} \)：通过各自嵌入层编码文本提示和输入图像获得的前缀嵌入

推理过程被公式化为潜在流形 \( \mathbb{R}^d \) 中隐藏状态更新的序列。

**推理步骤**

给定固定前缀嵌入 \( \mathbf{P} \)（其中 \( L \) 是文本提示的序列长度），第 \( \tau \) 个推理步骤（\( \tau \in \{1, \dots, \mathcal{T}\} \)）递归更新思想状态 \( \mathbf{h}_\tau \in \mathbb{R}^d \)：

\[
\mathbf{h}_\tau = \mathbf{e}_{L+1}^\top f_\phi\left([\mathbf{P}; \mathbf{h}_{\tau-1}]\right), \quad \tau = 1, \dots, \mathcal{T} \quad \text{(4)}
\]

其中：
- \( [\cdot; \cdot] \)：沿序列维度连接
- \( \mathbf{e}_{L+1} \)：用于提取第 \( (L+1) \)-th序列位置的隐藏状态的单热基向量

**关键特性**

重要的是，\( \mathbf{h}_{\tau-1} \) 直接作为 \( f_\phi \) 第一层的高维输入，绕过离散嵌入查找表。

**推理步骤与扩散时间步的区别**

注意，我们的推理步骤 \( \mathcal{T} \) 不同于扩散模型的内部降噪时间步。

每个推理步骤 \( \tau \) 涉及从噪声到图像的完整降噪轨迹，条件化于当前思想状态 \( \mathbf{h}_\tau \)。

**条件流生成**

在每个推理步骤 \( \tau \)，我们条件化扩散模型于当前思想状态 \( \mathbf{h}_\tau \)，通过求解流ODE生成中间视觉输出 \( \mathbf{I}_\tau \)：

\[
\frac{d \mathbf{z}_\tau(t)}{dt} = v_\psi(\mathbf{z}_\tau(t), t, \mathbf{h}_\tau), \quad \mathbf{z}_\tau(1) \sim \mathcal{N}(\mathbf{0}, \mathbf{I}), \quad \mathbf{I}_\tau = \mathbf{z}_\tau(0) \quad \text{(5)}
\]

其中 \( \mathbf{z}_\tau(t) \) 表示流时间 \( t \in [0,1] \) 时的潜在状态。

**训练目标**

通过监督生成输出与真实中间目标 \( \mathbf{I}_\tau^* \) 对比来优化模型，使用基于公式(3)的条件流匹配损失：

\[
\mathcal{L}_{\text{reasoning}} = \mathbb{E}_{\tau, t, \mathbf{z}_\tau(0), \mathbf{z}_\tau(1)} \left[ \|(\mathbf{z}_\tau(0) - \mathbf{z}_\tau(1)) - v_\psi(\mathbf{z}_\tau(t), t, \mathbf{h}_\tau)\|^2 \right] \quad \text{(6)}
\]

**中间目标的获取**

中间目标 \( \mathbf{I}_\tau^* \) 可以通过逐步真实分解获得（如迷宫的部分路径段）。

**训练策略**

受第3节分析的启发，我们对 \( f_\phi \) 和 \( f_\psi \) 都应用LoRA微调。这种联合适应实现了条件生成和生成阶段的协同推理。

#### 6.2.2 终端思想Grounding

虽然增强推理的框架实现了迭代视觉优化，但其对纯视觉监督的依赖引入了两个主要挑战：

1. **模态差距**：视觉目标与潜在推理状态之间的模态差距
2. **显式文本真实值的利用不足**：对显式文本真实值的利用不足

受文本领域潜在推理中显式监督成功的启发，我们引入了一个辅助对齐目标，将最终推理状态与文本监督对齐：

**参考状态计算**

\[
\mathbf{h}_{\text{ref}} = \mathbf{e}_{L_{\text{gt}}+1}^\top f_\phi\left([\mathbf{P}_{\text{gt}}, \mathbf{I}_{\text{input}}]\right) \quad \text{(7)}
\]

其中：
- \( \mathbf{P}_{\text{gt}} \in \mathbb{R}^{L_{\text{gt}} \times d} \)：真实推理步骤的嵌入
- \( \mathbf{I}_{\text{input}} \)：任务输入（如初始迷宫配置）

**对齐损失**

我们使用L2（语义）损失将最终推理状态 \( \mathbf{h}_{\mathcal{T}} \) 与此参考对齐：

\[
\mathcal{L}_{\text{align}} = \| \mathbf{h}_{\mathcal{T}} - \mathbf{h}_{\text{ref}} \|^2 \quad \text{(8)}
\]

**总体训练损失**

总体训练损失结合了流匹配损失与这个对齐项：

\[
\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{FM}} + \mathbb{I}_{\{\tau = \mathcal{T}\}} \cdot \lambda_{\text{align}} \mathcal{L}_{\text{align}} \quad \text{(9)}
\]

其中：
- \( \mathbb{I}_{\{\tau = \mathcal{T}\}} \)：仅在最终推理步骤激活对齐损失的指示函数
- \( \lambda_{\text{align}} \)：平衡视觉生成质量和文本grounding的权重

**经验设置**

经验上，我们设置 \( \lambda_{\text{align}} = 1 \)。

**关键作用**

如我们的消融研究（第5.2节）所示，这一项对于防止潜在推理状态的漂移和确保推理轨迹保持在grounding之上至关重要。

#### 6.2.3 渐进式训练策略

**动机**

推理步骤和最终输出服务于不同的目的：
- **中间步骤**：专注于探索解决方案路径
- **最终步骤**：强调产生正确答案

直接同时优化两个目标可能导致冲突的梯度。

**策略概述**

因此，我们采用两阶段渐进式训练策略。

**阶段1：推理发展（Reasoning Development）**

如图3(a)所示，我们监督所有推理步骤 \( \tau = 1, \dots, \mathcal{T} \)，使模型学习逐步视觉推理。

训练目标：

\[
\mathcal{L}_{\text{stage1}} = \sum_{\tau=1}^{\mathcal{T}} \left( \mathcal{L}_{\text{FM}}^\tau + \mathbb{I}_{\{\tau = \mathcal{T}\}} \lambda_{\text{align}} \mathcal{L}_{\text{align}} \right) \quad \text{(10)}
\]

其中 \( \mathcal{L}_{\text{FM}}^\tau \) 表示推理步骤 \( \tau \) 的流匹配损失。

通过在每个中间步骤提供监督，我们鼓励模型发展连贯、渐进的推理轨迹。

**阶段2：终端巩固（Terminal Consolidation）**

一旦模型发展了鲁棒的推理能力，我们将训练焦点转移到巩固最终输出的视觉准确性。

如图3(b)所示，虽然中间推理过程在前向传递中保持，但梯度仅为最终输出计算：

\[
\mathcal{L}_{\text{stage2}} = \mathcal{L}_{\text{FM}}^\mathcal{T} + \lambda_{\text{align}} \mathcal{L}_{\text{align}} \quad \text{(11)}
\]

中间步骤 \( \tau < \mathcal{T} \) 仅在前向传递中执行，作为推理脚手架而不传播梯度。

**防止退化**

为了防止之前学习的推理链的退化，阶段2采用有限迭代的短周期微调策略。

**训练策略的优势**

这种渐进式训练策略具有以下优势：
1. **分阶段目标**：早期关注推理能力发展，后期关注输出质量
2. **避免冲突**：防止中间步骤和最终步骤目标的冲突
3. **保留推理能力**：通过短周期微调防止推理链的退化
4. **提高最终质量**：专注于优化最终输出的准确性

#### 6.2.4 推理过程

**推理流程**

如图3(b)所示，EndoCoT在推理过程中不需要解码中间视觉状态。

通过指定推理步数 \( \mathcal{T} \)，模型递归更新其潜在思想状态以生成最终结果。

**关键特性**

1. **无需中间输出**：推理过程不需要生成中间图像
2. **完全在潜在空间**：所有推理都在潜在空间进行
3. **灵活的推理深度**：可以通过调整 \( \mathcal{T} \) 控制推理深度
4. **高效**：避免不必要的中间视觉解码

**算法伪代码**

```python
def inference(visual_encoder, mllm, dit, prompt, input_image, T):
    # 编码前缀
    P = encode_prefix(prompt, input_image, visual_encoder)
    
    # 初始化思想状态
    h_0 = initialize_thought_state()
    
    # 迭代推理
    for tau in range(1, T+1):
        # 更新思想状态
        h_tau = mllm(concatenate(P, h_{tau-1}))
        
        # 如果是最后一步，生成最终输出
        if tau == T:
            I_tau = dit_generate(dit, h_tau)
        # 否则，继续推理（不生成中间输出）
    
    return I_T
```

**推理时间扩展**

推理过程支持推理时间扩展：
- 可以动态调整推理步数 \( \mathcal{T} \)
- 更多推理步数通常带来更好的性能
- 实现精度与效率的权衡

**与训练的关系**

推理过程与训练阶段的对应关系：
- 阶段1训练：学习多步推理能力
- 阶段2训练：优化最终输出质量
- 推理：应用学习的推理能力生成结果

### 6.3 完整推理算法

**算法总结**

EndoCoT的完整推理算法可总结如下：

1. **输入准备**
   - 文本提示 \( P \)
   - 输入图像 \( I_{\text{input}} \)
   - 推理步数 \( \mathcal{T} \)

2. **前缀编码**
   - 使用视觉编码器和MLLM编码文本提示和输入图像
   - 获得前缀嵌入 \( \mathbf{P} \)

3. **迭代推理**
   - 对于每个推理步骤 \( \tau = 1, \dots, \mathcal{T} \)：
     - 使用当前思想状态 \( \mathbf{h}_{\tau-1} \) 更新前缀
     - 通过MLLM获得新的思想状态 \( \mathbf{h}_\tau \)
     - 如果 \( \tau = \mathcal{T} \)，使用DiT生成最终输出

4. **输出生成**
   - 返回最终生成的解决方案 \( I_\mathcal{T} \)

**关键创新点**

EndoCoT的关键创新点在于：
1. **内生推理**：推理过程内生地嵌入到扩散模型中
2. **潜在空间操作**：所有推理都在潜在空间进行，高效且可扩展
3. **端到端训练**：推理和生成在统一框架中端到端训练
4. **可扩展性**：支持推理时间和训练时间的资源扩展

- MLLM的MLP层
- DiT的自注意力层
- DiT的MLP层

#### 7.1.2 数据集

**评估基准**

遵循He等（2025）的评估协议，我们选择以下四个多样化的推理任务作为评估基准：

**1. 迷宫导航（Maze）**

**任务描述**
- 在网格中找到从起点到终点的路径
- 必须避免穿过墙壁
- 要求空间推理和路径优化

**数据规模**
- 不同迷宫大小：8×8, 16×16, 32×32
- 使用DFS生成迷宫，BFS计算最短路径

**难度递增**
- 8×8：较简单，适合基准测试
- 16×16：中等复杂度
- 32×32：高复杂度，测试推理极限

**2. 旅行商问题（Traveling Salesperson Problem, TSP）**

**任务描述**
- 访问所有城市并返回起点
- 优化总路径长度
- 典型的NP难组合优化问题

**数据规模**
- 城市数量：12, 15, 18, 45
- 随机分布在2D网格上
- 使用Held-Karp算法求解最优解

**挑战**
- 组合爆炸：城市数量增加时解空间指数增长
- 全局优化：需要考虑所有城市的相对位置

**3. 数独求解（Sudoku）**

**任务描述**
- 填充9×9网格，满足行、列、3×3块的唯一性约束
- 需要全局一致性推理
- 错误填入会导致整个谜题无解

**数据规模**
- 难度等级：30, 40, 35（网格规模）
- 通过回溯生成有效棋盘
- 移除数字确保唯一解

**推理特点**
- 约束满足：严格遵循数独规则
- 逻辑推导：通过排除法和唯一性求解
- 全局一致性：填入必须与已有数字一致

**4. 视觉空间规划（Visual Spatial Planning, VSP）**

**任务描述**
- 在有障碍物和危险洞的环境中规划安全路径
- 实时感知和决策
- 涉及复杂的空间推理和风险评估

**数据规模**
- 地图大小：3, 4, 5, 6, 7, 8, 16, 32
- 使用Dijkstra算法计算安全路径

**环境特点**
- 安全地块：可以行走的区域
- 危险洞：必须避免的区域
- 起点-终点：需要连接的安全点

**扩展版本（VSP-Super）**
- 更大的地图尺寸（最大32×32）
- 更密集的障碍物分布
- 更高的复杂度要求

**数据统计**

如表5所示，我们构建的综合数据集总计182.4K实例，包含中间推理步骤：

| 任务 | 实例数量 | 中间步骤 | 总样本 |

| TSP | 42,000 | ~N | 420,000 |

#### 7.1.3 评估设置

我们评估模型在三种不同设置下的性能：

**设置1：零样本（Zero-Shot）**
- **描述**：没有任何任务特定训练的原始基线
- **目的**：测试模型的内在推理能力
- **代表性模型**：
  - ThinkGen（Jiao等，2025）
  - ChronoEdit（Wu等，2025）
  - Qwen3-VL-8B（Bai等，2025）
  - Qwen-Image-Edit-2511（Wu等，2025）

**设置2：任务特定训练（Task-Specific Training）**
- **描述**：在各个任务上分别进行训练和评估（如在Maze数据集上训练，仅在Maze上评估）
- **目的**：测试模型在特定任务上的最大潜力
- **DiffThinker的默认设置**

**设置3：统一训练（Unified Training）**
- **描述**：单个模型在所有任务的组合数据集（Maze+TSP+Sudoku+VSP）上训练，同时在所有任务上评估
- **目的**：测试模型是否能够学习可迁移的推理技能
- **更具挑战性**：要求模型处理广泛的问题类型

#### 7.1.4 评估指标

**主要指标**

1. **准确率（Accuracy）**
   - 解决方案的正确性
   - 是否满足所有约束条件
   - 是否达到优化目标（如最短路径）

2. **路径重复率（Path Repetition Rate）**
   - 仅适用于路径规划任务（Maze, TSP, VSP）
   - 生成路径与真实路径的重叠比例
   - 衡量空间推理的精确性

**辅助指标**

3. **推理效率**
   - 推理步数 \( \mathcal{T} \)
   - 总计算时间
   - GPU内存使用

4. **泛化能力**
   - 不同复杂度任务上的性能
   - 对输入变化的鲁棒性
   - 在未见配置上的表现

## 9. 与Spatial AGI领域的关联性

### 9.1 Spatial AGI的核心挑战

**9.1.1 空间理解的复杂性**

Spatial AGI（空间通用人工智能）面临的核心挑战包括：

**多尺度空间推理**
- 需要同时处理局部细节和全局结构
- 在不同抽象层次上进行推理
- 平衡精细感知和宏观规划

**动态环境交互**
- 环境可能随时间变化
- 需要实时感知和决策
- 适应不确定性和噪声

**多模态信息整合**
- 融合视觉、语言、其他模态信息
- 跨模态的对齐和推理
- 维持一致性

**长视距规划**
- 需要考虑长期目标和约束
- 避免短视决策
- 平衡即时需求和未来规划

**9.1.2 当前方法的局限**

现有的Spatial AGI方法通常面临以下局限：

1. **推理与生成的分离**
   - 推理和生成往往是独立的模块
   - 缺乏闭环的交互和反馈
   - 无法利用生成结果验证推理

2. **静态推理过程**
   - 推理过程往往是预先计算好的
   - 无法动态适应环境变化
   - 缺乏灵活性和适应性

3. **泛化能力有限**
   - 在训练环境上表现良好
   - 但在未见场景上性能下降
   - 缺乏真正的泛化能力

4. **计算效率问题**
   - 复杂推理需要大量计算资源
   - 难以实时应用
   - 扩展性差

### 9.2 EndoCoT对Spatial AGI的贡献

**9.2.1 内生推理机制**

**核心贡献**

EndoCoT提出的内生推理机制对Spatial AGI具有重要意义：

1. **统一的推理-生成框架**
   - 将推理和生成无缝集成
   - 形成闭环的认知-行动循环
   - 实现真正的端到端推理

2. **动态推理过程**
   - 推理过程是动态的、迭代的
   - 可以根据生成结果调整推理
   - 具有适应性和灵活性

3. **可解释的推理轨迹**
   - 推理过程是透明和可追踪的
   - 可以可视化中间推理状态
   - 便于调试和理解

**对Spatial AGI的启示**

- 空间推理应该是内生到感知-行动循环中的
- 而不是外部注入的或预计算的
- 推理和感知应该相互促进

**9.2.2 多模态协同推理**

**核心贡献**

EndoCoT实现了MLLM和DiT的协同推理，这对Spatial AGI具有启发性：

1. **认知-行动分解**
   - MLLM负责高级认知规划和语义理解

## 10. 局限性与未来方向

### 10.1 当前局限性

尽管EndoCoT在复杂视觉推理任务上取得了显著成功，但仍存在一些重要的局限性：

**10.1.1 推理步数的优化**

**问题描述**

当前方法需要手动调整最优的推理步数 \( \mathcal{T} \)。

**具体表现**
- 不同任务和实例可能需要不同的推理深度
- 过少推理步数可能导致性能不足
- 过多推理步数可能造成计算浪费
- 缺乏自动确定最优推理步数的机制

**影响**
- 降低了方法的易用性
- 可能导致次优的性能或效率
- 增加了用户的调参负担

**10.1.2 数据质量依赖**

**问题描述**

当前方法依赖于高质量的显式中间监督数据。

**具体表现**
- 需要为每个任务提供中间推理步骤的真实值
- 数据集的构建需要大量人工或算法设计
- 数据质量直接影响模型性能
- 难以扩展到缺乏中间监督的任务

**影响**
- 限制了方法的适用范围
- 增加了数据收集和标注的成本
- 阻碍了在大规模真实世界数据上的应用

**10.1.3 任务特异性**

**问题描述**

虽然统一训练显示了跨任务迁移的潜力，但当前方法仍然在某些任务上表现出特异性。

**具体表现**
- 统一训练的性能略低于任务特定训练
- 不同任务的推理模式可能存在差异
- 难以完全通用地学习推理机制
- 某些任务的推理策略可能难以迁移

**影响**
- 限制了方法的通用性
- 需要为不同任务设计不同的策略
- 难以实现真正的通用推理

**10.1.4 计算复杂度**

**问题描述**

尽管推理过程高效，但训练和推理的计算复杂度仍然较高。

**具体表现**

- 在高分辨率任务上计算成本仍然可观
- 实时应用可能面临挑战

**影响**
- 限制了在资源受限环境中的应用
- 增加了部署成本
- 难以满足实时性要求

**10.1.5 可解释性的局限**

**问题描述**

虽然推理轨迹是可追踪的，但潜在空间中的推理状态仍然难以直接解释。

**具体表现**
- 隐式token的语义不明确
- 难以直接理解中间推理步骤的含义
- 可解释性仍然有限
- 调试和分析仍然具有挑战性

**影响**
- 限制了方法在需要高可解释性场景中的应用
- 难以诊断和纠正错误
- 降低了用户的信任度

## 11. 技术亮点与创新

### 11.1 核心创新点

**11.1.1 首次实现真正的内生推理**

**创新描述**

EndoCoT是首个通过迭代潜在状态优化实现真正链式思维推理的扩散框架。

**技术突破**
1. **推理内生化**
   - 推理过程内生到扩散模型中
   - 不依赖外部预计算的推理
   - 形成端到端的推理-生成循环

2. **潜在空间推理**
   - 所有推理在潜在空间进行
   - 避免离散token的局限性
   - 实现连续优化

3. **迭代优化**
   - 通过多步迭代逐步优化解决方案
   - 每步基于前一步的结果
   - 避免过早承诺

**与传统方法的对比**

| 方法 | 推理方式 | 特点 |
|------|---------|------|
| 传统扩散模型 | 无推理 | 纯生成，缺乏推理 |
| DiffThinker | 外部注入 | 预计算推理，静态注入 |
| EndoCoT | 内生推理 | 迭代优化，动态推理 |

**优势**
- 真正的推理过程，而非模式匹配
- 动态适应，非静态决策
- 泛化能力强，不易过拟合

**11.1.2 迭代潜在状态优化**

**创新描述**

通过在MLLM中迭代优化潜在思想状态来激活推理潜力。

**技术细节**
1. **状态更新机制**
   - 递归更新潜在状态 \( \mathbf{h}_\tau \)
   - 将前一步的状态作为下一步的输入
   - 形成推理链

2. **条件化生成**
   - 每步基于当前状态生成输出
   - 实现推理与生成的交互
   - 形成闭环

3. **潜在空间操作**
   - 所有操作在高维潜在空间
   - 避免离散化的局限
   - 保持连续性

**数学表示**
\[
\mathbf{h}_\tau = \mathbf{e}_{L+1}^\top f_\phi\left([\mathbf{P}; \mathbf{h}_{\tau-1}]\right), \quad \tau = 1, \dots, \mathcal{T}
\]

**优势**
- 灵活的表达能力
- 避免错误累积
- 支持连续优化

**11.1.3 终端思想Grounding**

**创新描述**

引入辅助对齐目标，将最终推理状态与文本监督对齐。

**技术细节**
1. **参考状态计算**
   - 使用真实推理步骤计算参考状态
   - 提供明确的目标
   - 防止漂移

2. **语义对齐**
   - 使用L2损失对齐推理状态
   - 确保语义一致性
   - 维持推理轨迹

3. **仅在最终步应用**
   - 仅在最终推理步骤激活
   - 避免过度约束
   - 保持灵活性

**重要性**
- 防止推理状态的累积漂移
- 确保推理轨迹的可控性
- 提高最终输出的准确性

**消融验证**
- 移除语义损失导致性能严重下降
- 尤其在复杂任务上影响显著
- 证明了其关键作用

### 11.2 技术优势

**11.2.1 跨模态协同推理**

**优势描述**

实现MLLM和DiT的协同推理，发挥各自优势。

**协同机制**
1. **分工明确**
   - MLLM：高级认知规划和语义理解
   - DiT：低级空间定位和视觉生成
   - 形成自然的认知-行动分解

2. **联合优化**
   - 联合训练两个组件
   - 学习跨模态的协同
   - 优化整体性能

3. **信息反馈**
   - 生成结果影响下一步推理
   - 形成闭环系统
   - 相互促进

#### 12.1.2 Maze数据集

**生成算法**

使用深度优先搜索（DFS）在网格上雕刻完美连接、无环的迷宫。

**步骤**
1. **迷宫生成**
   - 在N×N网格上应用DFS生成完美迷宫
   - 确保任意两点间有且仅有一条路径

2. **起点和终点**
   - 随机选择起点s和终点g
   - 确保它们不同

3. **最短路径计算**
   - 使用广度优先搜索（BFS）计算最短路径P
   - P = (v₁, ..., v_K)

4. **中间状态记录**
   - 迭代记录路径P，模拟逐步空间探索
   - 为每个前缀P_{1:k}渲染I_k
   - 生成中间视觉状态集合S

**算法伪代码**

```python
def generate_maze(N):
    # 初始化网格图
    G = (V, E_empty) where V = {1..N}^2
    
    # DFS生成完美迷宫
    M = DFS_SpanningTree(G)
    
    # 随机选择起点和终点
    s, g = random.sample(V, 2)
    
    # BFS计算最短路径
    P = BFS(M, s, g)  # P = (v1, ..., vK)
    
    # 生成中间视觉状态
    S = empty set
    for k in range(1, K+1):
        I_k = Render(M, P[1:k])
        S.add(I_k)
    
    return Dataset instance D = (M, P, S)
```

**数据规模**
- 迷宫大小：8×8, 16×16, 32×32
- 每个规模：15,000实例
- 总计：45,000实例

#### 12.1.3 TSP数据集

**生成算法**

在2D平面上随机分布N个城市，使用Held-Karp算法求解。

**步骤**
1. **城市分布**
   - 在W×H的2D平面上随机采样N个城市
   - C = {c₁, ..., c_{Nc}}

2. **距离矩阵计算**
   - 计算所有城市对之间的欧氏距离
   - D ∈ R^{Nc×Nc}

3. **最优路径求解**
   - 使用Held-Karp动态规划算法求解
   - DP[m][i] = min_{j∈m,j≠i} {DP[m\{i}][j] + dist(j,i)}
   - 其中m是访问城市的位掩码

4. **中间状态记录**
   - 逐步渲染最优路径
   - 为每个前缀P_{1:k}渲染I_k
   - 生成中间视觉状态集合S

**算法伪代码**

```python
def generate_tsp(W, H, N_c):
    # 随机分布城市
    C = random_sample(W * H, N_c)
    
    # 计算距离矩阵
    D = compute_distance_matrix(C)
    
    # Held-Karp动态规划
    P = HeldKarpDP(D)  # 最优路径
    
    # 生成中间视觉状态
    S = empty set
    for k in range(1, N_c+1):
        I_k = RenderTrajectory(C, P[1:k])
        S.add(I_k)
    
    return Dataset instance D = (C, P, S)
```

**数据规模**
- 城市数量：12, 15, 18, 45
- 每个城市数：10,500实例
- 总计：42,000实例

#### 12.1.4 Sudoku数据集

**生成算法**

通过回溯生成有效的9×9拉丁方格，满足标准约束。

**步骤**
1. **有效棋盘生成**
   - 通过回溯生成有效的9×9棋盘L
   - 满足行、列、块的数字唯一性约束

2. **约束验证**
   - ∀i, Σ_{j=1}^9 L_{i,j} = 9
   - ∀j, Σ_{i=1}^9 L_{i,j} = 9
   - ∀block_k, Σ L ∈ block_k = 9

3. **挖洞**
   - 迭代移除数字，同时确保谜题保留唯一解
   - 如果移除后解不唯一，则恢复
   - 目标：移除H_target个数字

4. **中间状态记录**
   - 通过逆过程模拟求解
   - 逐步从真实解填充空单元格
   - 记录中间状态

**算法伪代码**

```python
def generate_sudoku(H_target):
    # 生成有效拉丁方
    L = GenerateLatinSquare()  # 9×9解
    
    # 迭代移除数字
    M = L
    h = 0
    while h < H_target:
        c = random_sample(NonEmptyCells(M))
        v = M[c]
        M[c] = empty  # 挖洞
        
        if NumSolutions(M) > 1:
            M[c] = v  # 恢复以维持唯一性
        else:
            h += 1
    
    # 模拟求解
    S = SimulateSolving(M, L)
    
    return Dataset instance D = (M, L, S)
```

**数据规模**
- 难度等级：30, 40, 35（网格规模）
- 每个难度：~11,667实例
- 总计：35,000实例

#### 12.1.5 VSP数据集

**生成算法**

生成包含安全地块、危险洞、起点和终点的FrozenLake网格。

**步骤**
1. **地图生成**
   - 生成N×N的地图M
   - 为每个位置分配类型：Safe（安全）或Hole（危险）
   - 类型分布：T(v) ~ Bernoulli(p_hole)

2. **图构建**
   - 构建安全地块的图G = (V_safe, E_safe)
   - 仅包含安全地块作为节点

3. **起点和终点**
   - 在安全地块中随机采样起点s和终点g
   - 确保存在有效路径

4. **最短路径计算**
   - 使用Dijkstra算法计算安全路径
   - 距离松弛：d(v) = min{d(v), d(u) + w(u,v)}

5. **中间状态记录**
   - 逐步记录代理的状态转换
   - 捕获环境感知和危险回避
   - 为每个前缀P_{1:k}渲染I_k

**算法伪代码**

```python
def generate_vsp(N, p_hole):
    # 生成地图
    V = {1..N}^2
    for v in V:
        T(v) = Bernoulli(p_hole)  # Safe or Hole
    
    # 构建安全地块图
    V_safe = {v in V | T(v) == Safe}
    E_safe = {(u,v) | u, v in V_safe and adjacent(u,v)}
    G = (V_safe, E_safe)
    
    # 随机选择起点和终点
    s, g = random_sample(V_safe, 2)
    
    # Dijkstra计算最短路径
    P = Dijkstra(G, s, g)
    
    assert P != empty  # 确保存在有效路径
    
    # 生成中间视觉状态
    S = empty set
    for k in range(1, len(P)+1):
        I_k = RenderEnv(M, P[1:k])
        S.add(I_k)
    
    return Dataset instance D = (M, P, S)
```

**数据规模**
- 地图大小：3, 4, 5, 6, 7, 8, 16, 32
- 每个大小：7,500实例
- 总计：60,000实例

#### 12.5.3 关键实现细节

**1. 思想状态更新**

```python
def update_thought_state(mllm, prefix, h_prev, l_pos):
    """
    更新思想状态
    
    Args:
        mllm: MLLM模型
        prefix: 前缀嵌入 [P; h_prev]
        h_prev: 前一个思想状态
        l_pos: 提取位置 (L+1)
    
    Returns:
        h_curr: 当前思想状态
    """
    # 前向传播
    outputs = mllm(prefix)
    
    # 提取L+1位置的隐藏状态
    h_curr = outputs.hidden_states[-1][:, l_pos, :]
    
    return h_curr
```

**2. 条件流生成**

```python
def conditional_flow_generation(dit, thought_state, timesteps):
    """
    条件流生成
    
    Args:
        dit: DiT模型
        thought_state: 当前思想状态
        timesteps: 采样时间步
    
    Returns:
        latent: 生成的潜在表示
    """
    # 初始化噪声
    latent = torch.randn(batch_size, channels, height, width)
    
    # 流匹配采样
    for t in timesteps:
        # 条件化于思想状态
        v_pred = dit(latent, t, thought_state)
        
        # Euler步
        alpha_t = alpha[t]
        sigma_t = sigma[t]
        d = (latent - sigma_t * v_pred) / alpha_t
        latent = latent - (sigma_{t-1} - sigma_t) * d
    
    return latent
```

**3. 终端对齐损失**

```python
def terminal_alignment_loss(h_final, h_ref):
    """
    终端对齐损失
    
    Args:
        h_final: 最终推理状态
        h_ref: 参考状态
    
    Returns:
        loss: L2对齐损失
    """
    loss = torch.nn.functional.mse_loss(h_final, h_ref)
    return loss
```
