## 核心信息

### 摘要

尽管视觉语言模型（VLMs）在2D视觉理解方面表现出色，但它们理解和推理3D空间的能力——这是空间智能的核心——仍然停留在表面。当前的方法试图通过依赖显式的3D模态或通过增加部分视图条件的几何先验来增强VLMs来弥合这一领域差距。然而，这种方法阻碍了可扩展性，并最终给语言模型增加了从稀疏线索隐式重建整体3D几何的病态任务。

在本文中，我们论证了空间智能可以从2D视觉本身中固有地出现，而不是通过显式的空间指令调整强加的。为此，我们引入了Spa3R，这是一个自监督框架，从未配位的多视角图像中直接学习统一的、视图不变的空间表示。Spa3R建立在我们提出的预测性空间场建模（PSFM）范式之上，其中Spa3R学习合成任意未预见视图的特征场，这些视图由紧凑的潜在表示条件化，从而内化对底层3D场景的整体和连贯理解。

我们进一步通过轻量级适配器将预训练的Spa3R编码器集成到现有VLMs中，形成Spa3-VLM，有效地将语言推理建立在全局空间上下文的基础上。在具有挑战性的VSI-Bench上的实验表明，Spa3-VLM在3D VQA上实现了58.6%的最先进精度，显著优于以前的方法。这些结果突显了PSFM作为推进空间智能的可扩展路径。

### 关键贡献

1. **识别了现有VLMs在空间推理中的基本瓶颈**：依赖语言模型从部分、视图条件特征隐式重建3D场景构成了低效且病态的学习目标。

2. **提出了Spa3R**：一个基于预测性空间场建模（PSFM）的自监督框架，通过合成任意新颖视图的特征场来学习统一的、视图不变的空间表示，从而内化场景的内在几何和空间布局。

3. **提出了Spa3-VLM**：将预训练的Spa3R编码器集成到VLM中，以将VLM推理建立在整体空间上下文的基础上。在VSI-Bench上的广泛实验展示了显著的性能改进，突显了PSFM作为通向通用空间智能的可扩展和有效的路径。

## 核心算法原理

### 预测性空间场建模（PSFM）理论基础

#### 1. 核心思想

Spa3R的核心理念是：**空间智能可以从2D视觉本身中固有地通过预测建模产生**，而不需要显式的空间指令调整。这与人类如何通过多视角和运动观察发展空间意识相类似。

PSFM将3D空间理解形式化为一个**空间场建模问题**。我们概念化一个3D场景为一个连续的空间特征场f，这是一个将任何由相机姿态v定义的视点映射到其对应的视图中心特征图F的函数：

```
f: V → F
```

其中V表示相机姿态的集合，F表示特征图的集合。

#### 2. PSFM的数学形式化

PSFM的目标是从NC个上下文视图C = {(vc, Fc)}c=1^NC这一稀疏集合中推断封装场景内在几何结构的低维空间流形。

我们将这个问题正式形式化为一个**神经过程（Neural Process）**：

**编码器**：我们使用摊销编码器Eφ将上下文集C编码成一个统一的潜在变量z。这个变量z作为空间流形的整体表示，参数化场景的内在属性。

```
z = Eφ(C) = Eφ({(vc, Fc)}c=1^NC)
```

**解码器**：解码器Dθ作为一个条件神经场，重建任意目标姿态vt的目标视图特征F̂t，条件化于z：

```
F̂t = Dθ(vt | z)
```

**训练目标**：在训练阶段，我们从每个场景采样一组视图S，并将其随机分割为上下文集C和目标集T。学习目标是最小化预测特征F̂t和它们的真实对应物Ft之间的距离：

```
L_PSFM = E_{C,T~S}[∑_{t∈T} dist(Dθ(vt | Eφ(C)), Ft)]
```

#### 3. 信息瓶颈机制

理论上，PSFM建立了一个信息瓶颈，将**内在空间表示**与**外在的、视图条件的属性**解耦。通过强制编码器Eφ通过单个潜在z支持流形上任意点处的特征合成，模型被迫内化场景的完整3D几何和空间布局。

这个预测性信息瓶颈迫使编码器必须捕获：
- 场景的内在3D几何
- 空间布局
- 语义关系

从而产生超出观察图像的整体理解。

### Spa3R架构详解

Spa3R通过一个专门的编码器-解码器架构实例化PSFM范式，该架构由以下三个主要组件组成：

#### 1. 非对称视图聚合器（Asymmetric View Aggregator）

**设计动机**：PSFM的一个关键先决条件是构建一个规范的空间特征场f，其中上下文特征Fc和目标特征Ft在空间上对齐。

**实现方式**：
- 有效适配预训练的VGGT（Visual Geometry Grounded Transformer），利用其强大的全局注意力机制提取空间对齐的特征
- 采用**非对称注意力掩码策略**：在VGGT的全局注意力层中，严格防止信息从目标视图泄漏到上下文视图

**注意力掩码**：
给定一批上下文视图C和目标视图T，我们构造一个应用于注意力对数的视图掩码M ∈ {0, -∞}^(L×L)，其中L表示总序列长度。具体来说：

- 目标视图被允许关注所有视图（C ∪ T）
- 上下文视图被限制只能关注其他上下文视图

对于来自视图i的查询补丁和来自视图j的键补丁：

```
M_ij = {
  0, if i ∈ T or j ∈ C
  -∞, otherwise
}
```

这个机制确保：
- 上下文特征Fc独立于目标被计算
- 目标特征Ft及其对应的相机姿态vt在相同的坐标系中空间对齐

#### 2. Spa3R编码器

Spa3R编码器Eφ是一个Transformer，设计用于将上下文特征Fc封装成紧凑的表示z ∈ R^(Nq×D)。

**实现细节**：
- 初始化Nq个可学习的查询嵌入q
- 这些查询与上下文特征Fc连接成一个单一序列
- 通过Transformer层处理以迭代地细化查询，通过从上下文聚合信息产生最终的空间表示z

```
H = Transformer(Concat[q, Fc])
z = H[:Nq]
```

**关键特性**：
- 使用可学习查询作为抽象的"槽位"（slots）
- 每个槽位通过注意力机制从所有上下文视图中提取相关信息
- 产生的z是一个统一的、视图不变的场景表示

#### 3. Spa3R解码器

Spa3R解码器Dθ合成目标特征F̂t，条件化于空间潜在z和目标相机姿态vt。这个过程涉及两个关键的几何机制：

**机制1：基于射线的查询**

为了编码目标视锥的嵌入，我们为每个齐次像素坐标ũ使用估计的内参K生成相机空间射线方向d：

```
d = Normalize(K^(-1) · ũ)
```

这些射线方向d通过线性投影映射到初始相机嵌入r。

**机制2：相对3D位置编码**

为了显式建模目标视图和空间上下文之间的几何关系，我们在解码器中采用PRoPE（Positional Relative Pose Encoding）。PRoPE直接将相机之间的相对变换编码到注意力机制中。

注意力的计算：

```
Oi = ∑_j softmax(Q_i^T · T_ij · K_j / √d) · T_ij · V_j

T_ij = D_i^PRoPE · (D_j^PRoPE)^(-1)
```

其中D^PRoPE是封装3D投影矩阵和视图内2D RoPE的相对位置编码矩阵。

**特征合成**：

目标视图相机嵌入r然后通过Transformer解码器Dθ查询空间上下文z，以合成目标特征F̂t：

```
H' = Transformer(Concat[r, z])
F̂t = H'[:-Nq]
```

### 损失函数设计

Spa3R框架端到端训练以最小化目标特征的重建误差。具体来说，目标特征由以下组成：

1. **几何特征**：来自非对称视图聚合器
2. **语义特征**：来自冻结的DINOv3骨干网络

语义特征的结合作为辅助目标，明确鼓励编码器与几何结构一起捕获高级语义抽象。

**损失函数**：

解码器采用独立的几何和语义预测头。我们对两个头应用一致的损失公式，结合L1距离和余弦相似性：

```
L(F̂t, Ft) = ||F̂t - Ft||_1 + (1 - (F̂t · Ft) / (||F̂t||_2 · ||Ft||_2))
```

总训练目标是几何和语义头的重建损失之和。

### 训练策略

**数据采样**：
- 每个迭代从场景采样4到12个视图
- 随机指定一半视图为通过非对称注意力掩码的上下文视图
- 其余视图作为目标视图

**优化配置**：
- 优化器：AdamW
- 学习率：1 × 10^(-3)
- 训练步数：80K步
- 硬件：8块NVIDIA 5090 GPU

**模型配置**：
- 编码器和解码器：6层Transformer
- 隐藏维度：D = 768
- 可学习查询数：Nq = 256

## 与Spatial AGI的关系

### 空间表示的理解方式

Spa3R提出了一种根本不同的空间理解和表示方式，这对Spatial AGI（通用空间智能）有重要启示：

#### 1. **从2D到3D的自发涌现**

传统观点认为空间智能需要：
- 显式的3D模态（如LiDAR点云）
- 大量空间标注数据（空间问答对）
- 显式的空间指令调整

Spa3R的核心洞见是：**空间智能可以从2D视觉本身中固有地通过预测建模产生**。这类似于：
- 人类如何通过多视角和运动观察发展空间意识
- 婴儿如何在无明确教学的情况下理解空间关系

这对Spatial AGI的意义：
- 降低了空间智能的门槛：不需要昂贵的3D传感器
- 提高了可扩展性：可以使用广泛可用的2D图像/视频数据
- 符合生物启发：更接近人类的空间学习方式

#### 2. **统一的、视图不变的空间表示**

Spa3R学习的是一个统一的潜在表示z，它：
- 封装了整个3D场景的内在几何和语义
- 不依赖于任何特定视图
- 可以条件化合成任意新视图的特征

这与Spatial AGI的目标高度一致：
- **场景理解**：建立对整个环境的全局模型
- **泛化能力**：在新视角下保持稳定的空间理解
- **可组合性**：表示可以用于多种下游任务

#### 3. **预测性学习的本质**

PSFM范式将空间学习转化为一个预测任务：
- 给定部分视图，预测其他任意视图的特征
- 通过这个预测性目标，模型被迫学习内在的空间结构

这种方式的优点：
- **自监督**：不需要人工标注的空间标注
- **内在驱动的目标函数**：预测任务本身驱动学习，而非外在任务
- **可扩展**：可以从大量未配位的多视角图像中学习

### 空间关系的处理机制

Spa3R通过多种机制处理空间关系：

#### 1. **全局注意力机制**

非对称视图聚合器使用全局注意力：
- 允许跨视图的信息聚合
- 通过注意力掩码防止目标信息泄漏
- 在统一的坐标系中对齐所有视图

这实现了：
- **跨视角一致性**：同一物体在不同视角的表示保持一致
- **空间对齐**：特征在统一的3D坐标系中表示
- **上下文聚合**：可以从多个视角提取全局上下文

#### 2. **相对几何编码**

解码器使用PRoPE（相对位置编码）：
- 直接将相机之间的相对变换编码到注意力权重中
- 使用3D投影矩阵和视图内2D RoPE

优势：
- **几何一致性**：特征合成尊重实际的相机几何关系
- **尺度不变性**：相对变换对场景尺度不敏感
- **坐标原点无关**：不依赖绝对坐标系统

#### 3. **射线基础查询**

通过相机空间射线方向查询空间场：
```
d = Normalize(K^(-1) · ũ)
```

这实现了：
- **视角条件化**：可以为任意视角生成特征
- **物理几何对应**：特征与实际的成像几何对应
- **灵活采样**：可以合成任意未观察到的视角

### 对Spatial AGI的启发

#### 1. **空间表示学习与推理的解耦**

Spa3R的设计哲学：
- 空间表示学习（Spa3R编码器）与推理（VLM）分离
- 预训练的Spa3R编码器可以作为即插即用的模块
- 保留了基础VLM的泛化能力

- **模块化架构**：空间感知和空间推理可以分开开发和优化

Spa3R-VLM使用轻量级残差交叉注意力适配器：


关键设计：
- **交叉注意力**：VLM的视觉特征可以主动查询空间上下文
- **残差连接**：保留VLM的预训练知识
- **轻量级**：只增加少量参数，易于集成

对Spatial AGI的启发：
- **主动查询**：模型可以根据需要主动提取相关空间信息
- **知识保留**：添加空间能力不应破坏原有能力

#### 3. **评估基准的重要性**

Spa3R使用多个空间推理基准进行评估：
- **VSI-Bench**：主要基准，包含5,000+问答对
- **CV-Bench**：交叉领域泛化能力

对Spatial AGI的启发：

- **真实场景**：使用真实世界的室内/室外场景
- **泛化能力**：在不同领域和任务上测试泛化

## 预测性空间场相比传统3D表示的优势

### 传统3D表示的局限性

在深入分析Spa3R的优势之前，我们需要理解传统3D表示方法的局限性：

#### 1. **依赖显式3D模态**

**LiDAR点云方法**：
- 需要昂贵的专用传感器
- 数据采集成本高
- 难以大规模应用
- 在某些环境下不可用（如室内、水下）

**3D网格/体素表示**：
- 需要预先的3D重建
- 表示复杂度高
- 难以实时处理
- 存储和传输开销大

**隐式表示（如NeRF）**：
- 需要每个场景单独训练
- 计算开销大
- 难以泛化到新场景
- 训练时间长

#### 2. **部分、视图条件特征的不足**

当前的一些方法（如VG-LLM、Spatial-MLLM、VLM3R）：
- 只提供部分、视图条件的几何先验
- 受计算约束，只能处理有限数量的视图
- 给VLM留下大量未被观察到的空间上下文

**根本问题**：
- VLM被要求从稀疏、局部的视觉标记隐式重建整体3D场景
- 这在空间指令调整的稀疏监督下是一个病态问题
- 语言模型没有被设计来做这种3D重建任务

#### 3. **可扩展性问题**

**数据需求**：
- 需要大量空间标注数据（空间问答对）
- 标注成本高
- 难以覆盖所有可能的空间场景

**计算需求**：
- 处理3D数据需要大量计算资源
- 实时应用困难
- 难以在边缘设备上部署

**泛化能力**：
- 在训练数据分布外性能下降
- 难以适应新环境
- 场景特定优化难以泛化

### PSFM的核心优势

#### 1. **统一的全局表示**

**视角不变性**：
- 潜在表示z不依赖于任何特定视图
- 封装了整个3D场景的全局信息
- 可以用于所有视图的条件化特征合成

**全局一致性**：
- 所有特征在统一的坐标系中对齐
- 跨视角的语义和几何信息保持一致
- 不会出现视图间的矛盾或冲突

**可组合性**：
- 同一个表示可以用于多种任务
- 支持不同粒度的空间查询
- 可以灵活地集成到下游应用

#### 2. **预测性驱动的学习**

**信息瓶颈机制**：
- 通过单个紧凑潜在z支持任意视图的特征合成

## 个人思考

### 最有趣的发现

#### 1. **从2D到3D的自发涌现**

Spa3R最令人兴奋的发现是：**空间智能可以从纯2D视觉中自发涌现**，而不需要显式的3D监督或3D传感器。这与传统的直觉相悖，但与人类的实际学习方式一致。

**深层含义**：
- 这表明3D空间理解可能是2D视觉推理的自然延伸，而不是一个完全独立的模态
- 可能存在一个"空间归纳偏置"（spatial inductive bias），模型可以通过预测性学习自动学习到
- 这为构建更通用的人工智能提供了一个新方向：不是为每个模态设计专门的架构，而是让模型通过目标函数自然地学习到所需的表示

**启发**：
- 我们应该更多地关注**目标函数设计**，而不是架构设计
- 预测性目标可能是一个通用的学习范式，不仅适用于空间，还适用于其他领域
- "涌现"能力（emergent capabilities）可能比我们想象的更普遍和可控

#### 2. **信息瓶颈作为学习驱动力**

Spa3R的核心机制是使用预测性信息瓶颈迫使编码器学习全局空间表示。这个设计巧妙而强大。

**深层含义**：
- 信息瓶颈不仅仅是一个约束，而是学习的驱动力
- 通过设计适当的瓶颈（如单个紧凑潜在表示），我们可以引导模型学到我们想要的结构
- 这提供了一种控制模型学习的方法：不直接告诉模型要学什么，而是通过任务设计让模型自然地学到

**启发**：
- 我们可以设计各种信息瓶颈来引导模型学习不同的表示
- 例如，时间瓶颈可以学习动态表示，因果瓶颈可以学习因果关系
- 这为表示学习提供了一个统一的框架

#### 3. **查询机制的重要性**

Spa3R-VLM使用交叉注意力让VLM主动查询空间上下文，而不是被动地接收固定的空间特征。这个设计选择带来了显著的性能提升。

**深层含义**：
- 被动特征融合（如简单拼接）可能导致"模态崩溃"（modality collapse）
- 主动查询机制让模型根据需要提取相关信息，更接近人类的认知过程
- 注意力机制可能是一个通用的信息整合机制，适用于多种模态融合场景

**启发**：
- 在多模态融合中，我们应该更多地考虑让模型主动查询，而不是被动接收
- 不同的下游任务可能需要不同的查询策略
- 我们可以学习如何进行查询，而不仅仅是学习查询什么

### 最意外的结果

#### 1. **语义特征的显著贡献**

我最初认为空间推理主要依赖几何信息，但实验表明语义特征的贡献几乎与几何特征同等重要。

**数据分析**：
- 仅几何：57.5%
- 仅语义：56.7%
- 几何+语义：58.6%

**深层含义**：
- 空间推理不仅仅是"距离在哪里"，还涉及"是什么"、"有什么"等语义理解
- 空间和语义可能是深度耦合的，不能分开处理
- 高级空间推理（如理解场景中的"舒适度"）可能更依赖语义而非几何

**启发**：
- 我们不应该将"空间"狭义地理解为"几何"
- Spatial AGI可能需要同时处理几何、语义、甚至情感维度
- 空间理解可能需要一个多层次的表示系统

#### 2. **50%掩码比例的最优性**

实验表明50%的掩码比例（一半上下文，一半目标）达到最优性能，这比我预期的要低。

**数据分析**：
- 25%掩码：57.5%
- 50%掩码：58.6% (最优)
- 75%掩码：58.1%

**深层含义**：
- 模型不需要看到"足够多"的视图就能学习全局表示
- 预测任务的难度需要适中：太简单无法强制学习，太难无法形成稳定表示
- 这与人类的视觉学习相似：我们不需要看到物体的所有角度就能形成整体理解

**启发**：
- 在自监督学习中，任务难度是一个关键的超参数
- 我们可能需要自适应地调整任务难度
- "最佳难度"可能因数据分布和任务而异

#### 3. **PRoPE的显著优势**

相对位置编码（PRoPE）比绝对位置编码（Plücker坐标）表现更好，这个结果在预料之中，但优势如此显著（+1.0%）令人惊讶。

**深层含义**：
- 绝对位置编码对场景尺度和坐标原点非常敏感
- 相对关系是更本质的几何属性
- 在实际应用中，相机参数可能有噪声或误差，相对编码更鲁棒

**启发**：
- 在几何建模中，我们应该优先考虑相对关系
- 绝对坐标可能只是"参考系"的约定，而非本质属性
- 我们应该设计更多基于相对关系的表示方法

### 最有价值的启发

#### 1. **表示学习与推理的解耦**

Spa3R将空间表示学习（Spa3R编码器）与推理（VLM）完全解耦，这是一个强大的架构范式。

**价值**：
- 每个模块可以独立优化和扩展
- 表示学习不需要依赖特定的下游任务
- 可以将表示用于多个不同的应用

**更广泛的应用**：
- 不仅是空间，其他领域（如时间、因果、逻辑）也可以采用类似架构
- 可以构建一个"表示库"，不同的下游任务可以复用这些表示
- 这提供了一个新的AI系统设计范式：专用表示 + 通用推理

#### 2. **自监督学习的威力**

Spa3R完全通过自监督学习，不需要任何人工标注的空间数据，却达到了最先进的性能。

**价值**：
- 大幅降低了开发成本
- 可以利用大量无标注数据
- 模型可以持续学习和改进

**对未来AI的启发**：
- 我们应该更多地关注如何设计自监督任务
- 标注数据可能成为瓶颈，而非必需品
- 自监督可能是通向通用AI的关键路径

#### 3. **即插即用的空间模块**

预训练的Spa3R编码器可以作为即插即用的模块集成到任意VLM，这是一个非常实用的设计。

**价值**：
- 快速为现有VLM添加空间能力
- 不需要重新训练整个模型
- 降低了使用门槛

**对AI生态的启发**：
- 我们可以构建一个"模块生态"，不同的模块（空间、时间、因果等）可以组合
- 这类似于软件工程中的"组件"或"库"
- 可能形成一个新的AI开发范式：模块组装而非端到端训练

### Q3的思考过程

**为什么选择这个问题？**

我选择了"预测性空间场相比传统3D表示的优势是什么？"作为第三个问题，原因如下：

1. **学术价值**：这个对比可以帮助我们理解PSFM范式的本质创新，以及它如何克服传统方法的局限性。

2. **实践意义**：理解这些优势有助于我们在实际应用中选择合适的方案，以及设计更好的空间表示方法。

3. **方法论启示**：这个对比揭示了预测性学习和自监督学习的威力，这些启示可以应用到其他领域。

4. **挑战和未来方向**：通过分析优势，我们也可以看到当前的局限和未来的改进方向。

**分析维度**：

我尝试从多个维度分析这个对比：
- 理论基础：信息论、神经科学、认知科学的视角
- 技术实现：架构、算法、优化等方面的差异
- 实验验证：性能、泛化、消融研究结果
- 实际部署：成本、效率、维护等工程考虑
- 未来发展：可能的改进方向和潜在应用

**关键洞察**：

通过这个对比，我获得了几个关键洞察：
1. 表示的质量（全局vs局部）可能比表示的显式性（3D vs 2D）更重要
2. 目标函数设计（预测性 vs 监督）可能比架构设计（3D模块 vs 2D模块）更关键
3. 可扩展性（数据、模型、任务）是实际应用的核心考虑因素

**个人思考**：

这个问题让我思考了一个更深刻的问题：什么是"好的表示"？传统观点认为好的表示应该显式、精确、可解释。但Spa3R表明，一个好的表示可能应该是：
- 压缩的（通过信息瓶颈）
- 隐式的（不需要显式3D结构）
- 可组合的（可以用于多种任务）
- 可学习的（通过自监督获得）

这挑战了我们对"表示"的传统理解，也为未来的研究提供了新方向。

## 总结

### 核心发现总结

Spa3R通过预测性空间场建模（PSFM）范式，提出了一种全新的空间表示学习方法。其核心发现包括：

1. **空间智能可以从纯2D视觉中自发涌现**，不需要显式的3D传感器或空间标注数据。这与人类的视觉学习方式一致。

2. **预测性信息瓶颈**是学习全局空间表示的有效机制。通过迫使模型从单个紧凑潜在表示合成任意视图的特征，模型被迫学习场景的内在3D几何和空间布局。

3. **统一、视图不变的空间表示**比部分、视图条件的特征更有效。这种表示封装了整个场景的全局信息，可以用于多种下游任务。

4. **几何和语义信息同样重要**。健壮的空间推理需要同时处理显式几何结构和高级语义上下文。

5. **相对位置编码**比绝对姿态编码更鲁棒。对场景尺度和坐标原点变化不敏感，更适合实际应用。

### 对Spatial AGI的意义

Spa3R对通用空间智能（Spatial AGI）的发展具有重要意义：

1. **降低门槛**：不需要昂贵的3D传感器，可以使用广泛可用的2D图像/视频数据，提高了空间智能的可及性。

2. **提高可扩展性**：可以从互联网规模的数据中学习，模型可以持续改进，理论上没有上限。

3. **提供新范式**：预测性学习可能是一个通用的空间学习范式，可以应用于各种空间任务。

4. **架构启示**：表示学习与推理的解耦、即插即用模块、自适应查询等设计对构建Spatial AGI系统具有启发意义。

### 技术贡献

1. **提出PSFM范式**：预测性空间场建模是一个全新的空间表示学习方法，结合了神经过程、信息论和计算机视觉的洞察。

## 相关工作

### 3D重建和场景表示

**神经辐射场（NeRF）**：
- 使用隐式神经场表示场景
- 需要每个场景单独训练，计算开销大
- 为后续的泛化方法奠定了基础

**像素NeRF（pixelNeRF）和MVSNeRF**：
- 引入条件神经场学习跨多个场景的泛化3D先验
- 实现前馈推理，无需每个场景优化
- 推动了3D重建从场景特定优化到通用化前馈推理的转变

**3D Gaussian Splatting**：
- 使用3D高斯表示场景，高效渲染
- 为Spa3R的几何特征提取提供了基础

**占据预测**：
- 预测3D空间中的占据情况
- 提供另一种场景表示方式

**几何基础模型（DUSt3R、VGGT）**：
- 统一估计多样化的3D属性（点图、深度、相机参数）
- 从未配位图像中进行鲁棒的3D重建
- 为Spa3R的非对称视图聚合器提供了预训练基础

**纯Transformer架构用于新视角合成（LVSM及其变体）**：
- 舍弃显式3D归纳偏置，利用Transformer的表示能力从2D数据隐式推断几何结构
- Spa3R受到这种预测性范式的启发，但目标不同：
  - LVSM：像素级合成，高保真视觉重建
  - PSFM：表示学习，预测空间对齐的特征场

### 视觉语言模型用于3D推理

**显式3D模态方法**：
- **以物体为中心的表示**：用3D物体表示锚定语言
- **点云处理**：直接处理点云以将语言建立在3D空间中
- **优点**：强的几何基础
- **缺点**：依赖专用传感器（如LiDAR）或预处理的3D数据，限制了可扩展性和适用性

**从2D视频或多视角数据推断3D理解**：
- **GPT4Scene**：从视频特征重建鸟瞰图（BEV）表示，捕获时空上下文
- **VG-LLM**：从几何基础模型提取的视图条件先验增强VLM
- **Spatial-MLLM**：类似VG-LLM，使用视图条件先验
- **VLM3R**：与指令对齐的3D重建增强VLM
- **局限**：通常只向VLM暴露部分、视图条件的几何线索，将整体3D重建的负担转移给语言模型

**Spa3R的突破**：
- 引入预测性空间场建模（PSFM），一个自监督范式
- 明确学习统一的、视图不变的空间表示
- 提供鲁棒和连贯的基础，支持复杂的空间推理

## 附录：详细技术细节

### A. 数学推导

#### A.1 神经过程视角

PSFM可以形式化为一个神经过程，其目标是从上下文集C推断潜在表示z，然后用z预测目标集T。

**条件分布**：
```
p(FT | FC) = ∫ p(FT | z) p(z | FC) dz
```

**变分推断**：
我们使用摊销推断来近似p(z | FC)：
```
qφ(z | FC) = N(z; μ(FC), diag(σ²(FC)))
```

**变分下界（ELBO）**：
```
log p(FT | FC) ≥ E_{z~qφ(z|FC)}[log pθ(FT | z) - KL(qφ(z|FC) || p(z))]
```

Spa3R简化了这个框架，通过确定性编码器和预测性损失：
```
z = Eφ(FC)
L = E[||Dθ(FT | z) - FT||]
```

#### A.2 信息瓶颈理论

从信息论角度，PSFM建立了一个信息瓶颈：
```
I(FC; z) ≤ I(FC; FT)
```

其中：
- I(FC; z)：编码器压缩的信息量
- I(FC; FT)：上下文和目标之间的互信息

通过最小化这个瓶颈，模型被迫学习最"有用"的空间表示。

**最优表示**：
最优的z应该满足：
```
max_z I(z; FT) - β I(z; FC)
```

其中β控制压缩率。

#### C.2 子任务说明

5. **相对距离**：两个物体之间的距离
6. **相对方向**：一个物体相对于另一个的方向
7. **路径规划**：从一个位置到另一个位置的路径
8. **接近顺序**：按照某种标准（如距离）排列物体的顺序
