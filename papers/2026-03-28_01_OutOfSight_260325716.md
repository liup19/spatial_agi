## 摘要

视频世界模型在模拟物理世界方面展现出了巨大的潜力，然而现有的记忆机制主要将环境视为静态画布。当动态主体（如行人、动物）隐藏在镜头之外稍后重新出现时，当前的方法往往难以应对，导致主体冻结、扭曲或消失。为了解决这个问题，我们引入了Hybrid Memory（混合记忆）这一新范式，要求模型同时作为静态背景的精确档案管理员和动态主体的警惕追踪者，确保主体在视野外的时间间隔内保持运动连续性。为了促进这一方向的研究，我们构建了HM-World，这是首个专门为混合记忆设计的大规模视频数据集。该数据集包含59K个高保真视频片段，具有解耦的相机轨迹和主体轨迹，涵盖17个多样化场景、49个不同主体，以及精心设计的进入-退出事件以严格评估混合一致性。此外，我们提出了HyDRA（Hybrid Dynamic Retrieval Attention），一种专用记忆架构，通过压缩记忆到token并利用时空相关性驱动的检索机制。通过选择性关注相关的运动线索，HyDRA有效地保留了隐藏主体的身份和运动。在HM-World上的广泛实验表明，我们的方法在动态主体一致性和整体生成质量方面显著优于最先进的方法。

## 研究问题和动机

### 1.1 当前视频世界模型的局限性

视频世界模型（Video World Models）近年来在模拟物理世界方面取得了显著进展，特别是在以下领域展现了巨大潜力：
- **自动驾驶**: 生成高保真的驾驶场景，提供实时环境模拟
- **具身智能**: 为机器人和智能体提供环境交互的基础
- **游戏开发**: 创造动态、可交互的游戏世界

然而，现有的视频世界模型存在一个共同的盲点：**将世界视为静态画布**。这些方法在记忆和重构静止环境方面表现出色，但现实世界是一个繁忙、动态的舞台， populated by subjects（如行走的行人、奔跑的动物）等主体，它们遵循独立的运动逻辑。

### 1.2 动态主体隐藏带来的挑战

当动态主体隐藏在相机视野之外时，现有的记忆机制会失去对它们的追踪，导致以下问题：

1. **主体冻结**: 当主体重新出现时，保持之前的静止状态，无法恢复运动
2. **主体扭曲**: 主体变形、变形或产生不自然的动作
3. **主体消失**: 主体完全从画面中消失，无法重新出现

这些问题严重影响了视频生成质量，特别是在需要长期连贯性和动态一致性的应用中。

### 1.3 混合记忆的需求

为了解决上述问题，论文提出了**混合记忆（Hybrid Memory）**这一新范式，其核心要求是：

- **静态背景的一致性**: 模型需要精确地记忆和重建静止的背景元素（建筑物、道路、环境）
- **动态主体的连续性**: 模型需要持续追踪和预测动态主体（行人、动物）的运动轨迹
- **视野外的预测**: 当主体隐藏时，模型必须在潜在空间中隐式地模拟其运动

这要求模型同时执行两个看似矛盾的任务：
1. 精确地归档静态背景
2. 警惕地追踪动态主体

### 1.4 缺失的研究基础

当前在混合记忆研究方面存在以下空白：

1. **缺乏专门的数据集**: 现有的视频数据集大多集中在静态场景或简单动态环境，缺乏专门设计的进入-退出事件
2. **缺乏专门的方法**: 现有记忆方法主要针对静态场景或简单动态环境，未针对复杂动态场景和混合记忆进行优化
3. **缺乏评估基准**: 缺乏专门评估混合记忆能力（特别是动态主体在视野外的连贯性）的指标

## 核心方法和技术细节

## 4.1 基础架构和相机注入

### 4.1.1 整体架构

HyDRA方法基于一个全序列视频扩散模型构建，主要组件包括：

1. **因果3D VAE**: 用于编码和解码视频潜在表示
2. **Diffusion Transformer (DiT)**: 核心生成网络
3. **动态检索注意力模块**: 替代标准3D自注意力层
4. **投影器和前馈网络**: 标准Transformer组件

模型采用**Flow Matching**作为去噪目标，而不是传统的扩散模型。

### 4.1.2 流匹配学习目标

给定一个视频序列，3D VAE将其编码为潜在表示：
```
z0 ∈ ℝ^(C × f × h × w)
```

其中：
- C: 通道数
- f: 时间维度（帧数）
- h, w: 空间分辨率

在训练阶段，在时间步t∈[0,1]上的噪声潜在表示zt通过z0和标准高斯噪声z1∼𝒩(0,I)之间的线性插值获得：
```
zt = (1-t)z0 + tz1
```

模型学习预测在时间步t上的速度场：
```
vt = z0 - z1
```

损失函数为：
```
ℒθ = 𝔼[z0, z1, t] ‖u(zt, t; θ) - vt‖²
```

其中θ表示模型参数。

在推理阶段，随机采样的高斯噪声通过迭代去噪过程逐步恢复为干净的潜在表示，然后由3D VAE解码器解码为视频序列。

### 4.1.3 相机注入机制

为了实现精确的空间控制，模型将相机轨迹作为显式条件注入：

相机姿态序列P = {(Ri, ti)}i=1^f，其中：
- Ri ∈ ℝ^(3×3): 第i帧的旋转矩阵
- ti ∈ ℝ^3: 第i帧的平移向量

将姿态参数扁平化并连接形成统一的相机条件：
```
ccam ∈ ℝ^(f × 12)
```

使用MLP层实现的相机编码器Ec_CAM(ccam)对条件进行编码，然后将编码后的特征广播到空间维度，与潜在特征逐元素相加：
```
Hout = Hin + Ec_CAM(ccam)
```

这确保了模型能够根据相机轨迹生成正确视角的内容。

## 4.2 记忆标记化（Memory Tokenization）用于检索

### 4.2.1 问题的本质

在HyDRA框架中，编码的记忆潜在表示Z_mem作为记忆的主要表示。然而，直接将整个Z_mem注入生成过程存在以下问题：

1. **计算开销**: 需要处理大量潜在表示
2. **噪声干扰**: 可能引入不相关的噪声，误导模型的推理路径
3. **空间和时间不一致**: 未经处理的潜在表示缺乏对时空关系的明确建模

### 4.2.2 3D卷积记忆标记化器

为了克服这些限制，论文引入了基于3D卷积的记忆标记化器𝒯_mem，设计用于同时处理空间和时间维度。

**核心思想**: 通过3D卷积扩展时空感受野，捕获长期运动信息，将原始潜在表示压缩为密集的、时空感知的记忆token。

**数学定义**:
```
M = 𝒯_mem(Z_mem)
M ∈ ℝ^(C' × f'_mem × h × w)
```

其中：
- f'_mem: 时间维度（压缩后的时间步数）
- h × w: 下采样的空间分辨率
- C': 新的通道数（通过卷积核大小调整）

### 4.2.3 卷积核设计

记忆标记化器使用3D卷积核，其中：
- **时间维度kernel (T)**: 例如 2×4×4, 4×4×4, 8×8×8
- **空间维度kernel (H×W)**: 例如 2×2, 4×4, 8×8

实验结果表明：
- 空间维度对性能影响较小（2×2到8×8之间PSNR下降仅0.244）
- 时间维度至关重要（从4×4×4到1×4×4，PSNR下降1.281）

这证明了在tokenization过程中进行时空交互对于捕获长期动态信息的必要性。

### 4.2.4 优势

记忆标记化器带来了以下优势：

1. **特征增强**: 3D卷积提供了更丰富的运动感知表示
2. **降噪能力**: 通过卷积聚合，过滤掉不相关的上下文
3. **计算效率**: 压缩后的token数量显著减少
4. **时空对齐**: 同时捕获空间和时间关系

## 4.3 动态检索注意力（Dynamic Retrieval Attention）

### 4.3.1 检索机制的重要性

如4.2节所述，无选择地注入所有历史上下文会降低视频一致性并增加计算成本。因此，一个高效的检索机制对于优化信息流至关重要。

### 4.3.2 动态检索注意力的设计

动态检索注意力是一种时空引导的检索方法和记忆机制，直接替代基线模型中的标准3D自注意力层。

#### 输入定义

- **目标潜在表示**: Z_tgt ∈ ℝ^(C' × f_tgt × H' × W')
- **记忆token**: M ∈ ℝ^(C' × f'_mem × h × w)

#### 步骤1: 投影

将输入投影为Query、Key和Value：

- 目标潜在表示 → Queries Q
- 记忆token → Keys K_mem 和 Values V_mem

#### 步骤2: 空间池化

由于目标潜在表示和记忆token具有不同的空间分辨率，首先对目标query进行空间池化以对齐：

```
q̃_i ∈ ℝ^(C' × h × w)
```

#### 步骤3: 时空亲和度计算

计算目标query与每个记忆token时间切片之间的时空亲和度：

```
S(i, j) = (1/√d) Σ_(y=1)^h Σ_(x=1)^w ⟨q̃_i(x, y), k_mem,j(x, y)⟩
```

其中：
- ⟨⋅,⋅⟩: 通道间内积
- d: 通道维度（用于缩放）

亲和度S(i, j)有效量化了当前目标潜在表示与记忆token之间的时空对应关系。

#### 步骤4: Top-K选择

基于亲和度进行Top-K选择，筛选出与当前query最相关的记忆token：

```
ℐ_i = TopK(S_i, K)
K_sel = {k_mem,j | j ∈ ℐ_i}
V_sel = {v_mem,j | j ∈ ℐ_i}
```

其中ℐ_i表示最相关的K个记忆token的索引。

#### 步骤5: 局部窗口合并

为了保持局部去噪的稳定性，将检索到的记忆特征与局部时间窗口的特征合并：

```
K'_i = [K_sel, K_loc]
V'_i = [V_sel, V_loc]
```

其中K_loc和V_loc来自以当前帧为中心的局部窗口𝒲_i内的相邻潜在表示。

#### 步骤6: 标准注意力计算

最终计算动态检索注意力：

```
Attention(q_i, K'_i, V'_i) = Softmax(q_i(K'_i)^T / √d) V'_i
```

### 4.3.3 检索token长度的影响

论文通过实验研究了检索token长度的影响：

- **检索5个token**: PSNR = 19.309，性能较差，信息丢失严重
- **检索10个token**: PSNR = 20.357，最优配置
- **检索15个token**: PSNR = 20.333，性能接近最优，无明显优势

这表明适度的token数量（10个）能够提供必要的时空信息，同时避免引入冗余噪声。

### 4.3.4 检索方法对比

论文对比了两种检索方法：

1. **FOV重叠检索**: 基于视野重叠度计算相关性
2. **动态亲和度检索**: 基于Q-K交互的精细时空相关性

实验结果（表6）表明动态亲和度检索显著优于FOV方法：
- PSNR: 19.776 → 20.357（提升0.581）
- DSCctx: 0.820 → 0.827（提升0.007）
- DSCGT: 0.844 → 0.849（提升0.005）
- Subject Consistency: 0.908 → 0.926（提升0.018）

这证明了利用QK交互评估细粒度时空相关性的优势。

## 4.4 HyDRA框架完整工作流程

### 4.4.1 训练阶段

1. **输入**: 77个上下文帧 + 相机轨迹
2. **编码**: 3D VAE编码为潜在表示z0
3. **噪声注入**: 生成噪声潜在表示z1
4. **记忆构建**: 
   - 编码历史上下文为记忆潜在表示Z_mem
   - 使用记忆标记化器转换为记忆token M
5. **目标生成**: 针对目标帧生成噪声潜在表示zt
6. **检索**: 
   - 对目标潜在表示进行空间池化
   - 计算与记忆token的亲和度
   - 选择Top-K相关token
   - 与局部窗口特征合并
7. **去噪**: 使用动态检索注意力预测速度场v
8. **损失计算**: ℒθ = 𝔼 ‖u(zt, t; θ) - vt‖²

### 4.4.2 推理阶段

1. **输入**: 相机轨迹 + 初始噪声
2. **去噪**: 迭代应用动态检索注意力模块
3. **输出**: 干净的视频潜在表示
4. **解码**: 3D VAE解码器解码为视频帧

### 4.4.3 计算复杂度分析

相比标准3D自注意力：
- **空间复杂度**: 从O(N²·H·W)降低到O(K·H·W)，其中K是检索token数量
- **时间复杂度**: 显著减少，特别是在长序列生成中

## 实验设置和结果

## 5.1 实验设置

### 5.1.1 实现细节

模型基于**Wan2.1-T2V-1.3B**构建：

- **上下文帧数**: 77帧
- **3D VAE**: 时间下采样因子为4
- **记忆标记化器**: 3D卷积核大小为2×4×4
- **检索token长度**: 10
- **局部窗口大小**: 5
- **训练配置**: 10K迭代，32个GPU，批次大小32

### 5.1.2 评估协议

测试集包含从HM-World数据集中随机选择的1000个视频样本，包括训练中未见过场景和主体的样本，以评估泛化能力。

评估指标分为三类：

#### 1. 通用记忆容量

- **PSNR**: 峰值信噪比，衡量像素级重建质量
- **SSIM**: 结构相似性，衡量结构保真度
- **LPIPS**: 学习的感知图像块相似性，衡量感知质量

#### 2. 帧级一致性

- **Subject Consistency**: 主体一致性
- **Background Consistency**: 背景一致性

这些指标来自Vbench评估套件。

#### 3. 动态主体一致性（DSC）

论文提出了新的评估指标DSC（Dynamic Subject Consistency）来专门评估动态主体在进入-退出事件中的连贯性：

**定义**:
```
DSC_GT = sim(F_pred, F_gt)
DSC_ctx = sim(F_pred, F_ctx)
```

其中sim(⋅,⋅)是跨通道的余弦相似度，F表示从边界框裁剪的语义特征：

- **F_gt**: 来自Ground Truth视频的主体特征
- **F_ctx**: 来自上下文视频的主体特征

**两种评估方式**:
- **DSC_GT**: 与真实值比较，评估对运动和外观的保真度
- **DSC_ctx**: 与上下文比较，评估与历史的一致性

### 5.1.3 基线方法

比较方法包括：

1. **Baseline**: Wan2.1-T2V-1.3B + 相机编码器，直接连接上下文潜在表示和噪声潜在表示
2. **DFoT**: 历史引导的视频扩散（ICML 2025）
3. **Context-as-Memory**: 上下文即记忆（SIGGRAPH Asia 2025）
4. **WorldPlay**: 最先进的商业模型，零样本评估

- WorldPlay在上下文引用指标上表现较好（DSC_CTX 0.822），表明经过大量训练的模型具备一定的混合一致性能力
- 这间接验证了DSC指标的合理性，能够有效反映动态主体一致性

从生成的视觉对比可以看出：

**Baseline**:
- 主体进入时严重扭曲
- 运动不连贯
- 出现鬼影或变形

**Context-as-Memory**:
- 主体部分保持身份一致性

- 完全失去主体

- 保持外观一致性

- 完全保持主体的身份一致性
- 运动连贯流畅
- 重新进入时无缝衔接之前的运动轨迹
- 整体生成质量最佳

## 5.3 消融实验

### 5.3.1 记忆标记化器卷积核大小（表4）

| Kernel (T×H×W) | PSNR | SSIM | LPIPS | DSC_ctx | DSC_GT | Subject Consistency | Background Consistency |
|----------------|------|------|-------|---------|---------|---------------------|------------------------|
| 2×2×2 | 20.113 | 0.599 | 0.299 | 0.820 | 0.843 | 0.919 | 0.929 |
| **4×4×4** | **20.357** | **0.606** | **0.289** | **0.827** | **0.849** | **0.926** | **0.932** |
| 8×8×8 | 20.230 | 0.610 | 0.292 | 0.822 | 0.843 | 0.923 | 0.927 |
| 1×4×4 | 19.076 | 0.554 | 0.337 | 0.819 | 0.841 | 0.912 | 0.925 |

#### 关键发现：

1. **空间维度不敏感**: 从2×2到8×8，PSNR变化仅0.244，说明模型对空间卷积核大小具有较强的鲁棒性
2. **时间维度关键**: 从4×4×4到1×4×4，PSNR下降1.281，DSC_GT下降0.014，证明了时间交互的必要性
3. **最佳配置**: 4×4×4提供了最佳性能平衡

**解释**: 较大的空间卷积核可以提供更丰富的空间上下文，但过多的时间卷积可能导致信息丢失。因此，4×4×4是理想的配置。

### 5.3.2 检索token数量（表5）

| 检索token数 | PSNR | SSIM | LPIPS | DSC_ctx | DSC_GT | Subject Consistency | Background Consistency |
|-------------|------|------|-------|---------|---------|---------------------|------------------------|
| 5 | 19.309 | 0.566 | 0.339 | 0.817 | 0.836 | 0.913 | 0.927 |
| **10** | **20.357** | **0.606** | **0.289** | **0.827** | **0.849** | **0.926** | **0.932** |
| 15 | 20.333 | 0.612 | 0.291 | 0.828 | 0.842 | 0.925 | 0.935 |

#### 关键发现：

1. **5个token不足**: 性能显著下降（PSNR 19.309），表明信息丢失严重
2. **10个token最优**: 最佳性能配置
3. **15个token冗余**: 性能接近10个token，无明显优势

**解释**: 
- 5个token不足以提供足够的时空信息
- 10个token能够提供必要的上下文信息
- 超过10个token会引入冗余噪声，但边际效益递减

### 5.3.3 检索方法对比（表6）

| 方法 | PSNR | SSIM | LPIPS | DSC_ctx | DSC_GT | Subject Consistency | Background Consistency |
|------|------|------|-------|---------|---------|---------------------|------------------------|
| FOV Overlap | 19.776 | 0.586 | 0.300 | 0.820 | 0.844 | 0.908 | 0.930 |
| **Dynamic Affinity** | **20.357** | **0.606** | **0.289** | **0.827** | **0.849** | **0.926** | **0.932** |

#### 关键发现：

1. **动态亲和度显著优于FOV**: PSNR提升0.581，Subject Consistency提升0.018
2. **更精细的相关性评估**: QK交互能够捕捉细粒度关系
3. **纯几何的局限性**: FOV仅基于静态几何重叠，无法捕捉动态时空关系

**解释**: 
- FOV方法假设token和目标帧的几何关系决定相关性，但忽略了动态主体的运动模式
- 动态亲和度方法直接计算Query和Key之间的相关性，能够捕获更丰富的时空关系

## 关键创新点

### 创新点1: 混合记忆范式

**问题识别**: 论文首次系统性识别了现有视频世界模型在处理动态主体时的盲点 - 将世界视为静态画布。

**创新**: 提出Hybrid Memory范式，要求模型同时：
- 作为静态背景的精确档案管理员
- 作为动态主体的警惕追踪者

**意义**: 
- 填补了视频世界模型在复杂动态场景中的研究空白
- 为记忆机制的设计提供了新的理论框架
- 指明了未来研究的方向

### 创新点2: HM-World数据集

**创新**: 构建了首个专门为混合记忆设计的大规模视频数据集。

**数据集特点**:
- **规模**: 59K个高保真视频片段
- **多样性**: 
  - 17个多样化场景
  - 49个不同主体（人类、动物）
  - 10种运动路径
  - 28种相机轨迹
- **设计**: 严格解耦相机轨迹和主体轨迹
- **标注**: 精确标注每个主体的进入-退出时间戳

**意义**:
- 为混合记忆研究提供了标准化的基准
- 通过精心设计的相机运动诱导进入-退出事件
- 包含全面的标注信息（视频、标题、相机姿态、主体位置、时间戳）

**与现有数据集对比**:
- WorldScore: 仅3K，且无动态主体
- Context-as-Memory: 静态场景
- Multi-Cam Video: 动态主体但仅原地动作
- 360°-Motion: 相机静态，主体始终在视野中
- HM-World: 唯一包含进入-退出事件的动态主体数据集

### 创新点3: HyDRA记忆架构

**创新**: 提出了HyDRA（Hybrid Dynamic Retrieval Attention）专用记忆架构。

**核心技术**:

1. **记忆标记化器**: 
   - 基于3D卷积
   - 同时处理空间和时间维度
   - 生成密集的、时空感知的记忆token

2. **动态检索注意力**:
   - 替代标准3D自注意力
   - 基于QK交互的时空相关性评估
   - Top-K选择策略
   - 局部窗口合并

3. **双重视角一致**: 
   - DSC_CTX: 与上下文的一致性
   - DSC_GT: 与真实值的一致性

**意义**:
- 有效解决特征纠缠问题（静态背景与动态主体特征耦合）
- 显著减少计算复杂度
- 在保持静态背景一致性的同时，提升动态主体的连贯性

### 创新点4: DSC评估指标

**创新**: 提出了新的动态主体一致性评估指标。

**指标定义**:
```
DSC_GT = sim(F_pred, F_gt)
DSC_ctx = sim(F_pred, F_ctx)
```

**优势**:
- **语义级评估**: 使用CLIP提取语义特征，而非像素级
- **时间对齐**: 通过空间对齐和时间归一化确保比较的有效性
- **双重视角**: 同时评估与真实值和上下文的一致性
- **领域无关**: 适用于不同类型的动态主体

**意义**:
- 提供了专门评估混合记忆能力的标准化指标
- 能够准确捕获运动和外观的连贯性
- 为未来研究提供了评估基准

## 局限性和未来工作

### 当前局限性

#### 1. 复杂场景下的性能下降

**问题**: HyDRA在高度复杂的场景中性能会下降，特别是：
- 3个或更多主体的场景
- 严重的遮挡情况

**原因分析**:
- 检索机制在多主体场景中难以区分不同主体
- 遮挡导致主体外观和运动信息丢失
- 记忆token的空间分辨率在复杂场景中不足

**实验证据**: 消融实验和定性分析显示，随着场景复杂度增加，生成质量下降

#### 2. 记忆容量限制

**问题**: 
- 长时间序列生成中记忆逐渐衰减
- 超过一定时间步后，主体追踪能力下降

**原因分析**:
- 检索token数量固定（K=10），无法适应不同长度的序列
- 3D卷积的固定感受野限制了长时记忆捕获

#### 3. 实时性限制

**问题**: 
- 生成速度较慢，难以满足实时应用需求
- 检索和注意力计算耗时较长

**原因分析**:
- 基于Transformer的架构
- 检索和注意力机制的复杂计算

#### 4. 数据集偏差

**问题**: 
- HM-World基于Unreal Engine 5渲染
- 可能与真实世界数据存在分布差距

**影响**: 零样本评估中WorldPlay表现较好，但HyDRA仍保持优势

### 未来工作方向

#### 1. 多主体场景的优化

**目标**: 提升模型在多主体场景中的性能

**可能的解决方案**:
- **多头记忆机制**: 为每个主体分配独立的记忆槽
- **主体分离**: 在潜在空间中显式分离不同主体
- **身份感知检索**: 基于主体身份（如外观、运动模式）进行检索
- **注意力稀疏化**: 使用稀疏注意力机制减少计算复杂度

#### 2. 长时记忆机制

**目标**: 提升长时间序列的生成质量

**可能的解决方案**:
- **分层记忆**: 多级记忆结构（短期、中期、长期）
- **自适应检索**: 根据序列长度动态调整检索策略
- **记忆衰减建模**: 显式建模记忆的衰减和恢复
- **时序记忆池化**: 更高效的多时步信息聚合

#### 3. 实时优化

**目标**: 提升生成速度以满足实时应用需求

**可能的解决方案**:
- **蒸馏**: 将HyDRA蒸馏为更轻量的模型
- **并行优化**: 优化检索和注意力计算
- **硬件加速**: 利用GPU/TPU优化
- **近似算法**: 使用更高效的近似注意力机制

#### 4. 真实世界适应

**目标**: 提升模型在真实世界数据上的表现

**可能的解决方案**:
- **真实数据训练**: 在真实视频数据上训练
- **域自适应**: 缓解渲染数据与真实数据的差距
- **不确定性估计**: 处理真实世界中的复杂和不可预测性
- **人类反馈强化学习**: 使用RLHF优化生成质量

#### 5. 更多应用场景

**目标**: 扩展HyDRA的应用范围

**可能的场景**:
- **自动驾驶**: 长时间、大范围的路况生成
- **游戏开发**: 实时动态世界生成
- **电影制作**: 复杂场景的镜头控制
- **虚拟现实**: 实时、高保真的虚拟环境
