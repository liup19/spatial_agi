## 摘要

大型视觉-语言模型（LVLMs）在语义理解方面表现出色，但在细粒度的空间定位方面存在困难，因为模型必须隐式地推断复杂的几何结构，而从未产生空间解释。我们提出了Perceptio，一个具有2D-3D空间推理能力的感知增强型LVLM，通过在自回归序列中直接生成的显式语义分割tokens和深度tokens来实现。具体来说，我们（i）从强大的单目教师模型蒸馏出一个VQ-VAE深度码本，将密集深度标记为紧凑序列，以及（ii）在LLM中集成基于SAM2的语义分割tokens和VQ-VAE深度tokens，使模型首先发出空间tokens然后回答。为了稳定深度token生成，我们引入了新颖的复合深度token目标（标记、token和计数损失）和用于可微重构的软合并技术。我们采用跨多样化数据集的多任务协同训练策略，让模型学习感知tokens以处理多个下游任务。建立在InternVL之上，Perceptio在基准测试中实现了最先进的性能：在RefCOCO/+/g上的指代表达分割分别提高了+0.8/+1.4/+1.1 cIoU，HardBLINK空间理解准确率提高10.3%，MMBench准确率提高1.0%，证明显式的空间思维链显著增强了LVLMs中的空间定位能力。

## 核心信息

### 关键贡献

1. **LVLM中的显式空间感知**：我们引入了Perceptio，通过序列中的2D语义分割和离散化3D深度tokens来增强LVLMs，实现像素级和几何推理。据我们所知，Perceptio是第一个在LVLMs中的单个自回归序列中联合优化2D和3D感知信号的工作。

2. **新颖深度目标的多任务联合训练**：我们提出了一个联合文本-分割-深度目标以及一系列新颖的深度token损失函数（标记+token+计数），稳定了深度token的发射。软深度重构技术实现了完全端到端的可微深度训练。

3. **感知增强的数据**：我们策划了一个56K示例的联合数据集，将分割掩膜和深度先验与语言监督配对，通过添加对齐的深度tokens和属性描述来增强RefCOCO/+/g，以引导中间推理。

4. **最先进的性能**：Perceptio在所有三个指代表达分割基准测试（RefCOCO/+/g）上均达到SOTA，在HardBLINK空间推理上提高了+10.3%，在MMBench通用VQA上提高了+1.0%，证明显式的序列感知显著加强了不同任务中的空间定位。

## 📝 GLM WebReader深度分析

### Q1: 核心算法原理

**问题**: 这篇文章的核心算法原理是什么？

**答案**:

#### 1. 核心思想和动机

Perceptio的核心思想是将显式的2D语义分割和3D深度感知整合到大型视觉-语言模型（LVLM）的自回归生成过程中。其动机源于现有LVLM在空间理解方面的局限性——尽管这些模型在语义理解（识别图像中的物体）方面表现出色，但在空间理解（理解物体的位置、深度、距离和比例）方面存在明显不足。例如，BLINK研究显示流行LVLM在简单的空间推理任务上 barely surpass random guessing。

#### 2. 主要技术方法

Perceptio采用了一个三路径的感知增强架构：

**路径1：标准图像编码器**
- 用于语义外观特征提取
- 保留原有的视觉语义理解能力

**路径2：冻结的SAM2编码器**
- 提供分割感知的表示
- 用于2D语义分割任务

**路径3：冻结的预训练深度VQ-VAE码本**
- 离散化图像深度信息
- 用于3D深度推理

核心LLM消费编码后的图像特征和查询，产生一个交织自然语言tokens和感知控制tokens的自回归序列。

#### 3. 算法流程和关键步骤

**步骤1：深度码本训练**

- 使用Depth Anything V2作为教师模型，从训练数据集生成可靠的连续深度图
- 训练VQ-VAE将这些连续深度图离散化为深度tokens
- VQ-VAE码本大小为K=128
- 每个深度图被编码为n×n的网格代码索引，结果为n-token序列
- 每个token代表深度码本中K个离散深度值之一

**步骤2：感知增强生成模式**

模型强制执行特定的输出序列格式：

```
[seg][dstart,d1,d2,…,dn,dend][t1,t2,…,tm]
```

其中：
- `[seg]`：特殊控制token，其嵌入条件化分割解码器以输出查询定位的掩膜
- `[dstart,d1,…,dn,dend]`：离散化的深度tokens序列
- `[t1,t2,…,tm]`：文本回答tokens

模型被训练总是按以下顺序发出这些tokens：先分割，再深度，然后是答案。这种强制排序的动机来自于解码器的自回归性质——通过首先生成感知tokens，模型有效地基于场景的空间结构执行思维链推理，然后再制定最终答案。

**步骤3：分割token生成**

- 基于[seg] token，SAM2解码器重构分割掩膜
- 训练期间，使用交叉熵和DICE损失微调SAM2解码器
- 掩膜类型（如指代、实例或语义）由提示词暗示的任务决定

**步骤4：深度token生成**

- 深度tokens序列d1:dn被解释为VQ-VAE码本的索引
- 通过VQ-VAE解码器重构密集深度图
- 深度分支（码本和解码器）保持冻结
- LVLM仅训练生成索引预训练码本的深度tokens

**步骤5：多任务联合训练**

总损失函数：

```
Ltotal = LLM + LSegRecon + λd * Ldepth + λr * LDepthRecon
```

其中：
- LLM：标准的LLM损失（负对数似然）
- LSegRecon：2D分割重构损失（交叉熵 + DICE）
- Ldepth：深度token生成损失（复合损失）
- LDepthRecon：可微深度重构损失

#### 4. 输入输出

**输入**：
- 图像I
- 文本查询q

**输出**：
- 分割掩膜 M̂（通过[seg] token触发SAM2解码器）
- 深度图 D̂（通过深度tokens触发VQ-VAE解码器）
- 文本回答 t（自然语言响应）

**特殊设计**：
- 在推理时，生成的[seg]和[depth] tokens可用于通过其各自的教师模型创建2D和3D定位可视化
- 但训练后的LVLM模型独立于教师分支操作，以生成下游任务所需的文本响应

#### 5. 关键技术创新

**创新1：复合深度token损失**

深度token生成损失是一个复合损失，用于对齐何时开始/结束（Lmarker）、用什么代码填充（Ltoken）以及产生多少（Lcount）：

```
Ldepth = λm * Lmarker + λt * Ltoken + λc * Lcount
```

**标记损失（Marker Loss）**：
- 确保深度开始token dstart和深度结束token dend在正确位置生成
- 使用token级交叉熵确保边界token的准确性

**Token损失（Token Loss）**：
- 确保LLM生成正确的深度token值
- 仅在深度序列内部应用

**计数损失（Count Loss）**：
- 鼓励LLM产生所需长度n的序列
- 当生成长度偏离n时激活惩罚项

**创新2：软深度重构**

为了以可微方式解码深度，用码本嵌入的软合并技术替换硬码字选择：

- 模型预测码本上的概率分布
- 通过用其预测概率对每个嵌入进行加权来形成软token
- 这种连续松弛将离散tokens映射到平滑嵌入空间
- 允许来自深度重构目标的梯度流过token化阶段
- 实现完全端到端的训练

**创新3：感知条件化生成**

通过在文本回答之前强制模型生成感知tokens，Perceptio实现了感知增强的条件生成：
- 模型首先解释感知信号
- 基于显式的2D和3D线索锚定后续语言
- 提高VQA、定位和空间推理性能

**创新4：多任务数据集策划**

- 将RefCOCO/+/g（指代表达分割基准测试）增强为联合数据集
- 对于每个指代表达：
  1. 将真值掩膜转换为紧凑的分割tokens序列
  2. 附加编码相同区域量化深度的对齐深度tokens
  3. 添加简洁的、属性集中的单句子对象描述
- 所有信号统一在单个指令-输出格式中
- 总共56K示例的训练数据

#### 6. 训练细节

**数据集组成**（约1.1M图像-文本对）：
- 665K LLaVA-1.5指令调优样本（图像QA和聊天）
- 214K定位对话生成样本（图像级文本驱动的分割）
- 60K ADE20k与感知tokens数据集
- 56K联合数据集（RefCOCO/+/g增强版）

**实现细节**：
- 在64个NVIDIA A100 GPU上训练约24小时
- 最大序列长度：8,192 tokens
- 优化器：AdamW，学习率4×10^-5
- 批次大小：每设备1，梯度累积8步，有效批次大小512
- LoRA秩：256
- 深度VQ-VAE码本维度：K=128
- LLM生成深度tokens数量：n=100
- 损失权重：λm=0.3, λt=0.5, λc=0.2, λd=1.0, λr=1.0

**推理成本**：
- 尽管生成额外的感知tokens，Perceptio的推理开销可以忽略不计
- 对于密集字幕生成，Perceptio-8B每100 token需要3.52秒，与Sa2VA-8B的3.53秒相当
- FLOPs也相当（4.06T vs 4.66T）

### Q2: 与Spatial AGI的关系

**问题**: 这篇文章与通用空间智能（Spatial AGI）有什么关系？

**答案**:

#### 1. 如何理解和表示空间

Perceptio提供了一种新的空间表示方法，将2D语义和3D几何信息整合到统一的token序列中：

**2D空间表示（语义分割tokens）**：
- 通过[seg]特殊token编码图像中的语义边界
- 基于SAM2的语义分割tokens捕捉物体级别的空间组织
- 提供像素级的空间定位能力
- 将空间表示为物体实例的集合

**3D空间表示（深度tokens）**：
- 通过VQ-VAE码本将连续深度图离散化为token序列
- 每个深度token代表特定的深度值
- 将3D几何信息编码为可学习的离散符号
- 提供场景的深度结构和相对位置信息

**统一空间表示**：
- 2D和3D空间信息在同一自回归序列中联合表示
- 空间信息作为条件先验嵌入到语言生成过程中
- 实现语义（是什么）和几何（在哪里）的统一建模

这种方法与Spatial AGI的核心需求高度契合——Spatial AGI需要同时理解场景中的物体语义和它们的空间关系，而Perceptio恰好提供了这种能力。

#### 2. 如何处理空间关系

Perceptio通过显式的空间思维链（Spatial Chain-of-Thought）来处理空间关系：

**空间思维链机制**：

传统LVLM依赖从池化特征隐式推断几何关系，而Perceptio强制模型首先显式生成空间表示：

```
输入: 图像 + 查询
↓
[seg tokens] → 2D语义理解（识别物体边界）
↓
[depth tokens] → 3D几何理解（理解深度和距离）
↓
[text tokens] → 基于显式空间信息的推理
```

**空间推理的具体实现**：

1. **深度推理**：
   - HardBLINK任务测试相对深度理解（如"哪个点更靠近相机"）
   - 深度tokens提供明确的3D几何线索
   - 模型通过生成准确的深度序列来理解空间层次

2. **定位推理**：
   - RefCOCO/+/g任务测试基于自然语言的指代表达分割
   - 分割tokens提供像素级的空间定位
   - 模型通过生成[seg] token来锚定文本查询到特定空间区域

3. **多模态空间推理**：
   - MME、MMBench等基准测试要求视觉-语言空间推理
   - 模型结合语义和几何信息进行复杂推理
   - 显式空间tokens作为推理的中间步骤

**处理复杂空间关系的优势**：

- **相对位置理解**：深度tokens明确编码物体间的深度关系
- **空间层次理解**：2D分割+3D深度提供多层次的空间结构
- **跨模态对齐**：空间tokens自然地对齐视觉和语言模态

#### 3. 对Spatial AGI的启发

Perceptio为Spatial AGI的发展提供了几个关键启发：

**启发1：显式空间表示的重要性**

- 仅仅依靠规模不会自动产生空间智能
- 显式地将空间信息整合到模型架构中至关重要
- 空间应该作为第一类公民（first-class citizen）被建模

**启发2：感知增强的条件生成**

- 在语言生成之前显式生成空间表示可以显著提高空间推理能力
- 这种"先感知后推理"的模式模仿人类的空间认知过程
- 为Spatial AGI提供了一个可行的架构范式

**启发3：多模态空间的统一建模**

- 同时优化2D语义和3D几何可以产生协同效应
- 分割tokens提供"是什么"（语义），深度tokens提供"在哪里"（几何）
- 两者的结合比单独使用任一都更有效

**启发4：可学习的空间tokens**

- 通过VQ-VAE将连续空间信息离散化为可学习的tokens
- 这些tokens可以像语言tokens一样被训练和推理
- 提供了一种将空间知识整合到LLM框架的通用方法

**启发5：多任务学习的价值**

- 在多样化的空间相关任务上联合训练（分割、深度、VQA）
- 多任务学习提高了模型的空间泛化能力
- 为Spatial AGI需要处理广泛空间任务的需求提供了解决方案

#### 4. 应用到Spatial AGI的场景

- 多模态LLM：主要关注语义理解，空间推理能力有限
- Perceptio：显式加入空间tokens，显著提升空间推理能力

**对比机器人系统**：
- 机器人系统：通常模块化设计，感知和规划分离
- Perceptio：端到端的感知与推理，更接近人类认知

#### 7. 局限性与未来方向

**当前局限**：

1. **优化权衡**：
   - 深度token生成与纯文本任务之间存在轻微优化张力
   - 移除深度tokens轻微改善通用VQA指标（+0.4% MMBench）

2. **数据限制**：
   - 训练和评估仅限于静态图像
   - 扩展到视频面临新的挑战（时序一致性、物体追踪）

3. **教师模型依赖**：
   - 依赖冻结的教师模型（Depth Anything V2, SAM2）
   - 教师模型的错误会传播到学生模型

**未来方向**：

1. **任务自适应课程学习**：
   - 通过课程学习缓解优化张力
   - 根据任务需求动态平衡空间和语言能力

2. **视频扩展**：
   - 扩展到时序一致的深度tokens
   - 结合物体追踪能力

3. **鲁棒性改进**：
   - 提高对教师噪声的鲁棒性
   - 开发自适应的教师模型更新策略

4. **空间模态扩展**：
   - 扩展到其他空间信号（表面法线、光流）
   - 向统一的Spatial AGI框架发展

### Q3: 自由问题与深度思考

**问题**: Perceptio通过在自回归序列中显式生成空间tokens（分割+深度）来增强LVLM的空间推理能力。这种"显式空间思维链"方法相比传统的隐式推断方法，在哪些任务上表现最好？为什么？这是否意味着我们应该在所有需要空间推理的LVLM中引入类似的显式空间表示？

**答案**:

#### 为什么问这个问题？

这个问题的提出基于对Perceptio方法和结果的深入分析。Perceptio的核心创新是将空间信息从隐式推断转变为显式生成，并在多个基准测试上取得了显著的性能提升。理解这种方法的适用边界和局限性对于指导未来的Spatial AGI研究至关重要。

#### Perceptio表现最好的任务

**1. 指代表达分割（RefCOCO/+/g）**

**性能表现**：
- RefCOCO: 82.7% cIoU (vs Sa2VA-8B: 81.6%)
- RefCOCO+: 77.9% cIoU (vs Sa2VA-8B: 76.2%)
- RefCOCOg: 80.0% cIoU (vs Sa2VA-8B: 78.7%)
- 平均提升：+1.1/+1.7/+1.3 cIoU

**为什么表现最好**：

- **任务本质**：指代表达分割需要精确的像素级定位，这正是分割tokens设计的目标
- **显式对齐**：[seg] token直接将自然语言查询映射到像素空间，建立了语言-视觉的显式对齐
- **语义-几何协同**：结合分割（语义边界）和深度（空间深度）提供了完整的空间上下文
- **端到端优化**：在统一框架中优化整个pipeline，避免了模块化系统的信息损失

**具体例子**：
查询："那个穿红色衣服的人"
- 隐式方法：从图像特征推断，可能受到池化损失
- Perceptio方法：显式生成[seg] token → 精确掩膜 → 深度tokens理解3D位置 → 更准确的理解和回答

**2. 相对深度推理（HardBLINK）**

**性能表现**：
- 3点：75.8% (vs LLaVA-Aurora: 66.9%)
- 4点：71.0% (vs LLaVA-Aurora: 60.5%)
- 5点：66.1% (vs LLaVA-Aurora: 54.8%)
- 平均：71.0% (vs LLaVA-Aurora: 60.7%)
- 平均提升：+10.3%

**为什么表现最好**：

- **任务本质**：HardBLINK测试"哪个点更靠近相机"这类相对深度问题，完全依赖于深度推理
- **深度tokens的必要性**：消融实验显示移除深度tokens导致准确率从71.0%降至45.2%（下降25.8个百分点）
- **显式3D表示**：深度tokens提供了明确的3D几何线索，而传统方法必须从2D特征隐式推断
- **量化精度**：VQ-VAE将连续深度离散化为128个级别，提供足够的精度来区分相对深度

**具体例子**：
问题："点A还是点B更靠近相机？"
- 隐式方法：从图像纹理和大小推断，容易出错
- Perceptio方法：显式生成深度tokens → 准确的深度图 → 直接比较深度值 → 高精度回答

**3. 通用视觉问答（MMBench, MME）**

**性能表现**：
- MMBench: 83.4% (vs Sa2VA-8B: 82.4%, InternVL2-8B: 81.7%)
- MME Perception: 1654 (vs Sa2VA-8B: 1651)
- MME Cognition: 628 (vs Sa2VA-8B: 578)

**为什么表现良好**：

- **多样化的空间任务**：这些基准测试包含大量需要空间推理的问题
- **感知-推理协同**：先显式生成空间表示再进行推理，类似人类的认知过程
- **上下文增强**：空间tokens为语言生成提供了丰富的视觉上下文
- **多任务泛化**：在多样化数据上训练提高了泛化能力

#### 为什么这些任务表现最好？

**原因1：任务与设计的完美匹配**

Perceptio的设计恰好满足了这些任务的核心需求：

| 任务 | 核心需求 | Perceptio的解决方案 |
|------|---------|---------------------|
| 指代表达分割 | 像素级定位 | [seg] token + SAM2解码器 |
| 相对深度推理 | 3D几何理解 | VQ-VAE深度tokens |
| 视觉问答 | 综合+空间推理 | 2D分割 + 3D深度 + 语言推理 |

**原因2：显式与隐式的权衡**

对于需要精确空间信息的任务，显式表示具有显著优势：

**显式表示的优势**：
- 信息保真度高：空间信息不会在池化或压缩中丢失
- 可解释性强：可以检查生成的分割掩膜和深度图
- 精确控制：可以精确控制空间表示的质量
- 梯度传播：可以端到端优化空间生成过程

**隐式表示的局限**：
- 信息损失：池化操作会丢失细节
- 黑盒性质：难以理解和调试空间推理过程
- 误差积累：隐式推断的误差会在后续步骤中积累
- 训练困难：难以直接优化空间推理目标

**原因3：空间思维链的认知合理性**

Perceptio的"先感知后推理"模式模仿人类的空间认知过程：

1. **人类视觉处理**：
   - 低级视觉：边缘、颜色、纹理
   - 中级视觉：物体识别、分割
   - 高级视觉：场景理解、空间推理

2. **Perceptio的处理**：
   - 低级：图像编码器
   - 中级：[seg] tokens（分割） + [depth] tokens（深度）
   - 高级：基于显式空间表示的语言推理

这种分层处理符合认知科学原理，因此在这些任务上表现优异。

#### 是否应该在所有需要空间推理的LVLM中引入显式空间表示？

**答案是：有条件的肯定**

**应该引入的情况**：

1. **需要精确空间定位的任务**

- 指代表达分割
- 目标检测与追踪
- 机器人抓取与操作
- AR/VR中的虚拟物体放置

**理由**：这些任务的核心是"在哪里"，精确的空间定位至关重要。隐式推断难以达到像素级精度。

2. **需要深度理解的任务**

- 3D场景理解
- 距离估计
- 相对位置推理
- 障碍物避让

**理由**：深度是3D理解的核心，深度tokens提供了明确的3D几何线索。

3. **需要复杂空间推理的任务**

- 场景图生成
- 空间关系理解
- 多步骤空间任务
- 空间规划

**理由**：复杂推理需要明确的中间表示，显式空间tokens可以作为推理的"工作记忆"。

4. **需要可解释性的应用**

- 医学诊断
- 自动驾驶决策
- 安全关键系统

**理由**：显式空间表示提供了可解释的中间步骤，增强了系统的可信度。

**可能不需要引入的情况**：

1. **纯语义理解任务**

- 图像分类（如"这是一只狗吗"）
- 物体识别（如"这是什么物体"）
- 简单的属性识别（如"是什么颜色的"）

**理由**：这些任务主要关注"是什么"，空间信息不是核心需求。引入额外空间表示可能增加复杂度而收益有限。

2. **计算资源极其受限的场景**

- 边缘设备（如手机、物联网传感器）
- 实时性要求极高的应用

**理由**：虽然Perceptio的推理开销很小，但在极端资源受限场景中，任何额外的计算都可能不可接受。

3. **空间信息已经充分编码的场景**

- 已经有精确的深度传感器（如LiDAR）
- 3D模型已经预先构建

**理由**：在这些场景中，空间信息已经通过其他方式获得，不需要从图像中推断。

3. **通用对话场景**

- 闲聊、创意写作
- 不涉及空间推理的对话

**理由**：显式空间表示对这类任务的贡献有限，可能引入不必要的复杂度。

**权衡考虑**：

1. **性能 vs 复杂度**

引入显式空间表示会带来：
- 性能提升：在空间相关任务上显著提升
- 复杂度增加：需要训练额外的深度tokens和分割tokens
- 计算开销：虽然很小，但并非零

**建议**：在空间推理是核心能力的应用中，性能提升远超复杂度增加，值得引入。在空间推理次要的场景中，需要仔细权衡。

2. **通用性 vs 专门性**

Perceptio是多任务模型，在多个任务上都达到SOTA：
- 通用性强：可以处理多种空间相关任务
- 专门性：在特定的空间推理任务上表现最佳

**建议**：如果目标是构建通用的Spatial AGI，应该引入显式空间表示。如果目标是构建专门的视觉系统（如纯分类器），可能不需要。

3. **训练成本 vs 应用价值**

Perceptio的训练成本：
- 64个A100 GPU，约24小时
- 需要策划的多模态数据集

但应用价值高：
- 端到端的感知与推理
- 在多个基准测试上达到SOTA
- 可部署到实际应用

**建议**：对于有资源和需求的研究团队，训练成本是合理的投资。对于资源受限的项目，可以考虑使用预训练的Perceptio模型。

#### 未来研究方向

**方向1：自适应空间表示**

- 根据任务需求动态决定是否生成空间tokens
- 在需要空间推理的任务上使用显式表示，在纯语义任务上使用隐式表示
- 实现复杂度和性能的动态平衡

**方向2：层次化空间表示**

- 不同粒度的空间tokens（从粗到细）
- 根据任务需求选择适当的粒度
- 提高效率和灵活性

**方向3：条件化空间生成**

- 基于查询类型条件化空间tokens的生成
- 只生成任务所需的空间信息
- 减少不必要的计算

**方向4：多模态空间tokens**

- 扩展到其他空间模态（法线、光流、表面曲率等）
- 提供更丰富的空间理解
- 向更全面的Spatial AGI发展

**方向5：可学习的空间推理策略**

- 学习何时使用显式表示，何时使用隐式推断
- 优化空间推理的计算效率
- 提高模型的适应性

## 个人思考

### 最有趣的发现

**发现：空间推理能力的非涌现性**

最有趣的发现是，尽管LVLM在语义理解方面表现优异，但空间推理能力并不能简单地随着模型规模的扩大而自然涌现。论文中提到InternVL2.5-26B在HardBLINK的"closer-to-camera"点选择任务上仅达到33.1%的平均准确率，这凸显了空间理解在多模态LLM中的显著弱点。

**为什么有趣？**

1. **颠覆了规模直觉**：
   - 我们通常认为"scaling law"适用于所有能力
   - 但空间推理似乎遵循不同的规律
   - 这挑战了"更大模型=更好能力"的简单假设

2. **提示了架构的本质差异**：
   - 语义理解和空间推理可能需要不同的架构设计
   - 空间推理需要显式的空间表示
   - 仅仅增加规模无法解决架构层面的缺陷

3. **对AGI发展的启示**：
   - AGI不仅仅是规模的扩展
   - 需要精心设计每个核心能力的架构
   - Spatial AGI需要专门的空间处理模块

**深层含义**：
这一发现暗示了AGI发展中的一个关键洞察：不同的认知能力可能需要不同的架构设计和训练策略。不能期待所有能力都通过简单地扩大规模来实现。这对未来的AGI研究提供了重要指导。

### 最意外的结果

**结果：深度tokens与纯文本任务之间的优化张力**

最意外的结果是消融实验显示，移除深度tokens会轻微改善通用VQA指标（如+0.4% MMBench）。这表明深度token生成与纯文本任务之间存在轻微的优化张力。

**为什么意外？**

1. **直觉上的预期**：
   - 空间tokens应该增强而不是削弱语言能力
   - 多任务学习通常带来正向迁移
   - 预期看到的是全面的性能提升

2. **实际观察到的现象**：
   - 深度tokens对空间任务有巨大帮助（HardBLINK +10.3%）
   - 但对纯文本任务有轻微负面影响（MMBench -0.4%）
   - 这种不对称性值得深入研究

**可能的原因**：

1. **表示空间的竞争**：
   - 深度tokens占用模型容量
   - 可能在某些情况下与纯语言表达竞争资源
   - 模型的注意力分配在多任务间需要权衡

2. **训练目标的冲突**：
   - 深度生成和语言生成有不同的优化目标
   - 可能导致模型在纯语言任务上的次优表现
   - 需要精细的损失权重调节

3. **数据分布的偏移**：
   - 深度tokens主要在特定数据集上训练
   - 可能与通用VQA数据分布不完全匹配
   - 需要更好的数据平衡

**对未来研究的启示**：

1. **需要更精细的多任务优化策略**：
   - 任务自适应的课程学习

## 总结

### 核心发现总结

Perceptio通过在LVLM的自回归序列中显式生成2D语义分割tokens和3D深度tokens，实现了感知增强的空间推理。核心发现包括：

1. **显式空间表示的有效性**：在需要精确空间推理的任务上，显式空间tokens显著提升性能
   - HardBLINK准确率提升10.3%
   - RefCOCO/+/g分别提升+1.1/+1.7/+1.3 cIoU

2. **2D-3D协同的价值**：语义分割和深度推理互补，结合效果优于单独使用
   - 移除深度tokens导致HardBLINK准确率下降25.8%
   - 移除分割tokens导致通用VQA性能下降2.3%

3. **感知增强条件生成的成功**："先感知后推理"模式类似人类认知，效果显著
   - 在多个基准测试上达到SOTA

5. **多任务学习的泛化性**：联合训练提高了空间泛化能力

### A. 深度重构损失的详细算法

深度重构损失通过软token平均方法变得可微。算法流程如下：

**输入**：
- 解码器logits z ∈ ℝ^(B×T×V)
- 真值tokens y ∈ ℕ^(B×T)
- 深度代码索引集 D
- 码本 {e_k}_{k∈D}

**算法步骤**：

1. 对于每个批次b从1到B：
   - 在y_b中解析深度span (s_b, e_b)
   - 找到dstart和dend的位置

2. 如果找到有效的深度span：
   - 计算长度L_b = e_b - s_b - 1
   - 对于每个时间步t从s_b+1到e_b-1：
     * 获取预测logits ℓ = z_{b,t-1}
     * 掩码非深度索引：ℓ_j = -∞ 如果 j ∉ D
     * 计算softmax: p = softmax(ℓ)
     * 计算软token: z̃_{b,t} = Σ_{k∈D} p_k * e_k
   - 截断{z̃_{b,t}}到n个tokens
   - 重塑为√n × √n网格
   * 使用VQ-VAE解码器重构深度图: Ŷ_b = VQVAE_Decoder({z̃_{b,t}})

3. 如果没有找到有效的深度span：
   - 设置Ŷ_b = 0

4. 计算损失：
   L_DepthRecon = (1/B) * Σ_{b=1}^B ||Ŷ_b - Y_b||_2^2
