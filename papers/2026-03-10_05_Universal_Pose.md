## 摘要解读

### 核心问题

现有的Vision-Language-Action (VLA)模型普遍存在两个关键问题：

1. **特征坍塌 (Feature Collapse)**：模型将高级感知与稀疏的、特定具身的动作监督纠缠在一起
2. **低训练效率**：训练数据利用率低，难以有效迁移

这些问题的根源在于VLA模型通常依赖于为视觉问答（VQA）优化的视觉语言模型（VLM）主干。虽然这些模型在语义识别方面表现出色，但往往忽略了决定不同动作模式的细微3D状态变化。

### 提出方案

**Pose-VLA** - 一个解耦的学习范式，将VLA训练分为：

1. **预训练阶段**：在统一的相机中心空间中提取通用3D空间先验
2. **后训练阶段**：在特定机器人动作空间中进行高效具身对齐

### 关键创新

引入**离散姿态token（Discrete Pose Tokens）**作为通用表示，无缝集成：
- 来自多样化3D数据集的空间基础
- 来自机器人演示的几何级轨迹

### 核心贡献

1. 提出统一的VLM框架，整合RGB图像、深度图和相机内参，赋予内在3D意识
2. 引入离散姿态token作为通用接口，在一致的空间观测空间中统一3D空间先验
3. 策划了大规模预训练语料库：140万图像 + 650万3D标注 + 155万机器人轨迹
4. 在RoboTwin 2.0上达到79.5%平均成功率，在LIBERO上达到96.0%竞争力
5. 真实世界实验中仅需每个任务100个演示即可实现鲁棒泛化

## 一、引言

### 1.1 背景：VLA模型的兴起

通用具身智能的追求因Vision-Language-Action (VLA)模型的出现而显著加速。最先进的框架包括：

- **π 系列** [[6, 7]]: 采用双系统架构
- **GR00T 系列** [[5]]: 展示了端到端机器人操控的卓越潜力

**双系统架构**：
- VLM作为高级语义解释器
- 专用动作专家使用生成技术（如流匹配 Flow Matching [[23]]）作为低级控制器进行精确动作去噪

### 1.2 核心挑战

尽管取得了这些进展，将视觉语言表示扩展为物理基础的、可泛化的动作策略仍然是一个基本挑战。

**关键发现** [[47]]:
- VLM在VQA任务上表现优异，但其通用语义特征无法转化为鲁棒的下游策略性能
- 在辅助任务（如深度估计或VQA）上微调并不能保证改进控制

**核心开放问题**：如何高效地适应VLM以获得可转移的具身先验，直接促进下游策略学习？

### 1.3 问题根源：三个错配

研究者观察到这种失败不是偶然的，而是结构性的，源于三个持续的错配：

#### 1.3.1 粒度不匹配 (Granularity Mismatch)

**VLM预训练**：
- 主导任务：图像-文本对齐、视觉问答
- 强调：类别识别和高级语义

**机器人操控**：
- 关键依赖：细粒度3D状态变化
- 例如：姿态的细微变化、接触几何、相对运动

**结果**：VLM可能正确识别物体是什么，但对其物理状态如何演化保持不敏感。

#### 1.3.2 数据异质性差距 (Data Heterogeneity Gap)

**互联网规模视觉语料库**：
- 缺乏物理基础
- 丰富多样但抽象

**机器人演示数据集**：
- 稀缺、分布狭窄、收集成本高昂

**冲突**：现有VLA难以调和这两个极端，阻碍了它们在保持与现实世界控制相关性的同时吸收多样的空间体验。

### 1.4 提出方案：解耦学习范式

针对上述挑战，Pose-VLA提出了**解耦的学习范式**，将学习分为两个阶段：

#### 阶段1：大规模预训练
- 目标：在统一的、相机中心的观测空间中预训练通用空间先验
- 特征：大规模、非特定具身

#### 阶段2：轻量级后训练
- 目标：具身对齐
- 特征：特定机器人、数据高效

**核心思想**：
这种解耦从根本上重塑了大规模预训练的目标。通过引入3D空间监督，模型学会区分决定不同动作模式的细微3D变化。一旦建立了这些鲁棒的空间先验，VLM就成为一个强大的初始化，使得适应新的机器人策略比从头学习显著更高效。

### 1.5 Pose-VLA框架：以姿态为中心

基于上述原则，Pose-VLA引入了**以姿态重新中心的VLA学习框架**。

在这个架构中，**姿态作为结构桥梁**，将高级视觉感知映射到底层物理执行。

具体来说：

1. **解决粒度不匹配**：
   - 将物体状态和动作表示为3D姿态和时间序列
   - 显式约束模型在细粒度空间监督上推理

2. **解决数据异质性差距**：
   - 姿态token提供跨越多样化3D源的通用语言
   - 使Pose-VLA能够摄取大规模非机器人3D数据集与有限的机器人演示

**统一效果**：Pose-VLA将VLM转化为几何感知主干，针对下游操控进行优化。

### 1.6 技术实现

#### 1.6.1 架构基础
- 基于PaliGemma [[4]]架构
- 统一语义理解与空间基础

#### 1.6.2 多模态输入
- RGB图像
- 深度图
- 编码为射线图的相机内参

#### 1.6.3 3D意识注入
- 射线图提供几何先验
- 深度图提供度量信息

#### 1.6.4 训练策略
- **模态掩码策略**：训练期间使用，确保推理时对不同传感器可用性的鲁棒性
- **扩展词表**：用离散姿态token扩展语言模型词表，表示相机帧中的3D变换

### 1.7 训练流水线

#### 1.7.1 第一阶段：空间基础预训练
- 在大规模3D数据集上建立几何基础

#### 1.7.2 第二阶段：姿态对齐预训练
- 利用密集多视角监督将机器人轨迹投影到相同的相机中心观测空间

**协同效果**：这两个阶段将VLM从语义描述符转换为具身控制的基础。

### 1.8 后训练阶段

遵循 [[6]] 的方法，附加轻量级动作专家：
- 将预训练表示映射到特定机器人命令
- 确保VLM获得丰富的、3D感知特征
- 显著减少下游策略适应所需的演示数据量

### 1.9 主要贡献总结

1. **统一VLM框架**：
   - 整合RGB图像、深度图和相机内参
   - 注入内在3D意识
   - 促进视觉语言知识到机器人控制的有效转移

2. **离散姿态Token**：
   - 作为通用接口（在一致的相机中心观测空间内）
   - 对齐和摄取异构非机器人3D数据的空间先验
   - 整合专用机器人演示

3. **大规模预训练语料库**：
   - 140万图像 + 650万3D标注（用于空间基础）
   - 约155万多样化机器人轨迹（用于运动对齐）

4. **卓越性能**：
   - RoboTwin 2.0：平均79.5%成功率（SOTA）
   - LIBERO：平均96.0%（竞争力）
   - 真实世界：单个多任务模型仅需100个演示/任务

## 二、相关工作

### 2.1 用于3D基础的视觉语言模型

#### 2.1.1 发展趋势

越来越多的方法试图将通用VLM扩展到3D理解 [[15, 34, 38, 43, 12, 42]]。

#### 2.1.2 专门方法

几种方法专注于弥合视觉语言推理和3D几何任务之间的差距：

**SpatialLM** [[30]]:
- 从点云执行3D基础
- 生成3D边界框
- 限制：仅限于少量物体类别

**Spatial-Reasoner** [[28]]:
- 估计物体位置和方向作为中间输出
- 增强空间意识

**N3D-VLA** [[41]]（并发工作）:
- 利用大规模深度提升数据集
- 通过边界框实现3D定位
- 缺乏显式方向建模

#### 2.1.3 Pose-VLA的独特性

与之前方法相比，Pose-VLA的关键特征：
- **使用姿态token**统一物体姿态和运动轨迹于共享表示中
- **联合预训练**于异构3D数据集和机器人演示
- **结合相机射线和深度**作为空间先验
- 提供额外的几何感知用于3D基础和机器人操控

### 2.2 视觉语言动作模型

#### 2.2.1 VLA概述

Vision-Language-Action (VLA)模型扩展了Vision-Language Models (VLMs)的推理能力，直接预测机器人动作。

#### 2.2.2 动作输出类型

**离散方法** [[51, 18, 31, 45, 40, 21, 25, 14]]:
- 将动作预测视为下一个token预测任务
- 与大语言模型（LLM）的原生训练范式对齐

**连续方法** [[39, 6, 5, 7, 19, 27, 35, 25, 44, 32]]:
- 添加生成头（如扩散或流匹配）
- 实现更高的控制保真度

#### 2.2.3 核心挑战：VLM适应

尽管取得了进展，如何有效地将VLM的语义和推理能力适应动作模态仍然是一个开放问题。

**关键发现** [[47]] (VLM4VLA):
- 虽然VLM预训练是基础的
- 模型的通用VQA能力并不总是预测其下游控制性能
- 突出了持续的领域差距

#### 2.2.4 桥接差距的策略

**视觉文本数据共同训练** [[7, 50, 22, 32]]:
- 保留视觉语言知识

**具身思维链 (ECoT)** [[45, 19, 11, 37]]:
- 保存推理轨迹

**VLM潜在空间正则化** [[20, 17, 2]]:
- 将中层特征与视觉教师对齐
- 利用隐藏空间世界建模

#### 2.2.5 Pose-VLA的定位

与这些策略不同，Pose-VLA表明：
- **3D姿态预训练**为下游控制建立了至关重要的几何基础
- 这不是简单的辅助任务微调，而是根本性的几何基础

## 三、方法

### 3.1 预备知识

#### 3.1.1 问题形式化

给定多模态观测 **O** 和语言指令 **L**，Pose-VLA旨在建模输出token序列 **S** 的联合概率。

遵循自回归范式，生成过程表述为：

```
P(S|O,L) = ∏_{k=1}^K P(s_k|O,L,s_<k)
```

其中：
- s_k 表示长度为K的序列中的第k个token
- L 表示自然语言指令

#### 3.1.2 输出序列结构

为了弥合高级语义与细粒度3D控制，将输出定义为结构化的元组序列：

```
S = (τ₁, τ₂, ..., τ_T)
```

每个元组 τ_t 作为空间表示的基本单元：

```
τ_t = {c_t, b_t, p_t}
```

其中：
- **c_t**: 物体类别
- **b_t**: 图像坐标中的框中心 ∈ ℝ²
- **p_t**: 相机中心帧中的姿态 ∈ SE(3)

#### 3.1.3 设计原理

这个设计使 τ_t 能够作为通用几何原语：

**静态上下文中**：
- 定位物体用于空间基础

**时间序列中**：
- 表征轨迹航点用于运动估计

通过统一2D质心和3D姿态，τ_t 提供了一个多功能表示，在统一的生成框架中描述离散物理实体和连续运动路径。

### 3.2 VLM架构

#### 3.2.1 主干选择

采用 **PaliGemma** [[4]] 作为主干：

- **视觉编码器**: SigLip [[46]]
  - 从图像中提取丰富的语义特征
- **原始训练**: 主要在VQA和检测任务上

#### 3.2.2 问题识别

经验观察表明，PaliGemma在文本级或2D位置级的监督：
- 专注于高级语义
- 失去控制导向特征的细粒度细节

#### 3.2.3 解决方案：姿态作为通用表示

使用姿态作为通用表示来桥接：
- **非机器人数据**：计算机视觉任务
- **机器人数据**：操控任务

**关键优势**：
- 姿态在3D世界中描述
- 强制模型理解3D世界中的空间关系

#### 3.2.4 词表复用

为了保持固有的2D视觉基础能力，复用原始PaliGemma模型的词表：
- **文本词表**：表示 c_t（类别）
- **<loc> 词表**：表示 b_t（2D框中心）

#### 3.2.5 3D先验注入

为了融入3D先验，利用辅助输入：
- **深度图**
- **相机内参**

这些在3D标注的非机器人数据集中通常可用。研究表明，包含此类辅助信息显著提升了测试时的3D预测性能。

#### 3.2.6 坐标系选择

对于3D标注的非机器人数据，姿态定义在相机坐标系帧内。

**关键设计**：
- 要求模型直接在图像视图的相机帧中预测动作
- 而非估计机器人基帧中定义的机器人中心动作

**优势**：
- 确保跨异构数据源共享相机中心表示
- 使模型能够充分利用VLM预训练期间获得的3D意识

### 3.3 统一姿态表示

#### 3.3.1 核心理念

采用 **3D姿态作为通用表示**来统一非机器人3D数据集和机器人操控数据的特征空间。

#### 3.3.2 动机

这一选择由以下事实驱动：
- 物体和机器人夹爪都可以在此共享几何空间中有效参数化
- 通过强制此表示，约束模型直接从相机帧内的视觉观测估计空间坐标
- 自然促进鲁棒3D空间基础能力的发展

#### 3.3.3 相机中心姿态Token

引入专门的姿态token，定义在相机坐标系中：

**关键益处**：
- 显式解决预训练期间遇到的坐标不匹配挑战
- 促进多样化数据源无缝集成到一致框架中

#### 3.3.4 表征参数化

使用平移和旋转的参数化来表示物体状态和机器人运动轨迹。

##### 物体级表示

每个物体由其刚体变换表征，包括平移和旋转。

**离散化策略**：

**旋转**：
- 对欧拉角执行均匀量化
- 为每个主轴分配专门的 <rot> token

**平移**：
- 经验分析显示横向(x, y)和纵向(z)轴之间存在明显的分布差异
- x和y分布具有可比的均值和方差
- z轴（深度）分布显著偏移，反映相机固有的透视投影和深度范围

**适应策略**：
- 为x和y维度使用共享的 <trans_xy> token
- 为深度维度使用不同的 <trans_z> token

**尺度表示**：
- 为了在一般3D基准 [[8]] 中进一步评估3D基础性能，在表示中加入物体尺度
- 由于物体在所有三个轴上的物理尺寸显示出相似的统计特征
- 使用统一的 <size> token表示大小

##### 轨迹级表示

机器人动作可以由夹爪的6自由度姿态参数化。

**重用策略**：
- 通过采用此约定，夹爪姿态可以直接重用为物体级表示建立的离散token

**动态操控**：
- 对于动态操控，将动作轨迹表示为这些统一姿态token的时间有序序列

#### 3.3.5 与现有方法的对比

**传统方法** [[3, 29]]:
- 依赖将数值坐标分解为单个数字token
- 问题：效率低，token序列长度长

**Pose-VLA优势**：
- 通过减少每个坐标所需的token序列长度显著增强效率
- **关键优势**：确保所有任务数据在共享的现实世界度量空间中优化
- 在感知和动作之间实现统一的物理词汇表
- 通过监督模型估计目标物体姿态，驱动网络隐式学习3D空间特征
- 导致细粒度几何推理能力远远超过VQA监督主导的传统VLM

#### 3.3.6 相机中心动作表示

**现有VLA框架** [[18]]:
- 将动作定义为关节角度或基帧姿态
- 问题：观测空间（相机中心）和动作空间（机器人中心）之间存在根本不匹配

**空间差异**：
- 往往阻碍跨具身泛化
- 模型必须隐式桥接局部视觉观测和全局机器人基坐标系之间的差距

**Pose-VLA解决方案**：
- 将所有动作轨迹投影到每个视图的相应相机坐标系帧中
- **对齐效果**：确保非机器人3D基础数据和机器人演示数据共享一致的几何参考帧

#### 3.3.7 密集监督的优势

通过监督模型直接在图像观测空间中估计未来轨迹：

**相比传统方法的优势** [[18]]:
- 提供显著更密集、更基础的监督信号（而非映射像素到具身特定token）
- 增强跨多样化视角（如头戴式和腕载相机）的隐式3D对应学习
- 鼓励模型学习代表性、控制导向的特征，对连续观测之间的细微变化敏感

**统一优化**：
- 在共享token空间中表示夹爪和物体
- 统一大规模计算机视觉和机器人任务的优化目标
- 促进3D几何知识可扩展地转移到下游机器人操控

### 3.4 添加先验条件

#### 3.4.1 目标

为了增强框架的几何意识，融入辅助空间先验。

#### 3.4.2 融合策略

灵感来自Pow3R [[16]]，这些嵌入通过加性融合与初始Transformer块之前的RGB token嵌入集成，类似于标准的位置编码。

##### 相机射线编码

**目的**：建立像素与其物理观察方向之间的直接对应关系

**构建方法**：
从内参矩阵 **K ∈ ℝ³×³** 导出相机射线。

每个像素(u, v)的射线 **r** 计算为：

```
r = K⁻¹ [u, v, 1]ᵀ
```

**编码含义**：
- 编码相对于相机光中心的观察方向
- 使模型能够有效处理非中心裁剪
- 通过提供绝对几何锚点促进高分辨率推理

**处理流程**：
- 遵循RGB处理流水线
- 密集射线图被分块化并投影到潜在空间
- 与视觉特征融合

##### 深度先验集成

**输入**：深度图 **D** 和其对应的稀疏掩码 **M**

**关键决策**：
- 故意避免值归一化以保持与度量空间的显式对齐

**表示方法**：
将深度和掩码堆叠为联合表示：

```
[D, M] ∈ ℝᴴ×ᵂ×²
```

**处理流程**：
- 分块化并嵌入
- 通过集成有效像素掩码，使编码器能够鲁棒地处理不同级别的稀疏性

#### 3.4.3 多模态条件化效果

这种多模态条件化确保：
- 语义和3D几何信息的无缝集成
- 显著增强模型在复杂操控场景中进行精确3D推理的能力

### 3.5 训练策略

#### 3.5.1 预训练阶段

**目标**：使用下一个token预测损失向模型注入基本空间先验

**监督策略**：
为了保留基础VLM的广泛视觉语言能力：
- 联合监督2D定位token的生成
- 扩展的3D姿态token的生成

**损失函数**：
给定多模态输入 **I** 和语言指令 **L**，使用标准自回归目标：

```
L_pre(θ) = -∑_{k=1}^K log p_θ(s_k|φ(O), L, s_<k)
```

其中：
- s_k 表示结构化序列 **S** 中的第k个token
- φ(O) 表示模态掩码策略

**模态掩码策略**：
- 训练期间：随机零填充射线图或深度值
- 目的：确保模型即使仅有RGB输入也保持鲁棒推理能力
- 对于深度图像：采样稀疏深度点以增强对多样化传感器噪声模式的鲁棒性

**训练效果**：
- 为VLM配备粗略但语义基础的空间意识
- 使VLM能够从语义描述符转换为几何推理器

#### 3.5.2 后训练阶段

**目标**：具身对齐

**方法**：

- 将预训练表示从VLM主干映射到特定机器人命令

**解耦优势**：
允许主要焦点在预训练期间保持在大规模空间先验获取上

**具体实现**：
- 使用流匹配目标 [[23, 26]] 对动作token进行去噪
- VLM主干通过自注意力层与动作专家交互

## 六、核心问题详细解答

### Q1: 核心算法原理是什么？特别是通用姿态预训练

#### 6.1.1 算法架构概览

Pose-VLA的核心算法基于一个**解耦的学习范式**，将VLA训练分为两个截然不同的阶段：

```
┌─────────────────────────────────────────────────────────────┐
│                    Pose-VLA 两阶段架构                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  阶段1: 预训练                        │
│  ├─ 目标: 提取通用3D空间先验                                   │
│  ├─ 空间: 统一的相机中心观测空间                                │
│  ├─ 数据: 大规模3D数据集 + 机器人轨迹                          │
│  └─ 方法: 下一个token预测                                    │
│                                                              │
│  阶段2: 后训练                          │
│  ├─ 目标: 具身对齐                                           │
│  ├─ 空间: 机器人特定动作空间                                   │
│  ├─ 数据: 特定机器人演示                                       │
│  └─ 方法: 流匹配 + 动作专家                                   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

#### 6.1.2 通用姿态预训练的核心机制

##### 6.1.2.1 离散姿态Token化

**原理**：
将连续的3D姿态信息转换为离散的token，使其能够融入VLM的token序列生成框架。

**实现细节**：

**旋转Tokenization**:
```
旋转 → 均匀量化欧拉角 → 为每个主轴分配 <rot> token
```

**平移Tokenization**:
```
x, y轴 → 共享 <trans_xy> token
z轴 (深度) → 独立 <trans_z> token
```

**尺度Tokenization**:
```
物体尺寸 → 统一 <size> token
```

##### 6.1.2.2 两阶段预训练流水线

**第一阶段: 空间基础预训练**
```
目的: 建立基本的3D几何理解
数据:
  ├─ Omni3D (通用3D检测)
  ├─ Omni6DPose (6D姿态估计)
  └─ BOP Challenge (操控中心感知)
方法: 下一个token预测
输出: 具有粗略但语义基础空间意识的VLM
```

**第二阶段: 姿态对齐预训练**
```
目的: 将空间特征与运动控制对齐
数据:
  ├─ AgibotWorld Beta (100万轨迹)
  └─ InternData-A1 (55万轨迹)
方法: 密集多视角监督
输出: 具有运动先验的VLM
```

##### 6.1.2.3 统一观测空间

**关键设计**：所有动作轨迹投影到相机坐标系帧

**优势**：

### 8.1 姿态表示的理论基础

#### 8.1.1 为什么选择姿态？

**物理世界的本质**:
- 物体在3D空间中存在和运动
- 姿态（位置+方向）是物理实体的基本属性
- 动作本质上是姿态随时间的变化

**姿态的优势**:
```
姿态表示的优势:
├─ 几何完备性: 完整描述3D状态
├─ 度量意义: 直接对应物理世界
├─ 统一性: 物体和夹爪都可以用姿态表示
└─ 可解释性: 直观的物理意义
```

#### 8.1.2 离散化的必要性

**连续姿态的挑战**:
- 高维连续空间（6自由度）
- 需要大量数据覆盖
- 难以与离散token序列对齐

**离散化的好处**:
```
离散姿态Token:
├─ 高效表示: 减少token序列长度
├─ 可学习: 适应数据分布
├─ 可扩展: 易于集成到VLM
└─ 统一性: 创建共享词汇表
```

#### 8.1.3 Token化策略的理论支撑

**分布驱动的离散化**:

```
平移分布分析:
x, y轴分布: 相似均值和方差
       → 共享 <trans_xy> token

z轴(深度)分布: 显著偏移
       → 独立 <trans_z> token

旋转分布: 均匀(理论)
       → 均匀量化 <rot> token
```

**理论优势**:
- 适应数据的实际分布
- 提高表示效率
- 保持物理意义

### 8.2 两阶段学习的理论框架

#### 8.2.1 为什么需要两阶段？

**单阶段学习的问题**:
```
端到端学习VLA:
高级语义 ←─────┐
               ├──→ 纠缠
细粒度空间 ────┘

结果:
├─ 特征坍缩

```
阶段1 (预训练):

优势: 无需考虑特定具身
结果: 鲁棒的3D几何特征

阶段2 (后训练):
目标: 具身对齐
优势: 利用预训练表示
结果: 高效适应特定机器人
```

#### 8.2.2 预训练阶段的理论分析

**学习目标**:
```
L_pre(θ) = -∑_{k=1}^K log p_θ(s_k|φ(O), L, s_<k)
```

**理论基础**:
1. **下一个token预测**: 自回归生成

预训练学到:

**理论基础**:

### 8.4 多模态融合的理论分析

#### 8.4.1 RGB的局限性

**纯RGB的挑战**:
```
RGB图像:
├─ 丢失深度信息
├─ 缺乏度量意义
└─ 歧义性高
```

#### 8.4.2 深度的作用

**深度信息的优势**:
```
深度图:
├─ 显式3D信息
├─ 度量深度
└─ 减少歧义
```

**实验验证**:
- SUN RGB-D: RGB (39.0) → RGB+Depth (45.1)
- 真实世界长视野任务: 85% → 60% (移除深度)

#### 8.4.3 射线的作用

**射线编码的作用**:
```
相机射线:
├─ 编码观察方向
├─ 提供几何锚点
└─ 支持高分辨率推理
```

**实验验证**:
- Objectron: RGB (86.4) → RGB+Raymap (86.7)
- 完整模型: 87.3 (深度+射线最佳)

#### 8.4.4 融合策略的理论优势

**加性融合 (类似位置编码)**:

Fused = RGB_Embedding + Depth_Embedding + Ray_Embedding

优势:
├─ 保持多模态信息
├─ 统一表示空间
└─ 易于训练
```
