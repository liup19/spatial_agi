**摘要**:
在杂乱环境中的导航通常需要机器人容忍与可移动或可变形物体的接触以保持效率。现有的接触容忍运动规划（CTMP）方法依赖于间接的空间表示（如预构建地图、障碍物集），导致不准确性和对环境不确定性的适应性不足。

为了解决这个问题，我们提出了一个直接接触容忍（DCT）规划器，它将视觉-语言模型（VLMs）集成到直接点感知和导航中，包括两个关键组件：

1. **VLM点云分区器（VPP）**：使用VLM在图像空间中进行接触容忍推理，缓存推理掩码，使用里程计在帧间传播掩码，并将其投影到当前扫描以生成接触感知的点云。

2. **VPP引导导航（VGN）**：将CTMP表述为直接接触感知点云约束下的感知到控制优化问题，并由专门的深度神经网络（DNN）进一步解决。

我们在Isaac Sim和一个真实的类车机器人中实现了DCT，表明DCT在具有可移动障碍物的杂乱环境中实现了鲁棒和高效的导航，在多样化指标上优于代表性基线。代码可从 https://github.com/ChrisLeeUM/DCT 获得。

**关键贡献**:
- 提出了VPP，一个实时点分区器，使用VLM接触容忍推理和基于记忆的掩码传播来识别可移动物体
- 提出了VGN，一个快速学习规划器，直接在接触分区点云上操作，支持实时机器人规划和控制
- 在Isaac Sim和真实世界环境中实现了DCT，广泛评估表明在杂乱场景中比各种基线具有卓越的导航性能

## 核心算法原理

### 问题定义

**任务描述**:
考虑一个CTMP系统，由以下组成：
- 1个自主机器人
- 接触不耐受的固定障碍物集合 ℳ^fix
- 接触容忍的可移动障碍物集合 ℳ^mov

给定一个目标 s* 和机器人传感器输入 {𝒫, 𝒞, ℐ}，其中：
- 𝒪 表示激光雷达点云
- 𝒞 表示相机图像
- ℐ 表示惯性测量

CTMP的目标是生成一个机器人状态序列 𝒮 和可执行动作序列 𝒰 以到达目标。

这可以通过重复的有限时域规划来实现，其中系统仅规划一个有限的H个离散未来步骤序列：

ℋ_t ≜ {t, ⋯, t+H}

其中Δt是两个状态之间的时间长度。

具体地，在时间t，机器人状态表示为：

s_t = (x_t, y_t, θ_t)

其中(x_t, y_t)是位置，θ_t是方向。

机器人动作为：

u_t = (v_t, ψ_t)

其中v_t是线速度，ψ_t是角速度。

**接触容忍碰撞避免**:
在CTMP中，我们需要处理不同的障碍物ℳ^fix和ℳ^mov，碰撞避免与传统运动规划不同。

给定从激光雷达扫描获得的当前点集 ℙ_t = {p_t^1, ..., p_t^M}，其中p_t^i ∈ ℝ³是第i个点的位置，M是障碍物点的总数。

我们将ℙ_t分区为：

ℙ_t = ℙ_t^mov ∪ ℙ_t^fix

其中：
- ℙ_t^mov 表示可接触点
- ℙ_t^fix 表示接触不耐受点

对于CTMP，与可接触点ℙ_t^mov的接触是被允许的。因此，硬碰撞避免应该仅应用于ℙ_t^fix = ℙ_t \ ℙ_t^mov：

dist(𝔾_t(s_t), ℙ_t^fix) ≥ d_min, t ∈ ℋ_t

其中：
- 𝔾_t 表示机器人集合
- d_min 是预定义的安全边距

为了计算距离dist，等效于计算𝔾_t和ℙ_t^fix内任意两点之间的距离，然后取最小值：

dist(𝔾_t(s_t), ℙ_t^fix) = min{‖x - y‖ | x ∈ 𝔾_t, y ∈ ℙ_t^fix}

### 系统架构

DCT系统如图2所示，由两个主要模块组成：

#### 1. VPP（VLM Point Cloud Partitioner）

**目标**: 识别对应于可移动障碍物的激光雷达点。

**挑战**: VLM推理速度慢，而点云过滤需要实时更新。

**解决方案**: 引入两个互补策略：
1. **VLM驱动的过滤**: 利用VLM的推理能力准确分类障碍物
2. **实时掩码生成**: 实现高效、高频的激光雷达数据过滤

##### VLM驱动的障碍物过滤器

在时间t，给定一个定位提示ℒ_g和图像ℐ_t，使用定位模型GRD来定位候选障碍物区域：

𝕆_t = {o_t^i}_{i=1}^{N_t^0}

其中N_t^0是检测到的物体数量。

GRD是一个开放集检测器，结合视觉和语言来查找由ℒ_g指定的物体。由于"可移动"取决于任务和上下文（障碍物类型、环境、材料和机器人能力），引入一个VLM 𝒱_filt来过滤候选对象。

基于任务提示ℒ_filt以及ℐ, {o_t^N_0}，VLM返回可移动障碍物掩码列表：

𝕆^_t = {ô_t^j}_{j=1}^{N_t^1} = 𝒱_filt(ℒ_filt, ℐ, 𝕆_t)

其中N_t^1是过滤后的障碍物数量。然后记录时间t为刷新时间τ，障碍物列表𝕆^_τ和相应的机器人位置s_τ。

##### 记忆驱动的掩码生成

由于板载计算有限，VLM不能总是为每个点帧触发。在DCT中，只有当移动距离超过d_thres或自上次检测的时间超过t_thres时才触发VLM检测。

在两次视觉推理之间，因此重用和传播最后一个受信任的掩码，并在新检测到达时协调它们。由于机器人的移动，这个设计给多模态数据对齐带来了挑战。我们通过图3所示的三阶段程序来解决它：

**阶段I: 视点扭曲**

给定时间t的机器人状态和从激光雷达-惯性里程计获得的校准内参/外参，计算帧间单应性H_{τ→t}并扭曲每个掩码：

ô_t^n = 𝒲(ô_τ^n, H_{τ→t})

𝕆^_t = {ô_t^j}_{j=1}^{N_τ^1}

其中𝒲(·)对每个像素(x, y) ∈ ô_τ^n应用齐次投影。由于不可避免的状态估计误差，𝕆^_t可能仍然与当前场景错位。

**阶段II: 检测驱动的协调**

给定新图像ℐ_t，定位模型产生候选可移动区域：

𝕆_t = {o_t^i}_{i=1}^{N_t^0}

使用交并比（IoU）将这些与传播集合𝕆^_t匹配：

IoU(𝔸, 𝔹) = |𝔸 ∩ 𝔹| / |𝔸 ∪ 𝔹|

对于𝕆^_t中的每个ô_t^j，定义：

i*(j) = argmax_i IoU(ô_t^j, o_t^i)

其中i*(j)表示𝕆_t中与传播掩码ô_t^j达到最高IoU的检测的索引。

如果IoU(ô_t^j, o_t^{i*(j)}) ≥ σ_IoU（例如σ_IoU = 0.90），我们将该对视为成功匹配并更新受信任掩码几何：

ô_t^j ← o_t^{i*(j)}

没有有效匹配的传播掩码为了安全被丢弃。相反，𝕆_t中未匹配的检测作为新的受信任掩码插入。结果是协调集合𝕆^_t。

**阶段III: 点级细化和发布**

设时间t的激光雷达扫描为：

ℙ_t = {p_t^m}_{m=1}^M, p_t^m = [x_t^m, y_t^m, z_t^m]^T ∈ ℝ³

每个点被增强为p̃_t^m = [x_t^m, y_t^m, z_t^m, 1]^T并投影到图像：

ξ_t^m = π(𝐊 𝐓_{L→C}(t) p̃_t^m)

π([a, b, c]^T) = [a/c, b/c]^T

其中ξ_t^m ∈ ℝ²是像素坐标。

使用协调掩码𝕆^_t（设𝒰^_t = ⋃ ô_t^i），扫描被分区为：

ℙ_t^mov = {p_t^m ∈ ℙ_t | ξ_t^m ∈ 𝒰^_t}

ℙ_t^fix = ℙ_t \ ℙ_t^mov

虽然掩码门控有效，但图像级决策忽略3D连续性，可能包括孤立斑点或遗漏来自同一物体的附近返回。因此我们在ℙ_t^mov上执行3D欧几里得聚类（例如，半径为ε_cluster和最小大小为N_min的DBSCAN）以获得聚类𝒞_t^mov = {C_k}。然后应用两个对称修正：

1. **异常值抑制**: 被掩码选择但不属于任何|C_k| ≥ N_min的聚类C_k的点从可移动中移除：

ℙ_t^mov ← ⋃_{k: |C_k| ≥ N_min} C_k

2. **聚类完成**: ℙ_t^fix中位于某个可移动聚类C_k的距离ε_cluster内的点（即dist(p, C_k) ≤ ε_cluster）被添加到可移动：

ℙ_t^mov ← ℙ_t^mov ∪ {p ∈ ℙ_t^fix | dist(p, C_k) ≤ ε_cluster, ∃k}

最后，以激光雷达速率发布精炼流(ℙ_t^mov, ℙ_t^fix)。该流水线在稀疏视觉更新下保持时间一致性，同时为下游规划和控制生成空间一致的点集。

#### 2. VGN（VPP Guided Navigation）

**目标**: 使用快速学习规划器将过滤的点集ℙ_t^fix映射到控制信号u_t。

还包括一个点纠正块，在推送失败时更新障碍物可移动性属性。

##### 学习距离

给定由VPP生成的ℙ_t^fix，距离dist的计算是一个凸优化问题：

∀p_t^i ∈ ℙ_t^fix: min_x ‖𝐑_t(s_t)x + 𝐭_t(s_t) - p_t^i‖²₂

s.t. 𝐆x ⪯ 𝐠

其中：
- 𝐑_t(s_t) 是表示机器人方向的旋转矩阵
- 𝐭_t(s_t) 是表示机器人位置的平移向量
- 矩阵(𝐆, 𝐠)定义机器人边缘

然而，当ℙ_t^fix中的点数很大时，复杂性很高，导致不可接受的延迟。

为了解决这个问题，我们提出了一种更快距离计算的DNN方法。DNN训练一个代理通过模仿最优算法来解决问题，将耗时的迭代计算转换为实时前馈推理。

DNN的系统架构如图4所示，包括三个阶段：

**阶段I: 双重变换**

首先在强对偶性属性下将问题转换为其双重表述：

∀p_t^i ∈ ℙ_t^fix: max_{λ_t^i, μ_t^i} μ_t^i^T(𝐆𝐑_t(s_t)^T[p_t^i - 𝐭_t(s_t)] - 𝐠)

s.t. μ_t^i ⪰ 0, ‖λ_t^i‖ ⪯ 1, μ_t^i^T𝐆 + λ_t^i^T𝐑(s_t) = 0

其中{μ_t^i, λ_t^i}是对偶变量。

**阶段II: 演示数据生成**

解决双重问题以生成演示数据集𝒟 = {𝒜^{(j)}}_{j=1}^J。

第j个样本由𝒜^{(j)} = {𝒳^{(j)}, 𝒴^{(j)}}给出，其中输入元组由：

𝒳^{(j)} = {𝐆, 𝐠, s_t^{(j)}, p_t^{(j)}}

输出元组：

𝒴^{(j)} = {μ_t^{(j)}, λ_t^{(j)}}

**阶段III: DNN训练和推理**

DNN由6个具有32个单元的全连接层组成，具有ReLU、归一化和tanh激活函数。对于模型训练，使用均方误差作为损失函数，并使用随机梯度下降优化网络参数。

预训练的DNN模型部署在机器人上进行实时推理。

使用DNN，{λ_t^i, μ_t^i}对问题的最优解在微秒内获得。因此距离为：

dist(𝔾_h(s_h), ℙ_h^fix) = DNN(𝐆, 𝐠, s_h, ℙ_h^fix)

其中DNN(·)取所有p_h^i ∈ ℙ_h^fix的目标的最小值。

##### 运动规划

VGN问题可以表述为优化问题：

𝖯_t: min_{𝒰_t, 𝒮_t} C_t(𝒮_t)

s.t.
s_{h+1} = s_h + f(s_h, u_h)Δt, h ∈ ℋ_t

𝐮_min ⪯ 𝐮_h ⪯ 𝐮_max, h ∈ ℋ_t

DNN(𝐆, 𝐠, s_h, ℙ_h^fix) ≥ d_min, h ∈ ℋ_t

其中C_t(𝒮_t)是鼓励机器人接近目标s*的系统效用。它测量与给定参考路径的距离偏差：

C_t(𝒮_t) = Σ_{h∈ℋ_t} ‖s_{h+1} - s_{h+1}^♢‖²₂

其中s_{h+1}^♢ = {s_{t+1}^♢, ..., s_{t+H}^♢}是使用全局路径规划器（例如A*）计算的航点，s*是目标。

注意，第III节中的运动集合ℱ_t现在显式变为：

ℱ_t = {(12b), (12c), (12d)}

其中(12b)表示状态演化（f(·)定义车辆动力学），(12c)表示物理约束（𝐮_min, 𝐮_max是控制限制）。我们集体定义这些时不变参数为：

𝒫 = {f, 𝐮_min, 𝐮_max, d_min, 𝐆, 𝐠}

这个问题可以通过交替优化{𝒰_t, 𝒮_t}并执行DNN来解决。

##### 点纠正

当机器人未能推动或穿过物体时，VGN进入纠正模式。我们也将机器人视为卡住，如果线速度不为零但两个时间步之间的姿态变化非常小。

设Δd_t = ‖s_t - s_{t-1}‖₂。如果v_t ≥ v_min且Δd_t ≤ d_stuck连续τ_stuck步，则触发纠正。

首先，更新点标签。找到属于在失败的推送期间未移动的障碍物的点，并将它们表示为ℙ_t^{fail}。这些点被重新标记为不可移动。点集然后更新为：

ℙ_t^mov ← ℙ_t^mov \ ℙ_t^{fail}

ℙ_t^fix ← ℙ_t^fix ∪ ℙ_t^{fail}

接下来，机器人向后移动到先前的安全姿态。我们保持状态的短历史：

𝕊* = {s_t, ..., s_{t-F}}

其中F是回溯步数，s_{t-F}是最后一个可行检查点。我们暂时用𝕊*（反向遍历）替换参考路径，并以速度限制v_t ∈ [-v_rev, 0], v_rev > 0执行反向导航。机器人沿历史状态向后移动直到到达s_{t-F}。

最后，全局规划器计算避免新固定点的新参考路径𝕊^{new}，机器人沿此路径恢复正常导航。

### 算法流程

**算法1: DCT运动规划器**

输入:
- RGB图像{ℐ_t}
- 点云{ℙ_t}
- 机器人姿态{s_t}
- 目标s*
- 定位模型GRD
- 任务提示ℒ_filt
- 模型𝒱_filt
- 阈值d_thres, t_thres, σ_IoU
- 参数𝒫

输出: 规划轨迹s_t，控制序列u_t

**初始化**:
- 记忆列表𝕆^_τ
- 最后刷新(τ, s_τ)

**每个时间步t的主循环**:

1. **VLM驱动的障碍物过滤器**:
   - 如果dist(s_t, s_τ) > d_thres或(t - τ) > t_thres或𝕆^_τ = ∅:
     - 𝕆_t ← GRD(ℐ_t)
     - 𝕆^_t ← 𝒱_filt(ℒ_filt, ℐ_t, 𝕆_t)
     - (τ, s_τ) ← (t, s_t), 𝕆^_τ ← 𝕆^_t

2. **记忆驱动的掩码生成**:
   - 使用姿态变化将𝕆^_τ扭曲到t
   - 新检测𝕆_t ← GRD(ℐ_t)
   - 通过IoU协调掩码（≥ σ_IoU）
   - 丢弃未匹配，获得𝕆^_t，并更新：
     - ℙ_t^mov ← PointsInside(𝕆^_t)
     - ℙ_t^fix ← ℙ_t \ ℙ_t^mov
   - 通过聚类精炼ℙ_t^mov, ℙ_t^fix
   - 发布ℙ_t^mov, ℙ_t^fix

3. **VPP引导导航**:
   - (𝒮_t, 𝒰_t) ← VGN(s_t, ℙ_t^fix; s*, DNN, 𝒫)

## 与Spatial AGI的关系

### 空间理解与表示

**直接点云表示的优势**:
DCT采用直接点云表示，这代表了一种更接近原始传感器数据的空间表示方法，这与Spatial AGI的核心思想一致：

1. **从底层感知到高层理解**: DCT从激光雷达点云和相机图像直接学习可移动性推理，避免了传统方法中预建地图和障碍物集合的间接表示。这符合Spatial AGI中从原始传感器数据直接理解空间的要求。

2. **多模态空间感知**: 系统融合了视觉（RGB图像）和激光雷达点云，使用VLM进行跨模态推理。这体现了Spatial AGI中多模态空间理解的能力。

3. **空间语义理解**: VLM不仅识别物体，还理解其可移动性等语义属性。这是Spatial AGI中空间语义理解的重要方面。

### 空间关系处理

**接触容忍推理**:
DCT的核心创新之一是能够理解和处理空间中的接触关系：

1. **空间交互推理**: 系统能够判断哪些障碍物可以接触，哪些必须避免。这需要理解物体属性（如可移动性、可变形性）、机器人能力以及任务要求之间的复杂关系。

2. **动态空间推理**: 通过记忆驱动的掩码传播，系统能够在机器人移动时保持对空间的理解。这种时间连续性是Spatial AGI中动态空间推理的关键。

3. **适应性空间规划**: 根据环境变化（如发现某些障碍物实际上不可移动），系统能够动态调整空间策略。这种适应性是真正空间智能的标志。

### 对Spatial AGI的启发

**技术路线启发**:

1. **VLM用于空间推理**: DCT展示了将视觉-语言模型用于空间语义推理的有效性。这为Spatial AGI提供了一种利用大语言模型进行空间理解的路径。

## 创新点和局限性

### 主要创新点

#### 1. VLM点云分区器（VPP）

**创新1: VLM驱动的接触容忍推理**
- **传统方法**: 依赖预定义规则或学习模型来判断物体可移动性
- **DCT创新**: 使用视觉-语言模型进行开放集、上下文感知的可移动性推理
- **优势**: 能够理解"可移动"的复杂语义，考虑物体类型、环境、材料和机器人能力等多重因素

**创新2: 记忆驱动的掩码传播**
- **挑战**: VLM推理慢，无法为每个点帧触发
- **DCT创新**: 缓存VLM推理结果，通过单应性在帧间传播掩码，并结合新检测进行协调
- **优势**: 实现了高频点云分区，同时保持VLM的准确性

**创新3: 三阶段掩码生成**
- **阶段I - 视点扭曲**: 考虑机器人移动，通过单应性变换掩码
- **阶段II - 检测驱动的协调**: 使用IoU匹配新旧检测，处理错位问题
- **阶段III - 点级细化**: 通过3D聚类处理图像级决策的局限性
- **优势**: 保证了时间一致性和空间连贯性

#### 2. VPP引导导航（VGN）

**创新1: 学习距离计算**
- **传统方法**: 直接优化求解距离，计算复杂度高
- **DCT创新**: 通过DNN模仿优化过程，将迭代计算转换为前馈推理
- **技术细节**:
  - 将原始问题转换为双重表述
  - 生成演示数据集
  - 训练6层全连接网络（32个单元）
- **优势**: 实现了微秒级的距离计算

**创新2: 直接点约束**
- **传统方法**: 将点云转换为间接表示（如占用网格、凸集）
- **DCT创新**: 直接在分区点云上约束规划问题
- **优势**: 避免了间接表示的近似误差，提高了精度

**创新3: 在线纠正机制**
- **问题**: 推送失败时需要重新评估障碍物可移动性
- **DCT创新**: 检测失败情况，重新标记点，后退到安全状态，重新规划
- **优势**: 提高了系统鲁棒性和适应性

#### 3. 系统级创新

**创新1: VLM与学习规划的结合**
- 结合了VLM的语义理解能力和学习规划器的实时计算能力
- 实现了感知、推理、规划的端到端集成

**创新2: 多模态融合**
- 融合视觉（RGB）、激光雷达点云、惯性测量
- 通过单应性投影和掩码投影实现跨模态对齐

**创新3: 实时性能**
- 通过记忆机制和学习推理实现了实时系统
- 避免了传统CTMP方法的高计算开销

### 技术细节创新

**1. 掩码协调策略**:
- 使用IoU匹配（σ_IoU = 0.90）平衡准确性和稳定性
- 未匹配的传播掩码被丢弃（安全优先）
- 未匹配的新检测被添加（探索优先）

**2. 聚类修正**:
- 异常值抑制：移除不属于大聚类的孤立点
- 聚类完成：添加靠近可移动聚类的固定点
- 参数：ε_cluster（聚类半径）、N_min（最小聚类大小）

**3. 纠正触发条件**:
- v_t ≥ v_min（速度不为零）
- Δd_t ≤ d_stuck（姿态变化小）
- 连续τ_stuck步

**4. 点重新标记**:
- ℙ_t^{fail} ← 未移动的障碍物点
- ℙ_t^mov ← ℙ_t^mov \ ℙ_t^{fail}
- ℙ_t^fix ← ℙ_t^fix ∪ ℙ_t^{fail}

### 局限性

#### 1. 依赖VLM质量

**问题**:
- 系统性能高度依赖所选VLM的质量
- 实验表明不同VLM的性能差异很大（GPT-5: F1=90.32%, Llama 4: F1=78.94%）

**影响**:
- 错误的可移动性判断可能导致不安全接触或错失机会
- 需要选择高质量的VLM（如GPT-5）

**缓解措施**:
- 使用纠正机制重新评估失败情况
- 选择高精度的VLM（如GPT-5）

#### 2. 内存和计算资源

**问题**:
- VLM推理需要GPU支持
- DNN训练需要大量演示数据
- 实时系统需要高算力平台（实验使用RTX 4090）

**影响**:
- 部署到低功耗设备可能困难
- 实际机器人可能有资源限制

**缓解措施**:
- 使用云端VLM推理（可能增加延迟）
- 优化DNN模型大小
- 离线训练，在线推理

#### 3. 环境假设

**问题**:
- 假设障碍物要么可移动要么固定
- 未考虑部分可移动、可变形物体的复杂动力学
- 假设机器人能够推动某些物体（不考虑重量、摩擦等）

**影响**:
- 在复杂物理环境中可能失败
- 对机器人能力估计不准确可能导致失败

**缓解措施**:
- 在线纠正机制
- 扩展VLM提示考虑更多物理因素

#### 4. 点云-图像对齐

**问题**:
- 依赖准确的相机-激光雷达标定
- 状态估计误差会导致掩码错位
- 需要高精度的里程计

**影响**:
- 对齐误差可能导致错误的点分类
- 影响系统可靠性和安全性

**缓解措施**:
- 检测驱动的协调机制
- 3D聚类修正
- 使用高精度传感器和标定

#### 5. 训练数据依赖

**问题**:
- DNN需要大量演示数据（需要优化求解双重问题）
- 训练数据分布影响模型性能
- 可能难以覆盖所有场景

**影响**:
- 泛化能力可能有限
- 在新环境中可能需要重新训练

**缓解措施**:
- 使用域随机化
- 在线学习和适应

#### 6. 动态环境处理

**问题**:
- 虽然有记忆机制，但对快速变化的环境（如人移动）处理有限
- VLM更新频率低（基于距离或时间阈值）

**影响**:
- 在高度动态环境中可能滞后
- 可能错过新出现的可移动物体

**缓解措施**:
- 降低更新阈值
- 使用更快的目标检测器

### 与基线的比较

**vs. NeuPAN**:
- NeuPAN: 直接点导航，但无接触容忍
- DCT优势: 能处理可移动障碍物，在约束环境中成功率更高

**vs. Ellis22**:
- Ellis22: 需要占用网格地图，保守的障碍物膨胀

- 在混合杂乱环境中，随着可移动障碍物增加，DCT性能提升明显
- F1M3（1固定，3可移动）: 100%成功率，最短导航时间（7.69s）

## 个人思考

### 最有趣的发现

**1. VLM用于空间语义推理**

将VLM用于判断"可移动性"这一抽象空间语义是一个很巧妙的想法。传统的计算机视觉方法通常识别"是什么"（物体类别），而VLM可以理解"能做什么"（功能、属性）。这展示了大语言模型在空间理解中的潜力。

这让我想到，VLM还可以用于其他空间语义推理任务：
- "这个空间能容纳几个人？"
- "这个物体能用来垫脚吗？"
- "这条通道有多拥挤？"

**2. 记忆机制的优雅设计**

VLM推理慢但准确，目标检测快但语义理解弱。通过缓存VLM结果并通过几何变换传播，DCT巧妙地平衡了准确性和实时性。这种"慢推理、快传播"的思想可以应用到很多领域。

类比：
- SLAM中的关键帧选择（稀疏优化 + 密集插值）
- 人体姿态估计中的关键点检测 + 时序平滑
- 目标跟踪中的检测关联

**3. 学习替代优化的范式**

将复杂的优化问题转换为学习问题是机器学习中的经典思想，但DCT将其应用到了一个具体的空间规划问题上。通过训练DNN模仿优化求解器，实现了从迭代计算到前馈推理的转换。

这种范式的优势：
- 实时性：微秒级推理
- 可扩展性：DNN推理复杂度不随问题规模线性增长
- 泛化能力：可以处理训练数据中未见的情况

- H: 预测时域长度

- ψ_min, ψ_max: 最小/最大角速度限制

**纠正参数**:
- v_min: 最小速度阈值（用于检测卡住）
- d_stuck: 姿态变化阈值
- τ_stuck: 连续步数阈值
- F: 回溯步数
- v_rev: 反向速度限制

### DNN架构

**网络结构**:
- 层数: 6个全连接层
- 每层单元: 32个
- 激活函数: ReLU, 归一化, tanh
- 损失函数: 均方误差（MSE）
- 优化器: 随机梯度下降（SGD）

**输入维度**:
- 𝐆: 机器人边缘矩阵
- 𝐠: 机器人边缘向量
- s_t: 机器人状态（位置和方向）
- ℙ_t^fix: 固定障碍物点集

**输出维度**:
- μ_t: 对偶变量1
- λ_t: 对偶变量2

| F4M0 | 4 | 0 | 0.7 | 8.87 | 0.82 | 7.27 |

| F2M2 | 2 | 2 | 1.0 | 7.99 | 0.87 | 6.95 |
| F1M3 | 1 | 3 | 1.0 | 7.69 | 0.90 | 6.92 |

**关键观察**:
- 随着可移动障碍物增加，DCT性能显著提升
- F1M3达到最佳性能：100%成功率，最短时间，最高速度
- NeuPAN在约束环境中失败（场景2）

## 实验结果分析

### VPP有效性验证

**VLM选择实验**

论文评估了四个主流VLM在可移动性推理任务上的表现：

**GPT-5表现卓越**:
- Precision: 100% - 所有识别的可移动障碍物都是正确的
- Recall: 82.54% - 找到了大部分可移动物体
- F1-score: 90.32% - 综合性能最佳
- Accuracy: 90.00% - 总体分类准确率最高

这表明GPT-5在安全关键的机器人应用中是最佳选择，因为它不会错误地将固定障碍物识别为可移动。

**Gemini 2.5的激进策略**:
- Recall: 95.24% - 找到了几乎所有的可移动物体
- Precision: 75% - 但有25%的误判
- 这可能适用于探索导向的场景，但对安全有风险

**Qwen-vl和Llama 4**:
- 表现较为平衡但不如GPT-5
- Precision: 75% vs GPT-5的100%
- Recall: 80.95% / 83.33% vs GPT-5的82.54%

**选择结果**: GPT-5被选为默认VLM，平衡了安全性和覆盖率。

### 场景对比实验

**场景1: 可移动障碍物 + 宽侧路径**

这是最友好的场景，三种方法都成功了：
- **DCT**: 4.22秒，最快（相比Ellis22节省14%）
- **NeuPAN**: 4.37秒，接近DCT
- **Ellis22**: 4.917秒，最慢

DCT的优势来自其学习优化，保持了接近1 m/s的恒定速度，而Ellis22由于Pure Pursuit控制策略，在接近目标时明显减速。

**场景2: 可移动障碍物 + 窄侧路径**

这是关键的测试场景，侧路径太窄，不接触无法通过：
- **DCT**: ✓ 3.54秒，成功接触并推动障碍物
- **NeuPAN**: ✗ 失败，因为将可移动障碍物建模为硬约束
- **Ellis22**: ✓ 4.42秒，成功但较慢

这个场景凸显了接触容忍的价值。NeuPAN无法处理这种情况，因为它对所有障碍物都采用严格的碰撞避免。

**场景3: 固定障碍物 + 宽侧路径**

测试系统处理接触不耐受障碍物的能力：
- **DCT**: ✓ 5.72秒，5.33米
- **NeuPAN**: ✓ 5.62秒，5.34米
- **Ellis22**: ✓ 15.42秒，11.24米（绕路）

Ellis22的保守性明显：
- 首先转弯（0-4.2秒）
- 然后绕过货架（4.2-16秒）
- 导航时间是DCT的2.7倍
- 距离是DCT的2.1倍

这是间接表示（占用网格）的局限性导致的。占用网格需要保守的障碍物膨胀，降低了精度和效率。

### 混合杂乱环境实验

这是最全面的测试，评估不同比例的固定/可移动障碍物：

**F4M0（4固定，0可移动）**:
- 成功率: 70%（最低）
- 时间: 8.87秒
- 速度: 0.82 m/s
- 距离: 7.27米

这是最困难的场景，所有障碍物都必须避免。系统难以找到完全碰撞自由的路径。

**F3M1（3固定，1可移动）**:
- 成功率: 90%（提升20%）
- 时间: 8.53秒（减少4%）
- 速度: 0.83 m/s（提升1%）
- 距离: 7.08米（减少3%）

只需将一个固定障碍物改为可移动，性能就有显著提升。

**F2M2（2固定，2可移动）**:
- 成功率: 100%（达到最高）
- 时间: 7.99秒（减少10%）
- 速度: 0.87 m/s（提升6%）
- 距离: 6.95米（减少4%）

两个可移动障碍物使系统达到100%成功率。

**F1M3（1固定，3可移动）**:
- 成功率: 100%（保持最高）
- 时间: 7.69秒（最短，相比F4M0减少13%）
- 速度: 0.90 m/s（最高，相比F4M0提升10%）
- 距离: 6.92米（最短，相比F4M0减少5%）

这是最佳场景，系统可以最大化利用接触容忍。

**关键洞察**:
1. **线性改进**: 随着可移动障碍物增加，性能几乎线性提升
2. **阈值效应**: 从2个可移动障碍物开始，成功率达到100%
3. **复合效益**: 时间、速度、距离同时改善
4. **接触容忍的价值**: 在杂乱环境中，适当的接触可以大幅提升效率

### 真实机器人验证

论文在真实机器人的两个代表性场景中验证了DCT：

**场景1: 穿过窗帘**
- 窗帘被识别为可穿透（可接触）
- 系统预测掩码投影到扫描数据
- 整个运行过程中保持这个记忆
- 机器人穿过窗帘到达目标

这展示了：
- VLM可以理解"可穿透"这一空间属性
- 记忆机制能够持续跟踪可移动物体
- 系统能够处理可变形障碍物

**场景2: 在可移动箱子之间导航**
- 小箱子被分类为可移动
- 机器人与小箱子进行无害接触
- 同时对所有其他障碍物保持硬碰撞约束
- 机器人从椅子下方穿过，敏捷地避开椅子腿

这展示了：
- 精细的物体分类能力
- 同时处理不同类型障碍物的能力
- 直接点表示处理任意形状障碍物的优势

**真实世界挑战**:
- 传感器噪声
- 不完美的标定
- 状态估计误差
- 环境光照变化

DCT在真实环境中成功运行，展示了其鲁棒性。

## 相关工作

### 接触容忍运动规划（CTMP）

CTMP是机器人导航中的一个重要研究方向，核心是在杂乱环境中通过允许受控的接触来提高效率。现有方法可以分为三类：

#### 1. 基于搜索的方法

**核心思想**:
- 将CTMP表述为全局搜索任务
- 在执行前显式推理障碍物的位置和可移动性
- 构建完整的交互序列

**技术方法**:
- **反向顺序搜索**: 通过限制每个物体为单次交互，使用反向搜索减少搜索空间
- **LLM引导采样**: 使用大语言模型引导非均匀采样，高效决定哪些障碍物移动以及按什么顺序推动

**优势**:
- 全局最优性（理论上）
- 完整的交互计划

**局限**:
- 需要完全可观察的环境（地图）
- NP难组合问题，计算复杂度高
- 障碍物数量增加时扩展性差

**代表工作**:
- Stilman et al., 2008: 反向顺序搜索
- Zhang et al., 2025: LLM引导的交互规划
- Zhang et al., 2025: NAMO (Navigation Among Movable Obstacles)

#### 2. 基于强化学习的方法

**核心思想**:
- 将CTMP表述为策略学习问题
- 神经网络通过与接触容忍和接触不耐受障碍物的交互来学习策略

**技术方法**:
- **课程学习框架**: 从简单的碰撞避免任务逐步学习到更复杂的CTMP任务
- **域随机化**: 增强策略鲁棒性，使策略能够有效适应真实世界场景的不确定性
- **分层RL**: 结合高级推送策略生成和低级全身控制

**优势**:
- 能够处理复杂动力学
- 通过交互学习
- 不需要预定义规则

**局限**:
- 奖励稀疏问题
- 在真实世界环境中训练困难
- 训练分布敏感，泛化能力有限
- 如何为各种可移动障碍物设置适当的惩罚函数仍然具有挑战性

**代表工作**:
- Frank et al., 2014: 学习交互策略
- 10057016: 课程RL框架
- Yang et al., 2025: 分层RL框架
- Yao et al., 2024: 域随机化

#### 3. 混合规划方法

**核心思想**:
- 集成离线全局搜索和在线局部规划
- 可移动障碍物在执行期间被视为自由空间或低成本区域

**技术方法**:
- **物体定位 + 可移动性推理**: 结合物体定位和可移动性推理的扩展路径规划
- **有向可见性图（DV-graph）**: 边编码通行性和障碍物操作成本，交互规划更新障碍物可供性
- **优化框架**: 将可移动物体建模为交互元素，将接触动力学纳入轨迹优化

**优势**:
- 减少了详尽全局搜索的计算开销
- 提高了不确定环境中的灵活性
- 平衡全局最优和局部适应

**局限**:
- 通常依赖于间接空间表示，这会不可避免地给机器人动作带来误差
- 占用网格地图需要保守的障碍物膨胀，降低精度
- 限制在高精度任务中的适用性

**代表工作**:
- Ellis et al., 2022: 物体定位和可移动性推理
- 10802169: 有向可见性图
- Wang et al., 2025: 交互轨迹优化

### DCT的定位

**与基于搜索的方法对比**:
- DCT不需要全局地图，适合真实世界
- 使用学习规划器，避免NP难搜索
- 在线实时规划，更灵活

**与基于RL的方法对比**:
- DCT结合了优化和学习的优势
- 不依赖复杂的奖励设计
- 使用VLM进行显式推理，更可解释

**与混合方法对比**:
- DCT使用直接点表示，避免间接表示的误差
- 不需要预构建地图
- 更适合未知或半已知环境

### 技术组件相关工作

#### 1. 视觉-语言模型（VLMs）

**用于空间推理**:
- Grounding DINO (GRD): 开放集目标检测
- GPT-5, Gemini 2.5: 多模态推理
- Qwen-vl, Llama 4: 开源VLM

**应用**:
- 物体定位和识别
- 空间语义理解
- 任务条件推理

#### 2. 点云处理

**分割和聚类**:
- DBSCAN: 基于密度的聚类
- PointNet: 点云深度学习
- 3D欧几里得聚类

**多模态融合**:
- 点云投影到图像
- 单应性变换
- 掩码传播

#### 3. 学习优化

**学习求解器**:
- 深度展开网络
- 模仿优化求解器
- 端到端学习

**应用**:
- 距离计算
- 约束求解
- 实时推理

#### 4. 记忆机制

**时序一致性**:
- 掩码缓存
- 状态估计
- 关联和匹配

**在线学习**:
- 失败检测
- 重新标记
- 自适应更新

## 总结

### 核心贡献

1. **VPP（VLM Point Cloud Partitioner）**:
   - 首次将视觉-语言模型集成到点云分区中
   - 创新的记忆驱动掩码传播机制
   - 平衡了VLM的准确性和实时性要求

2. **VGN（VPP Guided Navigation）**:
   - 学习替代优化，实现微秒级距离计算
   - 直接点约束，避免间接表示误差
   - 在线纠正机制，提高鲁棒性

3. **系统级创新**:
   - 端到端的接触容忍导航系统
   - 在仿真和真实环境中验证
   - 性能优于多个代表性基线

### 技术优势

**准确性**:
- GPT-5实现90%以上的分类准确率
- 直接点表示避免了间接表示的近似误差
- 三阶段掩码生成保证空间连贯性

**实时性**:
- DNN实现微秒级推理
- 记忆机制减少VLM调用频率
- 整个系统适合实时导航

**鲁棒性**:
- 在线纠正机制处理失败情况
- 聚类修正处理对齐误差
- 在真实环境中成功运行

**适应性**:
- 支持多种VLM（可替换）

- 不依赖预构建地图

### 对Spatial AGI的意义

DCT为Spatial AGI提供了以下启示：

1. **VLM作为空间推理引擎**: 展示了如何利用大语言模型进行空间语义理解

DCT代表了朝着真正空间智能迈出的重要一步。通过结合VLM的语义理解能力、学习规划器的实时计算能力和记忆机制的时间一致性，系统在杂乱环境中实现了鲁棒、高效的导航。
