## 摘要

WorldCam提出了一种基于相机位姿的统一几何表示方法，用于构建交互式3D游戏世界模型。该方法的核心创新在于将用户动作映射到精确的6自由度相机位姿，并利用这些位姿作为空间索引来检索相关的历史观测，从而实现：

1. **精确的动作控制**：通过基于李代数的物理连续动作空间，将复杂的用户输入（如键盘和鼠标的耦合动作）转化为几何准确的相机位姿
2. **长期3D一致性**：利用全局相机位姿作为空间索引，在长时间导航中实现几何一致的位置重访
3. **高质量长期推理**：通过渐进式噪声调度和注意力机制，维持长时间序列的视觉质量

为了支持这项研究，作者还引入了WorldCam-50h数据集，包含3,000分钟的真实人类游戏玩法视频，标注了相机轨迹和文本描述。大量实验表明，该方法在动作可控性、长期视觉质量和3D空间一致性方面显著优于最先进的交互式游戏世界模型。

## 核心贡献

### 1. 统一的相机位姿表示
WorldCam首次将相机位姿确立为连接即时动作控制和长期3D一致性的统一几何表示。这一设计解决了以往工作将动作视为抽象条件信号的问题，通过显式的几何约束实现了精确的动作对齐和3D一致性。

### 2. 基于李代数的动作到相机映射
与以往采用解耦线性近似的方法不同，WorldCam将用户动作建模为李代数中的空间速度，严格推导出精确的6自由度相对位姿。这种方法能够准确捕获耦合动力学（如螺旋运动），而这是简单的线性近似无法做到的。

### 3. 位姿锚定的长期记忆机制
利用全局相机位姿作为空间索引来检索相关的历史潜在表示，实现了几何一致的位置重访。检索到的潜在表示与其关联的相机位姿嵌入建立了当前潜在表示与历史潜在表示之间的几何对应关系。

### 4. WorldCam-50h数据集
引入了大规模的人类游戏玩法数据集，包含来自Counter-Strike、Xonotic和Unvanquished的3,000分钟视频，标注了相机轨迹和文本描述。该数据集涵盖了复杂场景，如一般导航、快速360°相机旋转和反向穿越多样化的几何结构。

## 📝 核心问题分析

### Q1: 核心算法原理

**问题**: 这篇文章的核心算法原理是什么？

**答案**:

WorldCam的核心算法建立在视频扩散变换器（Video Diffusion Transformer, DiT）基础之上，通过引入相机位姿作为统一的几何表示，实现了交互式3D游戏世界模型的三大关键能力：精确动作控制、长期3D一致性维持和高质量视觉生成。

#### 1.1 动作到相机的精确映射（Action-to-Camera Mapping）

**核心思想**：
交互式3D世界建模的核心挑战是将用户动作转化为物理一致的相机运动。以往工作通常直接将原始动作信号注入生成模型，或采用简单的线性近似，这会导致错位或几何不一致的相机轨迹，特别是在涉及耦合平移和旋转的复杂运动下。

**技术实现**：
WorldCam定义了一个基于李代数𝔰𝔢(3)的连续动作空间。在每个从帧I_{i-1}到I_i的过渡中，用户动作A_i被表示为扭转向量：

```
A_i = [v_i; ω_i] ∈ ℝ^6
```

其中v_i = [v_x, v_y, v_z]ᵀ ∈ ℝ³和ω_i = [ω_x, ω_y, ω_z]ᵀ ∈ ℝ³分别表示线速度和角速度。

然后通过矩阵指数映射推导出对应的相对相机位姿ΔP_i ∈ SE(3)：

```
ΔP_i = exp(Â_i) = [ΔR_i  Δt_i]
                   [0ᵀ    1 ]
```

其中Â_i ∈ 𝔰𝔢(3)表示扭转向量A_i的4×4矩阵。

**关键优势**：
与解耦的线性近似不同，这种表述直接在SE(3)流形上联合积分线速度和角速度。这种设计即使在涉及紧密耦合平移和旋转的复杂用户动作下，也能产生几何精确的相机轨迹。

#### 1.2 相机控制的视频生成（Camera-Controlled Video Generation）

**核心思想**：
设计了一个以从动作导出的相机运动为条件的视频生成模型。

**技术实现**：
给定通过动作到相机映射获得的相对相机位姿序列{ΔP_i}_{i=1}^F，将它们累加为与窗口第一帧对齐的全局相机位姿。为了提供显式的视相关几何条件，相机位姿被转换为Plücker嵌入P̂ ∈ ℝ^{F×6}。

为了将相机控制注入DiT，引入了一个由两个MLP层组成的轻量级相机嵌入模块c_φ。由于VAE在时间上以因子r压缩输入，因此通过为每个潜在帧连接r个连续的Plücker嵌入来将相机条件与潜在序列对齐，结果得到p̂ ∈ ℝ^{f×(6r)}，其中f = F/r。

相机嵌入在每个自注意力层后被添加到DiT特征d中：

```
d ← d + c_φ(p̂)
```

**设计原理**：
Plücker嵌入提供了一种对相机位姿的显式、可微分的表示，能够精确捕获相机运动与3D场景结构之间的几何关系。通过将这些嵌入注入到DiT的中间特征中，模型能够在生成过程中保持对相机运动的显式感知。

#### 1.3 位姿锚定的长期记忆（Pose-Anchored Long-Term Memory）

**核心思想**：
除了精确的动作控制，利用从3.2节导出的相机位姿作为几何锚点来维持3D一致性，确保在重访位置或视点时的空间连贯性。

**全局位姿累加**：
由于动作到相机映射严格遵循SE(3)几何，相对运动可以可靠地累加为全局相机位姿。具体而言，第j帧I_j的全局位姿通过位姿组合计算：

```
P_j^{global} = P_{j-1}^{global} ∘ ΔP_j, P_0^{global} = I
```

其中I是单位矩阵，∘表示位姿组合。

**位姿索引的记忆检索**：
为了利用长期空间上下文，维护一个长期记忆池M，存储以前生成的潜在表示及其全局相机位姿。由于视频VAE的时间压缩，每个潜在z_j对应于r个连续帧的短片段，因此与一组r个全局相机位姿相关联。为了简单起见，将每个记忆条目表示为(P_j^{global}, z_j)，其中P_j^{global}表示与r个帧关联的全局相机位姿集合：

```
P_j^{global} = [R_j  t_j]
               [0ᵀ   1]
```

考虑到游戏环境中的复杂布局和频繁遮挡，采用分层记忆检索策略，使用全局相机位姿作为记忆库上的空间索引：

1. **位置检索**：首先选择相机位置最接近当前位置t_i的前K个候选者M_trans：
   ```
   M_trans = TopK_K(-‖t_j - t_i‖_2; (P_j^{global}, z_j) ∈ M)
   ```

2. **方向检索**：从M_trans中，进一步选择L个条目（L ≤ K），其观察方向与当前方向R_i最对齐，使用相对旋转矩阵的迹来测量：
   ```
   M_rot = TopK_L(tr(R_jᵀR_i); (P_j^{global}, z_j) ∈ M_trans)
   ```

**长期记忆条件**：
检索到的记忆条目(P_j^{global}, z_j) ∈ M_rot提供在生成期间强制执行空间一致性所需的几何上下文。将检索到的潜在表示与当前输入潜在序列连接。它们关联的相机位姿被重新对齐到当前去噪窗口的第一帧，通过相机嵌入模块c_φ嵌入，并注入到DiT的中间特征中。这允许模型在当前潜在表示和长期记忆潜在表示之间建立显式的几何对应关系。

#### 1.4 渐进式自回归推理（Progressive Autoregressive Inference）

**渐进式噪声调度**：
为了支持长期自回归视频扩散，采用渐进式每帧噪声调度，为每个去噪窗口内的潜在帧分配单调递增的噪声水平。这种设计在早期帧中提供可靠的低噪声锚点，同时保持未来帧处于较高噪声水平，从而使其可校正。这通过大窗口重叠实现高效的跨帧条件和稳定的前向展开。

具体而言，将扩散过程离散化为N个推理步骤，具有单调递增的噪声水平，并将它们划分为S个阶段，其中S等于序列中维护的噪声潜在帧的数量。噪声潜在序列z_t被重新表述为阶段性的z_s，其中s ∈ {0,…,S-1}表示噪声阶段，每个潜在帧必须完成所有阶段的所有去噪步骤才能完全去噪。

完成所有S个阶段后，潜在序列前移：最早的潜在帧被驱逐并由VAE解码，并将一个新初始化的纯噪声潜在帧附加到序列末尾。

**注意力接收器（Attention Sink）**：
虽然渐进式噪声改善了时间稳定性，但在大型和复杂的游戏运动下仍可能累积误差，导致视觉饱和和扭曲的UI元素。为了缓解这个问题，引入了受StreamingLLM启发的注意力接收器机制，通过锚定一小部分初始标记来稳定注意力。在推理期间，保留全局初始帧作为注意力锚点，帮助保持帧保真度、场景风格和UI一致性。

**短期记忆**：
在实践中，发现提供最近生成的潜在表示（称为短期记忆）对于减少自回归生成期间的误差漂移至关重要。凭经验将短期记忆潜在表示的数量设置为与生成的潜在表示数量匹配，在稳定性和计算效率之间取得平衡。

### Q3: 自由问题

**问题**: WorldCam的相机位姿统一表示机制如何应对极端或异常的相机运动？在用户输入导致不合理的相机轨迹时，系统如何保证生成的场景仍然保持视觉合理性和几何一致性？

**答案**:

这是一个非常深刻的问题，因为它触及了WorldCam在真实应用中的鲁棒性和泛化能力。通过仔细分析论文，我们可以从以下几个方面回答这个问题：

#### 3.1 李代数表示的内在约束

**数学上的合理性保证**：
WorldCam使用李代数𝔰𝔢(3)来表示相机运动，这个数学框架本身提供了一些内在的约束：

- **SE(3)流形约束**：所有的相机位姿都严格位于SE(3)流形上，这意味着即使输入了极端的线速度或角速度，生成的相机位姿始终保持为一个有效的刚体变换。这从数学上保证了不会产生"不合理"的相机位姿（例如，出现非正交的旋转矩阵或扭曲的缩放）。

- **连续性保证**：矩阵指数映射exp: 𝔰𝔢(3) → SE(3)是一个连续且光滑的映射。这意味着即使输入的速度向量发生突变，生成的相机位姿仍然保持连续和平滑。这种连续性有助于避免视觉上的剧烈跳跃。

**速度幅度的自然限制**：
虽然论文没有明确提到对速度向量的硬约束，但实际的物理意义为速度幅度提供了自然的限制：

- **游戏环境的限制**：在真实的游戏环境中，用户输入的速度幅度受游戏引擎和物理系统的限制。即使是最剧烈的鼠标甩动或键盘输入，也只是在一定范围内的变化。
- **训练数据的覆盖**：WorldCam-50h数据集包含了"快速360°相机旋转"等极端动作，这意味着模型在训练时已经见过并学会了处理这类极端但合理的运动。

#### 3.2 渐进式推理的稳定性机制

**噪声调度的缓冲作用**：
WorldCam的渐进式噪声调度提供了一个自然的缓冲机制，有助于平滑异常运动：

- **低噪声锚点**：早期帧的低噪声级别为生成过程提供了稳定的锚点。即使当前的运动输入导致了一些不稳定，低噪声的历史帧仍然会引导生成过程保持一定的稳定性。
- **可纠正的高噪声**：未来帧处于较高的噪声级别，这意味着它们更容易受到条件信息的影响和纠正。这为模型提供了"纠正机会"，能够根据上下文调整不合理的运动。

**短期记忆的平滑作用**：
短期记忆机制通过提供最近生成的潜在表示，在时间上起到了平滑作用：

- **时间连贯性**：即使某帧的运动输入异常，短期记忆中的历史潜在表示仍然会在生成过程中发挥作用，保持时间连贯性。
- **误差抵消**：如果某帧的运动导致了一定的误差，短期记忆可以帮助在后续帧中抵消或减轻这种误差。

#### 3.3 长期记忆的几何约束

**空间一致性约束**：
位姿锚定的长期记忆机制通过几何约束强制空间一致性：

- **历史观察的参考**：当相机接近之前访问过的位置时，检索到的历史潜在及其相机位姿嵌入会提供几何参考。即使当前的运动输入导致了一些偏移，历史观察仍然会引导生成过程保持与3D世界的一致性。
- **多视图几何约束**：检索到的多个历史观察（来自不同的视点）提供了多视图几何约束，这有助于稳定生成过程并减少不合理的几何畸变。

**分层检索的鲁棒性**：
分层检索策略（先按位置，再按方向）本身具有一定的鲁棒性：

- **位置优先**：即使当前的方向有些异常，只要位置接近，系统仍然可以检索到相关的历史观察。
- **容错性**：检索到的L个候选者提供了多个参考，而不是单一的历史观察。这种多样性增加了系统的容错能力。

#### 3.4 训练数据的分布

**真实人类行为的覆盖**：
WorldCam-50h数据集是从真实人类游戏中收集的，这意味着它自然地包含了各种类型的人类行为：

- **正常行为**：大多数时候，人类玩家的行为是合理的和可预测的。
- **极端但合理的行为**：数据集明确包含了"快速360°相机旋转"等极端行为，但这些都是人类能够理解和处理的。
- **罕见的行为**：虽然可能存在一些罕见的或看似不合理的行为，但它们在训练数据中的比例很小，对整体模型行为的影响有限。

**异常检测的可能性**：
基于训练数据的分布，系统可以潜在地识别和处理异常输入：

- **速度分布**：数据集中记录了线速度和角速度的分布（图3(c)和(d)）。理论上，可以使用这些分布来检测异常输入。
- **轨迹合理性**：通过比较生成的轨迹与训练数据中的典型轨迹模式，可以识别异常的行为模式。

#### 3.5 视觉质量与几何一致性的权衡

**模糊输出与一致性**：
论文在实验部分提到了一个有趣的现象：Yume模型虽然在PSNR指标上与WorldCam相当，但这主要是因为它的输出更模糊（Sharpness只有95 vs 577）。这表明：

- **模糊的一致性**：过度平滑或模糊的输出可能会在像素级指标上表现出良好的一致性，但这并不代表真正的几何一致性。
- **清晰度与一致性的平衡**：WorldCam在保持高清晰度（Sharpness 656）的同时实现了优异的几何一致性，这说明它在视觉质量和几何一致性之间取得了良好的平衡。

**动态场景的处理**：
论文明确说明数据集聚焦于"静态环境，没有动态物体或其他玩家"。这意味着：

- **已知场景假设**：系统在训练和推理时都假设场景是静态的。如果用户输入导致相机移动到训练数据中未见过的位置，系统需要"幻觉"或生成新的场景内容。
- **几何一致性优先**：在生成新场景内容时，系统优先保证与已知场景的几何一致性，而不是追求完美的视觉真实感。

#### 3.6 实际应用中的限制

**显式的速度约束**：
虽然论文没有详细讨论，但在实际实现中，可能会加入一些显式的约束：

- **速度限制**：对线速度和角速度设置合理的上限，防止极端输入。
- **加速度限制**：对速度变化率进行限制，防止突变。
- **连续性约束**：强制相邻帧之间的相机运动保持一定的连续性。

**用户反馈机制**：
在真实的应用中，可以通过用户反馈来处理异常情况：

- **交互式校正**：如果生成的场景明显不合理，用户可以通过交互来校正。
- **自适应调整**：系统可以检测异常行为并自动调整生成策略，例如增加历史信息的权重。

#### 3.7 总结与展望

**多层防护机制**：
WorldCam通过多层机制应对异常相机运动：

1. **数学约束**：SE(3)和李代数提供内在的数学约束，保证位姿的合理性。
2. **渐进式推理**：噪声调度和短期记忆提供时间上的平滑作用。
3. **几何约束**：长期记忆通过空间一致性约束引导生成过程。
4. **训练数据**：真实人类行为的数据分布为模型行为提供了自然限制。

**未来的改进方向**：
基于以上分析，可能的改进方向包括：

- **异常检测机制**：基于训练数据分布的异常检测，识别和处理极端输入。
- **自适应推理**：根据输入的异常程度，动态调整生成策略（例如，增加历史信息的权重）。
- **物理引擎集成**：集成物理引擎来验证和约束相机轨迹的物理合理性。
- **用户意图理解**：通过更高级的理解机制，区分"异常但有意为之"的行为和"错误"的行为。

WorldCam的设计已经在很大程度上解决了这个问题，但仍有改进空间。未来的研究可以进一步探索如何在保持灵活性的同时，增强系统对异常行为的鲁棒性。

## 核心技术发现

### 发现1：相机位姿作为统一几何表示

**核心发现**：
将相机位姿确立为连接即时动作控制和长期3D一致性的统一几何表示是有效的。这一设计解决了以往工作将动作视为抽象条件信号的问题。

**证据**：
- 实验结果显示，WorldCam在动作可控性（RPE_camera: 0.086）上显著优于所有基线方法（GameCraft: 0.100, Matrix-Game 2.0: 0.119, Yume: 0.137）。
- 在长期3D一致性方面，WorldCam在所有指标上都取得了最佳性能（PSNR: 16.69, LPIPS: 0.3277, MEt3R: 0.0342, DINO Sim: 0.8884）。

**意义**：
这一发现表明，显式的几何表示对于构建真正的空间智能系统至关重要。它将动作控制和世界建模两个看似独立的问题统一在一个框架下，简化了系统设计并提高了性能。

### 发现2：基于李代数的精确动作映射

**核心发现**：
使用李代数𝔰𝔢(3)表示和映射用户动作能够比传统的线性近似提供更精确的动作控制。

**证据**：
- 消融实验（表9）比较了线性近似和李代数近似：
  - WorldCam (Linear): RPE_camera = 0.102
  - WorldCam (Lie): RPE_camera = 0.086
- 李代数方法在旋转误差上的改进尤其显著（RPE_rot从0.962°降低到0.696°）。

**意义**：
这一发现强调了数学严谨性在实际AI系统中的重要性。通过使用正确的数学工具（SE(3)和李代数），系统能够准确捕获耦合动力学，这对于真实世界应用至关重要。

### 发现3：位姿索引的记忆检索

**核心发现**：
使用相机位姿作为空间索引来检索历史潜在表示能够显著改善长期3D一致性。

**证据**：
- 消融实验（表10）比较了三种检索策略：
  - Random retrieval: PSNR = 15.76, LPIPS = 0.3645, MEt3R = 0.041
  - Temporal retrieval: PSNR = 15.18, LPIPS = 0.3867, MEt3R = 0.040
  - Ours (pose-based): PSNR = 16.42, LPIPS = 0.3496, MEt3R = 0.038

**意义**：
这一发现证明了空间索引的有效性。与传统的时间索引相比，空间索引能够更好地捕捉场景的结构信息，这对于长期一致性至关重要。这为构建Spatial AGI系统的空间记忆组件提供了重要的参考。

### 发现4：渐进式推理的稳定性

**核心发现**：
渐进式噪声调度和注意力接收器机制能够有效稳定长期自回归推理。

**证据**：
- 在视觉质量方面，WorldCam达到了最高的VBench平均分数（0.844），优于GameCraft（0.781）8.1%。
- 消融实验（表7和表8）显示，增加短期记忆潜在表示数量和引入注意力接收器能够提高长期视觉质量。

**意义**：
这一发现为解决长期生成中的误差累积问题提供了有效的方法。渐进式推理策略可能被广泛应用于其他长期生成任务中，如长视频生成、长期轨迹预测等。

## 个人思考

### 最有趣的发现

**统一的几何哲学**：
WorldCam最有趣的发现是"相机位姿作为统一几何表示"这一设计哲学。它简洁而优雅，将两个看似不同的任务（动作控制和世界建模）统一在一个数学框架下。这让我想到了物理学中的"统一场论"思想——试图用一个统一的数学框架描述不同的物理现象。

这种统一性不仅仅是工程上的便利，更反映了对问题本质的深刻理解。作者意识到，在3D交互式环境中，动作控制和3D一致性不是独立的两个问题，而是通过相机位姿内在耦合的两个方面。这种洞见的价值远超技术细节本身。

### 最意外的结果

**线性近似的不足**：
最意外的结果是，即使是简单的线性近似（WorldCam Linear）也能够在动作控制上达到与GameCraft相当的性能（RPE_camera: 0.102 vs 0.100）。这意味着：

1. **基线水平很高**：GameCraft使用的线性近似虽然理论上不够精确，但在实践中已经能够提供不错的动作控制。这说明，对于许多应用场景，"足够好"的近似可能是合理的选择。

2. **李代数的额外价值**：李代数方法相比于线性近似，在性能上的提升虽然显著（14.2%），但并非革命性的。这意味着李代数的价值不仅在于性能提升，更在于其数学上的严谨性和对复杂运动的准确建模能力。

3. **其他因素的重要性**：动作控制性能不仅取决于动作到相机的映射精度，还取决于许多其他因素，如模型架构、训练数据、推理策略等。这提醒我们，在系统设计时需要考虑整个pipeline，而不仅仅是某个单一组件。

### 最有价值的启发

**空间记忆系统的可行性**：
WorldCam的位姿锚定长期记忆机制为构建Spatial AGI的空间记忆系统提供了重要的启发。具体来说：

1. **显式几何锚点**：记忆条目不仅包含视觉内容，还包含显式的几何信息（相机位姿）。这使系统能够建立空间索引，而不是仅依赖时间索引。

2. **分层检索策略**：先按位置检索，再按方向检索，这种分层策略在复杂环境中既高效又有效。这为设计其他空间检索系统提供了参考。

3. **几何对应关系**：通过在当前和历史的潜在表示之间建立显式的几何对应关系，系统能够更好地理解和推理空间关系。这是实现真正空间智能的关键一步。

4. **可扩展性**：这个框架可以扩展到更复杂的环境和任务中。例如，可以将相机位姿扩展到更一般的物体位姿，或将单相机扩展到多相机/传感器设置。

### Q3的思考过程

**为什么选择这个问题**：
我选择"如何应对异常相机运动"这个问题，是因为它触及了WorldCam在真实应用中的鲁棒性和泛化能力。这是一个非常实际的问题，因为在真实环境中，用户的输入可能是不完美的，甚至是错误的。

这个问题让我深入思考了WorldCam设计的多个方面：

1. **数学框架的鲁棒性**：SE(3)和李代数提供了内在的约束，这从数学上保证了系统的合理性。
2. **推理策略的稳定性**：渐进式推理和记忆机制提供了多层防护，帮助系统应对异常输入。
3. **训练数据的限制**：真实人类行为的数据分布自然地限制了模型的行为，这也是一种隐式的鲁棒性机制。
4. **实际应用的权衡**：在视觉质量和几何一致性之间需要找到平衡，模糊的输出可能在像素指标上表现良好，但并不代表真正的几何一致性。

这个思考过程让我认识到，构建鲁棒的AI系统需要多层次的防护机制，从数学约束到推理策略，从训练数据到实时反馈。WorldCam的设计已经包含了许多这样的机制，但仍有改进空间。

## 总结

### 核心发现总结

WorldCam通过引入相机位姿作为统一的几何表示，成功解决了交互式3D游戏世界建模中的三大核心挑战：

1. **精确动作控制**：通过基于李代数的动作到相机映射，实现了比传统线性近似更精确的6自由度相机控制。
2. **长期3D一致性**：通过位姿锚定的长期记忆机制，利用全局相机位姿作为空间索引来检索相关的历史观测，确保了在长时间导航中的几何一致性。
3. **高质量视觉生成**：通过渐进式噪声调度、注意力接收器和短期记忆机制，维持了长时间序列的视觉质量。

### 对Spatial AGI的意义

WorldCam对Spatial AGI研究具有多重意义：

1. **统一几何表示的价值**：证明了使用显式的、数学上严谨的几何表示（SE(3)和李代数）对于构建空间智能系统的价值。
2. **空间记忆系统的设计**：提供了一个可行的空间记忆系统设计，利用几何信息进行空间索引和检索。
3. **长期一致性维持**：展示了如何通过几何约束和记忆机制实现长期空间一致性。
4. **数据集的重要性**：强调了高质量、大规模、开放许可的数据集对于推动Spatial AGI研究的重要性。

### 技术创新点

1. **李代数动作映射**：首次将李代数应用于交互式3D环境的动作到相机映射，实现了对耦合动力学的精确建模。
2. **位姿锚定记忆**：创新性地使用相机位姿作为空间索引，实现了高效的、几何一致的历史检索。
3. **渐进式自回归推理**：结合渐进式噪声调度和注意力机制，实现了稳定的长期生成。

### 实验验证

通过大量实验，WorldCam在以下方面显著优于现有方法：

- 动作可控性提升14%以上（RPE_camera: 0.086 vs 0.100）
- 视觉质量提升8.1%（VBench Avg: 0.844 vs 0.781）
- 3D一致性在所有指标上均达到最佳
- 人类评估中所有三个维度都取得最高分（4.31, 4.44, 4.36）

### 未来方向

基于WorldCam的工作，未来的研究方向可能包括：

1. **动态场景**：扩展到包含动态物体的场景
2. **多用户交互**：支持多个用户的并发交互
3. **更复杂的空间任务**：如物体操作、工具使用等
4. **跨泛化能力**：在不同游戏环境或真实场景之间的泛化
5. **实时优化**：进一步优化推理速度以支持更实时的交互
6. **异常处理**：增强对异常用户输入的鲁棒性

## 相关工作

### 交互式游戏世界模型

1. **The Matrix / Matrix-Game / Matrix-Game 2.0**
   - 通过cross-attention或AdaLN直接注入原始动作信号
   - 缺乏显式几何约束，难以实现3D一致性

2. **Genie**
   - 类似的动作注入方法
   - 不支持长期3D一致性

3. **Yan (GameCraft)**
   - 通过线性近似将用户输入映射到相机位姿
   - 忽略了SE(3)几何，无法捕获耦合动力学
   - 相机运动仅用于即时控制，缺乏长期几何锚点

4. **Yume**
   - 通过文本提示描述动作
   - 使用FramePack压缩历史潜在表示
   - 动作控制相对粗糙

### 相机控制的视频生成

1. **CameraCtrl**
   - 在生成过程中条件化相机位姿
   - 聚焦于短视频（如16帧）
   - 不支持动作驱动控制和长期推理

2. **MotionCtrl**
   - 类似的相机控制方法
   - 同样受限于短视频生成

### 交互式游戏数据集

1. **Minecraft数据集**
   - 提供成对的动作标签
   - 受限的视觉多样性和简化的几何结构

2. **闭源许可的游戏视频数据集**
   - 高质量但不可用于可重复研究

### 视频生成基础模型

1. **CogVideoX**
2. **Wan / Wan-2.1**
3. **LTX**
4. **Cosmos**
5. **Genie**

### 空间几何表示

1. **Plücker坐标**（Sitzmann et al.）
   - 用于相机位姿的显式几何表示
   - WorldCam采用此表示进行相机嵌入

2. **SE(3)和李代数**（Hall）
   - 用于3D刚体变换的数学框架
   - WorldCam利用其进行精确的动作到相机映射

### 记忆和检索机制

1. **StreamingLLM**
   - 引入注意力接收器以稳定长期生成
   - WorldCam借鉴此思想用于视频生成

2. **Context-based检索**（Yu et al.）
   - 使用视场重叠进行检索
   - WorldCam采用更简单的基于位姿的方法以适应复杂环境
