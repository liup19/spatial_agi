**研究背景**:
视觉-语言-动作（Vision-Language-Action, VLA）模型已成为通用机器人操作的有前景方法。然而，这些模型的泛化能力不一致：虽然它们在某些设置中表现出色，但微调后的变体在未见过的对象、场景和指令上经常失败。

**核心问题**:
当前VLA模型存在严重的泛化问题。尽管在LIBERO或Robocasa等基准测试中取得了快速进展，但这些模型在监督微调（SFT）过程中经常失去语言理解和泛化能力。研究表明，超过90%成功率的模型在系统性扰动下可能崩溃至接近零，这意味着这些策略可能依赖于对动作序列和环境布局的死记硬背，而不是对新感知输入的泛化。

**研究动机**:
为了让机器人从结构化的实验室环境走向真实世界，必须理解如何设计可泛化、可靠和可解释的VLA策略。传统的"安全"定义通常指碰撞避免、力限制和稳定性。随着机器人向真实世界过渡，"安全"的概念必须适应更广泛的安全范式。"安全"不应仅仅是几何或控制理论层面的，还应包括语义、情境和任务依赖性。例如，家用机器人必须避免接触热表面，并适当处理锋利或易碎的物体。

## 摘要

本文应用机制可解释性技术来更好地理解VLA模型的内部工作机制。为了探测内部表示，作者在VLA的隐藏层激活上训练稀疏自编码器（Sparse Autoencoders, SAEs）。SAEs学习一个稀疏字典，其特征作为模型计算的紧凑、可解释的基础。

研究发现，提取的SAE特征中绝大多数对应于特定训练演示中记忆的序列。然而，一些特征对应于可解释、通用和可转向的运动原语和语义属性，为VLA的可泛化性提供了有希望的洞察。作者提出了一个度量标准来根据特征是否代表可泛化的可迁移原语或特定于片段的记忆来对特征进行分类。

通过在LIBERO基准测试上的转向实验验证了这些发现。研究表明，单个SAE特征会因果性地影响机器人行为。转向通用特征会诱导与其语义含义一致的行为，并且可以应用于跨任务和场景。这项工作提供了首个机械证据，证明VLA可以学习跨任务和场景的泛化特征。

观察到在小型机器人数据集上的监督微调不成比例地放大了记忆化。相比之下，在更大、更多样化的数据集（如DROID）上训练或使用知识隔离（Knowledge Insulation）促进了更通用特征的习得。

为了促进未来VLA机制可解释性的研究，作者提供了一个开源代码库和用户友好的界面，用于激活收集、SAE训练和特征转向。项目页面位于：http://drvla.github.io

## 贡献总结

1. **提出了一种使用稀疏自编码器从VLA模型中提取可解释、通用和可转向特征的技术**

2. **提出了几个泛化量化度量标准，用于SAE特征，使用微调数据上的激活统计，无需运行任何策略滚动**

3. **在所有情况下都观察到显著的片段级记忆。然而，在DROID等多样化数据集上微调会导致相对更多的通用特征**

## 关键发现

### 核心发现

1. **大多数SAE特征是记忆化特征**：在所有测试的模型和数据集中，89-99%的特征被分类为记忆化特征。这些特征对应于特定片段、视觉场景或狭窄任务变体。

2. **存在可解释的通用特征**：尽管记忆化特征占主导地位，但仍有一些特征表现出通用性，在多样的片段中响应语义连贯的事件。例如，抓取物体、将其放置在目标位置，或特定的物体类别进入视野。

3. **特征的可转向性**：通过在推理时向残差流添加特征向量，可以因果性地影响机器人行为。转向通用特征会诱导与其假设解释一致的行为修改。

4. **数据集多样性的影响**：在更大、更多样化的数据集（如DROID）上微调会导致相对更多的通用特征。DROID（1545个独特任务）比LIBERO（130个任务）产生更多泛化特征。

5. **知识隔离的作用**：知识隔离（Knowledge Insulation）技术似乎能够缓解微调过程中泛化能力的下降。

## 核心技术方法

### 方法概述

整体方法包含四个主要组件：
1. SAE训练（Section 3.1）
2. 泛化度量（Section 3.2）
3. 特征分类（Section 3.3）
4. 特征转向（Section 3.4）

工作流程：
1. 从VLA策略中提取激活，并在模型内的多个层上训练SAE
2. SAE将叠加特征分解为稀疏字典
3. 为了处理大量特征，利用多个特征级度量指标，可以自动计算
4. 使用这些度量将特征分类为通用或记忆化
5. 对这些特征进行转向实验，进一步确认它们对模型的因果影响

### 3.1 SAE AuxK架构

为了从VLA的内部表示中提取可解释特征，作者在残差流激活上训练稀疏自编码器（SAEs），遵循Gao等人引入的TopK架构和AuxK辅助损失。该架构通过TopK激活函数提供对特征稀疏性的直接控制，并通过针对非激活特征的辅助重建损失来解决死潜在问题。

#### 归一化过程

给定输入激活 **x** ∈ ℝ^d，SAE首先应用每样本归一化：
- 减去学习的预偏差 **b**_pre，初始化为训练激活的几何中位数
- 进行每样本均值减法和ℓ2归一化：

```
x̃ = ((x - b_pre) - μ) / ||(x - b_pre) - μ||₂
```

其中μ是d_model维度上的标量均值。

#### 编码过程

归一化输入被编码为稀疏表示：

```
z = ReLU(TopK(W_enc(x̃)))
```

其中：
- TopK函数保留k个最大的预激活值，其余归零
- 选择后的ReLU确保非负特征系数
- 归一化空间中的重建 x̂ = W_dec z 通过恢复保存的ℓ2范数、均值和预偏差进行非归一化
- 解码器列被约束为单位范数，确保每个特征的贡献纯粹由其激活系数决定
- 编码器和解码器都不使用偏差项

#### 损失函数

总损失结合了归一化重建误差和AuxK辅助损失：

```
L = ||x - x̂||₂² / C_MSE + α · ||ẽ - ẽ_aux||₂² / C_MSE
```

其中：
- C_MSE是在初始化时计算的居中激活的方差，α = 1/32
- ẽ = x̃ - x̂ 是归一化空间中的重建残差
- ẽ_aux 是应用于该残差的top-k_aux死潜在特征的辅助重建
- 如果潜在特征在过去500个优化步骤中没有激活，则被视为"死"潜在特征

#### 初始化和训练

编码器权重初始化为解码器的缩放转置：
```
W_enc = W_decᵀ · √(k/n)
```

其中n是残差流维度。解码器梯度被投影到单位范数约束的切平面上，梯度范数被裁剪为1.0。

作者使用1×扩展比（即潜在空间维度与输入维度相同）训练SAE，涉及π0.5模型的八个层：
- PaliGemma层 0, 5, 11, 17（d = 2048, k = 100）
- 动作专家层 0, 5, 11, 17（d = 1024, k = 64）

类似地为OpenVLA模型训练SAE。

#### 与传统SAE的区别

与典型的VLM可解释性工作相比，本文采用了显著更低的扩展比（ER = 1）。实验表明：
- 更高的扩展比导致显著更多的死特征
- 即使在这种紧凑的字典大小下，可解释特征也能可靠地出现
- 这可能归因于机器人学中使用的小得多的数据集规模

对于OpenVLA，作者使用ER = 0.5训练SAE以匹配π0.5模型的字典大小（2048个特征）。

### 3.2 泛化量化度量

每个SAE字典包含数千个单独的特征。手动检查所有这些跨模型和层的特征是不可行的，因此作者定义了一组每特征激活统计，总结了每个特征在片段内和数据集上的行为。

#### 度量定义

设f_j(x_t^((e))) ∈ ℝ_≥0表示特征j在片段e的时间步t的激活，T^((e))是片段e中的时间步数。E = {e₁, e₂, ..., e_N}表示微调数据集中的所有片段集合，E_j^+ = {e : ∃ t, f_j(x_t^((e))) > 0}是特征j至少触发一次的片段集合。

##### 1. 片段覆盖率（Episode Coverage, c_j）

片段覆盖率是特征在微调数据集中至少触发一次的片段分数：

```
c_j = |E_j^+| / |E|
```

更高的片段覆盖率表明该特征在多样化任务中活跃，因此具有更大的泛化性。

**意义**：
- 高覆盖率（c > 0.5）表明特征在许多不同片段中激活
- 通用特征通常显示0.2到0.9之间的片段覆盖率，具体取决于数据集多样性
- 记忆化特征通常有很低的覆盖率（c < 0.1），通常集中在单个片段或少数共享相同场景的片段中

##### 2. 平均起始计数（Mean Onset Count, ō_j）

特征的起始是指该特征从非活跃到活跃的过渡。为了抑制噪声影响，添加激活阈值τ_on。每个特征j维护一个二进制状态s，初始s₀ = 0。

状态转移：
```
s_t = 1, if f_j(x_t) > τ_on
     0, if f_j(x_t) = 0
     s_{t-1}, otherwise
```

激活阈值τ_on = 0.1。在每次0→1过渡时计算起始，给出每片段的起始计数：

```
o_j = Σ_{t=1}^{T} max(0, s_t - s_{t-1})
```

平均起始计数仅对活跃片段取平均，与片段覆盖率解耦：

```
ō_j = (1/|E_j^+|) · Σ_{e∈E_j^+} o_j^((e))
```

由于每个活跃片段至少有一个起始，对于任何c_j > 0的特征都有ō_j ≥ 1。

**意义**：
- 通用特征通常显示ō_j ≫ 1，表示爆发式、事件驱动的激活
- 记忆化特征通常显示ō_j ≈ 0–1，表示持续而不是爆发式激活
- 高起始计数表明特征在片段中多次触发，响应重复事件

##### 3. 平均激活幅度（Mean Activation Magnitude, ā_j）

对于每个活跃片段e ∈ E_j^+，记录特征j在所有时间步上的最大激活。平均激活幅度对这些每片段最大值取平均：

```
ā_j = (1/|E_j^+|) · Σ_{e∈E_j^+} max_t f_j(x_t^((e)))
```

该度量捕获特征触发时的典型峰值强度，在所有活跃片段中取平均。

**意义**：
- 通用特征通常有更高的平均激活幅度
- 记忆化特征显示其顶级片段和后续片段之间的最大激活幅度急剧下降
- 这个度量帮助区分强特征和弱特征

##### 4. 相对运行长度（Relative Run Length, ℓ̄_r,j）

对于每个活跃片段，运行长度是每个起始的连续活跃时间步的平均值：

```
r_j = (1/o_j) · Σ_{t=1}^{T} s_t
```

通过片段长度归一化得到相对运行长度，将平均激活持续时间表示为片段的分数：

```
ℓ̄_r,j = (1/|E_j^+|) · Σ_{e∈E_j^+} (r_j^((e)) / T^((e)))
```

接近0的值表示短暂、瞬态的激活；接近1的值表示整个片段的持续激活。

**意义**：
- 通用特征倾向于低ℓ̄_r（爆发式）
- 记忆化特征通常显示高ℓ̄_r（持续）
- 这个度量是区分爆发式通用特征和持续记忆化特征的关键指标

### 3.3 特征分类

#### 3.3.1 特征类别

作者严格定义了通用、记忆化或可解释特征的含义。在语言模型的SAE可解释性工作中，特征主要通过顶级激活示例来评估：如果人类能够识别最强烈激活它的输入之间的共同模式，则特征是可解释的。在VLA中，分析单位是观察-动作轨迹而不是文本序列。作者相应地调整了可解释性标准：当特征的激活模式，无论是在片段内还是跨片段，始终与可识别的传感器运动事件或视觉状态一致时，特征是可解释的。

从VLA残差流的SAE训练中出现了两个功能上不同的可解释特征类别：

##### 通用特征（General Features）

一个特征是通用的，如果它在多样化片段集中响应语义连贯的事件而激活。例如，抓取物体、将其放置在目标位置，或特定的物体类别进入视野。

**特征属性**：
1. **高平均起始计数**（ō ≫ 1）：表示爆发式、事件锁定的激活而不是持续激活
2. **广泛的片段覆盖率**（c）：表示触发事件在许多片段中发生
3. **高平均活跃分数**（φ̄）：表示特征在数据集的大部分时间步上激活

**经验观察**：
- 通用特征倾向于显示ō > 1.0
- 片段覆盖率范围从0.2到0.9，取决于数据集多样性
- 这些特征具有实际意义，因为它们的解码器权重向量作为有效的转向方向：在推理时将向量添加到残差流可跨片段可重复地调节相应行为

##### 记忆化特征（Memorized Features）

一个特征是记忆化的，如果它调整为特定片段、视觉场景或狭窄任务变体。

**特征属性**：
1. **低平均起始计数**（ō ≈ 0–1）：表示持续而不是爆发式激活
2. **低片段覆盖率**（c）：通常集中在单个片段或少数共享相同场景的片段中
3. **顶级片段和后续片段之间的最大激活幅度急剧下降**

尽管范围狭窄，记忆化特征仍然可解释。

**观察到的现象**：
- 作者观察到这些特征位于一个谱上，并不总是整齐地落入记忆化或通用的二元分类中
- 一些特征主要记忆视觉场景但也编码任务级affordances（例如，DROID F322最强烈地在特定办公室场景中激活棋子抓取，但在无关场景中也在降低幅度下在棋盘上触发）
- 相反，一些特征显示通用语义但由于数据集组成效应被分类为记忆化

#### 3.3.2 手动标记特征

作者通过三阶段视觉检查管道标记特征子集，检查片段级、跨片段和数据集级别的激活模式。

**阶段1：片段级筛选**
- 从随机选择的片段开始，检查显示给定层的前50-100个特征的特征激活热图
- 热图的每一行显示单个特征在所有时间步上的激活幅度，与相应的相机帧对齐
- 作者称之为激活查看器（Activation Viewer）
- 通过爆发式激活签名识别候选通用特征：与可识别行为事件（如抓取、放置、物体出现）一致的短促、高幅度峰值
- 通过持续签名识别候选记忆化特征：在大多数时间步上接近均匀激活，与行为过渡不对齐

**阶段2：跨片段验证**
- 对于每个候选，查询数据集中所有SAE激活的预计算索引，称为特征搜索索引
- 这返回全局顶级激活时间步和按最大激活排名的前10个多样化片段
- 通用特征应该显示跨越多个片段和任务的峰值激活，具有一致、均匀的对齐
- 记忆化特征应该显示集中在一个或少数片段中的激活，具有幅度的急剧下降
- 在激活查看器中验证这个新片段子集显示与原始随机选择片段相同的每片段时间模式（爆发式 vs 持续）

**阶段3：标记标准**

特征接收**通用**标签如果：
1. 激活爆发在前多样化片段中与语义一致事件对齐
2. 在激活查看器中查看的held-out片段中再现激活模式
3. 全局每特征度量（平均起始计数、平均活跃激活、片段覆盖率）表明该特征在训练数据集的大部分中持续存在

特征接收**记忆化**标签如果：
1. 在顶级片段中激活是持续的
2. 顶级片段聚集在≤ 2个视觉场景周围
3. 全局每特征度量表明该特征在训练数据集的小子集上激活

如果对特征搜索结果的解释有任何模糊性，则排除给定特征。

#### 3.3.3 自动分类

手动标记精确但不会扩展到跨多个模型和层的数千个特征。为了大规模分类特征，作者在3.2节定义的四个时间度量上拟合逻辑回归分类器，使用来自单个参考层的30个手动标记特征（15个通用，15个记忆化）作为训练数据。

**分类器形式**：

```
P(general | m) = σ(β₀ + β₁·ō + β₂·c + β₃·ā + β₄·ℓ̄_r)
```

其中：
- ō 是平均起始计数
- c 是片段覆盖率
- ā 是平均激活幅度
- ℓ̄_r 是相对运行长度
- σ 是逻辑sigmoid函数

**关键特点**：
- 分类器对非标准化度量值操作，因此学习的决策边界可以在同一模型的层间应用而无需每层归一化
- 为每个微调数据集（LIBERO和DROID）训练一个分类器
- 将其应用于在该数据集上训练的所有模型；OpenVLA分类器重用LIBERO边界，因为OpenVLA也在LIBERO上微调

**LIBERO分类器系数**：
- β₀ = -4.20
- β₁ = 1.89（平均起始计数）
- β₂ = 1.80（片段覆盖率）
- β₃ = 0.52（平均激活幅度）
- β₄ = -0.36（相对运行长度）
- **100%留一法交叉验证（LOO-CV）准确性**

**DROID分类器系数**：
- β₀ = -1.78
- β₁ = 0.74（平均起始计数）
- β₂ = 2.36（片段覆盖率）
- β₃ = 0.35（平均激活幅度）
- β₄ = -1.04（相对运行长度）
- **96.7%留一法交叉验证（LOO-CV）准确性**

**系数分析**：

**LIBERO数据集**：
- 平均起始计数和片段覆盖率贡献几乎相等（β₁ ≈ β₂），表明在LIBERO中，许多任务共享共同场景，爆发式激活和广泛片段存在都需要来区分通用和记忆化特征
- 相对运行长度上的负系数（β₄ < 0）确认持续、长持续激活是记忆化特征的特征

**DROID数据集**：
- 与LIBERO相比，片段覆盖率占主导地位（β₂是β₁的3.2×），反映了DROID中显著更大的场景和任务多样性（1545个独特任务 vs LIBERO的130个）
- 当数据集本身多样化时，高片段覆盖率是泛化性更强的信号
- 相对运行长度上更强的负权重（β₄ = -1.04 vs LIBERO中的-0.36）进一步惩罚具有持续、跨片段激活模式的特征
- 这与数据集特征一致：DROID片段明显更长和更多变（μ = 283.5, σ = 219.2时间步，CV = 0.77）比LIBERO片段（μ = 161.5, σ = 68.2, CV = 0.42），使得相对运行长度成为瞬态通用特征和持续记忆化特征之间更强的区分符

### 3.4 特征转向（Feature Steering）

为了进一步验证对特定特征的解释，可以直接转向该特征并观察模型输出。

**转向向量提取**：

给定具有编码器f_enc和解码器f_dec的训练SAE，将特征i的转向向量提取为解码器权重矩阵的第i列：

```
v_i = W_dec[:, i] ∈ ℝ^d
```

其中d是残差流维度（PaliGemma层为2048，动作专家层为1024，OpenVLA为4096）。

**转向过程**：

解码器列在构造上为单位归一化，因此转向幅度完全由标量系数α控制：

```
y′ = y + α · v
```

转向向量在序列维度上广播，在每个去噪步骤的每个标记位置应用相同的扰动。

**与LLM转向的区别**：
这与标准LLM转向方法的另一个不同领域，因为模型不自回归地预测动作。

## 实验结果

### 4.1 可解释特征

发现主要模型产生的SAE特征中绝大多数显示出强烈的可解释性。

**估计方法**：
作者采用基于采样的方法来估计SAE字典中可解释特征的数量。从π0.5 LIBERO PG5和PG11、π0.5 DROID PG5和PG11以及OpenVLA Layer 8和24中随机抽取20个特征样本。

**可解释性标准**：
一个特征是可解释的，当其激活模式，无论是在片段内还是跨片段，始终与可识别的传感器运动事件或视觉状态一致时。

**结果**：
- π0.5 LIBERO PG5: 90%可解释（18/20）
- π0.5 LIBERO PG11: 80%可解释（16/20）
- π0.5 DROID PG5: 85%可解释（17/20）
- π0.5 DROID PG11: 70%可解释（14/20）
- OpenVLA LIBERO Layer 8: 80%可解释（16/20）
- OpenVLA LIBERO Layer 24: 70%可解释（14/20）
- **总体：79.17%可解释**（95/120）

**重要性**：
值得注意的是，可解释性与泛化性不同，并且有更广泛的范围。作者在其网站上提供了LIBERO中10个特征的详细解释，标签为"labeled"。

### 4.2 模型级分类

将3.3.3节描述的每数据集分类器应用于每个模型的所有层的每个特征。

**分类结果汇总**：

**π0.5 – LIBERO**:
- PG5层：2044个特征，32个通用，2012个记忆化，**98.43%记忆化**
- PG平均层（0, 5, 11, 17）：7175个特征，188个通用，6987个记忆化，**97.37%记忆化**

**π0.5 – DROID**:
- PG5层：2046个特征，104个通用，1942个记忆化，**94.92%记忆化**
- PG平均层（0, 5, 11, 17）：6649个特征，719个通用，5930个记忆化，**89.19%记忆化**

**OpenVLA – LIBERO Goal**:
- Layer 8：1775个特征，8个通用，1767个记忆化，**99.55%记忆化**
- LM平均层（0, 8, 16, 24, 31）：9389个特征，42个通用，9347个记忆化，**99.55%记忆化**

**关键观察**：

1. **记忆化特征占主导**：
   - 总体而言，发现这些模型中只有一小部分特征是通用的
   - 对于LIBERO研究表明VLA模型可以强烈地视觉过拟合场景
   - 在DROID上微调的模型显示相对更低比例的记忆化特征，但记忆化特征的绝对比例仍然很高

2. **数据集规模的影响**：
   - 虽然DROID按机器人标准很大，但与语言模型训练数据集的规模相比仍然相对较小，只有中间任务和场景多样性
   - 作者认为解释这些结果的最好方式是比较模型和数据集之间记忆化特征的相对比例
   - 在这方面，作者发现清晰和明确的模式：增加微调数据集的大小和多样性会增加通用特征的份额

3. **从LIBERO-GOAL到完整LIBERO再到DROID的进展**：
   - 观察到通用特征份额的稳步增加
   - 这表明更大、更多样化的数据集促进更通用特征的习得

4. **训练步数和知识隔离的影响**：
   - 见Section 4.5和A.5.1讨论常见误分类和训练步数及知识隔离对通用特征比例的影响

### 4.3 通用特征详解

#### 4.3.1 LIBERO通用特征突出展示

**四个高度通用的特征**：

**F1129（抓取/放置）**：
- 在初始物体抓取和放置期间激活
- 起始计数与抓取和放置子目标的数量成比例（单物体任务两个起始，双物体任务四个起始）
- P(general | F1129) = 0.91
- 片段覆盖率 > 0.99，对场景、目标物体和抓取数量不变
- 代表了数据集中一些最通用的特征

**F1902（运输）**：
- 在F1129起始对之间激活，对应携带阶段
- 激活幅度朝向目标位置近似线性增加，表明特征也编码与放置目标的接近度
- P(general | F1902) = 0.89
- 片段覆盖率 > 0.99
- 功能上与编码携带阶段相关

**F128（预抓取对齐）**：
- 在每个F1129起始之前，当末端执行器定位在目标物体上方时激活
- 当物体进入并在腕相机框架中居中时幅度增加
- 如果手臂返回到垂直对齐位置，放置后也会重新激活
- P(general | F128) = 0.92
- 片段覆盖率 > 0.99
- 代表预抓取对齐表示

**F445（任务完成）**：
- 当末端执行器接近目标放置位置时激活，主要在最终子目标上
- 在复合任务中，以较低幅度激活并优先在第二个物体放置上，与编码整体任务成功而不是子任务完成一致
- P(general | F445) = 0.58
- 片段覆盖率 > 0.99
- 代表任务级成功信号

**关键特点**：
- 所有四个特征都有片段覆盖率 > 0.99
- 对场景、目标物体和抓取数量不变
- 代表了数据集中一些最通用的特征
- 激活模式始终与可识别的行为事件对齐

#### 4.3.2 DROID通用特征突出展示

尽管DROID的场景和任务多样性显著更大（1545个独特任务 vs LIBERO的130个），这些特征仍显示高片段覆盖率。平均激活比其LIBERO对应物更低，与编码更抽象的操作概念一致。

**四个DROID通用特征**：

**F158（子任务检查点）**：
- 所有LIBERO和DROID PG5 SAE中最高ō的特征
- 在操作子阶段之间的过渡时激活：接近抓取位置、获取物体和接近目标
- 在片段472（8瓶子抓取和放置）中，特征产生16个不同的起始，每个与一个瓶子进入腕相机或抓取期间目标盘子可见对齐
- P(general | F158) = 0.91
- 代表操作阶段转换

**F586（捏式抓取）**：
- 在薄物体（盘子边缘、糖包、餐具手柄、毛巾）的精度抓取期间激活并持续通过抓取持续时间
- 在降低幅度（~0.5×）下对更宽抓取（罐子、块）激活，与编码抓取孔径一致
- P(general | F586) = 0.78
- 代表精度抓取原语

**F165（打开夹爪在目标上方）**：
- 当目标物体在腕相机中的打开夹爪爪之间可见时激活
- 起始通常跟随F158激活
- 当夹爪在任务开始前经过场景物体时发生虚假早期片段激活
- P(general | F165) = 0.83
- 代表目标获取配置

**F399（抓取获取/放置）**：
- 在抓取的关闭阶段和物体放置期间激活；在运输期间停用
- 激活峰值与涉及捏式抓取的片段中的F586重叠
- 功能上类似于LIBERO F1129但具有更嘈杂的时间边界
- P(general | F399) = 0.79
- 代表抓取和放置原语

**与LIBERO特征的对比**：
- DROID特征显示高片段覆盖率，尽管场景和任务多样性显著更大
- 平均激活更低，与编码更抽象的操作概念一致
- 特征更具组合性和抽象性
- 反映了DROID数据集中更大的多样性

### 4.4 特征转向实验

在LIBERO上应用两个通用特征（F128和F1902）来转向跨几个片段的闭环行为。

**关键观察**：
整个实验中持续的一个观察是VLA模型对激活级扰动的令人印象深刻的鲁棒性。即使以单个特征在高幅度下转向，策略继续显示目标导向的行为，通常试图接近相关物体、抓取或追求替代子任务，而不是简单地产生退化或随机运动。

**实验设置**：
- 对于这些闭环评估，转向从第10个时间步开始并继续通过整个片段
- 从π0.5动作地平线执行五个步骤，然后更新观察并推理模型
- 转向挂钩在所有10个去噪迭代期间激活，因此扰动影响整个细化过程，从噪声到动作轨迹

**F128（LIBERO PG5）转向结果**：
- 如4.3.1节讨论，F128与预抓取对齐强烈相关，当末端执行器在抓取前定位在目标物体上方时激活
- 在三个任务上执行转向：
  1. 将黑色碗从抽屉移到盘子
  2. 打开炉子并将摩卡壶放在炉子上
  3. 捡起字母汤并将其放在篮子里
- **观察到的行为**：转向此特征时，观察到机器人始终接近目标抓取物体，而不是将其捡起，简单地悬停在它上面
- **解释**：这种行为完全与我们将该特征解释为预抓取对齐表示的预期一致

**F1902（LIBERO PG5）转向结果**：
- 如4.3.1节讨论，F1902是与将物体运输到目标区域相关的特征
- 发现该特征接近目标位置激活并编码与放置目标的接近度
- 在三个任务上转向此特征：
  1. 将黑色碗从抽屉移到盘子
  2. 将白色杯子放在盘子上
  3. 捡起字母汤并将其放在篮子里
- **观察到的行为**：
  - 在所有三个示例中，机器人完全跳过相关物体抓取并直接移动到目标位置
  - 在捡起字母汤并将其放在篮子里的轨迹中，机器人甚至与篮子碰撞，将其推到一边
- **解释**：这种行为表明转向F1902编码的"运输到目标"概念使策略优先考虑接近目标而不是物体获取

**转向实验的总体发现**：
- 这些特征的转向结果强化了4.3.1节的发现，基于特征搜索和激活查看器调查
- 然而，转向并不完美，仍然显示一些与给定特征叠加效应的证据
- 对记忆化特征也进行转向：发现记忆化转向有各种效果，有时驱动机器人朝向特定方向，而其他时间似乎重复记忆化的行为
- 记忆化转向结果在网站上显示

**转向的层深度和类型依赖性**：
- 检查转向效果如何随π0.5模型中的层深度和类型变化
- 使用四个PaliGemma和动作专家层中每个的SAE特征转向并呈现结果
- **发现**：
  - 在早期动作专家层中转向产生比后期层或PaliGemma层大得多的末端执行器位移
  - 在LIBERO和DROID中，AE0是最可转向的，但效果随深度急剧缩减
  - PaliGemma层在相同深度下始终较弱
  - PG L17在两个模型中产生零效果
- **含义**：
  - 动作专家层在控制行为方面比PaliGemma主干更直接
  - 早期层对转向更敏感
  - 后期PaliGemma层可能编码更抽象的概念，难以通过简单转向直接影响

### 4.5 训练步数和知识隔离的影响

**研究问题**：
提出的特征度量是否可以充当训练期间模型泛化的代理？

**观察结果**：
如图6所示，结果微弱地支持这一主张。正如预期，随着微调进行，泛化性下降。尽管π0.5基础在超过10,000小时的跨体现机器人数据上训练，一旦微调开始，灾难性遗忘似乎开始。相比之下，知识隔离（KI），设计用于保留通用能力，似乎逆转这种下降。

**度量的行为**：
在图6中，报告了跨训练检查点的片段覆盖率和相对运行长度：
- **片段覆盖率**：随着训练步数减少，但在KI下趋势逆转
- **相对运行长度**：随着训练步数增加，但在KI下减少

**解释**：
- 如3.3节讨论，更高的片段覆盖率通常与更通用的特征相关
- 更低的相对运行长度也与更大的泛化性相关
- 片段覆盖率随训练步数减少，而相对运行长度增加，但在KI下趋势逆转，与KI改善泛化的先前证据一致

**分类器分布的支持**：
这一结论进一步得到DROID PG5上每个模型所有特征的回归基础泛化分类器的P(general)分布的支持。KI模型将分布移向泛化性，具有比在可比训练步数下的AE微调模型更高的中位数。

**中位数分类器输出**：
- 10k步：0.190
- 30k步：0.181
- 60k步：0.179
- 90k步：0.181
- KI模型：0.206

**意义**：
- 虽然这些转移是微妙的，但它们表明特征级度量可以提供训练期间泛化的信号，而不需要现实世界滚动
- 然而，需要更严格的评估来确认这一点
- 这提供了一个有前景的方向，用于在训练期间监控模型泛化而无需昂贵的评估

## 与Spatial AGI的关系分析

### 直接贡献

#### 1. 理解空间表示的内部机制

这篇论文为理解VLA模型如何编码和表示空间信息提供了首个机械证据。通过SAE技术，研究者能够直接观察和干预模型的内部表示，揭示了以下关于空间理解的关键洞察：

**空间感知的分层表示**：
- 研究发现VLA模型在不同的层中编码不同层次的空间信息
- PaliGemma层（视觉-语言主干）更专注于视觉语义理解
- 动作专家层则编码与运动和空间操作直接相关的原语
- 后期PaliGemma层显示出与动作专家层相似的结构，表明知识隔离训练使主干学习了动作相关表示

**空间关系的可解释特征**：
- F128（预抓取对齐）：编码末端执行器与目标物体的空间关系
- F1902（运输）：编码物体到目标位置的空间接近度
- F445（任务完成）：编码与目标放置位置的接近度
- 这些特征显示了模型内部对空间关系的显式表示

**物体中心的空间表示**：
- 特征激活通常与特定物体进入视野相关
- 夹爪之间的物体可见性（F165）表明模型编码了从自身视角的空间几何
- 碗子、盘子等物体的空间配置被显式编码

#### 2. 空间泛化的机械证据

这是第一项提供机械证据证明VLA可以学习跨任务和场景的泛化特征的工作。这对Spatial AGI具有重要意义：

**泛化性的量化指标**：
- 提出了四个量化泛化性的指标：片段覆盖率、平均起始计数、平均激活幅度、相对运行长度
- 这些指标可以在不需要策略滚动的情况下计算，使得泛化性评估更加高效
- 为评估和改进Spatial AGI系统的泛化能力提供了实用工具

**跨场景的空间泛化**：
- LIBERO数据显示场景多样化不足（只有~20个视觉环境），导致大量记忆化
- DROID数据集（真实世界1545个独特任务）产生更多泛化特征（10.81% vs 2.62%）
- 这证明了增加场景和任务多样性对于空间泛化的重要性

**跨任务的空间泛化**：
- 通用特征在多个任务中激活，显示模型学习了可迁移的空间原语
- F158在操作子阶段过渡时激活，显示跨任务的空间操作模式
- F586编码捏式抓取，这是一个跨任务的空间运动原语

#### 3. 空间操作的可控性

论文通过特征转向实验证明了空间行为是可以被精确控制的：

**空间行为的因果控制**：
- 转向F128使机器人悬停在物体上方而不是抓取，证明了对接近行为的控制
- 转向F1902使机器人直接移动到目标位置，证明了对空间目标的控制
- 这意味着可以精确控制机器人的空间行为模式

**层级的空间控制**：
- 不同层对转向的敏感性不同，提供了控制粒度的层次

## 核心技术发现

### 1. VLA模型内部的二分结构

研究发现VLA模型的内部表示呈现清晰的二分结构：

**通用特征**：
- 占少数（1-11%，取决于数据集）
- 对应于可泛化的运动原语和语义属性
- 显示跨任务和场景的一致激活模式
- 可以通过转向精确控制行为

**记忆化特征**：
- 占主导（89-99%）
- 对应于特定训练演示中的动作序列和环境布局
- 在特定片段或场景中持续激活
- 转向效果不稳定且难以预测

**意义**：
这为理解VLA的泛化失败提供了机械解释：模型过度依赖记忆化而不是学习泛化原语。

### 2. 数据集多样性的关键作用

研究表明训练数据集的规模和多样性直接影响特征泛化：

**LIBERO vs DROID**：
- LIBERO（40任务，~20场景）：2.62%通用特征（PG平均）
- DROID（1545任务，多样化场景）：10.81%通用特征（PG平均）
- DROID的通用特征比例是LIBERO的4倍以上

**微调步数的影响**：
- 随着微调步数增加，泛化特征比例下降
- 知识隔离似乎缓解这种下降
- 片段覆盖率和相对运行长度显示与泛化性相关

**意义**：
为改进Spatial AGI提供了明确方向：需要更大规模和更多样化的训练数据。

### 3. 特征的可转向性

通过转向实验证明了特征与行为的因果关系：

**F128转向**：
- 转向此预抓取对齐特征使机器人悬停在物体上方
- 行为与特征的语义解释完全一致
- 证明特征编码了空间接近概念

**F1902转向**：
- 转向此运输特征使机器人直接移动到目标位置
- 跳过物体抓取，甚至与篮子碰撞
- 证明特征编码了目标接近和运输概念

**层的差异**：
- 动作专家层对转向更敏感
- PaliGemma层影响较小，后期层几乎无影响
- 提供了不同控制粒度的层次

**意义**：
这是首个证据证明单个SAE特征可以因果性地影响VLA行为，为行为控制提供了新途径。

### 4. 知识隔离的潜力

研究表明知识隔离技术可能有助于保留泛化能力：

**特征分布的转移**：
- KI模型的P(general)分布中位数更高（0.206 vs 0.181）
- 片段覆盖率和相对运行长度趋势逆转
- 表明KI可能缓解微调期间的泛化性下降

**架构分离的作用**：
- π0.5的分离架构（VLM主干 + 动作专家）可能有助于保留泛化
- 后期PaliGemma层显示动作专家类似的结构
- 可能允许主干保留语义知识同时专家学习专门化操作

**意义**：
为Spatial AGI系统设计提供了新思路：架构分离可能平衡专门化和泛化。

### 5. 可解释性的高比例

尽管泛化特征比例低，但可解释性比例高（79.17%）：

**广泛的可解释性**：
- 跨模型（π0.5, OpenVLA）、数据集（LIBERO, DROID）和层（从早期到后期）
- 即使记忆化特征也通常可解释
- 表明VLA学习高度结构化的表示

**运动原语的发现**：
- 抓取、放置、运输、对齐等基本操作原语被发现
- 子任务检查点（F158）等高级概念也被识别
- 显示VLA学习分层化的操作表示

**意义**：
即使泛化有限，VLA学习高度结构化和可解释的表示，为改进提供了基础。

## 个人思考

### 1. 最有趣的发现

**特征转向的精确控制能力**

最让我惊讶的是单个SAE特征竟然能够如此精确地控制机器人行为。转向F128使机器人"悬停"而不是"抓取"，转向F1902使机器人"跳过抓取"直接去"运输"——这些行为完全符合特征的语义解释。这证明了VLA内部确实有明确的行为控制机制，而不仅仅是黑盒相关性。

这种精确控制的含义是深远的：
- 我们可以用这个方法来"调试"机器人行为
- 可以在不重新训练的情况下调整行为
- 可以在运行时干预策略以确保安全或符合期望

**分层表示的自发出现**

研究发现不同层自然地编码不同层次的信息：
- PaliGemma早期层更接近原始视觉输入
- 中期层显示与语义相关的激活
- 后期层显示与动作专家层类似的结构
- 动作专家层显示运动原语的激活模式

这种分层不是设计出来的，而是自发出现的，暗示了深度学习的涌现特性。对于Spatial AGI，这很鼓舞人心——如果我们给模型足够的架构和数据，它可能自发学习到空间推理的不同层次。

**记忆化与泛化的清晰分界**

虽然直觉上我们可能认为记忆化和泛化是一个谱，但研究显示它们可以相当清晰地区分：
- 记忆化：低片段覆盖率、低起始计数、持续激活
- 泛化：高片段覆盖率、高起始计数、爆发式激活

逻辑回归分类器在这种二分上达到96-100%的准确性，表明这两类特征有本质不同的激活模式。这为诊断模型是否真正泛化提供了实用工具。

### 2. 最意外的结果

**可解释性高但泛化低**

预期中，如果79%的特征可解释，那么应该有相当一部分泛化特征。但结果令人意外：89-99%的特征是记忆化的，只有1-11%是泛化的。

这说明：
- 可解释 ≠ 泛化。一个特征可以完美解释一个特定场景，但完全不泛化
- VLA模型的主要学习模式是记忆化训练演示，而不是提取泛化原语
- 要构建真正的Spatial AGI，我们还需要显著改进训练方法

**知识隔离的微妙效果**

预期知识隔离（KI）应该有更明显的效果，但结果显示改善是"微妙的"（subtle）：
- P(general)中位数从0.181（30k步）到0.206（KI）
- 这是一个约14%的相对改善，但绝对差异很小

这可能暗示：
- 知识隔离有帮助，但不是银弹
- 更根本的改变（数据集规模、架构设计）可能更重要
- 我们对如何保留泛化能力的理解还有限

**OpenVLA的极低泛化**

OpenVLA只有0.45%的泛化特征，远低于π0.5的2.62%（LIBERO）和10.81%（DROID）。这很意外，因为：
- OpenVLA使用更大的基础模型（Llama 2 7B vs Gemma 2B）
- 它是单一模型架构而不是分离架构

可能的原因：
- 单一模型架构可能不利于泛化
- 在LIBERO Goal上的微调数据集太小（428片段 vs LIBERO的1,693）
- 不同架构对泛化的支持不同

### 3. 最有价值的启发

**数据集多样性是关键**

研究清晰地表明，增加数据集规模和多样性直接改善泛化：
- LIBERO（40任务）→ 2.62%泛化
- DROID（1545任务）→ 10.81%泛化

这对Spatial AGI开发有几个含义：
1. **质量 > 数量**：不是更多数据，而是更多样的数据
2. **场景多样化至关重要**：需要在多样化环境中收集数据
3. **任务多样化同样重要**：不同类型的任务促进泛化原语

这对于实际开发意味着：
- 应该优先考虑数据集多样性而不是单纯的规模
- 可以使用合成数据来增加场景多样性
- 跨任务的数据共享可能比单任务的更多数据更有效

**训练时监控可以替代评估**

特征度量可以在训练期间计算，无需策略滚动，这对于Spatial AGI训练有重要意义：

传统评估问题：
- 策略滚动（特别是在模拟中）耗时且昂贵
- 真实世界评估更是不可行
- 这限制了迭代速度和实验范围

新方法的潜力：
- 可以在训练期间实时监控泛化性
- 无需完整评估就可以调整训练策略
- 大大加快开发迭代速度

这可能导致：
- 更快的Spatial AGI开发周期
- 更实验性的训练方法（快速试错）
- 更好的超参数调优（基于内部信号而非外部性能）

**分离架构的优势**

π0.5的分离架构（VLM主干 + 动作专家）显示出相对于OpenVLA单一架构的优势：
- 更高比例的泛化特征
- 后期PaliGemma层学习动作相关表示
- 知识隔离可能更有效

这对Spatial AGI架构设计的启发：
1. **功能分离可能有助于泛化**：不同模块专注不同任务
2. **主干保留通用知识**：VLM保持泛化的视觉-语义理解
3. **专家学习专门化**：动作专家学习特定于机器人技能

可能的未来架构：
- 更细粒度的功能分离（感知、推理、规划、控制）
- 模块化的知识传递机制
- 动态的专家选择和组合

### 4. 限制和未解决挑战

**转向的不完美性**

转向实验虽然证明了因果性，但转向并不完美：
- F128转向虽然产生悬停，但其他行为可能不预期
- 叠加效应仍然存在（特征不独立）
- 记忆化特征的转向效果不稳定

可能的解释：
1. **非线性相互作用**：VLA通过流匹配产生动作，存在非线性相互作用
2. **预测 ≠ 因果**：特征预测行为但不完全控制行为
3. **特征重叠**：多个特征可能同时影响相似行为

含义：
- 精确行为控制需要更深入的理解
- 可能需要转向多个特征而不仅仅是单个特征
- 需要更好的控制方法来处理叠加效应

**特征分类的不准确性**

论文讨论了两个被误分类的例子：
- F1939（73%覆盖率但单起始）→ 被分类为记忆化
- F1381（22.6%覆盖率但语义一致）→ 被分类为记忆化

这暴露了当前分类方法的局限性：
- 低起始计数 ≠ 一定是记忆化（可能只是场景初始）
- 低覆盖率 ≠ 一定是记忆化（可能任务子集很小）

可能的改进：
1. **考虑语义一致性**：不只是统计模式，还要考虑跨场景的语义对齐
2. **数据集多样性感知**：调整决策边界考虑任务分布
3. **额外的度量指标**：需要更好的指标来捕捉"语义泛化"

**实验范围的局限**

研究在几个关键方面受到限制：
1. **无硬件评估**：只在模拟中测试，真实世界可能不同
2. **有限的层范围**：主要关注中期层，早期和后期层研究不足
3. **每token激活**：主要使用平均池化，每token分析有限
4. **有限的任务多样性**：主要是桌面操作，未涉及更复杂任务

这些限制的意义：
- 真实世界部署还需要更多验证
- 对VLA的完整理解需要更广泛的实验
- 空间推理能力需要更复杂的任务来测试

**关键设计决策**：

**扩展比选择**：
- 选择ER = 1（而非LLM研究中常用的4-8）是基于机器人数据集的规模
- 更高扩展比导致显著更多的死特征（见Figure 7）
- 可解释特征即使在紧凑字典大小下也能可靠出现
- 这归因于机器人学中使用的小得多的数据集规模

**TopK激活**：
- 提供对特征稀疏性的直接控制
- 通过仅保留k个最大激活来强制稀疏性
- 与ReLU结合确保非负特征系数

**AuxK损失**：
- 解决死潜在问题（长时间不激活的特征）
- 对重建残差应用辅助损失
- 使用最近500步中不活跃的特征

#### 激活收集策略

**π0.5的收集**：
- PaliGemma层：在预填充期间触发（处理所有图像和指令标记）
- 动作专家层：在动作扩散过程的最终去噪步骤期间触发

**平均池化 vs 每token**：
- **默认使用平均池化**：
  - 每时间步产生单个d维向量
  - 大大减少存储需求（DROID的2000片段：减少两个数量级）
  - 捕获每时间步的聚合状态
  - 原因：每个时间步代表单个观察-动作对，是机器人行为的自然分析单位

- **每token收集**：
  - 每时间步涉及收集超过768个标记的激活（每个相机256个图像块标记）
  - 需要~3.5TB存储（对于2000片段DROID数据集）
  - 为了解决存储问题，将每个图像的块求和为单个向量
  - 每时间步产生2个图像向量和~21个文本标记向量

**OpenVLA的收集**：
- 记录Llama 2主干五个层的预填充激活
- 涵盖整个输入序列
- 单次前向传递预测单个动作

#### 多种子实验

为了验证识别的特征反映模型表示的结构而不是SAE的想象，训练了七个从独立随机种子开始的SAE（在相同的激活数据上，π0.5 LIBERO PG5）。

**结果**（Figure 8）：
- 对于给定片段，顶级特征在种子间一致恢复
- 显示相似的时间激活模式
- 跨不同种子识别的共同模式提供了证据，表明学习的SAE特征来自底层模型

**意义**：
- 特征不是随机伪影
- 反映模型表示的真实结构
- SAE方法可靠地提取有意义的信息

### 跨层特征分析

#### 层级差异

**π0.5的层分析**（Figure 9）：

**PaliGemma层**：
- **PG0**：激活紧密 resembles输入嵌入；与SigLIP嵌入基线比较确认了这种相似性
- **PG5 & PG11**：包含最显著和可解释的特征；显示事件锁定的激活模式
- **PG17**：类似于动作专家层的结构；与知识隔离训练一致

**动作专家层**：
- 显示与PaliGemma层 qualitatively different pattern
- SAE特征在操作的不同阶段激活，由抓取事件打断
- 解释为编码特征运动方向的运动原语
- 这种运动原语类似结构在LIBERO和DROID动作专家层中一致（尽管依赖完全不同的动作表示）

**跨模型差异**：

**π0.5 vs OpenVLA**：
- **DROID激活**：明显比LIBERO对应物更密集
  - 归因于DROID真实世界场景的显著更大视觉多样性
  - 需要SAE同时招募更多特征来表示每个时间步
- **OpenVLA特征**（Figure 9c）：显示与π0.5 qualitatively different texture
  - 由于OpenVLA只是不包含动作专家的VLM，激活模式更类似于PaliGemma主干层
  - 虽然单个特征在热图中似乎更稀疏，分类为通用的特征比例实际上显著低于π0.5

#### 嵌入基线对比

为了确保识别的特征来自机器人微调而不是来自预训练视觉编码器中的现有特征，在两个基线上训练SAE：
1. π0.5嵌入层
2. 从未见过任何机器人数据的冻结预训练SigLIP编码器

**结果**（Figure 10）：
- **π0.5嵌入激活**：与PG0中的发现有些相似
  - 不令人惊讶，因为两者只被单个注意力层和MLP分开
- **预训练SigLIP模型**：产生 qualitatively different and much denser 激活模式
  - 缺乏在后期层中识别的通用特征的事件锁定结构
  - 表明主要结果中报告的可解释特征不是从预训练视觉表示继承的，而是在机器人微调期间出现的

**意义**：
- 可解释特征是在机器人微调期间学习的
- 不是从预训练视觉模型继承的
- VLA在微调期间学习了与机器人操作相关的特定特征

### 每token SAE探索

#### 设置

**动机**：
虽然平均池化激活揭示了几项激动人心的广泛泛化特征，探测视觉-文本语义对齐需要token级分辨率。

**方法**：
- 为了使问题更可处理，将所有图像输入聚合成单个"图像"token
- 为每个"文本"token收集单独激活
- 训练单个SAE来重建所有这些激活

#### 结果

**总体观察**（Figure 12）：
- 每token SAE提供更多关于模型的洞察
- 但需要更多调查和可能更多训练数据来完全评估
- 总体上，每token SAE有前景但目前比其平均池化对应物不太可解释

**三个示例特征**：

**F1881**：
- 在所有片段中突出但缺乏人类可解释性
- 在这个片段中，在主相机token上强烈且几乎均匀激活，在腕相机上更稀疏、较低幅度
- 所有突出文本token显示非零激活，与通用视觉-语义对齐角色一致
- 名词"pot"和其修饰符"black"显示几乎相同的激活模式
  - 暗示这个特征连接场景中的共指标记
- 然而，这些激活的时间结构不是清晰可解释的

**F225**：
- 同样在所有片段中突出，但似乎捕获纯文本语义结构
- 在两个相机token上零激活，但在任务指令的名词上有强激活
- 不在动词上激活，跨片段一致的模式
- "black"和"pot"同样共激活，当腕相机定位在pot上方时峰值

**F1659**：
- 一个可解释的记忆化特征：
  - 顶级相机token激活 exclusively 来自包含lids和pots的片段
  - 在lid放置在pot上时峰值
  - 顶级文本token激活对应"pot"或相邻空间，当"pot"是最终文本token时
  - 在这个片段中，"pot"是最高激活token，同样与"black"和腕相机共激活
  - 其余token显示 qualitatively similar 但显著更弱的激活
- 其他片段中的类似特征显示强视觉-文本对齐，同时保持任务特定

**与平均池化SAE的对比**：
- 相比平均池化SAE，这些每token SAE在一般情况下显著不太可解释
- 需要进一步调查来完全表征其视觉-语义对齐属性
- 泛化性证据存在但集中在每层不到十个特征中
- 这些通用特征的顶级激活文本token通常是功能词（"the"、"and"）而不是语义信息的内容词

**意义**：
- 每token分析提供更深层的洞察，但可解释性有限
- 需要更多数据和研究来理解视觉-文本对齐
- 目前，平均池化方法为理解机器人行为提供更好的可解释性

### 模型架构

#### π0.5架构

**组成**：
- **PaliGemma视觉-语言主干**：
  - Gemma 2B语言模型
  - 18个transformer层
  - 隐藏维度d = 2048
  - 400M SigLIP图像编码器
- **动作专家**：
  - 300M参数
  - 18个transformer层
  - 隐藏维度d = 1024

**输入**：
- PaliGemma主干处理三个相机图像（基座、左腕、右腕）
  - 对于研究案例，只使用两个输入图像
  - 加上自然语言指令

**动作生成**：
- 动作专家通过迭代去噪将表示解码为连续机器人动作
  - 通过key-value缓存关注PaliGemma
  - 使用扩散模型

**知识隔离**：
- 使用名为知识隔离的技术微调的两个变体
- π0.5-LIBERO：在LIBERO上微调
- π0.5-DROID：在DROID上微调

**选择原因**：
- LIBERO提供控制的模拟环境用于闭环评估
- DROID提供一些真实世界多样性，能够识别通用语义特征

#### OpenVLA架构

**组成**：
- **Llama 2 7B主干**：
  - 32个transformer层
  - 隐藏维度d = 4096

**特点**：
- 单一语言模型用于感知和动作预测
  - 不像π0.5的分离架构
- 自回归预测动作
- 使用公开可用检查点在LIBERO Spatial套件上微调

**包含原因**：
- 测试π0.5中观察到的特征和现象是否跨VLA架构泛化
  - OpenVLA有根本不同的动作解码机制

### 评估方法

#### 可解释性评估

**采样方法**：
- 随机抽取20个特征
- 从π0.5 LIBERO PG5和PG11、π0.5 DROID PG5和PG11，以及OpenVLA Layers 8和24
- 手动标记每个特征为可解释或不可解释

**可解释性标准**：
- 特征是可解释的，当：
  - 其激活模式，无论是在片段内还是跨片段
  - 始终与可识别的传感器运动事件或视觉状态一致时

**额外内容**：
- 在网站上提供了LIBERO中10个特征的详细解释
- 标签为"labeled"

#### 通用特征分类

**分类器训练**：
- 使用30个手动标记特征（15个通用，15个记忆化）
- 来自单个参考层
- 训练逻辑回归分类器
- 使用四个时间度量：平均起始计数、片段覆盖率、平均激活幅度、相对运行长度

**分类器应用**：
- 将训练的分类器应用于每个模型的所有层的所有特征
- 为每个微调数据集训练一个分类器（LIBERO和DROID）
- OpenVLA分类器重用LIBERO边界（因为OpenVLA也在LIBERO上微调）

#### 转向实验

**闭环评估**：
- 在LIBERO中评估转向效应
- **π0.5**：
  - 通过10次迭代去噪在单次前向传递中预测50个未来时间步的动作轨迹
  - 转向挂钩在所有10次去噪迭代期间激活
  - 因此扰动影响整个细化过程，从噪声到动作轨迹
  - 预测的50个时间步中，只有前5个在重新查询模型之前执行
- **OpenVLA**：
  - 以自回归方式在每次前向传递中预测单个动作
  - 在每个环境步中用当前观察重新查询模型

**转向设置**：
- 转向从第10个时间步开始并继续通过整个片段
- 从π0.5动作地平线执行五个步骤，然后更新观察并推理模型

**转向强度**：
- 在所有实验中使用单个特征转向
- 使用标量系数α控制转向幅度
- 解码器列在构造上为单位归一化

## 限制和未来工作

### 论文识别的局限性

#### 1. 可转向性 ≠ 可解释性

**问题**：
发现一个特征的有意义顶级激活不意味着可靠的可转向性。具体来说，发现许多清晰特征在用于转向时显示有限或不可预测的因果影响。

**可能的原因**：
1. **非线性下游相互作用**：由于VLA通过流匹配产生动作
2. **预测 ≠ 因果**：预测行为不意味着控制行为
3. **特征重叠**：多个特征可能同时影响相似行为

**含义**：
- 高可解释性不意味着高可控性
- 需要更好的理解特征如何影响最终动作
- 可能需要转向多个特征而不仅仅是单个特征

#### 2. 分类不准确性

**问题**：
发现一些特征不拟合提议的通用vs记忆化特征分类。例如：
- 一些特征显示高度通用，跨许多不同片段类型激活，但只在片段的结束或开始时激活
- 其他特征似乎通用但在数据集的如此小部分激活，以致被分类为记忆化

**具体例子**（Section A.5.1）：
- **F1939（LIBERO PG5）**：ō = 1.00, c = 0.732, ā = 0.037, ℓ̄_r = 0.127
  - 片段覆盖率73%，但起始计数恰好1.0
  - 所有激活在每个片段的前20个时间步内发生，不管场景或任务
  - 与编码机器人"home"位置一致——一个所有LIBERO环境共享的姿势
  - 视觉检查是通用的（场景和任务不变，近四分之三片段中存在）
  - 但其每片段单起始模式产生一个与记忆化特征无法区分的ō

- **F1381（DROID PG5）**：ō = 1.00, c = 0.226, ā = 0.065, ℓ̄_r = 0.990
  - 在抓取lid期间激活，不管lid类型（金属锅盖、玻璃盖、纸杯盖、绿色塑料盖）和场景
  - 在135个包含"lid"一词的任务指令片段中116个中触发（86%召回率）
  - 在336个额外片段中触发，指令如"把锅放在桌子右边并打开它"，通过lid抓取完成
  - 然而，因为lid相关片段只构成DROID数据集的6.7%
  - 片段覆盖率0.226低于分类器的决策边界
  - 近统一相对运行长度（ℓ̄_r = 0.990）进一步惩罚它

**结构性限制**：
两种情况暴露了相同的结构性限制：
- 以统一语义连贯但在数据集中比例小子集激活一次每片段（ō ≈ 1）的特征将被分类为记忆化
- 分类器将低爆发性与记忆化混淆，因为在标记训练集中，通用特征主要是多起始

**可能的解决方案**：
1. 数据集多样性感知的片段覆盖率归一化
2. 捕获跨场景一致性的额外度量，独立于激活频率
3. 更复杂的分类器，考虑语义一致性和激活模式

#### 3. 数据约束

**问题**：
与VLM研究相比，作者使用的 orders of magnitude 更少数据，由于机器人学特定的数据约束。

**存储和计算限制**：
- 由于存储和计算约束，主要使用平均池化token
- 每时间步基础评估SAE
- 包括一些初始每token实验（Section A.7）
  - 有前景但通常不太可解释，值得进一步研究

**含义**：
- 可能需要更多数据来完全理解VLA的内部表示
- 每token分析提供更深层的洞察但需要更多资源
- 机器人数据稀缺性是根本限制

#### 4. 评估范围限制

**问题**：
评估范围仍然有些受限。虽然研究在闭环模拟滚动中演示转向结果，但不评估硬件。

**高成功率通用VLA策略的缺乏**：
- 缺乏可以在真实硬件上可靠部署的高成功率通用VLA策略
- 可能进一步抑制可转向性
- 真实世界的噪声和不确定性可能影响结果

**含义**：
- 真实世界部署需要更多验证
- 当前结果可能在模拟中看起来比在现实中更干净
- 需要硬件实验来验证发现在现实世界中的有效性

### 未来工作方向

#### 1. 训练时应用

**诊断微调的脆弱性**：
- 在模型训练期间使用SAE（不需要任何滚动，模拟或其他）
- 轻量级应用
- 特定片段特征的存在可以用于诊断常见VLA微调过程的脆弱性

**泛化的代理指标**：
- 特征度量可能作为训练时泛化的代理，无需真实世界评估
  - 真实世界评估可能耗时且昂贵
- 使用此信息，未来工作可以探索：
  - 变化数据集混合和微调时间表以保留可迁移行为
  - 保留泛化的其他训练技术

**潜在影响**：
- 更快的开发迭代（无需完整评估）
- 更好的超参数调优（基于内部信号而非外部性能）
- 更实验性的训练方法（快速试错）

#### 2. 运行时应用

**运行时监控器**：
- 在测试时，这些特征可以用于运行时监控器
- 警告用户策略何时利用记忆化轨迹而不是使用通用特征
- 提供透明度并帮助用户理解策略行为

**安全应用**：
- 可以检测不安全或不可靠的行为
- 可以在危险情况下触发干预
- 为Spatial AGI的安全部署提供新机制

**自适应控制**：
- 根据检测到的特征类型调整策略行为
- 当使用通用特征时允许更自信的操作
- 当使用记忆化特征时触发谨慎模式

#### 3. 策略增强

**显式特征鼓励/消除**：
- 用户可以在微调期间明确鼓励或消除特定SAE特征
- 这可以调整模型更依赖哪些表示
- 可能用于：
  - 鼓励泛化特征
  - 消除记忆化特征
  - 强化安全相关特征

**强化学习整合**：
- 在强化学习设置中使用这些信息
- 可能设计奖励函数来鼓励使用通用特征
- 可以在训练期间动态调整特征重要性

**架构改进**：
- 基于特征发现设计新的VLA架构
- 可能更好的功能分离（感知、推理、规划、控制）
- 可能更有效的知识保留机制

#### 4. 更深入的可解释性

**每token分析**：
- 初始每token实验有前景但需要更多工作
- 需要更多数据和研究来完全评估
- 提供对视觉-语义对齐的更深洞察

**跨模态对齐**：
- 研究视觉、语言和动作表示如何对齐
- 理解多模态信息如何集成
- 可能导致更好的多模态模型

**因果干预**：
- 超越简单的转向，开发更复杂的因果干预方法
- 可能需要同时操纵多个特征
- 可能需要考虑特征之间的相互作用

#### 5. 硬件验证

**真实世界实验**：
- 需要在真实硬件上验证发现在现实世界中的有效性
- 真实世界的噪声和不确定性可能影响结果
- 对于实际部署至关重要

**跨平台泛化**：
- 测试发现在不同机器人平台上的泛化
- 验证SAE特征是否跨机器人架构和配置通用
- 对于构建通用Spatial AGI系统很重要

**长期部署研究**：
- 研究这些发现在长期部署中如何表现
- 特征如何随时间演变
- 如何维护和更新可解释性

## 总结

### 核心贡献总结

1. **首个VLA机械可解释性框架**：
   - 提出了使用SAE从VLA模型提取可解释、通用和可转向特征的技术
   - 提供了首个机械证据，证明VLA可以学习跨任务和场景的泛化特征

2. **泛化量化度量**：
   - 提出了四个泛化量化度量标准（片段覆盖率、起始计数、激活幅度、运行长度）
   - 可以在微调数据上计算，无需策略滚动
   - 为评估和改进Spatial AGI系统提供了实用工具

3. **大规模特征分类**：
   - 在多个VLA模型（π0.5, OpenVLA）和数据集（LIBERO, DROID）上训练SAE
   - 发现79.17%的特征可解释
   - 但89-99%是记忆化的，只有1-11%是泛化的

4. **特征转向验证**：
   - 通过转向实验证明单个SAE特征可以因果性地影响VLA行为
   - 转向通用特征诱导与其语义含义一致的行为
   - 可以跨任务和场景应用

5. **训练洞察**：
   - 观察到所有情况下都有显著的片段级记忆化
   - 但在DROID等多样化数据集上微调导致相对更多的泛化特征
   - 知识隔离似乎缓解微调期间的泛化性下降

6. **开源工具**：
   - 提供了Dr. VLA，一个用于VLA中SAE训练、特征评估和策略转向的开源软件包
   - 用户友好的界面
   - 促进未来VLA机械可解释性研究

### 对Spatial AGI的意义

**当前VLA的局限**：
- 泛化能力严重受限（只有1-11%泛化特征）
- 主要依赖记忆化训练演示
- 远未达到Spatial AGI所需的强泛化能力

**改进方向**：
1. **数据集设计**：
   - 更大规模和更多样化的训练数据集
   - 强调场景和任务多样性
   - 避免在有限场景上过度训练

2. **训练方法**：
   - 使用知识隔离等技术保留泛化
   - 在训练期间监控泛化性（使用SAE度量）
   - 改进微调策略以避免过度记忆化

3. **架构设计**：
   - 分离架构（VLM主干 + 动作专家）显示优势
   - 功能分离可能有助于泛化
   - 需要更好的架构来平衡专门化和泛化

4. **可解释性和安全性**：
   - SAE提供诊断和调试工具
   - 可以在运行时监控策略行为
   - 为实现语义和情境安全提供新机制

**未来展望**：
虽然当前VLA远未达到Spatial AGI，但这篇论文为理解和改进提供了重要基础：
- 我们现在有工具来诊断VLA的内部表示
- 我们有度量来量化泛化性
- 我们有方法来干预和指导行为
- 我们有方向来改进训练和架构

### 最有价值的启示

1. **机械可解释性是可行路径**：
   - SAE技术成功应用于VLA模型
   - 79%的特征可解释，即使泛化性低
   - 为理解和改进VLA提供了实用工具

2. **数据集多样性是关键**：
   - DROID的泛化特征比例（10.81%）是LIBERO（2.62%）的4倍
   - 场景和任务多样性对于空间泛化至关重要
   - 为Spatial AGI数据集设计提供了明确方向

3. **训练时监控可以替代评估**：
   - 特征度量可以在训练期间计算，无需策略滚动
   - 大大加快开发迭代速度
   - 为训练大型Spatial AGI系统提供了新方法

4. **特征转向提供新控制机制**：
   - 单个特征可以因果性地影响行为
   - 为精确控制机器人行为提供了新途径
   - 可能用于实时干预和安全保证

## 相关工作

### 2.1 VLM的机械可解释性

**文献中的可解释性方法**：
- 因果追踪
- 激活修补
- 探针

**稀疏自编码器（SAEs）**：
- 在模型激活中发现先验未知潜在特征或模式的非监督方法
- 现代SAE可解释性范式由Towards Monosemanticity推广
  - 对单层transformer应用SAE
  - 提取4000+可解释特征
- 进一步工作将此方法扩展到生产级模型
  - 包括视觉-语言模型（VLMs）的应用

**线性表示假设**：
- 建议概念在模型激活空间中表示为方向
- 直接检查这些方向被叠加阻碍
- 目标是将高度叠加的残差流表示分解为单语义特征

**金门Claude**：
- 金门大桥的单语义特征被隔离
- 用于特征级转向
- 突出特征级干预的能力

**与本文的联系**：
- VLAs通常重用相同的VLM主干
- 本文建立在这个可解释性框架上
- 将SAE基础分析和干预适应到体现控制设置
- 目标是显示VLA表现出与LLM相同的许多可解释性特征

### 2.2 VLAs的机械可解释性

**VLAs机械可解释性是新领域**：
- 过去一年中只有少量工作发布

**现有工作类别**：
1. **从内部表示解码任务相关信息**
2. **干预内部激活以因果转向行为**

#### 解码方面

**Lu等人 [20]**：
- 在OpenVLA激活上训练线性探针来解码状态
  - 物体位置和动作
- 发现探针准确性超过大多数层的90%
- 暗示像在LLMs中，VLAs通常线性编码与任务相关的结构化信息
  - 即使端到端策略仍然是黑盒

**Molinari等人 [21]**：
- 超越静态状态解码
- 探测向涌现的"世界模型"结构
- 测试状态过渡向量是否可从中间激活中恢复
- 提出SAE作为可解释规划的有前途的下一步
- 但不实现它们

#### 因果干预方面

**Häon等人 [22]**：
- 通过将FFN方向投影到标记空间来解释它们
- 按语义聚类方向（如更高/更低、更快/更慢）
- 显示操纵相应激活可以转向动作
- 问题：底层FFN特征仍然高度叠加
  - 可能不干净地扩展到更抽象或组合概念
  - 没有显式特征分解（如SAEs）

**Khan等人 [23]**：
- 提出使用SAE转向VLA模型
- 但不分析产生的SAE特征是否可解释
- 转向方法基于对比干预
  - 可以同时跨许多潜在方向起作用
  - 而不是针对单个孤立特征

**Buurmeijer等人 [24]**：
- 使用线性探针跨越VLAs的观察和干预
- 发现在闭环操作时可以可靠地应用控制理论技术
  - 以最小线性干预转向输出
- 提出利用SAE发现没有显式标签的特征
  - 作为未来工作的方向

#### 与本文的贡献

**相比现有工作**：
- 在解码和干预方面都有贡献
- 证明SAE特征可解释、通用和可转向
- 量化泛化性并提供分类方法
- 在多个模型和数据集上评估

### 2.3 改进VLA模型的泛化

**VLA策略脆弱性**：
- 特别是在监督微调（SFT）后的训练数据之外
- 许多工作寻求解决这个明显的脆弱性

#### 假设1：视觉表示退化

**Kachaev等人 [25]**：
- 提出视觉表示正则化器
  - 与冻结教师模型对齐模型表示
- 发现这有助于微调模型更好地泛化到空间推理任务

**知识隔离 [26]**：
- 防止微调退化内部模型表示的方法
- 提出机制如停止梯度和架构分离
  - VLM主干保留语义
  - 专家高效处理连续控制

**Grover等人 [27]**：
- 采取不同方法改进泛化
- 用双视觉编码器增强标准VLA架构
  - 一个编码器冻结，另一个可训练以适应
- 发现这增强视觉鲁棒性和指令泛化

#### 假设2：评估方法问题

**LIBERO-PRO [10]**：
- 发现标准LIBERO基准高估性能
- 提出系统性扰动强制真正的泛化
- 最大的挑战之一是提供真正分布外的场景

#### 与本文的联系

本文补充这些工作：
- 提供机械证据证明泛化问题
- 提出度量来量化泛化性
- 证明数据集多样性和知识隔离有助于泛化
- 为理解为什么某些方法有效提供基础
