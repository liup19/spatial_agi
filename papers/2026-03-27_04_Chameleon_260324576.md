## 1. 摘要

机器人操作通常需要记忆能力：遮挡和状态变化会使决策时的观测出现感知混淆，使得动作选择在观测层面呈现非马尔可夫性，因为相同的观测可能源于不同的交互历史。大多数具身智能体通过语义压缩的痕迹和基于相似性的检索来实现记忆，这会丢弃消除混淆所需的细粒度感知线索，并且可能返回感知相似但决策无关的情节。

受人类情景记忆的启发，我们提出了Chameleon，它将基于几何的多模态令牌写入记忆以保留消除混淆的上下文，并通过可微分的记忆堆栈产生目标导向的召回。我们还介绍了Camo-Dataset，一个真实机器人UR5e数据集，涵盖感知混淆下的情景召回、空间跟踪和序列操作。在各任务中，Chameleon在感知混淆的设置中持续提高决策可靠性和长视野控制性能，优于强基线模型。

**核心贡献**：
1. 感知混淆问题的形式化：我们将机器人记忆密集型任务形式化为具有感知混淆的决策问题，其中相似的决策时观测可能需要根据交互历史做出不同的动作，强调了对情景记忆的需求
2. 生物启发的分层情景记忆架构：我们提出了Chameleon，一个生物启发的分层记忆架构，集成了基于几何的消除混淆编码、通过HoloHead训练的可微分目标导向检索，以及用于长视野操作的内存调节整流流控制
3. 真实机器人情景记忆数据集和分析：我们介绍了Camo-Dataset，一个用于感知混淆下的情景召回、空间跟踪和序列操作的UR5e真实机器人数据集，以及定量评估和表征级分析

## 2. 研究问题和动机

### 2.1 机器人操作中的记忆需求

在真实世界的机器人操作中，智能体经常需要基于记忆进行行动。随着操作的展开，任务相关的状态可能被遮挡或覆盖，使得决策点在观测层面呈现非马尔可夫性，迫使正确的动作依赖于历史交互。这使得长视野记忆成为具身智能的中心瓶颈。

**典型案例：壳杯游戏 (Shell Game)**

如图1所示，在壳杯游戏中，当相同的杯子在抓取时间变得不可区分时，机器人必须依赖交换历史来选择正确的杯子。这种场景清楚地展示了感知混淆的挑战：相同的观测（三个看起来相同的杯子）需要不同的决策（抓取哪个杯子），这取决于之前的交互历史。

### 2.2 现有方法的局限性

**问题1：语义压缩丢失关键信息**

许多具身系统的记忆是语言中心化的：经验被总结成语义压缩的、类似文本的痕迹，反映了大型语言模型（LLMs）普及的检索增强生成（RAG）方法。这种压缩对于机器人记忆密集型任务中的感知混淆是适应不良的，因为它移除了确定正确动作所需的消除混淆的感知上下文，包括：
- 空间关系
- 视角相关的证据
- 之前交互的结果

在图1的例子中，诸如"球在杯子下"这样的总结可能在语义上是正确的，但在交换后仍然不足以识别正确的杯子。

**问题2：相似性驱动的检索导致干扰**

最近的工作通过视觉历史缓冲区增强具身智能体，但当重复的相似场景对应于不同历史时，相似性驱动的访问仍然会引入干扰。这种方法在两个关键方面暴露了失败：

1. **写入阶段的失败**：记忆必须被写成消除混淆的、可索引的事件以减少干扰
2. **检索阶段的失败**：记忆必须根据是否支持当前目标和近期未来结果来检索，而不是基于相似性

### 2.3 生物情景记忆的启发

生物情景记忆正好实现了这两个功能：

**进化视角**：在进化视角中，情景记忆通常被视为智能的基础，先于抽象、符号推理。它将经验组织为绑定到时空和因果上下文的事件，即使没有语言也能实现前瞻性行为。

**非语言动物的证据**：非语言动物提供了基于事件规划的具体证据，例如：
- 食物贮藏鸟类记住存储了什么、存储在哪里以及何时存储的
- 使用这些记忆来指导后续的检索决策

**神经机制**：从机制上讲，情景记忆通常归因于内嗅皮层（EC）、海马体形成（HC）和前额叶皮层（PFC）之间的协调动态：

| 神经结构 | 功能 |
|-----------|------|
| 内嗅皮层 (EC) | 整合多模态感官流 |
| 齿状回 (DG) | 支持模式分离，减少干扰，使相似体验存储为不同的痕迹 |
| CA3 | 通过来自部分线索的关联检索支持模式完成 |
| CA1 | 支持将召回内容绑定到上下文事件中，供下游使用 |
| 前额叶皮层 (PFC) | 控制使召回偏向于与当前目标和预期近期未来结果相关的记忆，而不是仅基于相似性 |

### 2.4 算法差距

通过这个视角，感知混淆暴露了当前具身系统中的两个算法差距：

1. **差距1：记忆形成中的消除混淆、可索引事件表示**
   - 如何在记忆形成期间形成消除混淆的、可索引的事件表示

2. **差距2：目标导向的检索学习**
   - 如何学习目标导向的检索，使召回由决策效用驱动

Chameleon正是为解决这两个差距而设计。

## 3. 相关工作

### 3.1 记忆的分类

随着神经科学的进展，记忆通常分为非陈述性记忆和陈述性记忆：

- **非陈述性记忆**：通过行为表现表达，包括技能和习惯
- **陈述性记忆**：涉及有意识地回忆事实和事件

在陈述性记忆中，情景记忆由模式分离和模式完成支持：
- **模式分离**：将多模态感官流集成到不同的事件表示中以消除混淆的感知
- **模式完成**：从部分线索检索事件以支持下游决策

### 3.2 AI中的记忆机制

在人工智能中，记忆通常分为参数化记忆和非参数化记忆（分别类似于知识和情景记忆）：

| 记忆类型 | 描述 | 示例 |
|-----------|--------|--------|
| 参数化记忆 | 在模型参数中编码跨任务知识 | 深度学习模型权重 |
| 非参数化记忆 | 存储特定任务的上下文 | RAG、记忆缓冲区 |

**LLM基智能体的RAG方法**：
- 通过基于嵌入的相似性搜索查询外部知识存储库来模拟长期记忆
- 在零样本设置中，由于缺乏相关性监督，检索仍然具有挑战性
- 语义相似性不一定对应于任务效用

最近的工作通过结构化知识或查询转换来增强检索，以更好地捕获长范围依赖性并减少语义差距。

### 3.3 具身智能中的记忆机制

在具身智能中，记忆机制通常继承自LLM范式，依赖于语义嵌入或稀疏拓扑映射来编码对象关系和空间结构。这种抽象将连续物理状态压缩为符号表示，并丢弃细粒度动态。

**不匹配问题**：
- 在非结构化环境中的操作需要跟踪对象位姿、表面属性、接触动态和不断演化的状态转换
- 语义记忆与物理动态之间的不匹配限制了VLA系统的泛化和操作精度

## 4. 核心方法

### 4.1 系统概览

Chameleon遵循**感知 → 记忆 → 策略**的流水线，用于感知混淆下的长视野操作（如图2所示）：

```
观测 (o_t)
    ↓
感知模块 (Φ_perc)
    ↓
融合令牌 (x_t)
    ↓
记忆模块 (Φ_mem)
    ↓
决策状态 (h_t)
    ↓
策略模块 (Φ_pol)
    ↓
末端执行器轨迹 (x̂_t:t+H)
```

**设计理念**：
1. **感知模块**：将两个相机视图和本体感觉转换为基于几何的、视图一致的补丁令牌x_t，在记忆形成之前减少混淆
2. **记忆模块**：通过完全可微分的动态将情节状态与工作状态耦合，并输出唯一的决策状态h_t作为策略的唯一接口
3. **策略模块**：条件仅基于h_t，通过条件流匹配生成未来末端执行器位姿轨迹

**单接口约束**：这个设计使记忆的作用显式化：如果智能体在长视野控制中成功，正确动作选择所需的信息必须在h_t中表示并随时间可靠传播。

### 4.2 感知模块 (Perception)

感知模块实现了记忆形成期间的消除混淆要求：决策状态只有在感官输入（视觉和几何）被编码到足够判别性的特征中时才能被分离。

#### 4.2.1 输入和输出

每个时间步t，智能体接收多模态观测：

```
o_t = {I^f_t, I^h_t, s_t, ψ_t}
```

其中：
- I^f_t：来自固定前视相机的RGB图像
- I^h_t：来自手挂载侧视相机的RGB图像
- s_t：机器人本体感觉状态，包括末端执行器位姿和夹持器状态
- ψ_t：可选的任务阶段指示器（例如，离散0/1信号表示观察阶段与操作阶段）

感知模块输出融合的视觉令牌x_t ∈ ℝ^{N×d}用于记忆模块。

#### 4.2.2 腹侧流：补丁外观令牌

对于每个视图v ∈ {f, h}，冻结的DINO编码器产生补丁令牌：

```
V^v_t = DINO(I^v_t) ∈ ℝ^{N_v×d}
```

#### 4.2.3 背侧流：末端执行器锚定的几何代码

从s_t我们获得末端执行器位姿p_t ∈ ℝ³，并将其投影到每个视图：

```
u^{EE,v}_t = Π(K^v, T^v_t, p_t) ∈ ℝ²
```

对于视图v中的补丁i，定义：

**单位射线计算**：

```
r^{v}_{t,i} = (K^v)⁻¹ũ^{v}_{t,i} / ‖(K^v)⁻¹ũ^{v}_{t,i}‖₂ ∈ ℝ³
```

其中ũ^{v}_{t,i}是补丁中心图像坐标的齐次形式。

**动作中心描述符**：

```
g^{v}_{t,i} = [u^{v}_{t,i}, r^{v}_{t,i}, ρ^{v}_{t,i}, cosθ^{v}_{t,i}] ∈ ℝ⁷
```

其中：
- ρ^{v}_{t,i} = ‖u^{v}_{t,i} - u^{EE,v}_t‖₂：测量到末端执行器投影的接近度
- cosθ^{v}_{t,i} = r^{v}_{t,i}ᵀ d^v_t：从相机中心到末端执行器位姿的单位方向余弦角

几何编码器输出：
- C^v_t ∈ ℝ^{N_v×d_c}：用于特征调制的条件代码
- b^v_t ∈ ℝ^{N_v}：用于注意力门控的一元补丁偏置

其中b^{v}_{t,i} = MLP(g^{v}_{t,i})。

#### 4.2.4 几何偏置的双向交叉视图增强

前视和手视图通过残差交叉注意力相互增强。对于从视图a到视图b（a ≠ b）的注意力：

定义：
- Q = V^a_t W_Q
- K = V^b_t W_K
- U = V^b_t W_V

并更新：V̄^a_t = V^a_t + Attn_{a→b}(Q, K, U)

**注意力logits**结合了解析的对极可行性和学习的一元补丁偏置：

```
Attn_{a→b}(Q, K, U) = softmax(
    (QKᵀ/√d) + B^{epi}_{a→b}
    + b^a_t 1ᵀ + 1(b^b_t)ᵀ
) U
```

其中B^{epi}_{a→b} ∈ ℝ^{N_a×N_b}是使用基础矩阵F_{a→b}从标定的相对几何计算的，作为负点到对极线距离。

**关键设计**：一元项来自b^v_t将通信偏向于源自末端执行器锚定的可靠、动作相关的补丁，而无需引入显式成对匹配器。

#### 4.2.5 几何条件调制和输出

应用FiLM风格的调制并输出融合的证据令牌：

```
Ṽ^v_t(i) = γ(C^{v}_{t,i}) ⊙ V̄^v_t(i) + β(C^{v}_{t,i})
x_t = Concat(Ṽ^f_t, Ṽ^h_t) ∈ ℝ^{N×d}
```

这个x_t保留局部化的外观证据，同时将交叉视图交换限制为几何一致的对应关系，为记忆模块产生稳定的写入时证据流。

**设计优势**：
1. 前相机提供稳定的全局上下文
2. 手挂载相机在遮挡下捕获接触级细节
3. 末端执行器位姿作为动作中心的几何锚点，防止交叉视图聚合混合不兼容的证据
4. 几何引导确保只有空间一致的补丁可以相互通信

### 4.3 记忆模块 (Memory)

记忆模块将融合的感知映射到单个读取决策状态h_t。

为了在感知混淆下减少干扰，它使用**空间锚点**和**时间锚点**对多模态输入进行编码，因此证据被写入到多个有效半衰期（即以不同时间尺度编码和存储）的时空状态中。

#### 4.3.1 多模态融合

记忆模块消费来自感知模块的x_t ∈ ℝ^{N×d}，其中：

```
x_t = Concat(Ṽ^f_t, Ṽ^h_t)
N = N_f + N_h
```

形成本体感觉令牌和，如果可用，阶段令牌：

```
p_t = Proj_{prop}(s_t) ∈ ℝ^d
φ_t = E_{phase}[ψ_t] ∈ ℝ^d
```

然后添加学习的加性标签以在模态和视图之间保留令牌身份：

```
x̄^v_t = Ṽ^v_t + A^{mod}_{vis} + A^{view}_v, v ∈ {f, h}
p̄_t = p_t + A^{mod}_{prop}
φ̄_t = φ_t + A^{mod}_{phase}
```

最后，连接和归一化：

```
z_t = LN(Concat[x̄^f_t, x̄^h_t, p̄_t, φ̄_t]) ∈ ℝ^{M×d}
```

产生用于所有层的融合令牌z_t。如果ψ_t不可用，则省略φ̄_t并设置M = N + 1。

#### 4.3.2 动态上下文调制

每一层使用从前一层传播的工作上下文来解释相同的融合证据。令h^{(ℓ-1)}_t ∈ ℝ^{d_w}为层ℓ-1在时间t输出的工作状态，令W^{(ℓ)}_c ∈ ℝ^{d×d_w}为学习的线性投影到令牌空间：

```
z̃^{(ℓ)}_t = z_t + 1 (W^{(ℓ)}_c h^{(ℓ-1)}_t)ᵀ ∈ ℝ^{M×d}
```

其中1 ∈ ℝ^M是全一向量。

#### 4.3.3 空间和时间锚点

**模式分离**首先将视觉证据分解为A个空间分量。令：

```
x̄_t = Concat[x̄^f_t, x̄^h_t] ∈ ℝ^{N×d}
N = N_f + N_h
```

路由权重π^{(ℓ)}_{t,a,i} ≥ 0满足：

```
Σ_{a=1}^A π^{(ℓ)}_{t,a,i} = 1
```

由以下产生：

```
π^{(ℓ)}_{t,·,i} = softmax(Router^{(ℓ)}(x̄_{t,i}))
```

每个空间锚点通过附加共享的全局上下文形成视觉摘要和锚点特定的令牌集：

```
u^{(ℓ)}_{t,a} = Σ_{i=1}^N π^{(ℓ)}_{t,a,i} x̄_{t,i} ∈ ℝ^d
z̃^{(ℓ,a)}_t = Concat[u^{(ℓ)}_{t,a}, p̄_t, φ̄_t] ∈ ℝ^{3×d}
```

然后，将每个空间锚点输出z̃^{(ℓ,a)}_t扩展为B个时间槽，由b索引，代表后续状态空间动力学中的不同特征半衰期。使用学习的时间查询{q^{(ℓ)}_{a,b}}_{b=1}^B，槽特征为：

```
f^{(ℓ)}_{t,a,b} = Attn(q^{(ℓ)}_{a,b}, z̃^{(ℓ,a)}_t) ∈ ℝ^d
a = 1,…,A, b = 1,…,B
```

集体地，{f^{(ℓ)}_{t,a,b}}形成一个A×B时空槽矩阵，其中a选择空间分量，b选择时间分辨率。

#### 4.3.4 带有情节和工作状态的时空记忆

在层ℓ中，每个槽(a, b)维护一个情节状态m^{(ℓ)}_{t,a,b}。集合{m^{(ℓ)}_{t,a,b}}形成隐藏印迹，仅通过读取表达。

给定f^{(ℓ)}_{t,a,b}，计算：

```
θ^{(ℓ)}_{t,a,b} = Proj^{(ℓ)}_{a,b}(f^{(ℓ)}_{t,a,b})
```

其中θ^{(ℓ)}_{t,a,b}遵循Mamba风格的选择性状态空间模型（SSM）参数化，具有内容依赖的输入/读取因子和步长项。为每个时间索引b ∈ {1,…,B}分配不同的基本步长Δ^{(0)}_t_b以实现显式半衰期调度，并通过Proj^{(ℓ)}_{a,b}进一步通过槽内容调制有效步长。

这产生结构化的、内容自适应的多时间尺度记忆，而不是A×B个相同SSM的简单堆叠。为了效率，我们实现融合的按槽扫描，在单次传递中更新所有A×B状态。

**情节记忆更新**槽状态：

```
m^{(ℓ)}_{t,a,b} = SSM^{(ℓ)}_{epi}(m^{(ℓ)}_{t-1,a,b}; θ^{(ℓ)}_{t,a,b})
```

并产生槽读取：

```
r^{(ℓ)}_{t,a,b} = Read^{(ℓ)}_{epi}(m^{(ℓ)}_{t,a,b}) ∈ ℝ^d
```

将这些读取聚合为情节召回向量：

```
r^{(ℓ)}_t = (1/AB) Σ_{a=1}^A Σ_{b=1}^B W^{(ℓ)}_{a,b} r^{(ℓ)}_{t,a,b} ∈ ℝ^{d_w}
```

并将其集成到工作记忆中，这是层ℓ的唯一输出状态：

```
h^{(ℓ)}_t = SSM^{(ℓ)}_{work}(
    h^{(ℓ-1)}_t;
    LN(h^{(ℓ-1)}_t + Proj^{(ℓ)}_r(r^{(ℓ)}_t)
)
```

其中h^{(ℓ)}_t ∈ ℝ^{d_w}。

**关键设计**：输入和读取通过状态空间递归隐式实现：印迹通过工作状态输入进入，在情节状态中持续，并通过读取表达，无需显式检索门。

#### 4.3.5 工作读取和分层记忆融合

层ℓ发出工作读取：

```
y^{(ℓ)}_t = LN(W^{(ℓ)}_o h^{(ℓ)}_t) ∈ ℝ^{d_w}
```

其中W^{(ℓ)}_o是学习的线性投影。

使用任务查询q_{task} ∈ ℝ^{d_w}融合来自L层的读取。定义层密钥和值：

```
k_{t,ℓ} = U_ℓ y^{(ℓ)}_t
v_{t,ℓ} = Enc_ℓ(y^{(ℓ)}_t)
```

计算注意力权重和最终状态：

```
α_{t,ℓ} = softmax_ℓ(⟨q_{task}, k_{t,ℓ}⟩)
h_t = Σ_{ℓ=1}^L α_{t,ℓ} v_{t,ℓ}
```

产生的h_t是策略消费的最终记忆读取。

#### 4.3.6 HoloHead

HoloHead通过潜在想象目标强制模式完成：条件在h_t上，它预测N_a + N_c路点在2D（前相机图像平面末端执行器投影）和3D（标准化世界框架末端执行器位姿）：

```
[ŵ^{2D}_{t:t+N_a} ŵ^{2D}_{t:t+N_c} ŵ^{3D}_{t:t+N_a} ŵ^{3D}_{t:t+N_c}]ᵀ = HoloHead(h_t)
```

并使用以下监督：

```
L_holo = Σ_{ξ∈{2D,3D}}(
    ‖ŵ^{ξ}_{t:t+N_a} - w^{ξ}_{t:t+N_a}‖₁
    + ‖ŵ^{ξ}_{t:t+N_c} - w^{ξ}_{t:t+N_c}‖₁
)
```

这迫使记忆动力学从部分线索重建连贯的未来，导致任务相关的检索。

**设计优势**：
1. 通过潜在想象目标训练记忆状态维护和召回任务相关信息
2. 模式完成：从部分线索检索相关事件
3. 预测性：使决策状态能够预测近期状态演化

### 4.4 策略模块 (Policy)

策略模块从记忆读取h_t生成末端执行器位姿轨迹x̂_{t:t+H} ∈ ℝ^{H×d_raw}供低级逆运动学（IK）控制器执行。

#### 4.4.1 单状态条件

我们仅将轨迹生成器条件在投影令牌上：

```
c_t = W_{ctx} h_t
```

其中W_{ctx}是学习的线性投影。

#### 4.4.2 条件整流流匹配

令x₁ ∈ ℝ^{H×d_raw}为地面真实未来轨迹，采样x₀ ~ 𝒩(0, I)和τ ~ 𝒰(0, 1)。

定义插值：

```
x_τ = (1-τ)x₀ + τx₁
u_τ = x₁ - x₀
```

并训练速度网络v_θ从(x_τ, τ, c_t)预测u_τ：

```
û_τ = v_θ(x_τ, τ, c_t)
L_flow = E[‖û_τ - u_τ‖²]
```

其中损失被屏蔽以忽略当轨迹具有可变有效长度时的填充步。在推断时，通过从x(0) = x₀使用固定步求解器求解ODE ẋ = v_θ(x, τ, c_t)来采样x̂_{t:t+H}。

**设计优势**：
1. 整流匹配提供稳定的轨迹生成
2. 一次性生成整个轨迹，提高长视野控制的稳定性
3. 条件仅基于压缩的决策状态，使记忆的作用显式化

## 5. 数据集设计

### 5.1 记忆密集型任务形式化

智能体决策制定源于目标、感知、记忆和先验知识的交互，使得完全记忆隔离的任务本质上不可实现。然而，这种相互依赖性开启了一个漏洞，允许智能体利用感知变化并绕过显式记忆检索。

为了评估智能体是否真正具有长期记忆推理能力，我们构建了一组记忆密集型任务，它们强制执行感知非马尔可夫决策阶段，其中智能体需要在相同的视觉观测下做出不同的决策。

### 5.2 Camo-Dataset

我们设计了三个机器人操作任务来全面评估三个不同的记忆能力：

#### 任务1：清理指定盘子 (Clean a specified plate)

**场景**：模拟厨房场景，其中用户将污染的盘子（例如，用于生肉但视觉上干净）放供机器人清理。

**记忆挑战**：这个任务体现了事件-对象绑定问题，要求智能体将特定事件与相同对象关联。

**决策阶段**：三个盘子看起来相似，机器人必须记住用户新放置的盘子。

#### 任务2：玩壳杯游戏 (Play shell game)

**场景**：模拟壳杯游戏，其中用户将立方体隐藏在一个杯子下并随机交换杯子。

**记忆挑战**：机器人必须通过记忆跟踪隐藏的立方体。这个任务代表空间跟踪问题，例如组织容器或从抽屉中检索物品。

**决策阶段**：三个杯子在决策阶段看起来相同。

#### 任务3：添加各种调料 (Add various seasonings)

**场景**：模拟在烹饪期间顺序添加三种调料。每次添加后，机器人必须将调料返回到其原始位置并移动回准备姿态，使得在不回忆先前动作的情况下推断是否已经添加了调料变得不可能。

**记忆挑战**：这个任务代表计数或顺序执行问题，例如制作咖啡或烤蛋糕。

**决策阶段**：三个勺子在决策阶段看起来相似。

### 5.3 记忆类别与生物基质的对应关系

| 记忆类别 | 生物基质 | Camo-Dataset任务 |
|-----------|----------|-------------------|
| 情节召回 (Episodic Recall) | CA1, PFC | 清理指定盘子 |
| 空间跟踪 (Spatial Tracking) | DG, CA3 | 玩壳杯游戏 |
| 序列操作 (Sequential Manipulation) | 多个区域协调 | 添加各种调料 |

## 6. 实验设置

### 6.1 数据集

我们在Camo-Dataset上使用真实世界机器人执行的一致训练和部署流水线评估Chameleon。

**数据收集**：
- 对于每个任务，使用UR5e机械臂在人在环控制下收集120个遥操作演示，以确保任务执行质量
- 演示确保任务相关的空间关系和操作序列被正确捕获

**测试阶段**：

- 对于每个任务类别执行36个独立真实世界试验

**一致性设计**：
- 在数据集和真实机器人评估中使用相同的决策设置和候选采样
- 两个设置中的随机级成功概率相同

### 6.2 基线模型

我们与三个最先进的具身智能体模型比较Chameleon：

| 基线模型 | 描述 |
|-----------|--------|
| Diffusion Policy | 基于扩散的策略生成方法 |
| Flow Matching | 基于流匹配的仿射方法 |
| ACT (Action Chunking with Transformers) | 基于Transformer的动作分块方法 |

我们还进行消融研究，通过评估五个变体来分析关键组件的贡献。所有模型的参数计数在附录中报告。

### 6.3 评估指标

除了标准成功率（SR）外，我们引入**决策成功率（DSR）**和**操作成功率（MSR）**以提供记忆密集型任务的细粒度评估，分别测量检索准确性和执行成功。

**指标关系**：

```
SR = DSR × MSR
```

由于DSR受随机级性能约束，我们采用机会调整的Cohen's kappa系数来反映记忆依赖的决策成功率：

```
κ = (p_o - p_e) / (1 - p_e)
```

其中：
- p_o：观测到的DSR
- p_e：随机级DSR

**指标解释**：
- **DSR (κ)**：记忆驱动的决策准确性，独立于执行能力
- **MSR**：执行精度，独立于记忆能力
- **SR**：整体任务成功率，结合决策和执行

### 6.4 实现细节

**训练配置**：
- 优化器：AdamW
- 学习率：1×10⁻⁴
- 批大小：4
- 学习率调度：余弦退火
- 精度：bfloat16混合精度
- 训练步数：每个任务20K步
- GPU：单个4090 GPU

**推理配置**：
- 轨迹生成：使用50个整流流ODE步骤
- 平均端到端推理延迟：每控制步82ms
- 实时性：能够在UR5e平台上进行实时操作

所有架构超参数、配置和训练细节在附录中进一步提供。

## 8. 关键创新点

### 8.1 感知混淆问题的形式化

**创新**：将机器人记忆密集型任务形式化为具有感知混淆的决策问题。

**贡献**：
- 明确定义了观测层面的非马尔可夫性
- 提供了系统性评估记忆能力的方法论
- 避免了智能体利用感知差异绕过显式记忆检索的漏洞

### 8.2 生物启发的分层记忆架构

**创新**：受人类内嗅皮层-海马体-前额叶皮层（EC-HC-PFC）情景记忆系统启发的完全可微分架构。

**生物对应**：
- EC → 多模态融合
- DG → 空间锚点（模式分离）
- CA3 → 情节记忆（关联检索）
- CA1 → 工作读取（上下文绑定）
- PFC → 分层融合（目标导向召回）

**技术优势**：
1. 完全可微分：端到端可训练
2. 多时间尺度：通过时间锚点实现
3. 时空结构化：通过空间和时间锚点实现
4. 无显式检索门：通过状态空间递归隐式实现

### 8.3 几何锚定的多视图感知

**创新**：使用末端执行器位姿作为动作中心的几何锚点来引导几何一致的跨视图通信。

**技术特点**：
1. 双相机设计：前相机提供全局上下文，手相机捕获接触细节
2. 几何引导注意力：对极可行性偏置和一元补丁偏置
3. FiLM风格调制：几何条件调制视觉特征
4. 视图一致性：确保只有空间一致的补丁可以相互通信

**优势**：
- 在遮挡下保留细粒度感知线索
- 防止跨视图聚合混合不兼容证据
- 为记忆形成提供稳定的写入时证据流

### 8.4 可微分目标导向检索

**创新**：通过HoloHead潜在想象目标学习目标导向检索。

**特点**：
1. 预测性训练：迫使记忆状态预测近期状态演化
2. 模式完成：从部分线索重建连贯未来
3. 任务相关性：召回由决策效用驱动，而非相似性

**优势**：
- 避免返回感知相似但决策无关的情节
- 提高记忆召回的决策效用

### 8.5 记忆条件整流流匹配

**创新**：使用整流流匹配一次性生成整个末端执行器轨迹。

**优势**：
1. 提高长视野控制的稳定性
2. 单状态条件：使记忆的作用显式化
3. 高效推理：平均82ms延迟，支持实时操作

### 8.6 真实世界数据集

**创新**：Camo-Dataset，专门设计用于评估感知混淆下的记忆能力。

**特点**：
1. 三个任务类别：情节召回、空间跟踪、序列操作
2. 强制感知非马尔可夫决策阶段
3. 真实机器人执行：UR5e上的120个演示和36个测试试验
4. 细粒度指标：DSR、MSR、SR

**价值**：
- 提供了系统评估记忆能力的基准
- 避免智能体利用感知变化绕过记忆的漏洞
- 与神经科学记忆分类对齐

### 8.7 多时间尺度状态空间模型

**创新**：内容自适应的多时间尺度记忆，而非简单堆叠相同SSM。

**特点**：
1. 显式半衰期调度：每个时间槽有不同基本步长
2. 内容自适应步长调制：通过Proj^{(ℓ)}_{a,b}进一步调制有效步长
3. 融合的按槽扫描：在单次传递中更新所有A×B状态

**优势**：
- 结构化多时间尺度记忆
- 高效实现
- 内容自适应记忆衰减

### 8.8 空间和时间锚点机制

**创新**：将视觉证据分解为空间分量和时间槽。

**特点**：
1. 空间锚点（A个）：路由权重将补丁分配到空间分量
2. 时间锚点（B个）：每个空间分量有多个时间槽
3. A×B时空槽矩阵：同时编码空间和时间结构

**优势**：
- 实现模式分离（空间维度）
- 实现多时间尺度（时间维度）
- 可索引的事件表示

## 9. 局限性和未来工作

### 9.1 当前局限性

**1. 任务范围限制**
- 当前数据集仅包含三个记忆类别
- 任务相对简单（盘子、杯子、勺子）
- 需要验证在更复杂场景中的泛化能力

**2. 对象多样性限制**
- 主要测试了相似的视觉对象（相同的盘子、杯子、勺子）
- 需要评估在对象类别、形状、材质变化下的性能

**3. 环境复杂性限制**
- 测试环境相对结构化（厨房场景）
- 需要验证在非结构化环境中的鲁棒性

**4. 记忆容量限制**
- 当前架构的情节记忆容量受时空槽数量限制
- 长期记忆的容量和检索效率未充分探索

**5. 交互历史长度限制**
- 主要测试了中短期交互历史
- 超长期记忆（数百步）的性能未充分评估

**6. 泛化能力**
- 任务特定训练：每个任务单独训练模型
- 跨任务泛化和零样本泛化能力未充分探索

**7. 传感器配置限制**
- 仅使用两个相机（前视和手挂载）
- 未探索其他传感器模态（力觉、触觉、深度）

**8. 计算开销**
- 虽然平均82ms延迟支持实时操作，但复杂场景可能需要更多优化
- 多时间尺度SSM和多层记忆的计算开销未充分分析

### 9.2 未来工作方向

**1. 扩展任务多样性**
- 开发更多类别的记忆密集型任务
- 包含更复杂的多步骤任务
- 测试多对象、多agent交互场景

**2. 改进记忆架构**
- 探索更强大的模式分离机制
- 设计动态可扩展的记忆容量
- 实现长期记忆的主动遗忘和重要性加权

**3. 增强泛化能力**
- 开发跨任务共享的记忆表示
- 探索元学习和少样本学习
- 设计任务不可知的基础记忆架构

**4. 多模态扩展**
- 集成力觉和触觉感知
- 探索音频线索（环境声音、操作声音）
- 结合深度信息和语义分割

**5. 理论分析**
- 分析记忆模块的数学性质
- 证明模式分离和完成的理论保证

#### 10.2.1 记忆架构的贡献

- **工作记忆**：集成情节召回进行即时控制，支持在线操作

**Spatial AGI关联**：在物理世界操作中，智能体必须记住过去与空间相关的交互以做出正确的未来决策。Chameleon的架构直接解决了这个核心需求。

#### 10.2.2 感知混淆处理的贡献

**核心问题**：Spatial AGI需要在遮挡和状态变化下操作，这正是感知混淆的定义。

**Spatial AGI关联**：在复杂空间任务中，智能体需要记住什么信息对未来决策是相关的。Chameleon的目标导向检索机制为Spatial AGI提供了这种能力。

#### 10.2.4 长视野控制的贡献

## 附录C：Camo-Dataset任务详情

### C.1 清理指定盘子

**设置**：
- 三个相同的盘子放置在桌子上
- 用户选择一个盘子并"污染"（标记为需要清洁）
- 机器人观察并记住哪个盘子被标记
- 延迟后，机器人必须选择并清洁正确的盘子

**记忆需求**：
- 事件-对象绑定：将特定事件（标记）与特定对象关联
- 长期保持：延迟期间保持记忆
- 消除混淆：在视觉相似的对象中识别正确的

### C.2 玩壳杯游戏

**设置**：
- 三个相同的杯子放在桌子上
- 用户将立方体隐藏在一个杯子下
- 用户随机交换杯子1-2次
- 机器人必须跟踪立方体最终位置
- 机器人必须抓取正确的杯子

**记忆需求**：
- 空间跟踪：跟踪对象通过遮挡
- 序列记忆：记住交换历史
- 消除混淆：在相同外观的杯子中识别正确的

### C.3 添加各种调料

**设置**：
- 三种不同的调料（盐、胡椒、辣椒粉）放在不同位置
- 机器人必须按特定顺序添加调料
- 每次添加后，机器人必须将调料返回到原始位置并移动回准备姿态
- 机器人必须记住哪些调料已经添加

**记忆需求**：
- 序列记忆：记住添加的顺序
- 计数：记住已经添加了多少种
- 状态推断：从当前观察推断哪些还需要添加

## 附录D：消融研究结果

### D.1 无空间锚点

**变体描述**：移除空间锚点机制，所有视觉证据直接路由到时间槽。

**预期效果**：
- 降低模式分离能力
- 增加视觉相似但历史不同状态之间的干扰
- 在情节召回任务中DSR显著下降

### D.2 无时间锚点

**变体描述**：移除时间锚点，所有槽使用相同的时间尺度。

**预期效果**：
- 失去多时间尺度记忆能力
- 无法在不同时间尺度上保持信息
- 在需要长期记忆的任务中性能下降

### D.3 无HoloHead

**变体描述**：移除潜在想象目标，仅使用策略损失训练。

**预期效果**：
- 降低模式完成能力
- 检索可能由感知相似性而非任务相关性驱动
- 在空间跟踪任务中DSR下降

### D.4 无几何锚定

**变体描述**：移除末端执行器锚定，使用标准的交叉注意力。

**预期效果**：
- 跨视图聚合可能混合不兼容证据
- 在遮挡下丢失细粒度感知线索
- 在所有任务中性能下降

### D.5 替换记忆模块

**变体描述**：使用Transformer替换时空SSM记忆模块。

**预期效果**：
- 失去多时间尺度和模式分离
- 检索可能基于相似性而非效用
- 性能显著低于完整Chameleon
