## 摘要

时空场景图提供了一种原理性表示方法来建模演化的对象交互，然而现有方法仍然根本上是帧中心的：它们只推理当前可见的对象，在遮挡时丢弃实体，并在2D中操作。为了解决这个问题，我们首先引入ActionGenome4D，这是一个通过前馈3D重建、世界坐标帧定向边界框（针对动作中涉及的每个对象）和密集关系注释（包括由于遮挡或相机运动而暂时未被观察到的对象）将Action Genome视频升级为4D场景的数据集。基于这些数据，我们形式化了世界场景图生成（WSGG），即在每个时间戳构建世界场景图的任务，该图包含场景中所有交互的对象，包括已观察和未观察的。然后，我们提出了三种互补的方法，每种方法探索了不同的归纳偏置来推理未观察的对象：PWG（持久世界图），它通过零阶特征缓冲区实现对象持久性；MWAE（掩码世界自动编码器），它通过交叉视图关联检索将未观察对象推理重新构造成掩码补全；4DST（4D场景Transformer），它用差异化每对象时间注意力（通过3D运动和相机姿态特征增强）替换静态缓冲区。我们进一步通过一系列基于图RAG的方法，设计和评估了强开源视觉-语言模型在WSGG任务上的性能，为未定位关系预测建立了基线。WSGG因此将视频场景理解推进到以世界为中心、时间持久且可解释的场景推理。

## 核心信息

### 主要贡献

1. **ActionGenome4D数据集**: 系统地将Action Genome扩展到4D，结合自动化伪标签生成与手动校正，为所有交互对象提供3D锚定的定向边界框，覆盖未观察对象关系的密集语义注释，以及具有相机姿态的时间对齐的每帧3D场景。

2. **WSGG任务**: 一个新任务，将视频场景图生成推广到以世界为锚定、时间一致的场景图，覆盖从单目视频的所有交互对象（已观察和未观察）。

3. **三种WSGG方法（PWG、MWAE和4DST）**: 探索不同归纳偏置进行未观察对象推理的互补方法，具有共享架构组件和全面消融实验。

4. **VLM评估**: 系统评估开源VLM在WSGG任务上的表现，为未定位关系预测建立基线。

### 核心思想

**问题**: 现有的视频场景图生成（VidSGG）方法根本上是帧中心的：
- 只推理当前可见的对象
- 在遮挡或对象离开相机视野时丢弃实体
- 在2D中操作，缺乏3D空间锚定
- 缺乏时间一致性

**解决方案**: WSGG将场景理解从帧中心转向世界中心：
- 在共享的世界坐标帧中表示所有对象（已观察和未观察）
- 维护对象的持久性（object permanence）
- 在3D空间中锚定场景理解
- 预测所有交互对象之间的密集关系

## 核心技术发现

### 1. 3D场景表示的持久性

**发现**: 通过持久3D几何脚手架，可以在世界坐标帧中保持所有对象的3D位置，即使它们暂时未被观察。

**技术细节**:
- 使用π³（pi3）模型进行前馈3D重建
- 通过迭代bundle adjustment（BA）优化相机姿态
- 构建共享世界坐标帧
- 每个对象都有定向3D边界框（OBB）
- 所有3D框都表示在同一世界帧中

**意义**: 这解决了传统VidSGG的帧中心限制，使得场景理解独立于相机视角。

### 2. 对象持久性的不同实现方式

**发现**: 三种方法探索了不同的归纳偏置来处理未观察对象：

#### PWG (Persistent World Graph)
- **原理**: 零阶保持（zero-order hold）
- **实现**: 保持每个对象的视觉特征在其最后可见外观处
- **优势**: 简单直接，实现对象持久性原则
- **局限**: 非差异化，无法端到端学习

#### MWAE (Masked World Auto-Encoder)
- **原理**: 将遮挡和相机运动视为自然的掩码
- **实现**: 使用关联检索器通过交叉视图重建和模拟遮挡目标来重建未观察对象表示
- **优势**: 掩码建模提供正则化，增强泛化
- **局限**: 需要训练多个损失函数

#### 4DST (4D Scene Transformer)
- **原理**: 用时间Transformer替换静态缓冲区
- **实现**: 对所有对象token（已观察和未观察）执行双向自注意力
- **优势**: 差异化，可端到端学习，时间上下文丰富
- **局限**: 计算复杂度较高

### 3. 多模态特征融合

**发现**: 结合视觉、几何、相机姿态和运动特征可以显著提升未观察对象推理。

**技术细节**:
- 视觉特征: 来自冻结的DINOv2/v3 backbones
- 几何特征: 3D OBB的8个角点
- 相机特征: 相机外参矩阵（SE(3)变换）
- 运动特征: 3D速度、加速度、相机相对速度

### 4. Spatial AGI相关的空间表示

**发现**: WSGG提供了以世界为中心的空间表示，这正是Spatial AGI所需要的。

**技术细节**:
- 世界坐标帧: 所有对象都在同一3D坐标系中表示
- 持久性: 对象即使在离开相机视野后仍然存在
- 时间一致性: 对象身份和位置在时间上一致
- 结构关系: 密集的成对关系（注意、空间、接触）

#### 1) 核心思想和动机

**核心思想**: 将视频场景理解从"以相机为中心、帧中心、2D"转向"以世界为中心、时间持久、3D"。

**动机**:
- **认知科学原则**: 对象持久性（object permanence）是人类和动物物理推理的基础 - 理解对象在离开直接感知后仍然存在。
- **机器人需求**: 机器人agent需要与环境中的对象交互，在时间上累积观察，维护现有对象的持久语义记忆，并推理空间关系如何演化，即使对它们不再可见的对象。
- **现有方法的限制**: 传统VidSGG方法是帧中心的，只推理当前可见的对象，在遮挡或离开相机视野时丢弃实体，在2D中操作。

**关键洞察**:
- 真实世界的agent需要世界中心的场景理解，而不是相机中心的
- 需要三种能力：3D空间锚定、时间一致的对象身份跟踪、对所有交互对象的密集语义注释（包括未观察对象）

---

#### 2) 主要技术方法

WSGG包含三个主要技术组件：

##### A. 数据集构建（ActionGenome4D）

**目标**: 将Action Genome从2D视频扩展到4D时空场景表示。

**三个阶段**:
1. **3D场景构建**
   - 使用π³模型进行前馈3D重建
   - 通过迭代bundle adjustment优化相机姿态
   - 输出：每帧3D点云和相机位姿

2. **几何注释**
   - 对象检测: GDINO（Grounding DINO）
   - 实例分割: SAM2（图像和视频模式）
   - 3D点提取: 使用分割掩码从π³重建中提取
   - 地板对齐: 通过SMPL mesh相似性估计
   - OBB拟合: PCA + 多尺度侵蚀策略
   - 时间平滑: Kalman滤波 + RTS反向平滑

3. **语义注释**
   - 使用VLM生成伪标签（带字幕的视频级事件图）
   - 手动校正界面
   - 覆盖所有对象（已观察和未观察）
   - 三类谓词：注意（3个）、空间（6个）、接触（17个）

##### B. 任务定义（WSGG）

**目标**: 在每个时间戳t构建世界场景图𝒢𝒲^t，包含所有交互对象（已观察和未观察）。

**形式化定义**:
- 世界状态: 𝒲^t = 𝒪^t ∪ 𝒰^t
  - 𝒪^t: 已观察对象（在帧I^t中可见）
  - 𝒰^t: 未观察对象（由于遮挡或相机运动不可见）
- 每个对象w_k^t有：
  - 类别c_k^t
  - 3D定向边界框𝐛_k^t（世界坐标帧）
  - （仅对已观察）2D边界框𝐝_k^t
- 世界场景图: 𝒢𝒲^t = {r_ijk^t | w_i^t, w_j^t ∈ 𝒲^t}
  - 包含所有对象的成对关系
  - 包括已观察-已观察、已观察-未观察、未观察-未观察

##### C. 三种方法

###### PWG (Persistent World Graph)

**核心**: 实现对象持久性的最简单方法 - 零阶保持。

**LKS (Last-Known-State) 记忆缓冲区**:
```
𝐦_n^(t) = {
    𝐟_n^(t)                         if visible at t,
    𝐟_n^(τ*)                        τ* = argmin_{τ: vis(n,τ)} |t-τ|,
    𝟎                              if never seen
}
```

**特点**:
- 非差异化缓冲区
- 保持每个对象在其最后可见外观处的视觉特征
- 记录每个对象的陈旧度Δ_n^(t) = |t - τ*|
- 使用3D几何作为脚手架进行全图推理

**输入**: 视觉特征（缓冲）、几何、相机、log-陈旧度
**输出**: 每个时间戳的世界场景图

###### MWAE (Masked World Auto-Encoder)

**核心**: 将遮挡和相机运动视为自然的掩码，通过关联检索重建未观察对象表示。

**架构**:
1. **脚手架Token化器**:
   - 每个对象在每个帧都接收一个token
   - 对于可见对象: 使用投影的视觉特征
   - 对于未观察对象: 使用可学习token
   - 训练时人工掩码一部分可见对象

2. **关联检索器**:
   - 非对称交叉注意力：查询跨越所有token，键/值仅限于可见token
   - 防止未观察token相互关注
   - 检索后添加可见性嵌入

**三重目标损失**:
- 场景图损失ℒ_SG（可见/未观察分割）
- 重建损失ℒ_recon（仅人工掩码对象）
- 模拟未见损失ℒ_sim（重新预测掩码但可见对象的关系）

**特点**:
- 差异化
- 掩码建模提供正则化
- 通过完整几何脚手架的结构先验重建

###### 4DST (4D Scene Transformer)

**核心**: 用时间Transformer替换静态缓冲区，实现端到端学习。

**时间对象Transformer**:
- 对每个对象n，形成T个token的序列
- token包含：视觉、结构、相机、运动、ego-motion特征
- 添加正弦位置编码和学习可见性嵌入
- 未观察token仍然接收完整几何和相机特征
- 双向Transformer编码器（无因果掩码）

**特点**:
- 完全差异化
- 端到端学习
- 每个对象独立的时间建模
- 丰富的上下文信息

#### 3) 算法流程和关键步骤

##### 共享架构组件

所有三种方法共享以下组件：

1. **全局结构编码器**
   - 输入: 3D OBB的8个角点
   - 处理: 中心化、扁平化、与质心连接（27-d）
   - 输出: 每对象结构token𝐠_n和全局token𝐠_global（通过max-pooling）

2. **空间位置编码**
   - 对每对对象(i, j)计算5-d特征:
     - 欧几里得距离d_ij
     - 单位方向𝐝̂_ij
     - log体积比ρ_ij = log V_i - log V_j
   - MLP编码 + 平均聚合

3. **空间GNN**
   - Transformer编码器 + 空间位置编码
   - 建模帧内对象交互

4. **节点预测器**
   - 两层MLP预测对象类别

5. **关系预测器**
   - 为每个人-对象对(p, o)形成关系token:
     - 𝐫_po = Proj([𝐡_p || 𝐡_o || 𝐮_po || 𝐞_p^txt || 𝐞_o^txt])
   - 使用冻结的CLIP文本嵌入
   - 三组MLP头预测注意、空间、接触关系

6. **时间边注意力**
   - 为每个跨帧的唯一对(p, o)，关系token通过时间Transformer增强

7. **相机姿态编码器**
   - 使用6D旋转表示编码SE(3)变换
   - 计算每对象相机相对特征（距离、视图对齐、方位角）

8. **运动特征编码器**
   - 每对象3D速度、加速度、相机相对速度（11-d）

##### 完整流程（以4DST为例）

**训练阶段**:
1. **数据准备**:
   - 加载ActionGenome4D数据集
   - 加载每帧的3D OBB、相机姿态、视觉特征（DINOv2/v3）
   - 加载关系标签（注意、空间、接触）

2. **特征编码**:
   - 编码3D OBB → 结构token
   - 编码视觉特征 → 视觉token
   - 编码相机姿态 → 相机token
   - 编码运动 → 运动token

3. **时间建模（4DST特有）**:
   - 对每个对象形成时间序列token
   - 添加位置编码和可见性嵌入
   - 双向Transformer编码

4. **空间建模**:
   - 将所有对象的token在同一帧内通过空间GNN
   - 使用空间位置编码

5. **关系预测**:
   - 形成关系token（人-对象对）
   - 时间边注意力增强
   - 预测三类关系

6. **损失计算**:
   - 分割为可见对（两者在𝒪^t）和未观察对（至少一个在𝒰^t）
   - 可见对使用干净GT
   - 未观察对使用VLM伪标签（加权λ_vlm）
   - BCE损失 + 节点损失

**推理阶段**:
1. **输入**: 单目视频V_1^T
2. **3D重建**: π³ + BA → 3D点云 + 相机姿态
3. **对象检测**: GDINO → 2D边界框 + 类别
4. **实例分割**: SAM2 → 分割掩码
5. **3D定位**: 从点云提取 + 地板对齐 + OBB拟合
6. **特征提取**: DINO视觉特征
7. **WSGG推理**:
   - 使用三种方法之一（PWG/MWAE/4DST）
   - 预测所有对象的关系（包括未观察）
8. **输出**: 每个时间戳的世界场景图𝒢𝒲^t

---

#### 4) 输入输出

**输入**:
- **视频**: V_1^T = {I^t}_{t=1}^T，T帧单目RGB图像
- **预处理**（离线）:
  - 3D点云: 来自π³重建
  - 相机姿态: SE(3)变换矩阵{𝐓^t}_{t=1}^T
  - 对象检测: 2D边界框{𝐝_k^t}
  - 实例分割: 像素级掩码
  - 视觉特征: DINOv2/v3特征{𝐟_k^t}

**输出**:
- **每个时间戳t的世界场景图** 𝒢𝒲^t:
  - **节点**: 所有对象{w_k^t}_{k=1}^{M(t)}在𝒲^t
    - 类别c_k^t ∈ 𝒞 (37类)
    - 3D OBB 𝐛_k^t ∈ ℝ^{8×3}（世界坐标）
  - **边**: 所有成对关系{r_ijk^t}_{ijk}
    - 注意关系p_ijk^t ∈ 𝒫_att (3类)
    - 空间关系p_ijk^t ∈ 𝒫_spa (6类，多标签)
    - 接触关系p_ijk^t ∈ 𝒫_con (17类，多标签)

**两种设置**:
- **PredCls**: 给定GT标签和3D定位，仅预测关系
- **SGDet**: 端到端检测和预测

**两种约束**:
- **With Constraint**: 每个对象对分配恰好一个谓词
- **No Constraint**: 每个对象对可以携带多个谓词（多标签）

### Q2: 与Spatial AGI的关系

**问题**: 这篇文章与通用空间智能（Spatial AGI）有什么关系？请分析：1) 如何理解和表示空间，2) 如何处理空间关系，3) 对Spatial AGI有什么启发，4) 可以应用到哪些Spatial AGI场景（机器人、AR/VR等）。

---

#### 1) 如何理解和表示空间

**世界中心的空间表示**:

WSGG从根本上改变了空间表示的方式，从传统的"以相机为中心"转向"以世界为中心"：

**传统VidSGG的空间表示**:
- **2D图像坐标**: 对象位置表示为2D边界框
- **帧中心**: 每帧独立处理，缺乏跨帧一致性
- **相机视角依赖**: 空间关系依赖于当前相机视角
- **非持久性**: 对象离开视野后消失

**WSGG的空间表示**:
- **3D世界坐标**: 所有对象在同一3D世界坐标帧中表示
- **持久性**: 对象即使在离开相机视野后仍然存在于世界状态中
- **视角无关**: 空间关系独立于相机视角
- **时间一致性**: 对象身份和3D位置在时间上一致

**具体的空间表示技术**:

**3D定向边界框（OBB）**:
- 8个角点在世界坐标帧中表示
- 不仅提供位置，还提供方向和体积信息
- 通过PCA和地板对齐获得
- Kalman滤波 + RTS反向平滑保证时间一致性

**世界坐标帧的建立**:
- π³前馈3D重建提供初始3D点和相机姿态
- 迭代bundle adjustment优化全局一致性
- 地板对齐通过SMPL mesh相似性估计
- 所有对象都转换到这个共享的世界帧

**空间特征的编码**:
- **成对几何特征**: 距离、方向、体积比（5维）
- **全局结构特征**: OBB的8个角点（27维）
- **相对相机特征**: 距离、视图对齐、方位角
- **运动特征**: 3D速度、加速度、相机相对速度

**对Spatial AGI的意义**:

WSGG的空间表示方式与Spatial AGI的核心需求高度契合：

1. **世界中心的感知**: Spatial AGI需要理解世界，而不是理解相机视野
2. **持久性记忆**: Agent需要记住对象即使它们暂时不可见
3. **统一坐标系**: 所有对象在同一3D坐标系中，便于规划和推理
4. **3D几何理解**: 真实的空间关系在3D中定义，而不是2D投影

---

#### 2) 如何处理空间关系

**密集的空间关系建模**:

WSGG将空间关系建模分为三个正交轴：

##### A. 注意关系（𝒫_att，3类，单标签）

关注关系：表示对象之间的视觉关注。
- 示例：person looking at computer, person watching TV

**特点**:
- 最直接的交互
- 视觉线索驱动
- 单标签（每个对象对最多一个注意关系）

##### B. 空间关系（𝒫_spa，6类，多标签）

几何和拓扑关系：表示对象在3D空间中的相对位置。
- 示例：in front of, behind, left, right, above, below

**特点**:
- 3D空间中的相对位置
- 多标签（一个对象对可以有多个空间关系）
- 使用3D几何信息（距离、方向、体积比）

##### C. 接触关系（𝒫_con，17类，多标签）

物理交互关系：表示对象之间的物理接触或交互。
- 示例：holding, touching, sitting on, standing on, leaning on

**特点**:
- 物理世界的真实交互
- 多标签（复杂交互可能涉及多种接触关系）
- 对机器人任务至关重要

**空间关系的处理方式**:

**成对关系预测**:
- 对每个人-对象对(p, o)形成关系token
- 融合视觉、几何、文本嵌入
- 时间边注意力增强时间一致性
- 三组独立的MLP头预测三类关系

**未观察对象的关系**:

这是WSGG最关键的贡献之一 - 预测未观察对象之间的关系。

**为什么重要**:
- 真实场景中，许多重要的交互发生在相机视野之外
- 机器人需要推理"房间里可能发生什么"而不仅仅是"现在看到什么"
- 长期规划和推理需要理解不可见对象的状态

**三种方法处理未观察关系**:

1. **PWG**: 使用最后已知状态，依赖3D几何脚手架
   - 优势: 简单、有效
   - 局限: 无法学习时间演化

2. **MWAE**: 通过关联检索重建未观察表示
   - 优势: 正则化强，泛化好
   - 局限: 需要复杂的损失设计

3. **4DST**: 时间Transformer建模完整时间序列
   - 优势: 端到端学习，时间上下文丰富
   - 局限: 计算复杂度高

**空间关系的空间锚定**:

与传统VidSGG不同，WSGG的关系锚定在世界坐标帧中：

- **传统**: 关系由2D图像中的相对位置决定（依赖于相机视角）
- **WSGG**: 关系由3D世界坐标中的相对位置决定（视角无关）

这使得关系预测更加鲁棒，不受相机运动的影响。

#### 3) 对Spatial AGI的启发

WSGG对Spatial AGI有多个重要启发：

##### 启发1: 世界中心的场景理解是必需的

**传统AI的问题**:
- 大多数视觉系统处理的是"相机看到的"
- 对于机器人agent来说，这远远不够
- 机器人需要理解"世界存在什么"

**WSGG的解决方案**:
- 将场景理解从相机中心转向世界中心
- 维护所有对象的持久表示，即使它们暂时不可见
- 在统一的3D世界坐标系中表示一切

**对Spatial AGI的启发**:
- Spatial AGI的核心能力应该是理解世界，而不是理解图像
- 需要构建和维护世界的持久3D表示
- 对象持久性（object permanence）是基本能力

##### 启发2: 3D几何是关键的结构先验

**发现**:
- 3D几何脚手架（OBB）提供了强大的结构先验
- 即使是简单的PWG方法（非差异化），在3D几何的帮助下也能达到接近最佳方法的性能
- 3D信息使得未观察对象的推理成为可能

**对Spatial AGI的启发**:
- Spatial AGI需要强大的3D感知和建模能力
- 3D几何不仅仅是输出，更是推理的基础
- 应该在训练和推理中都充分利用3D几何信息

##### 启发3: 时间建模和持久记忆是核心挑战

**发现**:
- 处理未观察对象是最难的挑战
- 三种方法探索了不同的归纳偏置：
  - 静态记忆（PWG）
  - 掩码建模（MWAE）
  - 时间Transformer（4DST）
- 4DST表现最好，表明端到端时间学习的价值

**对Spatial AGI的启发**:
- Spatial AGI需要强大的时间建模能力
- 需要维护和更新持久的世界模型
- 不同的归纳偏置可以互补，应该同时探索

##### 启发4: 多模态特征融合至关重要

**发现**:
- 单独使用任何一种特征（视觉、几何、相机、运动）都不够
- 多模态融合显著提升了性能
- 不同特征提供了互补的信息：
  - 视觉: 外观、语义
  - 几何: 3D位置、大小、方向
  - 相机: 视角、视野
  - 运动: 动态行为、轨迹

**对Spatial AGI的启发**:
- Spatial AGI需要整合多模态传感器数据
- 不仅仅是视觉，还包括：
  - 深度/3D传感器
  - IMU（惯性测量单元）
  - 里程计
  - 可能还有声音、触觉等
- 多模态融合是Spatial AGI的基础能力

##### 启发5: 评估方法需要考虑未观察对象

**发现**:
- 传统VidSGG评估只考虑可见对象
- WSGG评估包含未观察对象
- 这使得评估更加现实和严格

**对Spatial AGI的启发**:
- Spatial AGI的评估应该模拟真实世界的复杂性
- 不仅评估"看到了什么"，还要评估"理解了什么"
- 需要新的评估指标来衡量对未观察世界的推理能力

##### 启发6: 端到端学习的价值

**发现**:
- 4DST（完全可微分、端到端）在SGDet任务上表现最好
- 能够学习复杂的时间模式
- 提升了对象分类和尾类别recall

**对Spatial AGI的启发**:
- 端到端学习可能优于模块化方法
- 应该探索更大的端到端Spatial AGI模型
- 但也需要考虑模块化方法的可解释性和调试便利性

#### 4) 可以应用到的Spatial AGI场景

**任务**: 机器人在家庭环境中执行日常任务。

**WSGG的应用**:
- **场景理解**: 理解房间布局、物体位置、空间关系
- **持久记忆**: 记住即使暂时看不到的物体（例如：冰箱里的食物）

  - 人的位置和活动（例如：在厨房做饭）

**优势**:
- 不需要机器人"看到"所有东西就能理解场景

- **持久锚定**: 虚拟对象可以锚定在真实世界的3D位置
- **交互理解**: 理解用户与真实对象的交互
- **场景标注**: 自动标注场景中的物体和关系

**具体例子**:

  - 真实房间和家具的3D表示

  - 即使用户转头，虚拟物体仍然锚定在正确位置
  - 可以预测用户可能交互的对象

**优势**:
- 3D世界坐标使得虚拟对象锚定精确
- 持久记忆保证AR体验的连续性
- 理解关系使得交互更自然

##### 应用场景3: 自主导航和避障

**任务**: 机器人在动态环境中自主导航。

- **遮挡处理**: 理解被遮挡的对象（例如：门后面的人）

  - 人与物体的交互（例如：开门）

**优势**:
- 全局3D地图支持全局路径规划

##### 应用场景4: 视频监控和异常检测

**任务**: 在监控视频中检测异常行为。

**具体例子**:

### Q3: WSGG方法如何解决传统VidSGG的帧中心限制？三种方法在处理未观察物体方面的优势和局限性是什么？

---

#### 传统VidSGG的帧中心限制

传统VidSGG（Video Scene Graph Generation）存在以下根本性限制：

**1. 视野限制**:
- 只推理当前相机视野内的对象
- 对象离开视野后立即消失
- 无法理解相机外发生的事情

**2. 时间不连续**:
- 每帧独立处理
- 缺乏跨帧的对象身份一致性
- 无法跟踪对象的时间演化

**3. 2D依赖**:
- 关系基于2D图像坐标
- 依赖相机视角
- 视角变化会导致关系不稳定

**4. 遮挡处理差**:
- 遮挡后对象消失
- 无法推理遮挡对象的状态
- 缺乏对象持久性

---

#### WSGG如何解决这些限制

**1. 世界中心的表示**:
- 所有对象在统一的3D世界坐标中表示
- 不依赖相机视角
- 关系锚定在世界坐标中

**2. 持久的世界状态**:
- 维护所有对象的状态（已观察和未观察）
- 即使对象离开视野也保留
- 对象身份在时间上一致

**3. 3D几何脚手架**:
- 提供完整的3D场景结构
- 支持跨时间的对象跟踪
- 为未观察对象推理提供结构先验

**4. 时间建模**:
- 三种方法都建模时间维度
- 跟踪对象的时间演化
- 预测未观察对象的关系

---

#### 三种方法在处理未观察物体方面的对比

##### PWG (Persistent World Graph)

**核心思想**: 最简单的对象持久性实现 - 零阶保持。

**优势**:
1. **简单直接**: 实现最简单，易于理解和调试
2. **计算高效**: 非差异化，无需反向传播，推理速度快
3. **基线强**: 即使是非差异化设计，性能接近最佳方法
4. **可解释性强**: 明确知道使用的是哪个时间点的特征

**技术优势**:
- LKS缓冲区明确记录"最后已知状态"
- 陈旧度（staleness）可以用于权衡信任度
- 3D几何脚手架提供了强大的结构先验

**局限性**:
1. **非差异化**: 无法端到端学习，梯度无法流动
2. **特征冻结**: 特征不随时间演化，无法捕获时间模式
3. **信息丢失**: 陈旧度高的对象信息可能过时
4. **无上下文**: 未观察对象无法从其他对象的演化中学习

**适用场景**:
- 计算资源受限的环境
- 需要快速推理的场景
- 作为基线方法
- 对象变化不大的静态场景

**性能表现**:
- PredCls任务: 接近最佳方法（仅落后1-2个百分点）
- SGDet任务: 4DST表现更好
- 在多标签设置（No Constraint）中表现略逊

---

##### MWAE (Masked World Auto-Encoder)

**核心思想**: 将遮挡和相机运动视为自然的掩码，通过掩码建模重建未观察对象。

**优势**:
1. **差异化**: 可端到端学习
2. **正则化强**: 掩码建模提供了强大的正则化
3. **泛化好**: 通过重建和模拟未见损失增强泛化
4. **结构先验**: 利用完整3D几何脚手架

**技术优势**:
- 非对称交叉注意力：查询跨越所有token，键/值仅限可见
- 防止未观察token相互关注（避免错误传播）
- 训练时的人工掩码增强鲁棒性
- 三重目标损失（场景图 + 重建 + 模拟未见）

**局限性**:
1. **复杂度高**: 需要设计多个损失函数
2. **训练难度大**: 三个损失需要仔细平衡权重
3. **计算成本高**: 需要额外的重建头
4. **可能过拟合**: 掩码建模可能过度依赖训练模式

**适用场景**:
- 数据量充足的环境
- 需要强泛化能力的场景
- 复杂动态场景
- 作为正则化技术

**性能表现**:
- PredCls任务（No Constraint）: 表现最好（R@10 = 81.50 DINOv3-L）
- 多标签设置中表现出色
- 在多样化谓词预测上有优势

---

##### 4DST (4D Scene Transformer)

**核心思想**: 用时间Transformer替换静态缓冲区，实现端到端时间建模。

**优势**:
1. **完全差异化**: 整个模型可端到端学习
2. **时间上下文丰富**: 双向Transformer捕获完整时间信息
3. **可学习性**: 可以学习复杂的时间模式
4. **性能最好**: 在大多数设置中达到最佳性能

**技术优势**:
- 每个对象独立的时间序列建模
- 双向注意力（无因果掩码）
- 未观察token仍然接收完整几何和相机特征
- 融合视觉、结构、相机、运动、ego-motion特征
- 位置编码 + 可见性嵌入

**局限性**:
1. **计算复杂度高**: Transformer需要计算所有token对之间的注意力
2. **内存需求大**: 存储完整的时间序列
3. **训练时间长**: 端到端学习需要更多时间
4. **可解释性差**: Transformer的内部机制较难解释

**适用场景**:
- 计算资源充足的环境
- 需要最优性能的场景
- 复杂时间依赖的任务
- 需要端到端学习的场景

**性能表现**:
- SGDet任务（With Constraint）: 表现最好
- R@10 = 42.64 (DINOv2-L), 58.03 (DINOv3-L)
- 在对象分类和尾类别recall上有优势
- Mean Recall (mR@K)也是最高

---

#### 三种方法的互补性

这三种方法并非互相排斥，而是探索了不同的归纳偏置：

**PWG**: 探索"最简单够用"的假设
- 证明了3D几何脚手架本身就很强大
- 为更复杂方法建立了强有力的基线
- 展示了对象持久性的核心价值

**MWAE**: 探索"掩码建模"的归纳偏置
- 将遮挡视为自然的掩码
- 通过重建学习未观察表示
- 强调了正则化的重要性

**4DST**: 探索"时间建模"的归纳偏置
- 端到端学习时间模式
- 双向注意力捕获完整上下文
- 展示了深度学习的优势

**可能的改进方向**:
1. **混合方法**: 结合PWG的简单性和4DST的表达力
2. **分层建模**: 对不同重要性对象使用不同方法
3. **自适应机制**: 根据陈旧度和不确定性动态选择方法
4. **预训练**: 在大规模数据上预训练时间Transformer

---

#### 实验结果分析

从实验结果可以看出：

**1. 性能排名**:
- **SGDet (With Constraint)**: 4DST > MWAE ≈ PWG
- **PredCls (No Constraint)**: MWAE > 4DST ≈ PWG
- **整体**: 4DST最稳定，MWAE在多标签上有优势

**2. 主干网络影响**:
- DINOv3-L相比DINOv2-L有提升，但边际收益递减
- DINOv3-L显著优于DINOv2变体
- DINOv2从Base到Large的提升边际收益较小

**3. PredCls vs SGDet**:
- PredCls性能显著高于SGDet（给定GT标签和定位）
- 检测是主要瓶颈
- 4DST在SGDet上优势更明显（端到端学习的价值）

**4. 约束设置影响**:
- No Constraint（多标签）的recall显著高于With Constraint
- MWAE在No Constraint上优势明显
- 4DST在两种设置上都表现良好

---

#### 对Spatial AGI的启示

这些结果对Spatial AGI系统设计有重要启示：

**1. 简单方法也可以很有效**:
- PWG证明了3D几何脚手架的强大
- Spatial AGI不需要从一开始就追求最复杂的模型
- 先建立强基线，再逐步增加复杂性

**2. 不同任务需要不同方法**:
- 检测任务（SGDet）→ 4DST（端到端学习）
- 关系预测（PredCls）→ MWAE（正则化强）
- 应该根据任务特点选择合适的方法

**3. 计算和性能的权衡**:
- PWG: 低计算，接近最优性能
- 4DST: 高计算，最优性能
- 实际应用中需要权衡

**4. 多模态融合的重要性**:
- 视觉、几何、相机、运动特征的融合是必需的
- Spatial AGI需要整合多种传感器数据

**5. 持久记忆的价值**:
- 所有三种方法都强调持久记忆的重要性
- 这是Spatial AGI区别于传统视觉系统的关键

**6. 端到端学习的潜力**:
- 4DST的最佳表现展示了端到端学习的价值
- 未来的Spatial AGI应该探索更大的端到端模型

## 个人思考

### 1. 最有趣的发现

**3D几何脚手架的强大**:

最让我印象深刻的是，即使是简单的PWG方法（非差异化、零阶保持），在3D几何脚手架的帮助下也能达到接近最佳方法的性能。这说明：

- 3D几何信息提供了极强的结构先验
- 世界中心的表示本身就很有价值
- 对象持久性的核心价值在于"记住"什么，而不在于如何"更新"

这让我想到，人类的空间认知可能也是基于类似的原理：我们记住物体的位置，即使暂时看不到，我们仍然知道它们在哪里。这种"空间记忆"可能比复杂的推理更重要。

**意义**:
- 对于Spatial AGI来说，建立和维护准确的3D世界模型是首要任务
- 复杂的时间建模应该在强基线基础上添加，而不是替代
- 简单但正确的方法可能优于复杂但错误的方法

---

### 2. 最意外的结果

**MWAE在多标签设置上的优势**:

我原本预期4DST在所有设置上都会表现最好，但MWAE在PredCls No Constraint（多标签）上超越了4DST。这说明：

- 掩码建模提供了强大的正则化
- 对于多样化谓词预测（多标签），正则化比复杂建模更重要
- 重建和模拟未见损失起到了关键作用

**可能的解释**:
- 多标签任务更复杂，容易过拟合
- MWAE的重建目标强制模型学习对象的本质特征
- 模拟未见损失增强了泛化能力

**启示**:
- 不同的归纳偏置适合不同的任务
- 不应该盲目追求"最新"或"最复杂"的方法
- 应该根据任务特点选择合适的技术

**更广泛的意义**:
- 这不仅仅是视觉领域的问题，而是所有embodied AI的核心问题

## 关键数据

### 数据集统计

**ActionGenome4D**:
- 基础: Action Genome (AG) 数据集
- 扩展: 从2D视频到4D时空场景
- 对象类别: 37类
- 谓词类别:
  - 注意（Attention）: 3类
  - 空间（Spatial）: 6类
  - 接触（Contacting）: 17类
- 注释密度: 覆盖所有对象（已观察和未观察）

### 3D重建参数

**π³模型**:
- 输入: 单目RGB视频
- 输出: 每帧3D点云 + 相机位姿（SE(3)）
- 优化: 迭代bundle adjustment
- 帧采样: 光流引导的自适应策略

### 3D OBB参数

**定向边界框**:
- 表示: 8个角点（3D世界坐标）
- 提取: 从π³重建点云
- 对齐: 地板对齐（SMPL mesh相似性）
- 拟合: PCA + 多尺度侵蚀
- 平滑: Kalman滤波 + RTS反向平滑

### 模型参数

**视觉特征**:
- Backbone: DINOv2-L (ViT-L/14) 或 DINOv3-L
- 维度: d_roi = 1024 (ViT-L)

**结构特征**:
- 输入: 8个角点 + 质心 = 27维
- 输出: d_struct（未指定）

**关系特征**:
- 维度: d_rel（未指定）

**相机特征**:
- 表示: 6D旋转 + 平移
- 输出: d_camera（未指定）

**时间Transformer**:
- 序列长度: T（视频帧数）
- 维度: d_memory（未指定）

### 性能指标

**Recall @ K (R@K)**:
- K ∈ {10, 20, 50}
- 在Top-K预测中找到至少一个正确谓词的百分比

**Mean Recall @ K (mR@K)**:
- 所有谓词类别的平均R@K
- 更公平的评估（平衡类别不平衡）

**实验结果摘要**（DINOv3-L backbone）:

**PredCls (给定GT标签和3D定位)**:
- With Constraint:
  - 4DST: R@50 = 69.12
  - MWAE: R@50 = 68.59
  - PWG: R@50 = 68.58

- No Constraint:
  - MWAE: R@10 = 81.50（最佳）
  - 4DST: R@10 = 81.04
  - PWG: R@10 = 80.06

**SGDet (端到端)**:
- With Constraint:
  - 4DST: R@50 = 71.95（最佳）
  - MWAE: R@50 = 70.90
  - PWG: R@50 = 70.93

- No Constraint:
  - 4DST: mR@50 = 55.73（最佳）
  - MWAE: mR@50 = 54.08
  - PWG: mR@50 = 55.51

**关键观察**:
- 4DST在SGDet上表现最好（端到端学习的价值）
- MWAE在PredCls No Constraint上表现最好（正则化的价值）
- 三种方法性能差距不大（2-3个百分点）
- DINOv3相比DINOv2有提升，但边际收益递减

### 损失权重

**PWG和4DST**:
- λ_vlm: VLM伪标签的权重（未观察对象）
- 主要损失: 可见对象BCE + λ_vlm * 未观察对象BCE + 节点损失

**MWAE**:
- λ_recon: 重建损失权重
- λ_dom: 支配项权重
- 三重损失: 场景图损失 + λ_recon * λ_dom * 重建损失 + 模拟未见损失

### 消融实验（推断）

基于论文描述和结果，可能的消融结论：

**组件必要性**:
- 3D几何: 必需（提供脚手架）
- 视觉特征: 必需（提供语义）
- 相机姿态: 重要（视角信息）
- 运动特征: 有帮助（动态信息）

**方法对比**:
- 4DST > MWAE > PWG: 在SGDet上
- MWAE > 4DST > PWG: 在PredCls No Constraint上
- 性能差距: 2-3个百分点

**Backbone对比**:
- DINOv3-L > DINOv2-L
- DINOv2-L > DINOv2-Base/B
- 但边际收益递减

## 总结

### 核心发现总结

1. **世界中心的范式转变**: WSGG将场景理解从"以相机为中心"转向"以世界为中心"，这是一个根本性的范式转变。

2. **3D几何脚手架的价值**: 3D几何信息提供了极强的结构先验，即使是最简单的PWG方法也能达到接近最优性能。

3. **三种互补方法**: PWG（简单）、MWAE（正则化）、4DST（端到端）探索了不同的归纳偏置，各有优劣。

4. **未观察对象推理**: WSGG的核心挑战是推理未观察对象的关系，这是传统VidSGG无法做到的。

5. **多模态融合**: 视觉、几何、相机、运动特征的融合是必需的，不同特征提供了互补的信息。

6. **端到端学习的价值**: 4DST的最佳表现展示了端到端学习的潜力，特别是在SGDet任务上。

### 对Spatial AGI的意义

**核心能力**:
- 世界中心的场景理解（不是相机中心）
- 持久3D世界模型（不是每帧独立）
- 对象持久性（对象即使看不见也存在）
- 空间关系推理（基于3D世界坐标）

**技术路线**:
- 3D重建是基础设施（π³ + BA）
- 3D几何是结构先验（OBB + 世界坐标）
- 时间建模是核心挑战（三种方法探索）
- 多模态融合是必需的（视觉 + 几何 + 相机 + 运动）

**应用场景**:
- 家庭服务机器人（场景理解 + 任务规划）
- AR/VR环境（场景重建 + 虚拟锚定）
- 自主导航（地图构建 + 避障）
- 智能监控（异常检测 + 事件理解）
- 多机器人协作（共享世界模型）
- 语音交互（语言grounding + 指令执行）

**未来方向**:
1. **更大规模的数据集**: ActionGenome4D只是开始，需要更多样化和更大规模的数据
2. **更强大的3D重建**: π³的前馈重建可以进一步提升，结合NeRF/3D Gaussian Splatting
3. **更复杂的时间建模**: 4DST展示了潜力，可以进一步探索更复杂的时间模型
4. **端到端学习**: 探索更大的端到端Spatial AGI模型
5. **在线学习和更新**: 当前方法是离线的，实时Spatial AGI需要在线学习和世界模型更新
6. **因果推理**: 不仅预测关系，还要理解因果关系（为什么会发生这种关系）
7. **可解释性**: Transformer的决策过程需要更好的解释

## 附录：术语表

**术语定义**:

- **WSGG (World Scene Graph Generation)**: 世界场景图生成，在每个时间戳构建包含所有对象（已观察和未观察）的3D场景图

- **VidSGG (Video Scene Graph Generation)**: 视频场景图生成，传统方法，只处理可见对象

- **3D OBB (3D Oriented Bounding Box)**: 3D定向边界框，8个角点在世界坐标帧中表示

- **Object Permanence**: 对象持久性，认知科学概念，理解对象在离开直接感知后仍然存在

- **PWG (Persistent World Graph)**: 持久世界图，使用零阶保持实现对象持久性

- **MWAE (Masked World Auto-Encoder)**: 掩码世界自动编码器，将遮挡视为自然掩码

- **4DST (4D Scene Transformer)**: 4D场景Transformer，使用时间Transformer进行端到端学习

- **ActionGenome4D**: 数据集，将Action Genome扩展到4D时空场景

- **LKS (Last-Known-State)**: 最后已知状态，PWG中使用的记忆缓冲区

- **SE(3)**: 特殊欧几里得群，表示3D旋转和平移

- **R@K (Recall @ K)**: 在Top-K预测中找到至少一个正确谓词的百分比

- **mR@K (Mean Recall @ K)**: 所有谓词类别的平均R@K
